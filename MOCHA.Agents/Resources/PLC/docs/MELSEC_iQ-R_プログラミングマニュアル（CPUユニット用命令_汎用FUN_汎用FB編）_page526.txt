### テキスト
#### 桁移動命令 SMOV(P)
- 対応CPUはR00CPU、R01CPU、R02CPUでバージョン制限なし。RnCPU（R00CPU、R01CPU、R02CPU除く）、RnENCPUはファームウェアバージョン17以降で使用可能。エンジニアリングツールはバージョン1.020W以降で使用可能。
- データを4ビット単位で分配・合成する命令。

### 図
- **ラダー**: (s)、(n1)、(n2)、(d)、(n3)のパラメータを持つ命令ブロックの図。
- **ST言語**: 
  - ENO := SMOV(EN, s, n1, n2, n3, d);
  - ENO := SMOVP(EN, s, n1, n2, n3, d);
- **FBD/LD**: EN, ENO, s, d, n1, n2, n3の入出力ポートを持つ命令ブロックの図。

### 表
#### 実行条件
| 命令   | 実行条件           |
|--------|--------------------|
| SMOV   | 立ち上がりエッジ    |
| SMOVP  | 立ち下がりエッジ    |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容                         | 範囲  | データ型           | データ型(ラベル) |
|------------|------------------------------|-------|--------------------|------------------|
| (s)        | 指定した桁のデータを格納するデバイス | —     | 符号付きBIN16ビット | ANY16            |
| (n1)*1     | 移動させるデータが格納されている先頭位置 | 1〜4  | 符号なしBIN16ビット | ANY16_U          |
| (n2)*1     | 移動させる桁の個数             | 1〜4  | 符号なしBIN16ビット | ANY16_U          |
| (d)*1      | 桁移動したデータを格納するデバイス | —     | 符号付きBIN16ビット | ANY16            |
| (n3)*1     | 移動先の先頭桁位置             | 1〜4  | 符号なしBIN16ビット | ANY16_U          |
| EN         | 実行条件                     | —     | ビット              | BOOL             |
| ENO        | 実行結果                     | —     | ビット              | BOOL             |

*1 (n2) ≤ (n1)、(n2) ≤ (n3)となるように設定する必要がある。
