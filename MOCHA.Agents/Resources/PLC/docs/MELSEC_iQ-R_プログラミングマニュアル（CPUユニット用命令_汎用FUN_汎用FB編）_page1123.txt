### テキスト
#### TO(P)命令の概要
- (s2)で指定したデバイスから(n)点分のデータを、(U/H)で指定したユニットまたは自号機CPUユニット内のバッファメモリの(s1)で指定したアドレス以降に書き込みます。

- ユニットへのワードデータライトと自号機CPUユニットへのワードデータライトの2種類の動作があります。

- (s2)に定数を指定した場合は、同一データ((s2)で指定した値)を指定したバッファメモリアドレスから(n)ワード分に書き込みます。

- 命令実行時に対象ユニットの異常や処理中などでアクセスできない場合は無処理となります。

### 図の説明
1. **ユニットへのワードデータライト（デバイス→ユニットバッファメモリ）**
   - CPUユニットの(s2)で指定した(n)点分のデータを、ユニットバッファメモリの(s1)で指定したアドレス以降に(n)ワード分書き込むイメージ。
   - 左側にCPUユニットのメモリ領域、右側にユニットバッファメモリ領域が描かれており、矢印でデータ転送を示している。

2. **自号機CPUユニットへのワードデータライト（デバイス/ラベルメモリ→自号機CPUバッファメモリ）**
   - (s2)で指定した(n)点分のデータを自号機CPUバッファメモリの(s1)で指定したアドレス以降に(n)ワード分書き込むイメージ。
   - 左側にデバイス/ラベルメモリ、右側に自号機CPUバッファメモリが描かれている。

3. **定数指定時のユニットへのワードデータライト**
   - (s2)に定数「5」を指定した場合、ユニットバッファメモリの(s1)で指定したアドレスから(n)ワード分に同じ値「5」を書き込むイメージ。
   - CPUユニットの5が矢印でユニットバッファメモリの複数ワードにコピーされている。

4. **定数指定時の自号機CPUユニットへのワードデータライト**
   - (s2)に定数「70000」を指定した場合、同様に自号機CPUバッファメモリの(s1)で指定したアドレスから(n)×2ワード分に同じ値「70000」を書き込むイメージ。
   - デバイス/ラベルメモリの70000が自号機CPUバッファメモリの複数ワードにコピーされている。

### 注意事項
- 命令実行時に対象ユニットが異常や処理中でアクセスできない場合は無処理となる。
