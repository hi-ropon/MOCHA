### テキスト
#### 機能
- (s1)で指定された文字列データと、(s2)で指定された文字列データをa接点扱いで比較演算する。
- 比較は文字列のアスキーコードを先頭から1文字ずつ比較する。
- (s1)、(s2)の文字列は指定されたデバイス番号から00Hが格納されているデバイス番号までが対象。
- すべての文字列が一致した場合、比較結果も一致となる。

- 異なる文字列の場合は、キャラクターコードの大きい文字列が大きいと判定される。
- 異なる文字列の場合、最初に異なったキャラクターコードの大小で文字列の大小が決まる。

### 図（16進数コードの比較例）
- 文字列"ABCDE"の各文字を16進数コード（例：42H(B), 41H(A)など）で表し、(s1)と(s2)のコードを比較している。
- 例1: "ABCDE"と"ABCDE"が等しい場合の比較。
- 例2: "ABCDF"と"ABCDE"で、最後の文字が異なり(s1)の方が大きい場合の比較。
- 例3: "12345"と"12435"で、2文字目が異なり(s1)の方が小さい場合の比較。

### 表
#### 命令記号（ラダー、FBD/LD）と比較演算結果の対応

| 命令記号 | 比較演算結果 |
| :--- | :--- |
| $=, EQ | 導通状態(ENOはON) または 非導通状態(ENOはOFF)（条件により異なる） |
| $<>, NE | 非導通状態(ENOはOFF) または 導通状態(ENOはON)（条件により異なる） |
| $>, GT | 非導通状態(ENOはOFF) または 導通状態(ENOはON)（条件により異なる） |
| $<=, LE | 導通状態(ENOはON) または 非導通状態(ENOはOFF)（条件により異なる） |
| $<, LT | 非導通状態(ENOはOFF) または 導通状態(ENOはON)（条件により異なる） |
| $>=, GE | 導通状態(ENOはON) または 非導通状態(ENOはOFF)（条件により異なる） |

- 表は3回繰り返されており、それぞれ比較条件が異なる文字列の大小関係に対応している。
- 1回目の表は完全一致時の比較結果。
- 2回目の表は(s1)の文字列が大きい場合の比較結果。
- 3回目の表は(s1)の文字列が小さい場合の比較結果。

以上より、このページは文字列比較命令の動作原理と命令記号ごとの比較結果の状態を詳細に説明しています。
