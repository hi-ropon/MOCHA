### テキスト
#### TIME型→WORD型変換
関数名：TIME_TO_WORD(_E)  
対応CPU：RnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）  
説明：TIME型データをWORD型データへ変換します。

#### 使用例
- ラダー、FBD/LDではEN/ENOなしとEN/ENO付きの2種類のブロックが示されています。
- ST言語ではEN/ENOなしの場合は「d:=TIME_TO_WORD(s);」、EN/ENO付きの場合は「d:=TIME_TO_WORD_E(EN,ENO,s);」と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | TIME |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | WORD |

#### 機能
- 演算処理：入力されたTIME型データ(s)をWORD型データ(d)に変換して出力します。  
- 入力値sはTIME型のデータ値です。

#### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、dから演算出力値が出力されます。  
2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | 出力 d |
| :--- | :--- | :--- |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となります。その場合はdから出力されたデータを使用しないようプログラムで処理してください。

#### エラー
演算エラーはありません。

### 図
- ラダー/FBD/LDのEN/ENOなしブロックはs入力からd出力へ変換する単純なブロック図。  
- EN/ENO付きブロックはEN入力、ENO出力が追加されている。  
- ST言語の記述例も示されている。

### 図解
- TIME型の値（例：T#1s234ms）がWORD型の数値（例：1234）に変換されるイメージ図があり、TIME→WORD変換の動作を視覚的に説明している。
