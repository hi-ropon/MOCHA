### テキスト
#### 安全データ同一性チェック情報読出し命令（SP.SIDRD）
指定したファイルから安全データ同一性チェック用ファイル識別子を読み出す命令です。  
対応CPUはRnCPU、RnENCPU、RnPSFCPU（安全）、RnSFCPU（一般）などで、ファームウェアバージョン15以降、エンジニアリングツールはバージョン1.05C以降で使用可能です。

#### 実行条件
命令名：SP.SIDRD  
実行条件は特に指定なし。

### 図
- **ラダー図**  
  入力(s)、出力(d1,d2,d3)の構成を示す簡単なラダー回路図。  
- **ST（構造化テキスト）**  
  `ENO:=SP_SIDRD(EN,s,d1,d2,d3);` の形式で命令を呼び出す例。  
- **FBD/LD**  
  ファンクションブロック図で、EN（イネーブル）、s（ファイル名）、d1～d3（出力データ）、ENO（実行結果）を示す。

### 表
#### 設定データ：内容、範囲、データ型

| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s) | ファイル名を格納する先頭デバイス | なし | Unicode文字列 | ANYSTRING_DOUBLE |
| (d1) | 完了ステータスを格納するデバイス<br>0000:正常完了<br>0000以外:異常完了(エラーコード) | なし | ワード | ANY16 |
| (d2) | 読み出したファイル識別子を格納する先頭デバイス | なし | ダブルワード | ANY32 |
| (d3) | 処理完了にてONするビットデバイス<br>（異常完了時は(d3)+1もON） | なし | ビット | ANYBIT_ARRAY (要素数:2) |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |

#### 使用可能デバイス一覧

| オペランド | ビット (X,Y,M,L,SM,F,B,SB,FX,FY) | ワード (T,ST,C,D,W,SD,SW,FD,R,ZR,RD) | ワード (U□,G□,J□,U3E□,H) | ダブルワード (LT,LST,LC,LZ) | 間接指定 | 定数 (K,H,E,$) | その他 (U) |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| (s) | - | ○ | - | - | - | - | ○ |
| (d1) | - | ○ | - | - | - | ○ | - | - |
| (d2) | - | ○ | - | - | - | ○ | - | - |
| (d3) | ○ | - | - | - | - | - | - | - |

※○は使用可能、-は使用不可を示す。

以上の内容により、安全データ同一性チェック用ファイル識別子を指定ファイルから読み出し、完了ステータスや識別子を各デバイスに格納する命令の仕様が詳細に説明されています。
