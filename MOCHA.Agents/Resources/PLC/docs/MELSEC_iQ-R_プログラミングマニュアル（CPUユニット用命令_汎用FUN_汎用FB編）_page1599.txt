<overview>
このページは、PID制御プログラム例の一部であり、主にラダー図（プログラムの制御フローを示す図）とその下に関連する数式および説明文で構成されています。ページ上部から中段にかけてラダー図が配置されており、下部にテキストで数式とその説明が記載されています。表やグラフは含まれていません。
</overview>

<contents>
### ラダー図
- ラダー図は複数のリレー接点やコイル、演算命令が並んでおり、M41、M43、T1、C1、M0、X0A0、X1などの内部リレーやタイマー、カウンタが使用されています。
- 演算命令として、減算（-P）、除算（/P）、セット（SET）、リセット（RST）、MOV（データ移動）、S.PIDCONT（PID制御開始）などが使われています。
- 具体的には、D215、D216、D610、D612などのデバイスに対して演算結果や制御信号を格納・操作しています。
- ラダー図の最後は「{END}」でプログラム終了を示しています。

### テキスト・数式
- 下部には演算式の説明があり、D215に商、D216に余りを格納する処理について記載されています。
- 演算式は以下の通り：
  \[
  \frac{(SV_m) - (SV_a)}{10}
  \]
  ここで、
  - \(SV_m\)：手動モード時のSV（設定値）
  - \(SV_a\)：自動モード時のSV（設定値）
- 各番号（460、469、492、532）はラダー図の行番号に対応し、それぞれの処理内容を説明しています。
  - (460) 商をD215に、余りをD216に格納
  - (469) 自動モードの設定値(SV)に戻すための処理
  - (492) 1回目のみ設定値(SV)から余りを減算
  - (532) 手動から自動への切換え処理終了

### 全体の流れ
- 手動モードと自動モードのSV値の差を計算し、その結果を使ってPID制御の切替や制御値の設定を行うプログラム例であることがわかります。
</contents>
