このページは、BCD8桁データの乗算機能の説明と、その乗算結果の格納方法についてのテキストと図解、さらにエラーコードとその内容を示した表で構成されています。テキスト部分では機能の概要と注意点、例示があり、図はBCD8桁データの乗算イメージを示しています。表はエラーコードとその発生条件をまとめています。
</overview>

<contents>
### テキスト
#### 機能
- (s1)で指定したBCD8桁データと、(s2)で指定したBCD8桁データの乗算を行い、その結果を(d)で指定したデバイスに格納する。
- (d)がビットデバイスの場合、乗算結果は下位8桁（下位32ビット）までが対象であり、上位8桁（上位32ビット）を指定することはできない。

#### 例
- (d)がビットデバイスの場合の乗算結果のビット範囲は以下の通り：
  - K1…下位1桁（b0～b3）
  - K4…下位4桁（b0～b15）
  - K8…下位8桁（b0～b31）

### 図
- BCD8桁データの乗算イメージを示す図
  - (s1)と(s2)のBCD8桁データがそれぞれ9の数字で表されており、(s1)+1、(s2)+1の範囲も示されている。
  - 乗算結果は(d)で指定され、(d)+1、(d)+2、(d)+3の範囲に分割されて格納される様子が描かれている。
  - 乗算結果のビット配置が具体的に示されている。

### 表
#### エラーコードと内容
| エラーコード (SD0) | 内容 |
| :----------------- | :--- |
| 3405H              | (s1)のBCDデータが0～99999999以外のとき。 |
|                    | (s2)のBCDデータが0～99999999以外のとき。 |

- エラーコード3405Hは、(s1)または(s2)のBCDデータが指定範囲外の場合に発生する。
