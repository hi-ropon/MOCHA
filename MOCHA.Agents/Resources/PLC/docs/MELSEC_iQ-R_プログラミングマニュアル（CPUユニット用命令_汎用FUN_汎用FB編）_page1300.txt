<overview>
本ページは、系切替禁止時におけるDCONTSW命令とSP.CONTSW命令の動作例を示したテキスト説明とタイミング図（グラフ）で構成されています。テキスト部分では系切替許可状態になる条件が箇条書きで説明されており、グラフ部分では各系のプログラム実行状態や命令実行タイミング、系切替不可区間が視覚的に示されています。エラーに関する簡単な記述も含まれています。
</overview>

<contents>
### テキスト
#### 例：DCONTSW命令で系切替え禁止時に、SP.CONTSW命令による系切替えを実行した場合
- 下記の場合、系切替え許可状態になります。系切替え禁止に戻したい場合は、再度DCONTSW命令を実行してください。
  - 待機系の電源OFF→ON
  - 待機系CPUユニットのリセット
  - 待機系CPUユニットのRUN→STOP
  - 待機系CPUユニットの停止エラー発生
  - 運転モード切替え（バックアップモード→セパレートモード）
  - 系切替え（システム切替え）発生

#### エラー
- 演算エラーはありません。

### グラフ（タイミング図）
- A系、B系、待機系のプログラム実行状態を示す水平ラインがあり、プログラム実行中やEND状態が矢印や区切り線で表現されている。
- SP.CONTSW命令の命令実行タイミングが示されている。
- DCONTSW命令（系切替え禁止）とECONTSW命令（系切替え許可）の命令実行タイミングが示されている。
- 「系切替え不可」状態が楕円で囲まれ、系切替え不可区間が矢印で示されている。
- 矢印や点線で命令実行の流れや系切替え不可区間の範囲が視覚的に説明されている。

全体として、系切替え禁止状態における命令の実行タイミングと系切替え不可の状態遷移を理解するための説明資料となっている。
