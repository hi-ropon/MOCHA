### テキスト
#### 注意事項
SM705（変換桁数切換）がONのとき、(d)には演算結果を有効桁数分だけ格納する仕様である。そのため、DBINDA(P)(_U)命令を連続して実行し、演算結果を同一デバイスに格納する場合、(d)の一部に前回の演算結果が上書きされずに残る可能性がある。

### 図・表の説明
#### 例1：「-1234567890」をDBINDA(P)命令で文字列に変換
- (s)に「-1234567890」のビット配置を示す。
- (d)+0～(d)+5の各アドレスに16進数と括弧内の10進数で変換結果が格納されている。
- 例として、(d)+0に「31H(1)」「2DH(-)」、(d)+1に「33H(3)」「32H(2)」などが格納されている。

#### 例2：「-987654」をDBINDA(P)命令で文字列に変換
- (s)に「-987654」のビット配置を示す。
- (d)+0～(d)+3に変換結果が格納されているが、(d)+4と(d)+5には前回の変換結果の一部が残っている状態を示す。

#### 問題回避方法
- (d)+0～(d)+5の格納エリアをすべてクリア（00H）してからDBINDA(P)(_U)命令を実行するようプログラムを作成することが推奨される。

#### 例4～6：クリア処理を挟んだ場合の動作例
- (4)「-1234567890」を文字列に変換
- (5) (d)+0～(d)+5をすべて00Hでクリア
- (6)「-987654」を文字列に変換
- これにより、前回の変換結果が残らず正しく格納されることを示している。

### ポイント
- DBINDA(P)命令は有効桁数分のみ格納するため、連続実行時に残存データが発生する可能性がある。
- 連続実行時は格納エリアのクリアが必要。
