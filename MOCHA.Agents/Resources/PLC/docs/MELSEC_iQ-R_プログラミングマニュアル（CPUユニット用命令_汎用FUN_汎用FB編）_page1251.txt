### テキスト
#### 機能
- PIDCONT(P)命令は、サンプリング周期の計測とPID演算を行う。
- PIDCONT(P)命令は、指定したデバイス番号以降に設定されている入出力データエリアの設定値(SV)、測定値(PV)をもとにPID演算を行い、演算結果を自動操作量(MV)エリアに格納する。
- PID演算はサンプリング周期の設定時間経過後の最初のPIDCONT(P)命令実行時に行う。
- PID制御中は必ず制御指令をONにして、PIDCONT(P)命令を毎スキャン実行させる必要がある。複数回実行やスキャン未実行は正常なPID演算を妨げる。
- PIDCONT(P)命令は割込みプログラムで使用不可。割込みプログラムに記述すると正常なPID演算ができなくなる。
- (s)には入出力データエリアの先頭デバイス番号を指定する。
- 入出力データエリアとしてファイルレジスタを指定した場合、メモリプロテクトがかかっていると正常なPID演算ができなくなる。

### 表
#### 入出力データの割り付け
| 項目 | 入出力データ | 内容 |
| :--- | :--- | :--- |
| ライト | (s)+0 | 初回処理フラグ |
| リード/ライト禁止 | (s)+1～(s)+9 | PID制御用ワークエリア（ユーザ使用不可） |
| No.1ループ用入出力データエリア（18ワード） | ライト (s)+10 | 設定値(SV) |
|  | ライト (s)+11 | 測定値(PV) |
|  | リード (s)+12 | 自動操作量(MV) |
|  | リード (s)+13 | フィルタ後の測定値(PVf) |
|  | ライト (s)+14 | 手動操作量(MV_MAN) |
|  | ライト (s)+15 | 手動/自動選択(MAN/AUTO) |
|  | リード/ライト (s)+16 | アラーム(ALARM) |
|  | リード/ライト禁止 (s)+17～(s)+27 | No.1ループ用ワークエリア（ユーザ使用不可） |
| No.nループ用入出力データエリア（18ワード） | ライト (s)+(m+0) | 設定値(SV) |
|  | ライト (s)+(m+1) | 測定値(PV) |
|  | リード (s)+(m+2) | 自動操作量(MV) |
|  | リード (s)+(m+3) | フィルタ後の測定値(PVf) |
|  | ライト (s)+(m+4) | 手動操作量(MV_MAN) |
|  | ライト (s)+(m+5) | 手動/自動選択(MAN/AUTO) |
|  | リード/ライト (s)+(m+6) | アラーム(ALARM) |
|  | リード/ライト禁止 (s)+(m+7)～(s)+(m+17) | No.nループ用ワークエリア（ユーザ使用不可） |

- m = (n-1)×18 + 10
- 入出力データのデバイス点数は「10 + 18×n（使用ループ数）」で計算。
- 各データはBIN値で設定。
- 初回処理フラグはPID演算開始時の処理方法設定。0以外の場合は使用ループ数分のPID演算処理を分割して行う。
