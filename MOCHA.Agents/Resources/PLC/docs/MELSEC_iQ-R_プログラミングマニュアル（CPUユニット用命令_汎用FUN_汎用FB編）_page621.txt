### テキスト
#### データテーブルへのデータライト（FIFW(P)）
BIN16ビットデータを指定されたデータテーブルに格納する命令について説明している。

#### 実行条件
- FIFW命令は立ち上がりエッジで実行
- FIFWP命令は立ち下がりエッジで実行

### 図
- **ラダー図**  
  ラダー回路での命令表現が示されている。  
  (s)と(d)のオペランドを持つ命令ブロックの形状が描かれている。

- **FBD/LD図**  
  FBD/LDでの命令表現。EN（実行条件入力）、ENO（実行結果出力）、s（書き込みデータ）、d（テーブル先頭デバイス）を持つブロック。

### 表
#### 実行条件一覧
| 命令 | 実行条件 |
| :--- | :--- |
| FIFW | 立ち上がりエッジ |
| FIFWP | 立ち下がりエッジ |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s) | テーブルへの書き込みデータまたは、データが格納されているデバイス番号 | なし | 符号付きBIN16ビット | ANY16 |
| (d) | テーブルの先頭デバイス | なし | ワード | ANY16*1 |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要がある。
