### テキスト
#### 使用可能デバイス
比較演算命令で使用可能なデバイスの種類を表形式で示しています。オペランドごとにビット、ワード、ダブルワード、間接指定、定数、その他の対応状況が記載されています。注釈として、FX、FYは使用不可、T、ST、Cは使用不可の制限があります。

#### 機能
下限値(s1)のBIN32ビットデータと上限値(s2)のBIN32ビットデータの範囲に対して、比較データ(s3)のBIN32ビットデータで比較を行い、その結果に応じて(d)、(d)+1、(d)+2のいずれかがONになる動作を説明しています。

#### 注意事項
下限値(s1)は上限値(s2)より小さい値に設定する必要があり、もし下限値(s1)が上限値(s2)より大きい場合は、上限値(s2)は下限値(s1)と同じ値として扱われることが記載されています。

#### エラー
演算エラーは発生しない旨が記載されています。

### 図
#### 比較演算の動作ブロック図
X0入力に対して、DZCP命令が(s1)、(s2)、(s3)、(d)の各オペランドを用いて動作します。
- (d)がONになる条件は、(s1)、(s3)の値の大小関係により決定されます。
- (d)+1がONになる条件は、(s1)、(s2)、(s3)の値の大小関係により決定されます。
- (d)+2がONになる条件は、(s3)、(s2)の値の大小関係により決定されます。

この図は比較演算の結果に応じてどの出力がONになるかを視覚的に示しています。
