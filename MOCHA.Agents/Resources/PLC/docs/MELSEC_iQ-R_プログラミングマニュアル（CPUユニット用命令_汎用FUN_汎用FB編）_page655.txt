### 表
#### CSV形式変換書き込みに関するデータタイプ指定と処理単位の設定範囲

表は「実行/完了タイプ(d1)+0」「データタイプ指定(d1)+7」「処理単位および設定範囲」の3つの大きな項目に分かれています。

- **実行/完了タイプ(d1)+0**  
  ここではCSV形式変換書き込みのデータ形式を16進数で示しており、0100H、0101H、0110H、0111H、0120H、0121H、0130H、0140H、0141Hなど複数のコードが並んでいます。各コードは10進数の符号付き・符号なし、16ビット・32ビットデータ、文字列（アスキーデータ）、浮動小数点数（単精度・倍精度実数）などの種類を示しています。

- **データタイプ指定(d1)+7**  
  0から3までの番号でデータタイプの単位を指定しています。  
  0: ワード  
  1: 偶数バイト  
  2: 実行/完了タイプで指定したデータ型の単位  
  3: 奇数バイト  
  ただし、指定不可の組み合わせもあります。

- **処理単位および設定範囲**  
  - 要求書き込みデータ数(s3)+0: ワード数や要素数の範囲（例：1～65535、1～32767、1～1023など）  
  - ファイル位置(d1)+4, (d1)+5: 先頭または末尾からの位置指定（設定値は00000000H～FFFFFFFFHまたはFFFFFFFFHのいずれか）  
  - 書き込み結果データ数(d1)+2: 書き込み後のデータ数（ワード、バイト、ダブルワードなど）

### テキスト（注釈）
- 設定値は00000000H～FFFFFFFFH（ファイルの先頭から）、またはFFFFFFFFH（ファイルの最後に追加）のいずれかとなる。

### 全体のポイント
- CSV形式の変換書き込みにおいて、データの種類やサイズに応じて細かく処理単位や書き込み範囲を指定できることがわかる。
- 16進数コードでデータタイプを指定し、それに対応した処理単位（ワード、バイト、ダブルワードなど）を選択可能。
- ファイル内の位置指定は先頭または末尾からのオフセットで行う。
