### テキスト
#### SM705（変換桁数切換）がOFFの場合の動作
(d)には16進アスキーデータが固定桁数（4桁）で格納されます。  
(d)+2にはSM701（出力文字数切換）がOFFのときは00Hが格納され、ONのときは変化しません。  
(d)に格納される演算結果は4桁の16進数として処理され、有効桁数の左側の0は「0」として処理（0パディング）されます。

##### 例
(s)に02A6Hを指定した場合、(d)～(d)+2にそれぞれ32H(2)、36H(6)、00Hが格納されます。

#### SM705（変換桁数切換）がONの場合の動作
(d)には有効桁数の左側の0を省略した桁数（最大4桁）の16進アスキーデータが前詰めで格納されます。  
(s)の値と(d)に格納される値の例が表で示されています。

#### ASCIIコードについて
- 桁数が最大桁数（4桁）未満の場合、SM701のON/OFFに関わらず文字列の終端に00Hを格納します。  
- 文字列の終端が下位バイトの場合、上位バイトにも00Hを格納します。  
- 桁数が最大桁数（4桁）の場合、SM701がOFFのときは(d)+2に00Hを格納し、ONのときは変化しません。

#### エラー
演算エラーはありません。
