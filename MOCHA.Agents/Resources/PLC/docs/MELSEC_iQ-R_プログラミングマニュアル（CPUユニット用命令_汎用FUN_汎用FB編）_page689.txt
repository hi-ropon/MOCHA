### テキスト
#### 使用可能デバイス
オペランドとして使用可能なビット、ワード、ダブルワード、間接指定、定数、その他のデバイスが一覧表で示されています。各オペランドごとに対応するデバイス種類が○で示されています。

#### 機能
(s3)で指定した入力値（BIN16ビット値）を、(s1)、(s2)で指定した上下限リミット値の範囲内に制御し、(d)で指定したデバイスに格納する機能について説明しています。出力値は以下の条件により制御されます。

| 条件 | 出力値に格納される値 |
| --- | --- |
| 下限リミット値(s1) > 入力値(s3) のとき | 下限リミット値(s1) |
| 上限リミット値(s2) < 入力値(s3) のとき | 上限リミット値(s2) |
| 下限リミット値(s1) ≤ 入力値(s3) ≤ 上限リミット値(s2) のとき | 入力値(s3) |

- 上限リミット値のみで制御する場合は、(s1)に設定データ範囲の最小値を設定
- 下限リミット値のみで制御する場合は、(s2)に設定データ範囲の最大値を設定

#### エラー
| エラーコード (SD0) | 内容 |
| --- | --- |
| 3405H | (s1)で指定した下限リミット値が、(s2)で指定した上限リミット値よりも大きいとき |

### 図
入力値(s3)と出力値(d)の関係を示すグラフがあり、x軸に入力値(s3)、y軸に出力値(d)をとっています。  
- 入力値が下限リミット値(s1)より小さい場合、出力値は下限リミット値(s1)に固定される  
- 入力値が上限リミット値(s2)より大きい場合、出力値は上限リミット値(s2)に固定される  
- 入力値が下限リミット値(s1)と上限リミット値(s2)の間にある場合、出力値は入力値と同じになる
