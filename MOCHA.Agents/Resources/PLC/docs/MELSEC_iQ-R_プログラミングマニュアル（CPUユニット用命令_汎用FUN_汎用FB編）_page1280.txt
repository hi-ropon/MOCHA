### テキスト
#### ブロック起動命令「SET [BL□]」
- 対応CPUはR00CPU、R01CPU、R02CPUでバージョン制約なし。
- RnCPU（R00CPU、R01CPU、R02CPU除く）、RnENCPUはファームウェアバージョン12以降で使用可能。エンジニアリングツールはバージョン1.015R以降対応。
- プロセスCPU（プロセスモード）はファームウェアバージョン03以降、エンジニアリングツールはバージョン1.020W以降対応。
- プロセスCPU（二重化モード）はファームウェアバージョン18以降、エンジニアリングツールはバージョン1.050C以降対応。
- 指定したブロックを単独で活性化し、イニシャルステップから実行する。

#### 機能
- 指定ブロックを単独で活性化し、イニシャルステップから実行。イニシャルステップが複数ある場合は全て活性化。
- SFC用情報デバイスのブロック起動終了ビットを設定している場合、そのビットはOFFからONに変わる。
- 命令実行時に指定ブロックがすでに活性状態なら命令は無処理（NOP相当）。
- SFCブロックRUN中書込み中の場合も命令は無処理（NOP相当）。
- プロセスCPU（二重化モード）ではバックアップモードの待機系で実行しても無処理。

### 図
- ラダー図：指定ブロック(d)のビットデバイスをONにする接点を示す。
- FBD/LD図：EN（実行条件）入力に対し、ENO（実行結果）出力とd（指定ブロック）を制御するブロック起動命令の構造を示す。
