### 表
#### 命令処理時間一覧表
- 表は「命令名」「条件」「処理時間（μs）」の3つの大項目で構成されています。
- 処理時間はCPUユニット別に「R00CPU、R01CPU」「R02CPU」「左記以外のCPUユニット」に分かれ、それぞれ最小値と最大値が記載されています。

##### 主な内容
- **命令名**: DBIMPORT、DBEXPORT、DBTRANS、DBCOMMIT、DBROLBAK、RSET、QDRSET、ZRDRB、ZRWRB、ADRSET、DATERD、DATEWR、DATE+、DATE-、TIME2SEC、SEC2TIME、DATE2SEC、DATE2SEC_U、SEC2DATE、SEC2DATE_U、LDDT=、LDDT<>、LDDT> など。
- **条件**: ファームウェアバージョンや桁上げの有無、指定日付との比較条件（導通時、非導通時）、日付の具体的な値などが記載されています。
- **処理時間**:
  - DBIMPORT～DBROLBAKの命令はR00CPU、R01CPU、R02CPUの処理時間は記載なしで、左記以外のCPUユニットの処理時間のみ示されています。
  - RSETやQDRSETなどの命令は各CPUユニットで処理時間の最小・最大値が詳細に記載されています。
  - LDDT系命令は指定日付と現在日付の比較、導通時・非導通時の条件別に処理時間が異なり、詳細に数値が示されています。

##### 処理時間の例
- DBIMPORT（ファームウェアバージョン28以前のシーケンサCPU実行時）: 左記以外のCPUユニットで最小16.200μs、最大19.200μs。
- RSET命令: R00CPU、R01CPUで最小3.300μs、最大7.800μs、R02CPUで最小2.300μs、最大6.800μs、左記以外のCPUユニットで最小1.800μs、最大6.300μs。
- DATE+命令（桁上げなし）: R00CPU、R01CPUで最小4.800μs、最大8.300μs、R02CPUで最小3.800μs、最大7.300μs、左記以外のCPUユニットで最小3.300μs、最大6.800μs。

以上のように、命令ごとにCPUユニット別・条件別の処理時間が詳細にまとめられており、システムの性能評価や最適化に利用される資料となっています。
