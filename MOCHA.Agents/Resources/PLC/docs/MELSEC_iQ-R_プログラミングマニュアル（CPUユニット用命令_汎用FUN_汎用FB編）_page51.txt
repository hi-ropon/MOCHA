### テキスト
#### 1.5 プログラミング時の注意事項
##### 命令共通のエラー
命令実行時にエラーとなる条件を示しています。主なエラー内容は以下の通りです。

- 範囲外の入出力番号（0H～FFH、3E0H～3E3H以外）を指定した場合（エラーコード：2800H）
- 存在しないユニットの入出力番号を指定した場合（2801H）
- 命令で指定できないユニットの入出力番号を指定した場合（2803H）
- 範囲外のネットワーク番号（1～239以外）を指定した場合（2804H）
- 存在しないネットワーク番号を指定した場合（2805H）
- その他、指定デバイスやラベルが使用可能範囲を超えた場合やファイルレジスタ設定がされていない状態でアクセスした場合（2820H）
- 指定ユニットのバッファメモリ範囲を超えた場合やバッファメモリを持たないユニットを指定した場合（2823H）

※接点命令ではエラーは検出されず、演算結果が非導通状態となることに注意。

##### 命令実行時のデバイス、ラベルの範囲チェック
デバイスやラベルを命令で指定する際、範囲チェックは行われないため、演算結果が指定範囲内に収まるようプログラムを作成する必要があります。範囲を超えた指定をしてもエラーは発生しませんが、他のデバイスやラベルにデータが書き込まれます。ただし、領域外の場合はエラーコード2820Hが発生します。デバイスに割り付けたラベルを命令で指定した場合も同様です。

### 図
#### デバイス/ラベルメモリの割付イメージ
- メモリはD0から始まり、D1022、D1023の後にW0が続きます。
- D1023の次の転送先はD1023～D1032の範囲ですが、D1024～D1032が存在しなくてもデータは書き込まれ、W0～W0008のデータは上書きされます。
