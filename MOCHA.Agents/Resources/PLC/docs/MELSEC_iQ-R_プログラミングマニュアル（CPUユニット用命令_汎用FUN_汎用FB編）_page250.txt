### テキスト
#### BIN16ビット除算命令の概要
指定した2つのBIN16ビットデータを除算する命令です。対応CPUはRnCPU、RnENCPU（プロセス、三重化、一般、安全）です。バージョン1.035M以降のエンジニアリングツールで対応しています。

#### 実行条件
命令には「/」「/_U」「/P」「/P_U」の4種類があり、それぞれ実行条件が異なります。  
- 「/」と「/_U」は立ち上がりパルスで実行  
- 「/P」と「/P_U」は立ち下がりパルスで実行

### 図
#### ラダー図
- ラダー図では、3つの入力(s1, s2, d)があり、s1とs2が除算対象のBIN16ビットデータ、dが演算結果を格納する先頭デバイスです。

#### FBD/LD図
- FBD/LD図では、EN（実行条件）、s1、s2（除算対象データ）、d（結果格納先）、ENO（実行結果）という入出力ポートが示されています。  
- 図中の命令はDIVISION、DIVISIONP、DIVISION_U、DIVISIONP_Uのいずれかが入ります。

### 表
#### 実行条件一覧

| 命令   | 実行条件               |
|--------|------------------------|
| /      | 立ち上がりパルス       |
| /_U    | 立ち上がりパルス       |
| /P     | 立ち下がりパルス       |
| /P_U   | 立ち下がりパルス       |

#### 設定データ（内容、範囲、データ型）

| オペランド | 内容                                         | 範囲           | データ型           | データ型(ラベル)          |
|------------|----------------------------------------------|----------------|--------------------|---------------------------|
| (s1)       | 除算されるデータまたは除算されるデータが格納されているデバイス | -32768～32767 (符号付き) / 0～65535 (符号なし) | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S / ANY16_U          |
| (s2)       | 除算データまたは除算データが格納されているデバイス | -32768～32767 (符号付き) / 0～65535 (符号なし) | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S / ANY16_U          |
| (d)        | 演算結果を格納する先頭デバイス                 | —              | 符号付きBIN32ビット / 符号なしBIN32ビット | ANY16_S_ARRAY(要素数2) / ANY16_U_ARRAY(要素数2) |
| EN         | 実行条件                                     | —              | ビット               | BOOL                      |
| ENO        | 実行結果                                     | —              | ビット               | BOOL                      |

#### 備考
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、上記のデータ型の安全デバイスおよび安全ラベルのみ使用可能です。
