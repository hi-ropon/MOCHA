### テキスト
#### 34.5 剰余（MOD(_E)）
入力値の余り((s1)÷(s2))を出力する関数です。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化、一般）、RnPSFCPU（安全、一般）などです。

##### 機能
- (s1)、(s2)に入力されたINT型/DINT型/WORD型/DWORD型データの除算((s1)÷(s2))を行い、演算結果の余りを(d)から(s)と同一のデータ型で出力します。
- (s1)、(s2)への入力値はINT型/DINT型/WORD型/DWORD型のデータ値で、(s2)へ入力される値は0以外である必要があります。

##### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |
| :--- | :--- |
| EN | ENO | (d) |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、その場合は(d)から出力されたデータを使用しないようプログラムで処理してください。

### 図
- ラダー回路（FBD/LD）  
　EN/ENOなしとEN/ENO付きの2種類の回路図が示されています。  
　EN/ENOなしは単純にs1、s2を入力しdを出力。  
　EN/ENO付きはEN入力、ENO出力が追加され、s1、s2からdを出力。

- ST言語記述例  
　EN/ENOなし：演算子として記述可能。  
　EN/ENO付き：d:=MOD_E(EN,ENO,s1,s2);

### 表
#### 設定データ（引数、内容、種別、データ型）

| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s1(IN1) | 被除数 | 入力変数 | ANY_INT |
| s2(IN2) | 除数 | 入力変数 | ANY_INT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY_INT |

### 例示
- データ型がINT型の場合の例  
　5 ÷ 2 の商は2（出力しない）、余りは1（INT型で出力）という具体例が図示されています。
