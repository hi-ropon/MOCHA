本ページは主にテキストと図（ブロック図）で構成されており、ユニットアクセス命令のうち「FROMD(P)」および「DFROMD(P)」について説明しています。テキスト部分では命令の動作概要や条件、注意点が記載されており、図部分ではユニットバッファメモリからCPUユニットやデバイス/ラベルメモリへのデータ転送のイメージが示されています。表やグラフは含まれていません。
</overview>

<contents>
### テキスト
#### FROMD(P)
- (U/H)で指定したユニット、または他号機CPUユニット内のバッファメモリの(s)で指定したアドレスから、(n)ワード分のデータを読み出し、(d)で指定したデバイス以降に格納します。
- ユニットからのワードデータリード
- 他号機CPUユニットからのワードデータリードも可能。
- 読出しデータ(n)が0のときは無処理となる。
- 命令実行時に対象ユニットの異常や処理中などでアクセスできない場合は無処理となる。

#### DFROMD(P)
- (U/H)で指定したユニット、または他号機CPUユニット内のバッファメモリの(s)で指定したアドレスから、(n)×2ワード分のデータを読み出し、(d)で指定したデバイス以降に格納します。
- ユニットからのダブルワードデータリード
- 他号機CPUユニットからのダブルワードデータリードも可能。
- 読出しデータ(n)が0のときは無処理となる。
- 命令実行時に対象ユニットの異常や処理中などでアクセスできない場合は無処理となる。

### 図
#### FROMD(P)の図解
- ユニットバッファメモリの(s)アドレスから(n)ワード分のデータを読み出し、CPUユニットの(d)アドレス以降の(n)点に格納する流れを示す。
- 他号機CPUユニットのバッファメモリからも同様に(n)ワード分を読み出し、デバイス/ラベルメモリの(d)アドレス以降に格納する流れも示されている。

#### DFROMD(P)の図解
- ユニットバッファメモリの(s)アドレスから(n)×2ワード分のデータを読み出し、CPUユニットの(d)アドレス以降の(n)×2点に格納する流れを示す。
- 他号機CPUユニットのバッファメモリからも同様に(n)×2ワード分を読み出し、デバイス/ラベルメモリの(d)アドレス以降に格納する流れも示されている。
