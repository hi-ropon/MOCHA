### テキスト
#### BIN32ビットデータ合計値算出
命令名は「DWSUM(P)(_U)」で、指定したデバイスからn点のBIN32ビットデータをすべて加算します。

#### 実行条件
- DWSUM、DWSUM_U命令は立ち上がりエッジで実行されます。
- DWSUMP、DWSUMP_U命令は立ち下がりエッジで実行されます。

### 図
- ラダー図：3つのオペランド(s)(d)(n)を持つ命令のラダー表現が示されています。
- FBD/LD図：EN（入力）、ENO（出力）、s、d、nの各端子を持つ命令ブロックの図が描かれています。

| DWSUM_U    | 立ち上がりエッジ |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容                                   | 範囲       | データ型           | データ型(ラベル)    |
|------------|--------------------------------------|------------|--------------------|---------------------|
| (s)        | 合計値を算出するデータが格納されている先頭デバイス | —          | 符号付きBIN32ビット | ANY32_S*1            |
|            |                                      |            | 符号なしBIN32ビット | ANY32_U*1            |
| (d)        | 合計値を格納する先頭デバイス               | —          | 符号付きBIN64ビット | ANY32_ARRAY (要素数:2)|
|            |                                      |            | 符号なしBIN64ビット |                      |
| (n)        | データ個数                             | 0〜65535   | 符号なしBIN16ビット | ANY16                |
| EN         | 実行条件                             | —          | ビット              | BOOL                 |
| ENO        | 実行結果                             | —          | ビット              | BOOL                 |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定してください。

---

以上より、本命令は指定した先頭デバイスからn点の32ビットデータを合計し、64ビットの結果を別の先頭デバイスに格納する命令であることがわかります。実行条件や使用可能なデバイス種別も詳細に示されています。
