### テキスト
#### エラー
演算エラーはありません。

#### 注意事項
SM705（変換桁数切換）がONのとき、(d)には演算結果を有効桁数分だけ格納します。このため、DBINHA(P)命令を連続して実行し、演算結果を同一デバイスに格納する場合、(d)の一部に前回の演算結果が上書きされずに残ってしまう可能性があります。

#### 具体例
(s)が「13AC625E」のときにDBINHA(P)命令を実行し、さらに(s)が「F9E8D」のときにDBINHA(P)命令を実行する場合の動作例が示されています。

1. 「13AC625E」を文字列に変換
2. 「F9E8D」を文字列に変換
3. (d)+3に前回の変換結果の一部が残ったままになる

これを回避するには、格納エリア(d)+0～(d)+3をすべてクリアしてからDBINHA(P)命令を実行するようにプログラムを作成する必要があります。

### 図（表形式の説明）
1. (s)に「13AC625E」があり、b31～b0のビット列が示されている。
2. (d)+0～(d)+4の各アドレスに16進数と対応する文字コード（括弧内に文字）が格納されている。
   - 例：(d)+0に33H(3)、31H(1)
3. (s)に「F9E8D」があり、同様にビット列と(d)+0～(d)+4の格納内容が示されている。
4. (d)+3に前回の変換結果「45H(E)」「35H(5)」が残っていることが強調されている。
5. クリア処理後の(d)+0～(d)+3はすべて00Hで初期化されている。
6. クリア後に「F9E8D」を文字列に変換した結果が示されており、(d)+3の部分も00Hで上書きされている。

### 手順まとめ
(4) 「13AC625E」を文字列に変換  
(5) (d)+0～(d)+3をクリアする  
(6) 「F9E8D」を文字列に変換する

以上により、前回の変換結果が残る問題を防止できることが説明されています。
