### テキスト
#### ROL(P)命令の機能
- 指定されたデバイスのBIN16ビットデータを、SM700（キャリフラグ）を含めずに(n)ビット左へ回転させる命令です。
- SM700（キャリフラグ）は、ROL(P)命令実行前の状態によりONまたはOFFになります。
- (d)にビットデバイスが指定された場合は、桁指定で指定されたデバイス範囲内で回転します。
- 実際に回転するビット数は、(n)を桁指定で指定された点数で割った余りとなります。
  - 例：n=15、点数=12ビットの場合、15÷12=1余り3 → 3ビット左へ回転
- (n)は0～15の値を指定可能で、16以上の値が指定された場合は(n)÷16の余りの値で回転します。
  - 例：n=18のとき、18÷16=1余り2 → 2ビット左へ回転

### 図
- 16ビットのビット列（b15～b0）が左へ1ビットずつ回転していく様子を示しています。
- 各ビットの値が左へ1つずつずれていき、最上位ビットb15の値が最下位ビットb0に回り込む動きを視覚的に表現。
- (n)ビット左ローテーションのイメージとして、ビット列が左へn回転し、回転した分のビットが末尾に回り込む様子を示しています。
- SM700（キャリフラグ）部分は回転処理に含まれません。
