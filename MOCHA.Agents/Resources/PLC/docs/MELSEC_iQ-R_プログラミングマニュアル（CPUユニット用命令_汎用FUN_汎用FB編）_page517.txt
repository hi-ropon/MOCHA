### テキスト
#### 機能
- (s)で指定されたデバイス番号以降に格納されている、(n)ワード分のBIN16ビットデータの下位8ビットをワード単位に結合し、(d)で指定されたデバイス番号以降に格納する。
- (s)で指定されたデバイス番号以降に格納されている(n)ワードのデータの上位8ビットは無視される。なお、(n)が奇数の場合は、(n)バイト目のデータを格納したデバイスの上位8ビットに0を格納する。

#### 注意点
- (n)でバイト数を設定することで、(s)で指定するバイトデータの範囲と(d)で指定する結合データを格納するデバイスの範囲が自動的に決まる。
- (n)が0の場合は処理を行わない。
- (s)で指定されたバイトデータ格納デバイスの上位8ビットは無視され、下位8ビットのみが対象となる。
- 結合するデータが格納されているデバイス範囲と、結合されたデータを格納するデバイス範囲が重複していても正常に処理される。

#### エラー
- 演算エラーは発生しない。

### 図
1. **データ結合のイメージ図**
   - 左側に(s)で指定されたデバイス範囲のビット配置が示されている。
   - 各ワードの下位8ビット（b7～b0）が抽出され、(d)で指定されたデバイス範囲に連結して格納される。
   - (n)が奇数の場合、最後のバイトの上位8ビットは0で埋められる。
2. **例：(n)=5の場合のデータ結合**
   - (s)の5ワード分の下位8ビットデータ（00H、34H、56H、78H、FEH）が(d)の3ワード分に結合されて格納される様子を示す。
3. **例：D11～D16の下位8ビットをD12～D14に格納する場合**
   - D11～D16の下位8ビットの値（31H～36H）がD12～D14の上位8ビットと組み合わされて格納される様子を示す。

| 結合するデータが格納されているデバイス範囲 | 結合されたデータを格納するデバイス範囲 |
| :------------------------------------------- | :--------------------------------------- |
| (s)～(s)+(n)-1                               | (d)～(d)+((n/2)-1)                       |

- この表は、結合元のデバイス範囲と結合先のデバイス範囲の対応関係を示している。
