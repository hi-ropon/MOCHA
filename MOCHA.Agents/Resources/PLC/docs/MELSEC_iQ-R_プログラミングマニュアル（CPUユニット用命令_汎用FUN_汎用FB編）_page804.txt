### テキスト
#### 変換後の文字列データの格納方法
- 変換後の文字列データは、(d)以降のデバイス番号に格納されます。
- 符号は、BIN32ビットデータが正のときは20H（スペース）、負のときは2DH（-）を格納します。
- 小数部桁数を0以外に設定した場合、指定された桁数＋1桁目に自動的に2EH（.）を格納します。小数部桁数が0のときは2EHは格納されません。

##### 具体例
- 小数部桁数の値がBIN32ビットデータの桁数より大きい場合は、自動的に0を付加し、右詰めで「0.□□□□□」と変換します。
- 全桁数の値から符号、小数点分を除いた桁数がBIN32ビットデータの桁数より大きい場合、符号と数値の間に20H（スペース）を格納します。BIN32ビットデータの桁数の方が大きい場合はエラーとなります。
- 変換した文字列の最後には自動的に00Hを格納します。
