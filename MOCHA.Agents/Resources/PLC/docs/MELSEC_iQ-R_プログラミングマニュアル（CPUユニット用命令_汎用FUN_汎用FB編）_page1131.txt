### テキスト
#### 使用可能デバイス
オペランドごとに使用可能なデバイスの種類が表形式で示されています。ビット、ワード、ダブルワード、間接指定、定数、その他のカテゴリに分かれており、各オペランド（U/H、s1、s2、n）に対して使用可否が○や－で表記されています。なお、s2は一部の命令（DTOD(P)命令）のみ使用可能です。

#### 機能
(U/H)はユニットおよびCPUユニットの先頭入出力番号を16進数4桁で表したときの上3桁で指定することが説明されています。

(1)の図は、CPUユニットの先頭入出力番号の位置を示しており、書き込むユニットの先頭入出力番号はK4またはH4で指定することを示しています。

#### CPUユニットの先頭入出力番号
CPUユニットごとに先頭入出力番号が表でまとめられています。

| CPUユニット | 先頭入出力番号 |
| ----------- | -------------- |
| 1号機       | 3E0H           |
| 2号機       | 3E1H           |
| 3号機       | 3E2H           |
| 4号機       | 3E3H           |

### 図
CPUユニットの先頭入出力番号の位置を示すブロック図があり、PW、CPU、RX10などのユニットが並んでいます。RJ71EN71ユニットの位置が強調されており、その下に16進数アドレス（0000H、0010H、0020H、0030H、0040H、0060H）が示されています。先頭入出力番号は0040Hの位置に対応しています。
