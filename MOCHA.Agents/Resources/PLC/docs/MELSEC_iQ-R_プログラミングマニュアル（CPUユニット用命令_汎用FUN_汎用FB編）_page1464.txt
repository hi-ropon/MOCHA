### テキスト
#### 38 比較ファンクション
38.1 比較

比較関数GT(_E)、GE(_E)、EQ(_E)、LE(_E)、LT(_E)は、入力値のデータ比較結果を出力します。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、三重化）、RnPSFCPU（一般、安全）、RnSFCPU（一般）、RnSFCU（安全）です。

### 図
- ラダー、FBD/LDでの比較関数の使用例が2種類示されています。
  - EN/ENOなしの場合：s1、s2の2つの入力からdに比較結果を出力するシンプルな回路図。
  - EN/ENO付きの場合：EN入力が追加され、ENが有効な時に比較を行い、ENO出力がある回路図。
- ST言語での使用例も示されており、EN/ENOなしとEN/ENO付きの両方の書き方が記載されています。
  - 例：d:=GT(s1,s2); d:=GE(s1,s2); d:=EQ(s1,s2); d:=LE(s1,s2); d:=LT(s1,s2);
  - EN/ENO付きの場合は、d:=GT_E(EN,ENO,s1,s2);など。

### 表
#### 設定データ：内容、種別、データ型

| 引数          | 内容                          | 種別       | データ型          |
|---------------|-------------------------------|------------|-------------------|
| EN            | 実行条件(TRUE:実行、FALSE:停止) | 入力変数   | BOOL              |
| s1(IN1)～s28(IN28) | 入力                       | 入力変数   | ANY_ELEMENTARY*2   |
| ENO           | 出力状態(TRUE:正常、FALSE:異常) | 出力変数   | BOOL              |
| d             | 出力(TRUE:真値、FALSE:偽値)     | 出力変数   | BOOL              |

- 注意事項：
  - DXは使用できません。
  - 安全プログラムではREAL型/LREAL型/STRING型データを指定しないこと。指定するとエラーになります。
  - 入力変数s1は2～28の範囲で変更可能。
  - バージョン1.035M以降のエンジニアリングツールで対応。以前は演算子として記述。
