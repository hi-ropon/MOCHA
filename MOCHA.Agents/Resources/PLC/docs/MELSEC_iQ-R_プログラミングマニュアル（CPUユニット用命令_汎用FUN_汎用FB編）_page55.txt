### テキスト
#### ロングタイマデバイスの現在値を一括転送する場合
ロングタイマデバイスの現在値を一括転送する際の方法について説明しています。

- BMOVP命令を使用する場合は、現在値、接点、コイルを一括で転送します。
- DMOV命令を使用する場合は、現在値のみを一括転送します。

### 図
#### BMOVP命令を使用した一括転送のイメージ
- ラダー図にて、M0接点がONのときにBMOVP命令が実行されます。
- 転送元はLT0～LT3の各ロングタイマデバイスで、それぞれ「現在値」「FRT前回値」「接点」「コイル」の情報を含みます。
- 転送先はD0～D15のデバイス領域で、16個のデバイスに一括転送される構成です。
- 図中では、LT0～LT3の「接点」「コイル」部分がグレーで強調されています。

#### DMOV命令を使用した一括転送のイメージ
- ラダー図にて、SM402接点がONのときにFORループ（K4回）でDMOV命令が実行されます。
- 転送元はLT0～LT3の各ロングタイマデバイスの「現在値」のみで、「接点」「コイル」は転送対象外です。
- 転送先はD0～D7のデバイス領域で、現在値のみが一括転送される構成です。
- 図中では「現在値」のみがグレーで強調され、接点・コイルは白抜きで示されています。
- 転送先のDデバイスは、LT0～LT3の現在値に対応してD0、D2、D4、D6に割り当てられています。

以上により、BMOVP命令は現在値に加え接点・コイルも含めて一括転送したい場合に使用し、DMOV命令は現在値のみを効率的に転送したい場合に使用することが示されています。
