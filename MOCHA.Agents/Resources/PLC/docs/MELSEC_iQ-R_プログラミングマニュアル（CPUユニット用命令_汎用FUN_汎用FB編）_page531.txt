### テキスト
#### 安全デバイスおよび定数の使用制限
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、指定された安全デバイスおよび定数のみ使用可能。

#### 機能
- (s)で指定されたデバイスから(n)点分のBIN16ビットデータを、(d)で指定されたデバイスへ一括転送する。
- 転送元と転送先のデバイスが重複していても転送可能。
- デバイス番号の小さい方から大きい方へ転送する場合は(s)から、逆の場合は(s)+(n)-1から転送。
- (s)がワードデバイスで(d)がビットデバイスの場合、ワードデバイスはビットデバイスの桁指定で指定されたビット数が対象。
- (s)が桁指定ビットデバイスで(d)がワードデバイスの場合、ワードデバイスはビットデバイスの桁指定で指定されたビット数が対象。

#### 注意事項
- (s)、(d)が両方ビットデバイスの場合は、必ず(s)、(d)の桁数を同一にすること。
- リンクダイレクトデバイス、ユニットアクセスデバイス、CPUバッファメモリアクセスデバイスを使用する場合は、(s)か(d)の片方のみ指定。ただし、I/O No.指定にインデックス修飾を指定していない自号機CPUユニットのCPUバッファメモリアクセスデバイス(U3E0￥G□)は、(s)、(d)両方指定可能。

### 図
#### データ転送の動作イメージ
1. (s)から(n)点分のBIN16ビットデータを(d)へ転送する例
   - (s)の各アドレスに格納された16進数データ（例：1234H、5678H、7FF0Hなど）を、(d)の対応するアドレスに転送。
   - 転送範囲はb15からb0までの16ビット。

2. (d)にK1Y30が指定された場合の例
   - (s)で指定されたワードデバイスの下位4ビットが対象。
   - 転送先(d)のY30からY3Bまでのビットに対応して転送。

3. (s)にK1X20が指定された場合の例
   - (s)の桁指定ビットデバイスの各ビットが(d)のワードデバイスの下位4ビットに転送される。
   - 転送はビット単位で行われ、(1)の範囲に格納される。
