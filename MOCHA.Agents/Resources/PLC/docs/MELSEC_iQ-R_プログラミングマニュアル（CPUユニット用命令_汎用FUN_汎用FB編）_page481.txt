### テキスト
#### SM705（変換桁数切換）がONの場合の(s)の設定方法
(s)には任意の桁数（00H（NULLコード）まで）の16進アスキーデータを設定します。ただし、最大桁数（8桁）の場合は00H（NULLコード）の設定は不要です。
(s)の設定方法を以下に示します。

- ASCIIコードは、00H〜FFHの範囲で設定可能。
- (s)〜(s)+3で指定するアスキーデータは、00000000H〜FFFFFFFFHの範囲内。
- (s)+4以降のデータは無視される。
- 各位に設定するASCIIコードは、30H〜39Hおよび41H〜46Hの範囲内。
- 各位に設定するASCIIコードが00Hの場合は、16進アスキーデータの終端として処理される。

### 表・図
#### (s)に設定する値と(s)〜(s)+3の内容

| (s)に設定する値 | (s)〜(s)+3の内容 |
|-----------------|------------------|
| 00H〜FFH        | (s)の上位バイトに00Hを設定し、(s)+1以降は無視される。<br>図では、(s)に00H、(s)+1にASCII 1が入るが、(s)+1以降は無視。 |
| 10H〜FFH        | (s)+1の下位バイトに00Hを設定し、(s)+1の上位バイト以降は無視される。<br>図では、(s)にASCII 1、(s)+1に00Hが入る。 |
| 1000000H〜FFFFF | (s)+3の上位バイトに00Hを設定し、(s)+4以降は無視。<br>図では、(s)〜(s)+3にASCII 1〜ASCII 7が順に割り当てられている。 |
| 10000000H〜FFFFF | (s)+4以降は無視される。<br>図では、(s)〜(s)+3にASCII 1〜ASCII 8が割り当てられている。 |

### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3401H              | (s)〜(s)+3に変換できない不正なデータを入力したとき。<br>・各桁のASCIIコードが30H〜39H、41H〜46H以外の場合。 |
