### テキスト
#### ファイルレジスタの範囲チェック
ファイルレジスタを命令で指定する際は、演算結果が指定したファイルレジスタの範囲内に収まるようプログラムを作成する必要があります。範囲を超えた指定をするとエラー（エラーコード：2820H）が発生します。

- ファイルレジスタ(ZR)で範囲を超えた指定をした場合、エラーが発生する。
- ファイルレジスタ(R)で使用するブロックNo.の範囲を超えた指定をした場合もエラーが発生する。

### 図・説明
#### 例1: ファイルレジスタ(ZR)を指定した場合
- 命令例：
  - BMOV D0 ZR0 K2
  - BMOV D0 ZR65500 K50
- デバイス/ラベルメモリの割付イメージ：
  - ZR0～ZR65535はファイルレジスタファイルMAIN1の範囲
  - ZR0～ZR50000はファイルレジスタファイルMAIN2の範囲
- 注釈：
  1. 転送先がファイルレジスタMAIN1の範囲内（ZR0～ZR1）ならデータが書き込まれる。
  2. 転送先がファイルレジスタMAIN1の範囲外の場合、エラーとなる。

#### 例2: ファイルレジスタ(R)を指定した場合
- 命令例：
  - RSET K0
  - BMOV D0 R0 K2
  - BMOV D0 R32760 K50
- デバイス/ラベルメモリの割付イメージ：
  - R0～R32767はブロックNo.0の範囲
  - R0～R32767はブロックNo.1の範囲
- 注釈：
  1. 転送先がブロックNo.0のRデバイス範囲内（R0～R1）ならデータが書き込まれる。
  2. 転送先がブロックNo.0のRデバイス範囲外の場合、エラーとなる。

以上より、ファイルレジスタの指定は必ず割付範囲内で行う必要があり、範囲外指定はエラーを引き起こすため注意が必要であることが示されています。
