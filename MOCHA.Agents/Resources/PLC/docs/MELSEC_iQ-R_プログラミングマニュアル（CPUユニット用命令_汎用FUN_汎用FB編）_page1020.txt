### テキスト
#### 機能
- 実行指令がONのとき、指定した値(s1)から別の値(s2)まで、指定回数(n)で移行する処理を実行する。
- (n)はスキャン数（移行回数）を指定し、0<(n)<32768以外は無処理。
- (d1)+1はRAMPQ命令の実行回数が格納される。
- 1回（1スキャン）の変化値は (s2 - s1) / n で計算される。
- 例として0から350までを7スキャンで変化させる場合の数直線図を示す。
- 割り切れない場合は補正し、直線傾斜にならないことがある。
- RAMPQ命令使用時の動作詳細を記述。

#### 例
- X0がONしたとき、D0の内容を10から100まで6スキャンで変化させ、移行完了後はD0の内容を保持するプログラム例。
- ラダー図とタイミングチャートで動作を示す。
- スキャンごとのD0の値変化、完了デバイスのON/OFF動作、RAMPQ命令の再移行開始条件などを説明。
- (d1)+0の内容は指令OFF後は変化せず、再度ONで初期値から再移行開始。
- (s1)と(s2)の値はスキャンごとに変更しないことを推奨。変更すると急激な変化が起こる可能性あり。
- (d1)をビットデバイスで桁指定する場合はK4形式で指定。

#### 注意事項
- (d1)をビットデバイスで桁指定する場合は、桁指定をK4にしたときのみ使用可能。

### 図
- 数直線図：0から350まで7スキャンで変化させる例。各スキャンごとの値の増加と格納される値の関係を矢印と数値で示す。
- ラダー図：X0のONでRAMPQ命令が動作し、D0の値が変化する回路構成を示す。
- タイミングチャート：X0、D0、D1、M0、M1のON/OFF状態の変化をスキャンごとに示し、値の変化過程を視覚的に表現。
