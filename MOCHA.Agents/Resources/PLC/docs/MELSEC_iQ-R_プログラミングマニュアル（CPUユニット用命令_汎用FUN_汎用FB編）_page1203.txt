本ページは、FTPサーバ関連の設定項目について記載された表形式の説明資料です。ページ全体は表で構成されており、各行は「デバイス」「項目」「内容」「設定範囲」「セット側」の5つの列に分かれています。テキスト部分は各設定項目の説明文で、表部分はASCIIコードの格納方法やビット割り当てを示す小さなビットマップ表が複数含まれています。グラフや図はなく、主に文字列の格納方法や範囲を示す構造的な表が中心です。

### テキスト
#### FTPサーバ名（デバイス +5～+133）
FTPサーバ名（ドメイン名）をASCII文字列で設定します。文字数は1～256文字（半角英数字、半角記号）です。  
1文字目を(s1)+5の下位バイトに、2文字目を(s1)+5の上位バイトに格納し、最終文字の後に00Hを格納して文字列の終端を示します。  
なお、(s1)+4のFTPサーバ指定で"0000H:FTPサーバアドレス"を指定する場合は設定不要です。

#### FTPサーバアドレス（デバイス +134～+135）
FTPサーバのIPアドレスを設定します。範囲は000.000.000.001～223.255.255.254です。  
IPアドレスは4オクテットに分けて格納し、(s1)+135の上位バイト、(s1)+135の下位バイト、(s1)+134の上位バイト、(s1)+134の下位バイトにそれぞれ格納します。  
(s1)+4のFTPサーバ指定で"0001H:FTPサーバ名"を指定する場合は設定不要です。

#### ログイン名（デバイス +136～+152）
FTPサーバにログインするためのログイン名をASCII文字列で設定します。文字数は1～32文字（半角英数字、半角記号）です。  
1文字目を(s1)+136の下位バイトに、2文字目を(s1)+136の上位バイトに格納し、最終文字の後に00Hを格納して文字列の終端を示します。

#### パスワード（デバイス +153～+169）
FTPサーバにログインするためのパスワードをASCII文字列で設定します。文字数は0～32文字（半角英数字、半角記号）です。  
1文字目を(s1)+153の下位バイトに、2文字目を(s1)+153の上位バイトに格納し、最終文字の後に00Hを格納して文字列の終端を示します。

### 表（ビット割り当ての説明）
各文字列の格納方法を示す小さな表が複数あります。  
- 各表は16ビット（b15～b0）を2つの8ビットASCIIコードに分割して示しています。  
- 例えば、FTPサーバ名の(s1)+5のビット割り当ては、b15～b8が2文字目のASCIIコード、b7～b0が1文字目のASCIIコードとなっています。  
- 同様に、ログイン名やパスワードも同様のビット割り当てで格納されていることが示されています。  
- FTPサーバアドレスの表は4つのオクテットを2つの16ビットに分けて格納する構造を示しています。

### まとめ
本ページはFTPサーバ接続に必要な情報（サーバ名、IPアドレス、ログイン名、パスワード）をASCIIコードでメモリに格納する方法を詳細に説明しています。各文字列は2文字ずつ16ビットのデバイスに格納され、終端は00Hで示されます。設定範囲や文字数制限も明記されています。
