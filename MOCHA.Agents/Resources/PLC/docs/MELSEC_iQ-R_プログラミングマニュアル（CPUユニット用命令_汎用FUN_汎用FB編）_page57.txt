### テキスト
#### 同一デバイスのSET/RST命令を使用した場合
- ■SET命令の場合  
  実行指令がONのとき指定デバイスをONにし、OFFのときは無処理。  
  1スキャンに複数回SET命令を実行し、1つでもONなら指定デバイスはONになる。

- ■RST命令の場合  
  実行指令がONのとき指定デバイスをOFFにし、OFFのときは無処理。  
  1スキャンに複数回RST命令を実行し、1つでもONなら指定デバイスはOFFになる。

- ■1スキャンに同一デバイスのSET命令とRST命令がある場合  
  SET命令がONのとき指定デバイスをON、RST命令がONのとき指定デバイスをOFFにする。  
  両命令がOFFの場合はデバイスのON/OFF状態は変化しない。

### 図
- ラダー図  
  X0がONでSET M0命令、X1がONでRST M0命令を実行する回路例。

- タイミングチャート  
  X0、X1、M0の状態変化を時間軸で示す。  
  (1) X0がONのためM0はONになる。  
  (2) X1がONでもRST命令は無処理のためM0はONのまま。  
  (3) X0がOFFでもSET命令は無処理のためM0はONのまま。  
  (4) X1がONのためM0はOFFになる。

### 注釈
1. X0がONのため、M0はONする。  
2. X1がONのため、M0はONのまま（RST命令は無処理）。  
3. X0がOFFのため、M0はONのまま（SET命令は無処理）。  
4. X1がONのため、M0はOFFする。

### 補足
SET/RST命令で出力(Y)を指定すると、1スキャンの最後に実行したSET/RST命令のON/OFF状態が出力される。
