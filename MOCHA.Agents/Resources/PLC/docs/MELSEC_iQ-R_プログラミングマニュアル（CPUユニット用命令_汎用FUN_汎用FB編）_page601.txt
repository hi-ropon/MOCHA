### テキスト
#### プログラム待機命令 PSTOP(P)
PSTOP(P)命令は、指定したデバイスに格納されているファイル名のプログラム実行タイプを待機タイプに変更します。  
主にプロセスCPU（二重化モード）やSIL2プロセスCPUの両系実行プログラムで使用され、系切替時に実行タイプを引き継ぎません。

**機能のポイント**
- ファイル名で指定されたデバイスに格納されているプログラムを待機状態にします。
- プログラムメモリに格納されているプログラムのみ待機タイプに設定可能。
- 指定プログラムはEND処理で待機状態となる。
- パラメータで実行形式を指定している場合でもPSTOP(P)命令が優先される。
- ファイル名に拡張子(.PRG)は不要（.PRGファイルのみ対象）。

### 図
- **ラダー図**  
  ファイル名を指定するデバイス（FILE）を入力し、ST命令でENOにPSTOPまたはPSTOPPを設定する形を示す。

- **FBD/LD命令ブロック図**  
  EN（実行条件）、FILE（ファイル名）を入力し、ENO（実行結果）を出力する命令ブロック。

- **実行条件の波形図**  
  - PSTOP命令は立ち下がりエッジで実行  
  - PSTOPP命令は立ち上がりエッジで実行

| 命令   | 実行条件の波形           |
|--------|--------------------------|
