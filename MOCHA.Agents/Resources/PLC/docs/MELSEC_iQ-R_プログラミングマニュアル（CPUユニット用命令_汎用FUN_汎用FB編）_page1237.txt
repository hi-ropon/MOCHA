### テキスト
#### 機能概要
- 指定したデバイス番号以降に設定されている使用ループ数分のPID制御用データを一括でCPUユニット内部に登録し、PID制御を可能にする。
- PID制御用データは特定のデバイス番号に割り付けられている。

#### 注意事項
- PID制御用データの設定で使用するデバイスの点数は、「2 + 14 × 使用ループ数」で算出される。
- 各データはBIN値（2進数）で設定する必要がある。
- 使用ループ分のデバイス点数が指定したデバイスの最終番号を超えるとエラーにはならず処理されない。
- 1スキャン中に複数箇所でPIDINIT命令を実行した場合、PIDCONT命令に最も近いPIDINIT命令の設定値が有効となる。
- PIDINIT命令は必ずPIDCONT命令実行前に行う必要があり、実行しない場合はPID制御ができない。

### 表
#### PID制御用データの割り付け

| 項目               | PID制御用データ                         | 内容                                   |
|--------------------|--------------------------------------|--------------------------------------|
| 全ループ共通       | (s)+0                                | 使用ループ数                         |
|                    | (s)+1                                | 1スキャンの実行ループ数               |
| No.1ループ用(14ワード) | (s)+2 〜 (s)+15                     | 演算式選択、サンプリング周期(Ts)、比例定数(Kp)、積分定数(Ti)、微分定数(Td)、フィルタ係数(α)、操作量下限値(MVLL)、操作量上限値(MVHL)、操作量変化率リミット値(ΔMVL)、測定値変化率リミット値(ΔPVL)、0固定(0以外指定時はエラー)、微分ゲイン(KD)、0固定(0以外指定時はエラー)×2 |
| No.nループ用(14ワード)*1 | (s)+(m+0) 〜 (s)+(m+13)           | No.1ループ用と同様の項目をループ数に応じて割り当て |

*1 m = (n-1)×14 + 2  
(1)「0」固定となり、「0」以外を指定するとエラーになる。
