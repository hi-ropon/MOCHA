### テキスト
#### 演算処理
(s)に入力されたDINT型のデータをSTRING型のデータに変換し、(d)から出力します。

- (s)への入力値はDINT型のデータ値です。
- 符号データは、入力値が正の場合は20H（スペース）、負の場合は2DH（マイナス記号）を格納します。
- 有効桁数が少ない場合は、上位桁に20H（スペース）を格納します。

##### 例
-123456を入力した場合、上位バイトと下位バイトに分けてASCIIコードが格納されます。文字列の最後（6ワード目上位バイト）には、SM701（出力文字数切換）がOFFの場合、00Hが格納されます。

#### 演算結果
1. EN/ENOなしファンクション  
演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション  
実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |  
| :--- | :--- |  
| EN | ENO | (d) |  
| TRUE（演算実行） | TRUE | 演算出力値 |  
| FALSE（演算停止） | FALSE*1 | 不定値 |  

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、その場合は(d)から出力されたデータを使用しないようプログラム処理を行う必要があります。

#### エラー
演算エラーはありません。

### 図
- DINT型データ(s)の各バイトが、STRING型データ(d)の上位バイト・下位バイトに対応し、十億位から一の位までのアスキーコードに変換される構造を示しています。
- 例として-123456の入力時の各バイトのASCIIコード（16進数）が表形式で示され、符号や数字の位置が明確にされています。
- 文字列の最後の6ワード目上位バイトには、SM701がOFFの場合00Hが格納されることが示されています。

### 表
| 実行条件 | EN | ENO | (d) |
| :--- | :--- | :--- | :--- |
| 演算実行 | TRUE | TRUE | 演算出力値 |
| 演算停止 | FALSE | FALSE*1 | 不定値 |

*1 ENOがFALSEの場合、(d)のデータは不定値となるため注意が必要。
