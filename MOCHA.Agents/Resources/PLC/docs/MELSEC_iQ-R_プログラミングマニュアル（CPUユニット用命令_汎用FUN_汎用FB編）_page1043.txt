- テキスト部分：使用可能なデバイスの種類とその対応関係、更新対象のフィールド名の説明、更新後のデータの設定方法についての詳細な解説。
- 表部分：使用可能デバイスの一覧表、更新対象フィールド名の項目説明表、更新後のデータの項目説明表の3つの表が掲載されている。
- 図部分：更新後のデータ設定フォーマットを示す図解があり、フィールドごとのデータサイズと設定値の配置イメージを視覚的に説明している。

### テキスト
#### 使用可能デバイス
CPUユニットのデータベースアクセス命令で使用可能なデバイスの種類を示しています。オペランドごとにビット、ワード、ダブルワード、間接指定、定数などの対応が表形式で整理されています。

#### 更新対象のフィールド名
オペランド(s3)に対応し、更新対象のフィールド数やフィールド名の設定方法を説明しています。フィールド名は32文字固定でUnicode文字列を用い、足りない場合は前詰めし、パディングして32文字に調整します。フィールド数に応じて最終アドレスが変化します。

#### 更新後のデータ
オペランド(s4)に対応し、更新後のデータの設定方法を説明しています。データサイズや設定値の指定が可能で、データ型によって設定値の内容が変わります。

### 表
#### 使用可能デバイス
| オペランド | ビット | ワード | ダブルワード | 間接指定 | 定数 | その他 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| (s1)〜(s4), (d1), (d2) | 各種ビットデバイスの対応有無 | 各種ワードデバイスの対応有無 | ダブルワード対応 | 間接指定対応 | 定数対応 | - |

#### 更新対象のフィールド名（オペランド(s3)）
| デバイス | 項目 | 内容 | 設定範囲 | セット側 |
| :--- | :--- | :--- | :--- | :--- |
| +0 | フィールド数 | 更新対象のフィールド数を設定 | 1〜128（旧CPUは1〜16） | ユーザ |
| +1〜+□ | フィールド名 | 各フィールドの名前（32文字固定、Unicode） | なし | ユーザ |

#### 更新後のデータ（オペランド(s4)）
| デバイス | 項目 | 内容 | 設定範囲 | セット側 |
| :--- | :--- | :--- | :--- | :--- |
| +0 | データサイズ | 更新対象フィールドのデータサイズを設定 | なし | ユーザ |
| +1 | 未使用 | - | - | - |
| +2〜+□ | 設定値 | 更新後のデータを設定 | データ型による | ユーザ |

### 図
更新後のデータ設定フォーマットの図解では、ビット位置b15からb0までの範囲で1レコードのデータサイズを設定し、その後に各フィールドの設定値が順に並ぶ構造を示しています。n個目のフィールド設定値まで続き、各フィールドのデータサイズに従って配置されることが説明されています。
