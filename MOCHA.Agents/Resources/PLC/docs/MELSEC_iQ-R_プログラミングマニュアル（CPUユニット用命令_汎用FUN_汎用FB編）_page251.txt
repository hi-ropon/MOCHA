### テキスト
#### 使用可能デバイス
除算命令で使用可能なデバイスの種類を示しています。ビット、ワード、ダブルワード、間接指定、定数などのカテゴリごとに、各オペランド(s1, s2, d)で使用可能なデバイスが表形式でまとめられています。  
また、SIL2プロセスCPUおよび安全CPUの安全プログラムでは、特定の安全デバイスおよび定数のみ使用可能であることが記載されています。

#### 機能
(s1)で指定したBIN16ビットデータを被除数、(s2)で指定したBIN16ビットデータを除数として除算を行い、その結果を(d)で指定したデバイスに格納します。  
除算結果は32ビットで、商と余りを格納します。ビットデバイスの場合は桁指定したビット数を使用して商と余りを格納します。  
具体的には、商は下位16ビットに、余りは上位16ビットに格納されます。

#### エラー
エラーコード(SD0)として「3400H」があり、これは(s2)で指定した値（除数）が0のときに発生するエラーであることが記載されています。

### 図
BIN16ビットデータの除算処理のイメージ図が示されています。  
・(s1)に5678(BIN)  
・(s2)に1234(BIN)  
を指定し、除算を行うと、商が4(BIN)、余りが742(BIN)となり、(d)に格納される様子がビット単位で示されています。  
(d)は商、(d)+1は余りを表しています。
