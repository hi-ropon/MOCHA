### テキスト
#### 機能
■ROR(P)命令の説明
- (d)で指定されたデバイスのBIN16ビットデータを、SM700（キャリフラグ）を含めずに(n)ビット右へ回転させる命令です。
- SM700（キャリフラグ）は、ROR(P)命令実行前の状態によりONまたはOFFになります。
- (d)にビットデバイスが指定された場合は、桁指定で指定されたデバイス範囲内で回転します。
- 実際に回転するビット数は、(n)を桁指定で指定された点数で割った余りとなります。
  - 例：n=15、点数=12ビットの場合、15÷12=1余り3なので3ビット右へ回転します。
- nは0～15の値を指定可能で、16以上の値が指定された場合は、nを16で割った余りの値で回転します。
  - 例：n=18の場合、18÷16=1余り2なので2ビット右へ回転します。

### 図
- 16ビットのビット列b15～b0が示されており、初期状態のビット列と1ビット右回転後のビット列が矢印で示されています。
- 1ビット右回転では、b0の値が次のb15に移動し、他のビットは1つ右にずれます。
- SM700はキャリフラグとして、回転前のb0の値を保持し、回転後のb0の値は次の回転に影響します。
- (n)ビット右回転は、1ビット右回転をn回繰り返すイメージで示されています。
- ビットデバイス指定時の回転範囲と回転の仕組みも矢印で説明されています。

以上より、ROR(P)命令は16ビットデータのビット単位の右回転を行い、キャリフラグSM700は回転前のb0の値を保持することが理解できます。
