### テキスト
#### nビット左ローテーション（ROL(_E)）
入力値をnビット分左へ回転（ローテーション）して出力する命令について説明しています。

##### 命令の対応環境
- RnCPU、RnENCPU（プロセス、二重化、一般、安全）に対応しています。

##### 命令の動作
- 入力された16ビットまたは32ビットデータ（WORD型/DWORD型）を、指定されたビット数(n)だけ左へ回転し、同じデータ型で出力します。
- 左へ回転するビット数はnの入力値に依存します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力データ | 入力変数 | ANY_BIT |
| n(N) | シフトビット数指定 | 入力変数 | ANY_BIT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力データ | 出力変数 | ANY_BIT |

#### 機能
- (s)に入力された16ビットまたは32ビットデータを、(n)ビット左へ回転（ローテーション）し、(d)から(s)と同じデータ型で出力します。
- 左へ回転するビット数は(n)の入力値です。

#### 演算処理のイメージ
- ビット列が左へnビット分回転する様子を矢印付きの図で示しています。

#### 例
- (s)のデータ型が16ビットデータ(WORD型)、(n)への入力値が3の場合、3ビット分左へ回転します。
- 具体例として、入力値270FHが3879Hに変換されるビット列の変化を示しています。

#### 入力値の範囲
| (s)のデータが16ビットデータ(WORD型)の場合 | (s)のデータが32ビットデータ(DWORD型)の場合 |
| :--- | :--- |
| (n)の入力値は0～15の範囲内。下位4ビット分のデータが使用される。例：入力値6の場合は6、入力値220の場合は6 | (n)の入力値は0～310の範囲内。下位5ビット分のデータが使用される。例：入力値6の場合は6、入力値20の場合は22 |

### 図
- ラダー/FBD/LD命令ブロック図：EN/ENOなしとEN/ENO付きの2種類のブロック図を示し、入力s、n、出力dの接続を表現。
- ST言語の記述例：EN/ENOなしの場合はd:=ROL(s,n); EN/ENO付きの場合はd:=ROL_E(EN,ENO,s,n);と記載。
- ビット回転のイメージ図：WORD/DWORDのビット列が左へnビット分回転する様子を矢印で示す。
- 例のビット列図：16ビットWORD型の入力270FHのビット列と、3ビット左回転後の3879Hのビット列をビット単位で示し、どのビットがどこに移動したかを矢印で表現。
