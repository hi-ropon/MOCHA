### テキスト
#### 機能
- OUT命令の演算結果がONのとき、ロングタイマのコイルはONし設定値まで計測を行う。
- タイムアップ（現在値＝設定値）になるとa接点は導通し、b接点は非導通になる。
- OUT命令の演算結果がONからOFFに変化すると、表に示すようにa接点とb接点の状態が変わる。
- タイムアップ後はロング積算タイマの現在値クリアと接点OFFはRST命令で行う。
- 設定値が0のときはOUT命令実行時にタイムアップする。
- OUT命令実行時には、OUT LT命令、OUT LST命令のコイルON/OFFや接点ON/OFF、現在値の変更処理を行う。
- JMP命令などでOUT LT命令を飛ばした場合は現在値の更新や接点ON/OFFは行わない。
- 同一OUT LT命令を同一スキャン内で2回以上実行した場合は、実行回数分の現在値を更新する。

#### Point
- タイマの時限設定はエンジニアリングツールのパラメータ設定で行う。
- ロングタイマ/ロング積算タイマの設定範囲は0.001～1000ms（デフォルト0.001ms、設定単位0.001ms）。
- タイマの計数方法については「MELSEC iQ-R CPUユニットユーザーマニュアル（応用編）」を参照。

#### 注意事項
- ロングタイマの接点で別のロングタイマの計測を行う場合は、後で計測するロングタイマから順にプログラミングすること。
- 計測順にプログラミングしないと、すべてのタイマが同一スキャンでONする場合がある。
- 特にスキャンタイムより短い設定値や設定値が1の場合に注意。

### 図（ラダー図）
#### 例1：LT0～LT2のタイマを後から計測するロングタイマから順にプログラミングした場合
- LT1の接点がONした次のスキャンからLT2のタイマが計測開始。
- LT0の接点がONした次のスキャンからLT1のタイマが計測開始。
- X0がONするとLT0のタイマが計測開始。

#### 例2：LT0～LT2のタイマを計測順にプログラミングした場合
- X0がONするとLT0のタイマが計測開始。
- LT0の接点がONするとLT1、LT2のタイマの接点もONになる。

これらのラダー図は、タイマの計測開始順序と接点のON/OFFの関係を示し、プログラムの動作理解を助けるための例示となっている。
