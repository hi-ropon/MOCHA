### テキスト
#### 機能
- 指定ブロックの指定ステップを非活性にする。
- 命令実行により指定ブロックの活性ステップ数が0になるとブロックが非活性となる。SFC用情報デバイスのブロック起動終了ビットが設定されている場合、そのビットデバイスはONからOFFに変化する。
- すでに非活性のステップに命令を実行した場合は命令を無処理（NOP命令相当）とする。
- ブロック指定の場合は以下のように扱う。
  - シーケンスプログラム中で命令を実行した場合：ブロック0
  - SFCプログラム（動作出力内）で命令を実行した場合：命令を実行したブロック（自ブロック）
- プロセスCPU（二重化モード）の場合、バックアップモードの待機系で実行しても無処理となる。

#### 注意事項
- ステップ起動命令で並列分岐の中のステップを指定する場合、並列になっているすべてのステップを起動する必要がある。活性していない分岐回路があると並列結合が移行しない。
- 並列分岐の中のステップに対してステップ終了命令を実行すると、並列結合の条件が成立しない。
- SFCプログラムは動作出力内でステップNo.に自ステップNo.を指定できない。指定するとエラーになる。

| エラーコード | 内容 |
| :--- | :--- |
| 31A1H | 指定ブロックが存在しないとき。（SET/RST BL□¥S□） |
| 31A2H | ブロック指定時に指定ブロックNo.が指定可能なブロックNo.の範囲外のとき。 |
| 31B1H | SFCプログラムが存在しない（スキャン実行タイプのSFCプログラムが存在しない）、またはSM321（SFCプログラムの起動/停止）がOFFに設定して命令を実行したとき。 |
| 31B2H | 指定ステップが指定ブロック内に存在しないとき。（SET/RST BL□¥S□/S□） |
| 31B2H | 指定ステップNo.が指定可能なステップNo.の範囲外のとき。 |
| 31B5H | 動作出力内で指定ステップに自ステップを指定したとき。 |
