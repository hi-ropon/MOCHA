### テキスト
#### 機能：演算処理
- (s)に入力された16ビットデータ/32ビットデータ（WORD型/DWORD型）を、(n)ビット左へシフトして、(d)から(s)と同一のデータ型で出力する。
- 左へシフトする(n)ビット数は(n)へ入力された値。
- (s)への入力値は16ビット/32ビットデータである。
- (n)への入力値（シフトビット指定）は以下の範囲内である。

#### 入力値の範囲
| (s)のデータが16ビットデータ(WORD型)の場合 | (s)のデータが32ビットデータ(DWORD型)の場合 |
|--------------------------------------------|---------------------------------------------|
| (n)の入力値は0～15の範囲内。<br>下位4ビット分のデータが使用される。 | (n)の入力値は0～31の範囲内。<br>下位5ビット分のデータが使用される。 |
| 例：入力値が6の場合は6、22の場合は6（16ビットのため15まで） | 例：入力値が6の場合は6、22の場合は22 |

#### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、(d)から演算出力値が出力される。

2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の通り。

| 実行条件 | 演算結果 |  
| --- | --- |  
| EN | ENO | (d) |  
| TRUE（演算実行） | TRUE | 演算出力値 |  
| FALSE（演算停止） | FALSE*1 | 不定値 |  

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、(d)から出力されたデータを使用しないようプログラム処理を行う必要がある。

#### エラー
- 演算エラーは存在しない。

### 図
- 16ビットデータ(WORD型)の例として、(s)のデータ「270FH」を(n)への入力値8で左シフトした場合のビット列の変化を示す。
- 元のビット列（270FH）は「0000 0010 0111 0000 1111」と表され、左へ8ビットシフトすると「1111 0000 0000 0000」となる。
- 最下位から(n)ビット分は0になることを示している。

### 表
#### 入力値の範囲比較表
| (s)のデータが16ビットデータ(WORD型)の場合 | (s)のデータが32ビットデータ(DWORD型)の場合 |
| :--- | :--- |
| (n)の入力値は0～15の範囲内。下位4ビット分のデータが使用される。例：6の場合は6、22の場合は6 | (n)の入力値は0～31の範囲内。下位5ビット分のデータが使用される。例：6の場合は6、22の場合は22 |

#### 演算結果の条件と出力値
| 実行条件 | 演算結果 | (d) |
| :--- | :--- | :--- |
| EN | ENO | 演算出力値 |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOがFALSEの場合、(d)の出力は不定値となるため注意が必要。
