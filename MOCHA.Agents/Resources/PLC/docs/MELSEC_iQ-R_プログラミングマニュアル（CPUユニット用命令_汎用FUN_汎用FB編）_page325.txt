### テキスト
#### 機能
(s1)で指定されたデバイスから(n)点分の内容と、(s2)で指定されたデバイスから(n)点分の内容の排他的論理和（XOR）を求め、その結果を(d)で指定されたデバイス以降に格納します。

また、(s2)には-32768から32767までの符号付き16ビット定数を指定することも可能です。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 2821H              | (s1)から(n)点分のデバイス範囲と、(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s1)と(d)に同一デバイスを指定した場合は除く。<br>(s2)から(n)点分のデバイス範囲と、(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s2)と(d)に同一デバイスを指定した場合は除く。 |

### 図（ビット列の例示）
- (s1)と(s2)で指定されたビット列の例が示されており、それぞれのビット列に対してXOR演算を行った結果が(d)に格納される様子をビット単位で表現しています。
- 1つ目の図は、(s1)と(s2)がそれぞれ複数行のビット列で示され、XOR演算の結果が(d)に格納される例。
- 2つ目の図は、(s2)に符号付き16ビット定数が指定された場合のXOR演算例を示しています。

これらの図は、ビットの並び（b15からb0まで）を示し、各ビットの0/1の状態を視覚的に理解できるようになっています。
