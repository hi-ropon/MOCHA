### テキスト
#### SM705(変換桁数切換)がOFFの場合の動作
- アドレス(d)～(d)+3には16進アスキーデータが固定桁数（8桁）で格納される。
- ASCIIコードは各桁のアスキーコードで表される。
- (d)+4には00Hが格納され、SM701(出力文字数切換)がOFFの時のみ00Hが格納される。ON時は変化しない。
- 演算結果は8桁の16進数として処理され、有効桁数の左側の00は「0」として処理（0パディング）される。

##### 例
(s)に03AC625EHを指定した場合の格納例を示す。  
(d)～(d)+4にそれぞれ16進数のASCIIコードが格納される。

#### SM705(変換桁数切換)がONの場合の動作
- (d)には有効桁数の左側の0を省略した桁数（最大8桁）の16進アスキーデータが前詰めで格納される。
- (s)の値と(d)～(d)+3の内容の例を表形式で示す。

### 表
#### (s)の値と(d)～(d)+3の内容の対応例

| (s)の値       | (d)～(d)+3の内容                                                                                  |
|---------------|-------------------------------------------------------------------------------------------------|
| 0H～FF        | (d)の上位バイトに00Hが格納され、(d)+1以降は変化しない。                                        |
| 10H～FFH      | (d)+1に00Hが格納され、(d)+2以降は変化しない。                                                  |
| 1000000H～FFFFFFH | (d)+3の上位バイトに00Hが格納される。具体的に(d)～(d)+3にASCIIコードが順に格納される。          |
| 10000000H～FFFFFFFH | SM701(出力文字数切換)がOFFのとき(d)+4に00Hが格納される。ONのときは(d)+4は変化しない。          |

### 図
- BIN 32bitのビット構成図とASCIIコードの格納イメージ。
- 具体例として(s)に03AC625EHを指定した場合の各アドレス(d)～(d)+4に格納される16進ASCIIコードの配置図。
- (s)の値に応じた(d)～(d)+3の内容を示す表のビット構成図。

### 注意事項
- ASCIIコードは各桁のアスキーコードで表される。
- 桁数が最大桁数(8桁)未満の場合、SM701のON/OFFに関わらず文字列の終端に00Hを格納する。
- 文字列の終端が下位バイトの場合、上位バイトにも00Hを格納する。
- 桁数が最大桁数(8桁)の場合、SM701がOFFのときは(d)+4に00Hを格納し、ONのときは(d)+4は変化しない。
