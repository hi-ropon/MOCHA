### テキスト
#### 6.7 データ転送命令 BIN16ビットデータ転送 MOV(P)
指定されたデバイスのBIN16ビットデータを転送する命令について説明しています。

##### 命令概要
- MOV(P)命令は、転送元デバイスのBIN16ビットデータを転送先デバイスに転送します。
- 対応CPUはRnCPU、RnENCPU（プロセス、二重化、一般、安全）です。

##### 実行条件
- MOV命令は立ち上がりエッジで実行されます。
- MOVP命令は立ち下がりエッジで実行されます。

##### 設定データの内容、範囲、データ型
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s) | 転送元のデータまたはデータが格納されているデバイス番号 | -32768～32767 | 符号付きBIN16ビット | ANY16 |
| (d) | 転送先のデバイス番号 | ― | 符号付きBIN16ビット | ANY16 |
| EN | 実行条件 | ― | ビット | BOOL |
| ENO | 実行結果 | ― | ビット | BOOL |

- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、上記のデータ型の安全デバイスおよび安全ラベルのみ使用可能。

##### 使用可能デバイス
- 通常デバイスと安全デバイスに分かれており、それぞれビット、ワード、ダブルワード、間接指定、定数、その他の区分で使用可能なデバイスが表形式で示されています。
- 例：ビットデバイスはX, Y, M, L, SM, F, B, SB, FX, FYなど、ワードデバイスはT, ST, C, D, W, SD, SW, FD, R, ZR, RDなど。
- 安全デバイスはSA¥X, SA¥Y, SA¥M, SA¥SM, SA¥BなどのビットデバイスとSA¥T, SA¥ST, SA¥C, SA¥D, SA¥W, SA¥SDなどのワードデバイスが使用可能。

### 図
- ラダー図：MOV命令のラダー回路図が示されており、(s)と(d)のデバイスを転送する形状。
- ST言語：ENO:=MOV(EN,s,d); ENO:=MOVP(EN,s,d); の形式で記述。
- FBD/LD図：EN入力、ENO出力、s入力、d出力のブロック図。
