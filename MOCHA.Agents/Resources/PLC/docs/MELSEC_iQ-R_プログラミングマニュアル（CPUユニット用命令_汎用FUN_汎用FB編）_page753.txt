### テキスト
#### BIN16ビットデータ平均値算出
「MEAN(P)(_U)」命令は、指定したデバイスからn点分（16ビットデータ単位）の平均値を算出します。

#### 実行条件
命令ごとに実行条件が設定されており、MEANとMEAN_Uはパルス信号、MEANPとMEANP_Uは立ち上がりエッジで実行されます。

### 図
#### ラダー
ラダー図は3つの入力（s, d, n）と1つの出力（ENO）を持つ命令ブロックを示しています。

#### FBD/LD
FBD/LD図は、EN（実行条件）、s（平均値を算出するデータの先頭デバイス）、d（平均値を格納するデバイス）、n（データ数またはデバイス番号）を入力し、ENO（実行結果）を出力するブロックを示しています。

### 表
#### 実行条件
| 命令       | 実行条件       |
|------------|----------------|
| MEAN       | パルス信号     |
| MEAN_U     | パルス信号     |
| MEANP      | 立ち上がりエッジ |
| MEANP_U    | 立ち上がりエッジ |

| オペランド | 内容                             | 範囲       | データ型           | データ型（ラベル） |
|------------|--------------------------------|------------|--------------------|--------------------|
| (s)        | 平均値を算出するデータが格納されている先頭デバイス | —          | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S / ANY16_U  |
| (d)        | 平均値を格納するデバイス         | —          | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S / ANY16_U  |
| (n)        | データ数またはデータ数が格納されたデバイス番号 | 0〜65535   | 符号なしBIN16ビット | ANY16              |
| EN         | 実行条件                       | —          | ビット               | BOOL               |
| ENO        | 実行結果                       | —          | ビット               | BOOL               |
