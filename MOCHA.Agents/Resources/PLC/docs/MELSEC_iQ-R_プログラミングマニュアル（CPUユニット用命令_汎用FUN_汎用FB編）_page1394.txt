### テキスト
#### 32.57 STRING型→DINT型変換
STRING_TO_DINT(_E)関数は、STRING型データをDINT型データへ変換します。

##### ラダー、FBD/LD、STの使用例
- ラダーではEN/ENOなしとEN/ENO付きの2種類の回路図が示されています。
- ST言語ではEN/ENOなしの場合はd:=STRING_TO_DINT(s);、EN/ENO付きの場合はd:=STRING_TO_DINT_E(EN,ENO,s);と記述します。

##### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | STRING(11) |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | DINT |

##### 機能
- 演算処理として、(s)に入力されたSTRING型のデータをDINT型のデータに変換し、(d)から出力します。

##### 演算処理の詳細
- STRING型データ(s)は6バイトで構成されており、上位バイトから順に十億位、千万位、十万位、千位、十位、00H(文字列の終端)までのアスキーコードが格納されています。
- 下位バイトは符号のアスキーコードから一の位までのアスキーコードが格納されています。
- 入力値の範囲はアスキーコードで30H～39H、20H、2DH、00Hの範囲内で、STRING型のデータ値は-2147483648～2147483647の範囲です。

##### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション
- 実行条件と演算結果は以下の表の通りです。

| 実行条件 | 演算結果 |
| --- | --- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となります。その場合は(d)から出力されたデータを使用しないようプログラム処理を行う必要があります。
