本ページはテキストと図（ビット列のイメージ図）で構成されている。テキストはSFCプログラムのステップ活性状態を指定ブロックや指定ステップから一括で読み出す機能の説明で、読み出し結果の格納方法や例示が中心。図はビット列の並びと読み出し前後の対応関係を示し、例として存在しないステップの扱いや不定データの格納例を示している。

### テキスト
#### 機能
- 指定ブロックの活性状態を、指定ステップから指定ワード分一括で読み出す。
- ブロック指定なしの場合は、以下のブロックのステップ活性状態を読み出す。
  - シーケンスプログラム中で命令を実行した場合：ブロック0
  - SFCプログラム（動作出力内）で命令を実行した場合：命令を実行したブロック（自ブロック）
- 読み出した結果は、(d)に指定したデバイスに格納される。
- 読み出すデータ中で欠番となっているステップNo.に対応するビットは0になる。

#### 例示
- 指定ブロックのS5とS8が存在しない場合、該当ステップの活性状態は空欄（ビット0）として格納される。
- ブロック指定なしで読み出すデータ範囲がブロック中の最大ステップNo.を超えた場合、不定なデータが格納される。
- ブロック指定ありで読み出すデータ範囲がブロック内のステップ数を超えたり、存在しないステップの先頭を指定した場合も不定データが格納される。
- SFCプログラムが存在しない状態で実行した場合や、ブロック指定ありで存在しないブロックを指定した場合、読み出すビットデータはすべて"0"になる。

### 図の説明
- ビット列の並びを示し、(s)から(s)+(n)-1までのステップ活性状態ビットが、(d)から(d)+(n)-1のデバイスに対応して格納される様子を示す。
- 例では、ステップS5とS8が欠番の場合のビット配置や、ブロックの最終ステップNo.がS26でS0から2ワードをD0からD1に転送する場合のビット配置を示している。
- 不定データは「？」で表現されている。

以上より、SFCプログラムのステップ活性状態を効率的に読み出し、欠番や範囲外のステップに対しては0または不定データとして扱う仕様が説明されている。
