### テキスト
#### 単数値変数ファンクション 33.1 絶対値
ABS(_E)関数は、入力値の絶対値を出力します。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、三重化）、RnPSFCPU（一般、安全）、RnSFCU（一般、安全）です。

##### 機能
- 入力されたINT型、DINT型、REAL型、LREAL型データの絶対値を、出力変数(d)に同じデータ型で出力します。
- 入力値をA、演算出力値をBとすると、B=|A|となります。
- 入力値の例として、INT型で-32768を入力すると、出力は32768になります。
- DINT型で-2147483648を入力した場合も同様に絶対値が出力されます（演算エラーにはならない）。
- ABS_E関数では、出力変数ENOからTRUEが出力されます。

##### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション
- 実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
| :--- | :--- | :--- |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE | 不定値 |

※ ENOがFALSEの場合、(d)から出力されるデータは不定値となるため、その場合は(d)のデータを使用しないようプログラム処理を行う必要があります。

### 図
- ラダー、FBD/LDでのABS(_E)関数の表現
  - EN/ENOなし：入力sから出力dへ絶対値を出力するシンプルな図
  - EN/ENO付き：EN入力、ENO出力が追加された図
- ST言語での記述例
  - EN/ENOなし：d:=ABS(s);
  - EN/ENO付き：d:=ABS_E(EN,ENO,s);

### 表
#### 設定データ：内容、種別、データ型

| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | ANY_NUM*1 |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY_NUM*1 |

*1 安全プログラムではREAL型/LREAL型データを指定しないこと。指定した場合はエラーとなる。
