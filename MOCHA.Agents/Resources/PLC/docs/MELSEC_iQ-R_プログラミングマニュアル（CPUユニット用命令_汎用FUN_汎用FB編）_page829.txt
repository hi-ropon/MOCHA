### テキスト
#### 機能
(s)で指定されたデバイス番号以降に格納されている文字列データに対し、文字列の右（文字列の最終）から(n)文字分のデータを、(d)で指定されたデバイス番号以降に格納する処理について説明しています。

- 文字列の最終を示すNULLコード（00H）が文字列データの最後に自動付加されます。
- (n)で指定する文字数が0の場合は、(d)にNULLコード（00H）が格納されます。

#### 例（n=5の場合）
- (s)から始まるデバイスに「"ABCDEF12345"」の文字列が格納されているとします。
- そのうち右から5文字分「12345」を(d)で指定したデバイスに格納します。
- 文字はASCIIコードで格納され、最後にNULLコード（00H）が付加されます。

### 図
- 左側の図は、(s)で指定されたデバイス番号以降に格納されている文字列のASCIIコードのビット配置を示しています。b15～b0のビットに2文字分のASCIIコードが格納されているイメージです。
- 右側の図は、(d)で指定されたデバイス番号以降に、(s)の文字列の右から(n)文字分が格納される様子を示しています。
- 例では、(s)の文字列の5文字分が(d)に格納され、最後にNULLコード（00H）が付加されていることがわかります。

| エラーコード | 内容 |
| :--- | :--- |

| 3405H | (s)に指定可能な範囲外のデータを設定したとき。<br>・(s)の文字列が16383文字を超える<br>・(s)の文字列が0文字<br>・(n)が(s)で指定の文字数を超えているとき。 |
