### テキスト
#### SM705（変換桁数切換）がOFFの場合の動作
(d)～(d)+5には、10進アスキーデータが固定桁数で格納されます。

- ASCII S：符号のASCIIコード
- ASCII 10⁹～ASCII 10⁰：それぞれ十億の位から一の位までのASCIIコード
- (1) (d)+5の上位バイトは、SM701（出力文字数切換）がOFFのときは00Hが格納され、ONのときは20H（スペース）が格納される。
- 正の数の場合は20H（スペース）、負の数の場合は2DH（-）が格納される。
- 数値部が9桁以下の場合、ASCII 10⁹には20H（スペース）が格納される。
- 数値部が8桁以下の場合、ASCII 10⁸には20H（スペース）が格納される。
- 数値部が1桁の場合、ASCII 10¹には20H（スペース）が格納される。

#### 例
(s)に-12345678を指定した場合（符号付き指定）

### 図
- 32ビットのビット割り当て図：b31～b0までのビットがBIN 32bitとして示され、(s)+1と(s)の位置が示されている。
- ASCIIコードのビット配置表：(d)～(d)+5の各バイトに対応するASCIIコード（符号、十億の位～一の位）が示されている。
- 例のビット配置図：-12345678の各桁のASCIIコードが16進数でb15～b0に割り当てられている。

### 表
#### ASCIIコード対応表（(d)～(d)+5）
| アドレス | b15～b8 | b7～b0 |
| :------- | :------ | :----- |
| (d)      | ASCII 10⁹ | ASCII S |
| (d)+1    | ASCII 10⁷ | ASCII 10⁸ |
| (d)+2    | ASCII 10⁵ | ASCII 10⁶ |
| (d)+3    | ASCII 10³ | ASCII 10⁴ |
| (d)+4    | ASCII 10¹ | ASCII 10² |
| (d)+5    | 00H / 20H | ASCII 10⁰ |

#### 例のASCIIコード（-12345678の場合）
| アドレス | b15～b8 | b7～b0 |
| :------- | :------ | :----- |
| (d)      | 20H (SP) | 2DH (-) |
| (d)+1    | 31H (1)  | 20H (SP) |
| (d)+2    | 33H (3)  | 32H (2) |
| (d)+3    | 35H (5)  | 34H (4) |
| (d)+4    | 37H (7)  | 36H (6) |
| (d)+5    | 00H, 20H | 38H (8) |

### 注釈
- SPはスペース（20H）
- 符号はASCII Sに格納され、正の数はスペース、負の数はハイフン（2DH）で表される。
- 各桁の数値は対応するASCIIコードで格納される。
