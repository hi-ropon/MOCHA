### テキスト
#### パルスタイマ（TP(_E)）
パルスタイマは指定された時間の間、信号をONにする機能を持ちます。プロセスCPU（二重化モード）およびSIL2プロセスCPUの両系実行プログラムで使用した場合、バックアップモードの待機系では動作しません。

#### 動作概要
- (s)がONになると、(n)で設定された時間の間、(d1)をONにします。
- (d2)には(d1)がONしてからの経過時間がセットされます。
- 経過時間のカウントにはロングタイマを使用します。

#### 演算処理
1. 出力
   - (s)がONすると、(n)で設定された時間の間、(d1)をONにします。
   - (d2)には(d1)がONしてからの経過時間がセットされます。
2. 出力の終了
   - 経過時間が設定時間に到達すると(d1)をOFFにします。
   - (d1)がOFF後、(s)がOFFの場合は経過時間をリセットします。
   - (d1)がONの時に(s)がOFFしても、(d1)はOFFされません。
3. 出力時間の設定
   - (n)の有効設定範囲はT#1ms～T#2147483msです。
   - エンジニアリングツールのタイマ時限設定を変更することで有効範囲が変わる場合があります。

#### 注意点
- (n)の設定値は、(d1)がOFF→ON（立上り）となったときの値が使用されます。
- (d1)がON時に(n)の値を変更した場合、変更した値が有効になるのは次回出力開始時です。

### 表
#### 設定データ：内容、種別、データ型

| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 出力開始 | 入力変数 | BOOL |
| n(PT) | 出力時間設定値 | 入力変数 | TIME |
| ENO | 出力状態(TRUE:正常、FALSE:異常または停止) | 出力変数 | BOOL |
| d1(Q) | 出力 | 出力変数 | BOOL |
| d2(ET) | 経過時間 | 出力変数 | TIME |

#### 出力時間設定値の有効範囲

| 最小値 | 最大値 |
| :--- | :--- |
| タイマ時限のロングタイマ設定値[ms]と同じ。ただし、ロングタイマ設定値が1ms未満の場合は1msとなる。 | 出力時間設定値は時間型(32ビット値)のため、最大値は時間型の範囲に収まる値となる。<br>・出力時間設定値[ms] ≤ 2147483647[ms] × タイマ時限のロングタイマ設定値[ms]<br>【例】<br>・タイマ時限のロングタイマ設定値が0.001msの場合：T#1ms～T#2147483ms<br>・タイマ時限のロングタイマ設定値が1000msの場合：T#1000ms～T#2147483000ms |

### 図
- ラダー/FBD/LD言語でのパルスタイマ使用例
  - EN/ENOなしの場合：入力s、n、出力d1、d2のシンボル図
  - EN/ENO付きの場合：入力EN、s、n、出力ENO、d1、d2のシンボル図
- ST言語での使用例
  - EN/ENOなし：インスタンス名とパラメータ(IN:=s, PT:=n, Q:=d1, ET:=d2)
  - EN/ENO付き：インスタンス名とパラメータ(EN:=en, ENO:=eno, IN:=s, PT:=n, Q:=d1, ET:=d2)
