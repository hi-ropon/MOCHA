### テキスト
#### 41.2 二安定ファンクションブロック（リセット優先）
RS(_E)という名称のファンクションブロックについて説明しています。このブロックは2つの入力値を判別し、1（TRUE）または0（FALSE）を出力します。

### 図
#### ラダー、FBD/LD表現
- EN/ENOなしの場合のラダー図
  - 入力s1、s2があり、出力dがあるシンプルな構成
- EN/ENO付きの場合のラダー図
  - EN（実行条件）とENO（出力状態）が追加されている
- ST言語表現
  - EN/ENOなし: インスタンス名S、R1、Q1を用いた記述例
  - EN/ENO付き: インスタンス名EN、ENO、S、R1、Q1を用いた記述例

### 表
#### 設定データ：内容、種別、データ型
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件（TRUE:実行、FALSE:停止） | 入力変数 | BOOL |
| s1(S) | セット指令 | 入力変数 | BOOL |
| s2(R1) | リセット指令 | 入力変数 | BOOL |
| ENO | 出力状態（TRUE:正常、FALSE:異常または停止） | 出力変数 | BOOL |
| d(Q1) | 出力 | 出力変数 | BOOL |

このファンクションブロックは、セット指令とリセット指令の2つの入力を判別し、リセット優先で出力を制御します。
