<overview>
このページは主にテキストとビットデータの論理演算を示す図で構成されています。テキスト部分では、指定されたデバイスのBIN16ビットデータに対する否定排他的論理和（XNR）演算の機能説明とエラー情報が記載されています。図はビット単位のデータの具体例を示し、s1、s2の入力データと演算結果dのビット列を視覚的に表現しています。表やグラフはなく、テキストと図のシンプルな構成です。
</overview>

<contents>
### テキスト
#### 機能
- (s1)で指定されたデバイスのBIN16ビットデータと、(s2)で指定されたデバイスのBIN16ビットデータの否定排他的論理和（XNR）を求め、その結果を(d)で指定されたデバイスに格納します。
- ビットデバイスの場合、桁指定による点数以降のビットデバイスは0として演算します。

#### エラー
- 演算エラーはありません。

### 図
- 図は3つのビット列を示しています。
  - (s1): 16ビットの入力データ。b15からb0までのビットが1と0で表現されています。
  - (s2): 16ビットの入力データ。b15からb0までのビットが1と0で表現され、XNR演算の対象となる。
  - (d): (s1)と(s2)のXNR演算結果の16ビットデータ。b15からb0までのビットが1と0で示されている。

- 各ビットの並びにより、どのビットが1か0かが視覚的にわかるようになっており、XNR演算の結果がどのように導かれるかを示しています。
</contents>
