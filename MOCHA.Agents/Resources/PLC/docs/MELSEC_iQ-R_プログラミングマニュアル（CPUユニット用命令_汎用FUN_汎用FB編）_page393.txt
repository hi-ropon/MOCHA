### テキスト
#### 機能
(d)で指定したデバイスから(n1)ダブルワード分のデータ範囲内で、左へ(n2)ダブルワード分シフトします。シフト後、(s)から(n2)点分を空きデータ部分に格納します。
- (s)に定数を指定した場合、シフト後の最下位から(n2)点分は指定した値になります。
- (n2)で指定した値が00の場合は無処理となります。

#### 例
(n1)=9、(n2)=4の場合の動作を示しています。

#### エラー
| エラーコード(SD0) | 内容 |
|-------------------|------|
| 2821H             | (s)から(n2)点分の範囲と、(d)から(n1)点分の範囲が重複しているとき。 |
| 3405H             | (n1)、(n2)に指定した値が(n1)<(n2)のとき。 |

#### 注意事項
(s)と(d)をビットの桁指定で指定する場合、(s)と(d)の桁数を合わせる必要がある。

### 図
データシフトの動作を示す図で、以下の内容を表現しています。
- (d)から始まる9点分のデータがあり、これを左へ4点分シフトする。
- シフト前のデータは16進数で「30FFFFH」「1EEEEH」「20000H」「0H」「1FFFFH」「10000H」「1EEEEH」「7FFFFH」「2AAAAH」。
- シフト後は左端の4点分が消え、右端の4点分は(s)からのデータ「11111111H」「22222222H」「33333333H」「44444444H」で埋められる。
- 図中に「① (n2)ダブルワード分、左へシフト」「② コピー」と注釈がある。
