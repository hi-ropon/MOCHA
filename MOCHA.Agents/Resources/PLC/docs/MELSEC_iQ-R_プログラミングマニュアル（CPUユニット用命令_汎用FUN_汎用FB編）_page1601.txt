### テキスト
- (250) 入出力データを設定します。
- (338) 手動→自動切換え処理用に使用するデバイスをリセットします。
- (360) 外部から操作量(MV)をセットします。
- (401)～ 自動/手動切換え指令ON後、PID演算を行うことにより、初めて自動モードに切り換わるため、自動モードに切り換わる時間のずれを考慮して、手動→自動切換え処理を実行します。

### 表（ラダープログラム）
- 各行はラダーのステップ番号（例：250、293、306など）で区切られている。
- 接点（X3、X0、X80、X6、M0、M40、M44、T2など）やコイル（SET、RST、OUT、PLS）を用いて制御条件を表現。
- 命令例：
  - BINP K4X30 D610：K4X30のビット入力をD610に読み込み。
  - MOVP D610 D200：D610のデータをD200に移動。
  - SET M0：M0をセット。
  - MOV U8¥G400 D611：U8¥G400のデータをD611に移動。
  - MOVP K1 D615：定数K1をD615に移動。
  - RST M44、RST M41、RST C1：それぞれのデバイスをリセット。
  - BINP K4X20 D50：K4X20のビット入力をD50に読み込み。
  - MOVP D50 D614：D50のデータをD614に移動。
  - MOVP K0 D615：定数K0をD615に移動。
  - PLS M40：M40のパルス出力。
  - OUT T2 K10：タイマーT2にK10を出力。
- 比較命令（>= K2000 D50）や等価比較（<> D610 D200、= D610 D200）も使用されている。

### 処理概要
- 入出力データの設定、手動から自動切換え時のデバイスリセット、外部操作量のセット、自動/手動切換え指令ON後のPID演算による自動モード切換え処理を行うラダー制御例。
