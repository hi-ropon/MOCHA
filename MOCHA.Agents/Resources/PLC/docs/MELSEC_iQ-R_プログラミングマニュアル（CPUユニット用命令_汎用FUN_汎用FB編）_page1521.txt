### テキスト
#### タイマファンクションブロック TIMER_□_M
プロセッサCPU（二重化モードおよびSII 2プロセスCPUの両系実行プログラム使用時）での動作制約に注意が必要です。  
実行条件が成立すると、設定された時間までタイマカウントを実行します。

#### 機能
- s1の実行条件がONになると計測開始。s3×10msから計測開始し、s2×10msまで計測値が到達するとd2がON。
- s1の実行条件がOFFになると計測値は0になり、d2もOFF。
- 高速タイマの計測単位はエンジニアリングツールのデフォルト値から変更可能だが、変換/全変換時にワーニングが発生。
- s2は0～32767の値指定可能。
- s3は-32768～32767の値指定可能。負値指定時は初期値0。

### 表
#### 設定データ（内容、種別、データ型）

| 引数          | 内容                         | 種別     | データ型 |
|---------------|------------------------------|----------|----------|
| s1(Coill)     | 実行条件(TRUE:実行, FALSE:停止) | 入力変数 | BOOL     |
| s2(Preset)    | タイマ設定値                 | 入力変数 | INT      |
| s3(ValueIn)   | タイマ初期値                 | 入力変数 | INT      |
| d1(ValueOut)  | タイマ現在値                 | 出力変数 | ANY16    |
| d2(Status)    | 出力                         | 出力変数 | BOOL     |

### 図
- ラダー回路図：s1～s3の入力とd1、d2の出力端子を持つタイマブロックの図示。
- ST言語：インスタンス名とパラメータ（Coil, Preset, ValueIn, ValueOut, Status）を示す記述例。
- 回路例：TIMER_10_FB_Mのブロック図。M0がCoil、10がPreset、1がValueIn、D10がValueOut、M10がStatusに対応。
- タイミングチャート：M0のON/OFFによりタイマ計測が開始・停止し、D10に計測値が出力される様子を時間軸で示す。M0がONで10×10msの計測開始、D10が1から90msかけて増加、M10はOFF状態。
