### テキスト
DBINSERT(P)命令を使用して、ID=0003Hのレコード1件をデータベースの製品情報テーブル(proInfo)に登録する例を示しています。

### 表
#### 製品情報テーブル(proInfo)
| ID    | Product    | Size(x) | Size(y) | Size(z) |
|-------|------------|---------|---------|---------|
| 0001H | "製品名1"  | 80      | 100     | 60      |
| 0002H | "製品名2"  | 40      | 90      | 40      |
| 0003H | "製品名3"  | 40      | 80      | 40      |
| ...   | ...        | ...     | ...     | ...     |

- 各フィールドのデータ型は以下の通り
  - ID：WORD型
  - Product：WSTRIN型（15文字）
  - Size(x)、Size(y)、Size(z)：WORD型

### 図
- データベースは製品情報テーブル(proInfo)を格納しているイメージで示されています。
- ラダー図でDBINSERT命令の使用例を示し、各パラメータの意味を解説しています。
  - DBINSERT D0 "proInfo" D100 D300 M0 D500
    - D0：不明（おそらく命令の先頭）
    - "proInfo"：テーブル名
    - D100：フィールド数（0005H＝5フィールド）
    - D300：登録件数（0001H＝1件）
    - M0：不明（おそらく制御用）
    - D500：不明（おそらく結果格納用）
- フィールド名と値のメモリ配置を詳細に示しています。
  - フィールド名はD101～D164に16ワード分のWSTRIN型で"ID"、"Product"、"Size(x)"、"Size(y)"、"Size(z)"が格納されている。
  - 各フィールドの設定値はD301～D321に格納されている。
    - ID：0003H
    - Product："製品名3"（文字コードで分割）
    - Size(x)：K40（16進数で40）
    - Size(y)：K80
    - Size(z)：K40
- 注釈として、
  1. WSTRING型(16ワード) + WORD型(1ワード) ×4 = 20ワード
  2. フィールド「ID」の設定値
  3. フィールド「Product」の設定値
  4. フィールド「Size(x)」の設定値
  5. フィールド「Size(y)」の設定値
  6. フィールド「Size(z)」の設定値
が記載されています。

以上より、DBINSERT命令を用いて、指定したテーブルに対して複数フィールドのデータを1レコード分登録する方法を具体的に示した内容となっています。
