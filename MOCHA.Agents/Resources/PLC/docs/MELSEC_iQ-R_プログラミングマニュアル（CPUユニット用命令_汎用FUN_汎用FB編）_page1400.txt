<overview>
本ページは、STRING型データからREAL型データへの変換方法について説明しています。テキスト部分では変換ルールや注意点、演算結果の説明があり、表部分では演算条件と結果の対応表およびエラーコードとその内容が示されています。グラフや図はなく、主にテキストと表で構成されています。
</overview>

<contents>
### テキスト
- STRING型データにおいて、最初の0以外の数値の間に20H（スペース）または30H（0）が存在する場合は、これらを無視して変換する。
- 小数点型の場合の例として、「-1.35034」がREAL型に変換される。
- 指数型の場合の例として、「-1.35034E-10」がREAL型に変換される。
- STRING型データ（指数型）で「E」と数値の間に30H（0）が存在する場合も30Hを無視して変換する。
- 文字列中に20H（スペース）が含まれていた場合は20Hを無視して変換する。
- STRING型データは最大24文字まで入力可能で、20H（スペース）や30H（0）も1文字としてカウントされる。
- (s)への入力値はSTRING型のデータ値で、指定されたASCIIコード範囲内である必要がある。

#### 演算結果
1. EN/ENOなしファンクション：演算処理を実行し、(d)から演算出力値が出力される。
2. EN/ENO付きファンクション：実行条件と演算結果が表で示されている。

### 表
#### 演算条件と演算結果

| 実行条件 | 演算結果 |  
| --- | --- |  
| EN | ENO | (d) |  
| TRUE（演算実行） | TRUE | 演算出力値 |  
| FALSE（演算停止） | FALSE*1 | 不定値 |  

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、(d)のデータを使用しないようプログラム処理を行う必要がある。

#### エラーコードと内容

| エラーコード（SD0） | 内容 |  
| --- | --- |  
| 2820H | (s)から該当デバイス範囲内に00Hがないとき。 |  
| 3401H | (s)に変換できない不正なデータを設定したとき。以下の条件が含まれる。  
・整数部、小数部に30H(0)～39H(9)以外の文字がある  
・指定した文字列の中に2EH(.)が2つ以上ある  
・指定した文字列の指数部に、45H(E)、65H(e)、2BH(+)、2DH(-)以外の文字がある  
・指定した文字列の中に45H(E)、65H(e)の指数部が複数ある  
・指定した文字列の中に、指数部の数値を3桁以上記載している  
・指定した文字列の中に2BH(+)、2DH(-)の指数部の符号が複数ある  
・指定した文字列の中に2BH(+)、2DH(-)の符号が小数点形式の場合：整数部、指数部の場合：仮数部に複数ある  
・(s)以降の文字数が0、または24文字を超えているとき。 |  
| 3403H | (d)が下記範囲を超えるとき（オーバーフローが発生したとき）  
|(d)| < 2^128 |
