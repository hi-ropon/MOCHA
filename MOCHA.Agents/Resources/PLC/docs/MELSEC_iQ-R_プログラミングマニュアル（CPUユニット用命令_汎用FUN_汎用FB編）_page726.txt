### テキスト
#### 機能
(s)で指定されたデバイスから(n)点のBIN16ビットデータを検索し、最大値を見つけます。その最大値を(d)で指定されたデバイスに格納します。さらに、(s)で指定されたデバイスから検索し、最初に検出した最大値が格納されているデバイス番号(s)から何点目かを(d)+1に格納し、最大値の個数を(d)+2に格納します。

#### エラー
演算エラーはありません。

### 図
図は2つの部分に分かれています。

1. 左側の図は、(s)から始まる連続したデバイスのビットデータの一覧を示しています。例として、1234、5678、5678、-5214、5555（すべてBIN16）という値が並んでいます。これらは(n)点分のデータです。

2. 右側の図は、検索結果の格納先(d)のデバイスに最大値5678が格納され、その次の(d)+1に「2」（最大値が2番目に検出されたことを示す）、(d)+2に「2」（最大値の個数）を格納していることを示しています。

この図は、最大値の検索とその位置・個数の格納の流れを視覚的に説明しています。
