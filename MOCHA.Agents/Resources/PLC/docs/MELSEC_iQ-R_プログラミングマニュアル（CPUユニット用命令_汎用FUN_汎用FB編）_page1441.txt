### テキスト
#### 34.7 代入 MOVE(_E)
入力値の代入を出力します。

##### 対応CPU
RnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）

##### 命令概要
- 入力値(s)を出力(d)に代入する命令です。
- ラダー、FBD/LD、ST言語での記述例が示されています。

##### 引数の内容、種別、データ型
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | ANY*¹ |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY*¹ |

*¹ 安全プログラムではREAL型、LREAL型、STRING型データを指定しないこと。指定した場合はエラーとなる。

##### 機能
- 演算処理として、(s)に指定した変数の値を(d)に指定した変数へ代入する。
- (s)、(d)にはBOOL型、INT型、DINT型、WORD型、DWORD型、REAL型、LREAL型、STRING型、TIME型、構造体型、配列型が指定可能。
- (s)と(d)は同じデータ型のみ指定可能。

### 図（ラダー、FBD/LD、ST言語の代入命令例）
- ラダー、FBD/LDではEN/ENOなしとありの2種類のブロック図が示されている。
- ST言語ではEN/ENOなしの場合は「d:=MOVE(s);」、ありの場合は「d:=MOVE_E(EN,ENO,s);」と記述。

### 図（演算処理イメージ）
- 各データ型ごとに、(s)の値が(d)にそのまま代入される様子を示す。
- INT、DINT、WORD、DWORD、REAL、LREALの6種類のデータ型について、具体的な数値例を用いて代入のイメージを図示。
