このページは、BCD4桁データの減算機能について説明しています。ページ構成は以下の通りです。
- テキスト部分：BCD4桁データの減算処理の概要と動作説明、アンダーフロー発生時の挙動についての説明。

<contents>
### テキスト
#### 機能
(s1)で指定されたBCD4桁データと、(s2)で指定されたBCD4桁データの減算を行い、その減算結果を(d)で指定したデバイスに格納します。

- 減算結果にアンダーフローが生じた場合は、キャリフラグSM700はONしません。

### 図
- 正常減算例：
  - (s1)のBCDデータ「0678」から(s2)のBCDデータ「0234」を減算すると、結果(d)は「0444」となります。
  - 減算時に0が入ることを示しています。

- アンダーフロー発生例：
  - (s1)のBCDデータ「0001」から(s2)のBCDデータ「0003」を減算すると、結果(d)は「9998」となります。
  - この場合、キャリフラグSM700はONしません。

### 表
#### エラー
| エラーコード (SD0) | 内容 |
| :--- | :--- |
| 3405H | (s1)のBCDデータが0〜9999以外の場合。<br>(s2)のBCDデータが0〜9999以外の場合。 |

エラーコード3405Hは、(s1)または(s2)のBCDデータが指定範囲外（0〜9999以外）のときに発生します。
</contents>
