### テキスト
#### 倍精度実数データの構成
倍精度実数データは「符号部」「仮数部」「指数部」の3つの部分で構成され、以下のように表現されます。

- 符号部（1ビット）：数値の正負を表し、0は正数または0、1は負数を示す。
- 仮数部（52ビット）：単精度実数を2進数で1.XXXXX...×2^Nと表現したときのXXXXX...の部分。
- 指数部（11ビット）：単精度実数を2進数で1.XXXXX...×2^Nと表現したときのNを示す。

ビット構成は、符号部が最上位ビット（b63）、指数部がb62～b52、仮数部がb51～b0に割り当てられています。

#### 指数部の値とNの関係（表）
| 指数部(b52～b62) | 7FFH | 7FEH | 7FDH | ... | 401H | 400H | 3FFH | 3FEH | ... | 02H | 01H | 00H |
|------------------|-------|-------|-------|-----|-------|-------|-------|-------|-----|-------|-------|-------|
| N                | 未使用 | 1023  | 1022  | ... | 2     | 1     | 0     | -1    | ... | -1021 | -1022 | 未使用 |

### 注意事項
#### エンジニアリングツールから単精度実数の入力値を設定する場合
- 単精度実数データは32ビットで処理され、有効桁数は約7桁。
- 入力値が7桁を超えると8桁目を四捨五入する。
- 四捨五入後の値が-2147483648～2147483647の範囲を超えると意図した値にならない。

**例**
- 入力値「2147483647」は8桁目の「6」を四捨五入し、「2147484000」として扱う。
- 入力値「E1.1754943562」は8桁目の「3」を四捨五入し、「E1.175494」として扱う。

入力値の範囲：
- 小数表現：0.0000000001 ≤ 実数データの絶対値 ≤ 999999900000.0
- 指数表現：1.175494351E-38 ≤ 実数データの絶対値 ≤ 3.402823466E+38

#### エンジニアリングツールから倍精度実数の入力値を設定する場合
- 倍精度実数データは64ビットで処理され、有効桁数は約15桁。
- 入力値が15桁を超えると16桁目を四捨五入する。
- 四捨五入後の値が-2147483648～2147483647の範囲を超えると意図した値にならない。

**例**
- 入力値「2147483646.12345678」は16桁目の「6」を四捨五入し、「2147483646.12346」として扱う。
- 入力値「E1.7976931348623157+307」は16桁目の「5」を四捨五入し、「E1.79769313486232+307」として扱う。

入力値の範囲：
- 小数表現：0.0000000000000000000001 ≤ 実数データの絶対値 ≤ 999999999999999000000.0
- 指数表現：2.22507385850721E-308 ≤ 実数データの絶対値 ≤ 1.79769313486231E+308
