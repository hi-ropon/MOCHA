### テキスト
#### 文字列比較命令の説明
・(s1)と(s2)で文字列データの長さが異なる場合、長い文字列データが大きくなる。  
・(s1)、(s2)の文字列が16383文字を超える場合は非導通(ENOはOFF)となる。  
・LDSTRING_□命令をFBD/LDで使用した場合、ENに左母線または常時ONの変数部品/定数部品を指定する必要がある。  
・ORSTRING_□命令をFBD/LDで使用した場合、ENにTRUEを指定するとENOはONとなるが、ENは実行条件にはならない。

#### エラー
演算エラーはなし。

#### Point
文字列比較命令では、文字列の比較を行いながら以下のチェックを実施する。  
・デバイスエリアの範囲超過チェック  
・文字列が16383文字以内かのチェック  
デバイスエリア内に00Hが存在しない場合や文字列が16383文字を超える場合に文字列不一致を検出すると、非導通(ENOはOFF)にならずに比較演算結果が出力される。  
例として、デバイスエリアの最終デバイスNo.がD12287の場合の演算結果を示す。

### 表
| 口内命令記号（ラダー，FBD/LD） | 比較演算結果 |
| :------------------------------ | :----------- |
| $=、EQ                         | 非導通状態(ENOはOFF) |
| $<>、NE                        | 導通状態(ENOはON) |
| $>、GT                        | 導通状態(ENOはON) |
| $<=、LE                       | 非導通状態(ENOはOFF) |
| $<、LT                        | 非導通状態(ENOはOFF) |
| $>=、GE                       | 導通状態(ENOはON) |

### 図
1. 文字列データの長さが異なる場合の比較例  
   - (s1)は"1234567"、(s2)は"123456"で、(s1)の方が長いため大きいと判定される。  
   - 各文字は16進数で表現されている。

2. デバイスエリア範囲チェックの例  
   - (s1)の2文字目が(s2)と異なり、(s1)≠(s2)となるため演算結果は導通(ENOはON)となる。  
   - D12287以降はデバイスエリア外となるため、D12287まで文字列比較処理を実施し、不一致を検出すると条件成立で処理終了。  
   - 図中に"Z"や"A"といったデバイスエリア外の文字が示されている。
