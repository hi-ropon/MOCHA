### テキスト
#### 使用可能デバイス
マルチCPUシステムで使用可能なデバイスを、オペランドごとにビット、ワード、ダブルワード、間接指定、定数、その他に分類して一覧表で示しています。例えば、(U/H)オペランドではビットデバイスのX,Y,M,L,SM,F,SB,FX,FYが使用可能で、定数のK,Hも使用可能です。

#### コントロールデータ（オペランド：s1）
- +0 完了ステータス：完了時のステータスを格納。0000Hは正常完了、それ以外は異常完了（エラーコード）。
- +1 書込みデータ点数：書込みデータ点数をワード単位で設定。設定範囲は1～8192（マルチCPUシステム構成時の最大設定範囲）。

#### 機能
- マルチCPUシステム構成時、自号機CPUユニットで指定したデバイス(s2)以降のデータを他号機CPUユニット(U/H)の指定したデバイス(d1)以降に、(s1)+1で指定した書込みデータ点数分格納します。
- D(P).DDWR命令、M(P).DDWR命令の動作概要を示しています。

### 図
#### マルチCPUシステムにおけるデータ分配イメージ
- 左側に自号機CPUユニット（書込み要求元）、右側に他号機CPUユニット（書込み対象先）を示し、自号機の(s2)から始まるデータを他号機の(d1)から始まる領域に転送する様子を矢印で表現しています。
- 書込みデータ点数は(s1)+1で指定。

#### DP.DDWR命令の動作概要シーケンス図
- シーケンススキャンの各ステップ（0～END）における命令実行、要求信号送出、転送、応答信号受付処理、完了デバイス(d2)のON/OFF状態変化を時間軸で示しています。
- CPU間定周期通信はデフォルト0.888ms。
- 書込み要求から完了までの一連の流れを視覚的に理解できる構成。
