### テキスト
#### 機能
- 実行指令のON→OFF時に指定デバイスをONにし、実行指令がON→OFF以外（OFF→OFF、OFF→ON、ON→ON）のときはOFFにする。
- 1スキャン中に(d)で指定されたデバイスのPLF命令が1つの場合は、指定デバイスが1スキャンONする。
- 同一デバイスに対してPLF命令を1スキャンに複数回実行した場合の動作は別ページ（58ページ）を参照。

- Sc（1スキャン）について：
  - PLF命令実行後にRUN→STOPにしても、再度RUNにしてもPLF命令は実行されない。
  - PLF命令は命令実行後の次の命令実行時にOFF処理を行う。
  - SIL2プロセスCPUおよび安全CPUの安全プログラムでは、PLF命令は安全サイクル処理の安全プログラムが動作し、命令が実行されるまで指定デバイスの「ラベル」が1安全サイクル処理ONする。
  - PLF命令の引数に一般/安全共有ラベルを使用する場合、安全サイクル処理の割り込みタイミングによっては、一般/安全共有ラベルを使用している安全プログラムまたは一般プログラムで、一般/安全共有ラベルのONを検出できないことや複数周期にわたりONを検出することがある。

#### Point
- PLF命令をCJ命令でジャンプさせたり、実行したサブルーチンプログラムをCALL(P)命令でコールしなかった場合、(d)で指定のデバイスが1スキャン以上ONになることがあるため注意。

### 図
- X5入力がONからOFFに変わるタイミングでPLF命令が実行され、M0がONになる動作を示す。
- 波形図では、X5がOFFになるとM0がONになり、一定時間後にM0もOFFになる様子が示されている。
- Scは1スキャンのタイミングを示している。

### エラー
- 演算エラーはありません。
