### テキスト
#### 機能
- (s)で指定されたデバイス番号以降に格納されているBIN16ビットデータを16進数扱いでアスキー変換し、(d)で指定したデバイス番号以降に(n)で指定した文字数分を格納する。
- (n)で文字数を設定すると、(s)で指定するBINデータの範囲と(d)で指定する文字列の格納デバイス範囲が自動的に決まる。
- 変換するBINデータの格納範囲と変換後のアスキーデータの格納範囲が重複しても正常に処理される。
- (n)で指定した文字数が奇数の場合、文字列を格納するデバイス範囲の最終デバイス番号の上位8ビットには自動的に00Hが格納される。
- (n)で指定する文字数が0の場合は変換処理を行わない。

#### エラー
| エラーコード(SD0) | 内容 |
|-------------------|------|
| 3405H             | (n)に指定可能な範囲外のデータを入力したとき。<br>・指定した文字数が0～16383以外 |

### 図
- BIN16ビットデータのビット配置例とアスキーコードへの変換例を示す図。
  - 例として、4桁目から1桁目までのビットがどのようにアスキーコードに変換されるかを示している。
- BINデータの格納例と変換後のアスキーデータの格納例を示す図。
  - D11～D13のBIN16データがD10～D14のアスキーデータに変換される様子を表している。
- 文字数が奇数の場合の格納例。
  - (s)から(s)+3までのBIN16データが(d)から(d)+7までのアスキーデータに変換され、最終デバイスの上位8ビットに00Hが自動格納されることを示している。
