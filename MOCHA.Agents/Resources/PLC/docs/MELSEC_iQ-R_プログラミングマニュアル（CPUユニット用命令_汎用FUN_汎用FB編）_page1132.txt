### テキスト
#### TOD(P)命令の説明
- (s2)で指定したデバイスから(n)点分のデータを、(U/H)で指定したユニットまたは自号機CPUユニット内のバッファメモリの(s1)で指定したアドレス以降に書き込みます。
- 書き込みはユニットへのワードデータライトとして行われます。
- (s2)に定数を指定した場合は、同一データ((s2)で指定した値)を指定したバッファメモリアドレスから(n)ワード分書き込みます。
- 命令実行時に対象ユニットの異常や処理中などでアクセスできない場合は無処理となります。

### 図
#### ユニットへのワードデータライトのイメージ
- CPUユニットの(s2)で指定した(n)点分のデータが、ユニットバッファメモリの(s1)で指定したアドレス以降の(n)ワードに書き込まれる様子を示す。
- 自号機CPUユニットへのワードデータライトも同様に、デバイス/ラベルメモリの(s2)で指定した(n)点分のデータが自号機CPUバッファメモリの(s1)で指定したアドレス以降の(n)ワードに書き込まれる。

#### 定数指定時の書き込みイメージ
- CPUユニットの(s2)に定数「5」を指定した場合、ユニットバッファメモリの(s1)で指定したアドレス以降に同じ値「5」が(n)ワード分繰り返し書き込まれる。
- 自号機CPUユニットの場合も同様に、定数「70000」が(n)×2ワード分繰り返し書き込まれる。

以上の内容から、TOD(P)命令は指定したデータまたは定数をユニットのバッファメモリに連続して書き込む命令であり、図はその書き込み範囲とデータの繰り返しを視覚的に説明しています。
