### テキスト
#### 機能
下限値(s1)の時刻データと上限値(s2)の時刻データの帯域に対し、比較する時刻データ(s3)で比較し、その結果（下、帯域内、上）に応じて(d)、(d)+1、(d)+2のいずれかがONします。

#### 注意事項
- 下限値(s1)は上限値(s2)より小さな値に設定してください。もし下限値(s1)を上限値(s2)より大きな値にした場合、上限値(s2)は下限値(s1)と同じ値として扱います。

### 図
- ラダー図形式で、X0入力があり、TZCP命令が使われています。
- (d)、(d)+1、(d)+2の3つの出力があり、それぞれのON条件が時刻データ(s1)、(s2)、(s3)の比較によって決まります。
- 比較条件は以下の通りです：
  - (d)がONになるのは、(s1)+1または(s1)+2が(s3)より大きい場合
  - (d)+1がONになるのは、(s1)+1または(s1)+2が(s3)以下かつ(s3)+1または(s3)+2が(s2)より小さいか等しい場合
  - (d)+2がONになるのは、(s3)+1または(s3)+2が(s2)より大きい場合

| デバイス | 時計データ | データ範囲 |
| :------- | :--------- | :--------- |

| (s1)+1, (s2)+1, (s3)+1 | 分 | 0～59 |
| (s1)+2, (s2)+2, (s3)+2 | 秒 | 0～59 |
