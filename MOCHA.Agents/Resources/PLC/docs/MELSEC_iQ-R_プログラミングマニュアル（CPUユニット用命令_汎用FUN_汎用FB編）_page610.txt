### テキスト
#### BIN16ビットデータの左ローテーション
- ROL(P): BIN16ビットデータをキャリフラグを含めずに(n)ビット左へ回転させる命令。
- RCL(P): BIN16ビットデータをキャリフラグを含めて(n)ビット左へ回転させる命令。

### 図
#### ラダー回路図
- ROL(P)とRCL(P)の命令はラダー回路で(d)と(n)のオペランドを指定して使用する。
- ST言語では以下のように記述する。
  ```
  ENO:=ROLP(EN,n,d);
  ENO:=RCL(EN,n,d);
  ENO:=RCLP(EN,n,d);
  ```

#### FBD/LD図
- EN（イネーブル）とENO（イネーブル出力）、n、dの入力端子を持つブロックで表現される。
- ROL命令はST、FBD/LDでは対応していないため、汎用ファンクションのROLを使用する必要がある。

### 表
#### 実行条件
| 命令 | 実行条件 |
| :--- | :--- |
| ROL, RCL | 立下りエッジ（↓） |
| ROLP, RCLP | 立上りエッジ（↑） |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (d) | ローテーションするデバイス | － | 符号付きBIN16ビット | ANY16 |
| (n) | ローテーションさせるビット数 | 0～15 | 符号なしBIN16ビット | ANY16 |
| EN | 実行条件 | － | ビット | BOOL |
| ENO | 実行結果 | － | ビット | BOOL |

#### 使用可能デバイス
| オペランド | ビット | ワード | ダブルワード | 間接指定 | 定数 | その他 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| (d) | ○ (X,Y,M,L,SM,F,B,SB,FX,FY) | ○ (T,ST,C,D,W,SD,SW,FD,R,ZR,RD) | ○ (Z) | － | ○ (K,H) | － |
| (n) | ○ | ○ | ○ | － | ○ | － |

- ○は使用可能を示す。
- 定数はK,Hが使用可能で、Eや$は使用不可。
