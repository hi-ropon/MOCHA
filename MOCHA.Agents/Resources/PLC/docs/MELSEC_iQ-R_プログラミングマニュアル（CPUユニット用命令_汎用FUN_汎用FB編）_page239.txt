このページは、BIN16ビットデータの減算機能について説明しています。ページ構成は主にテキスト説明と、減算の動作を示す図（ビットデータの例）が含まれています。テキストは「機能」と「エラー」の2つのセクションに分かれており、図は減算の具体例を示しています。表やグラフは含まれていません。
</overview>

<contents>
### テキスト
#### 機能
- (s1)で指定したBIN16ビットデータと、(s2)で指定したBIN16ビットデータの減算を行い、減算結果を(d)で指定したデバイスに格納します。

- 減算結果にアンダーフローが生じた場合、桁下げは無視されます。この場合、SM700（キャリフラグ）はONしません。

#### エラー
- 演算エラーはありません。

### 図（減算の例）
- [(s1) - (s2) = (d)]の形式で示されている。
- 例1: 正のBIN16データ同士の減算
  - s1: 5678 (BIN)
  - s2: 1234 (BIN)
  - d: 4444 (BIN)
- 例2: 負の値を含む減算（-Pの場合）
  - s1: -12345 (BIN)
  - s2: 23456 (BIN)
  - d: 29735 (BIN)
- 例3: 負の値を含む減算（-Pの場合）
  - s1: 12345 (BIN)
  - s2: -23456 (BIN)
  - d: -29735 (BIN)
- 例4: アンダーフローが無視される場合（-P_Uの場合）
  - s1: 56789 (BIN)
  - s2: 56790 (BIN)
  - d: 65535 (BIN)

これらの図は、BIN16ビットデータの減算結果の具体的な数値例を示し、アンダーフロー時の動作も説明しています。
</contents>
