### テキスト
#### SM705(変換桁数切換)がOFFの場合の動作
(d)～(d)+1には、10進アスキーデータが固定桁数（4桁）で格納されます。  
(d)+2にはSM701(出力文字数切換)がOFFのとき00Hが格納され、ONのときは変化しません。  
(d)に格納する演算結果で、有効桁数の左側の0には20H（スペース）が格納され、0はサプレスされます。  
例として「0050」の場合、「00」はそれぞれ20H（スペース）となり、「50」が有効桁数となります。

#### 例
(s)に9105を指定した場合のビット配置例が示されています。

#### SM705(変換桁数切換)がONの場合の動作
(d)には、有効桁数の左側の0を省略した桁数（最大4桁）の10進アスキーデータが格納されます。  
(s)の値と(d)に格納される値の例が表形式で示されています。

### 表
| (s)の値 | (d)～(d)+1の内容 |
| --- | --- |
| 0H～9H | (d)の上位バイトに00Hが格納される。 (d)+1は変化しない。 |
| 10H～99H | (d)+1に00Hが格納される。 |
| 100H～999H | (d)+1の上位バイトに00Hが格納される。 |
| 1000H～9999H | SM701がOFFのとき(d)+2に00Hが格納される。ONのときは変化しない。 |

### 図
- ビット配置図：  
  - (s)のビット列を10進の千位、百位、十位、一の位に分けて示し、それぞれのASCIIコードが(d)～(d)+1に格納される様子を示す。  
  - 例では9105の各桁のASCIIコード（31H、39H、35H、30H）が対応するビット位置に格納されている。  
- ASCIIコードの説明：  
  - ASCII 10³：千の位のASCIIコード  
  - ASCII 10²：百の位のASCIIコード  
  - ASCII 10¹：十の位のASCIIコード  
  - ASCII 10⁰：一の位のASCIIコード  
- 注意点として、桁数が最大桁数未満の場合の00H格納ルールや、SM701のON/OFFによる(d)+2の値の変化についても図解で説明されている。
