### テキスト
#### 拡張時計データの減算
「S(P).DATE-」命令は時間データを減算する命令です。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）など複数のCPUに対応しています。

#### 実行条件
- S.DATE-命令は立ち上がりエッジで実行されます。
- SP.DATE-命令は立ち下がりエッジで実行されます。

### 図
#### ラダー図
- ラダー図では、(s1)、(s2)、(d)の3つのオペランドがあり、s1とs2が減算対象の時刻データ、dが結果格納先です。
- ST命令の記述例として、ENO:=S_DATEMINUS(EN,s1,s2,d); ENO:=SP_DATEMINUS(EN,s1,s2,d);が示されています。

#### FBD/LD図
- FBD/LD図は命令ブロックの形で、EN（実行条件）、s1、s2、d、ENO（実行結果）端子が示されています。
- 口にはS_DATEMINUS、SP_DATEMINUSが入ることが記載されています。

### 表
#### 設定データ：内容、範囲、データ型
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s1) | 減算される時刻データが格納されている先頭デバイス | 機能参照 | 符号付きBIN16ビット | ANY16_ARRAY（要素数5） |
| (s2) | 減算する時間(時刻)データの格納されている先頭デバイス | 機能参照 | 符号付きBIN16ビット | ANY16_ARRAY（要素数5） |
| (d) | 減算結果時刻(時間)データを格納する先頭デバイス | － | 符号付きBIN16ビット | ANY16_ARRAY（要素数5） |
| EN | 実行条件 | － | ビット | BOOL |
| ENO | 実行結果 | － | ビット | BOOL |
