### テキスト
#### 機能
- (s)で指定されたBIN32ビットデータを16進数で表現した数値をASCIIコードに変換し、(d)で指定されたデバイス番号以降に格納する。
- SM705（変換桁数切換）の状態により、(d)に格納される10進ASCIIデータの形式が異なる。

#### SM705の状態と動作概要
- SM705がOFFの場合は固定桁数（8桁）で格納。
- SM705がONの場合は(s)の値によって各桁を前詰めで格納。

#### 動作の概要
- SM705がOFFとONの場合の動作を例示。
- 例として、(s)にBIN32ビットデータ「000ABCDEH」が格納されている状態でDBINHA(P)命令を実行した場合の動作を説明。

### 表
| SM705の状態 | (d)の格納形式 | 参照先 |
| :--- | :--- | :--- |
| OFF | 固定桁数(8桁)で格納 | 797ページ SM705(変換桁数切換)がOFFの場合の動作 |
| ON | (s)の値によって各桁を前詰めで格納 | 797ページ SM705(変換桁数切換)がONの場合の動作 |

### 図
- SM705: OFFの場合  
  (d)+0から(d)+3までに16進数で「30H 30H」「41H 30H」「43H 42H」「45H 44H」が格納され、ASCIIコードで「000ABCDE」となる。
- SM705: ONの場合  
  (d)+0から(d)+3までに「42H 41H」「44H 43H」「00H 45H」が格納され、有効桁数部分「ABCDE」のみがASCIIデータに変換され、終端に「00H」が格納される。

### 補足
- SM705がOFFの場合は固定桁数で8桁がASCIIデータに変換されて格納される。
- SM705がONの場合は前詰めとなり、有効桁数部分のみがASCIIデータに変換され、終端に00Hが格納される。
