### テキスト
#### 使用可能デバイス
CPUユニット内蔵データベースアクセス命令で使用可能なデバイスの種類を示しています。オペランドごとにビット、ワード、ダブルワード、間接指定、定数などの使用可否が表形式でまとめられています。

#### 検索対象のテーブル名
- オペランド：(s2)
- デバイス：+0～+□
- 項目：テーブル名
- 内容：検索対象のテーブル名を32文字以内で設定可能。
- 設定範囲：特になし
- セット側：ユーザ

#### 検索対象のフィールド名
- オペランド：(s3)
- デバイス：+0、+1～+□
- 項目：
  - フィールド数：検索対象のフィールド数を設定。0を指定するとテーブルの全フィールドが対象になる。
  - フィールド名：各フィールド名を32文字で設定。フィールド数に応じて最終アドレスが変化。
- 設定範囲：
  - フィールド数は0～128（ファームウェアバージョン28以前は0～16）
- セット側：ユーザ

#### 検索対象のフィールド名設定項目表

### 図
#### (s3)のフォーマット図
- b15～b0のビット構成で示されている。
- (s3)にフィールド数nを設定。
- (s3)+1から1個目のフィールド名、(s3)+32からn個目のフィールド名が順に格納される。
- nフィールド分設定。ただし(s3)に0を設定した場合は設定不要。

この図はフィールド名のメモリ配置と設定方法を視覚的に説明している。
