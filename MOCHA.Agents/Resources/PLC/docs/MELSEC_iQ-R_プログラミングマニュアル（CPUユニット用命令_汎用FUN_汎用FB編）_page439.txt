### テキスト
#### 機能
(s)で指定された符号付きBIN16ビットデータを、符号なしBIN32ビットデータに変換し、(d)で指定されたデバイスに格納します。  
プログラム例では、M0がONしたとき、D0に格納された符号付きBIN16ビットデータを符号なしBIN32ビットデータに変換し、D100～D101へ格納します。

#### エラー
演算エラーはありません。

### 図
#### プログラム例のラダー図
- SM402の接点が閉じると、MOVP命令でH0A000の値をD0に格納。
- M0がONすると、INT2UDINT命令でD0の符号付きBIN16ビットデータを符号なしBIN32ビットデータに変換し、D100に格納。

#### ビットデータの変換イメージ
- (s)の符号付きBIN16ビットデータ（D0）は、b15からb0までの16ビットで構成され、値は-24576（16進数でA000H）。
- (d)の符号なしBIN32ビットデータ（D100、D101）は、b31からb0までの32ビットで構成され、上位16ビットは0000H、下位16ビットはA000Hで、値は40960。
- 変換処理は、(1)上位16ビットに0を格納、(2)下位16ビットに元の16ビットデータを格納する形で行われている。
