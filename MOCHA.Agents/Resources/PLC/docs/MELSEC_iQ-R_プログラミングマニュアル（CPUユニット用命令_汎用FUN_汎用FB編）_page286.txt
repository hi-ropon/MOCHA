### テキスト
#### 機能
- (s1)で指定したデバイスから(n)点分のBIN32ビットデータと、(s2)で指定したデバイスから(n)点分のBIN32ビットデータ、または定数の減算を行い、その結果を(d)で指定したデバイス以降に格納する。
- ブロック減算は32ビット単位で行う。

#### 注意点
- (s1)または(s2)が(d)と同一デバイス（完全一致）で指定された場合は演算可能。
- ただし、(s1)または(s2)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲が部分的に重複するとエラーになる。
- (n)で指定した値が0の場合は無処理。
- 演算結果にオーバーフローが生じた場合、SM700（キャリフラグ）はONしない。

### 図
#### 例1: (s2)にデバイスを指定した場合（符号付き指定時）
- (s1)と(s2)の各ビットデータの減算例を示す。
- b31からb0までの32ビットデータが並び、複数のデバイスアドレスに対応。
- 減算結果は(d)のデバイスに格納される。

#### 例2: (s2)に定数を指定した場合（符号付き指定時）
- (s1)のデバイスデータから定数(9999など)を減算し、結果を(d)に格納する例。

#### 例3: (s2)と(d)から4点分のデバイスが完全一致する場合
- W1,W0からW7,W6までの4点分のデバイスとD1,D0からD7,D6までの4点分のデバイスが完全に一致し、演算可能であることを示す。

#### 例4: (s2)と(d)から4点分のデバイスが部分的に一致する場合
- 一部のデバイス範囲が重複しているため演算エラーとなる例。

### 表
| 符号付きを指定した場合 | 符号なしを指定した場合 |
| :--------------------- | :--------------------- |
| K2147483647 (H7FFFFFFF) - K-2 (HFFFFFFFE) → K-2147483647 (H80000001) | K0 (H00000000) - K1 (H00000001) → K4294967295 (HFFFFFFFF) |
| K-2147483647 (H80000001) - K2 (H00000002) → K2147483647 (H7FFFFFFF) | なし |

- オーバーフロー発生時の結果を符号付き・符号なしで比較している。
