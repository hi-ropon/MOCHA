### テキスト
#### 機能
- (d)で指定されたテーブルの最先入データ(d)+1を、(s)で指定されたデバイスに格納します。  
- FIFR(P)命令実行後はデータテーブルのデータが1個ずつ前詰めされます。  
- (d)に格納されている値が0のときは、FIFR(P)命令を実行しないようにインタロックをかける必要があります。  
- データ数(d)には0～65535が格納可能です。

#### エラー
| エラーコード (SD0) | 内容                                   |
|--------------------|--------------------------------------|
| 3405H              | (d)の値が0でFIFR(P)命令を実行したとき。 |

### 図
- 左側の図は、(d)で指定されたデータテーブルの内容を示しており、最上位のデータが3、続いて5432、1234、4321、0と並んでいます。  
- 右側の図は、(d)で指定された別のデータテーブルの内容で、1234、4321、0、0が格納されています。  
- 矢印は、左のテーブルの最先入データ5432が(s)で指定されたデバイスに格納され、データテーブルの内容が1つ前詰めされる動きを示しています。  
- (1)の注釈は、0が格納されることを示しています。
