### テキスト
#### SM705（変換桁数切換）がOFFの場合の動作
(d)～(d)+2には、10進アスキーデータが固定桁数で格納されます。  
ASCIIコードの説明として、符号のASCIIコードや各桁のASCIIコード（万、千、百、十、一の位）が示されています。  
(d)+3にはSM701（出力文字数切換）がOFFのときは00Hが格納され、ONのときは変化しません。  
正の数の場合は20H（スペース）、負の数の場合は2DH（-）が格納されることが注記されています。

#### 例
BINDA(P)命令で(s)に-12345を指定した場合のビット配置と格納内容の例が示されています。

#### SM705（変換桁数切換）がONの場合の動作
(d)には10進アスキーデータが前詰めで格納されます。  
(s)の値と(d)～(d)+2の内容の例が表形式で示されており、正の数・負の数、桁数の違いによる格納内容の違いが詳細に説明されています。  
(d)+3にはSM701（出力文字数切換）がOFF時のみ00Hが格納されることが記載されています。

#### ASCIIコードの詳細
- ASCII 10⁴：万の位のアスキーコード
- ASCII 10³：千の位のアスキーコード
- ASCII 10²：百の位のアスキーコード
- ASCII 10¹：十の位のアスキーコード
- ASCII 10⁰：一の位のアスキーコード

#### 注意事項
- 演算桁数が最大桁数（符号+5桁）未満の場合、SM701のON/OFFに関わらず文字列の終端に00Hを格納し、上位バイトにも00Hを格納する。
- 演算桁数が最大桁数の場合、SM701がOFFのときは(d)+3に00Hを格納し、ONのときは(d)+3は変化しない。

### 図
- 16bitのビット配置図があり、(s)の16bitデータが(d)～(d)+3のASCIIコードに変換される様子を示している。
