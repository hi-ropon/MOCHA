### テキスト
#### 機能
(s)で指定されたデバイス番号以降に格納されているBIN16ビットデータを、(n)バイトに分離し、(d)で指定されたデバイス番号以降に格納します。

- (n)でバイト数を設定することで、(s)で指定するBIN16ビットデータの範囲と(d)で指定するバイトデータの範囲が自動的に決まる。
- (n)が0の場合は処理を行わない。
- (d)で指定されたバイトデータ格納デバイスの上位8ビットには自動的に00Hが格納される。

#### エラー
演算エラーはありません。

### 図
1. **ビットデータ分離のイメージ図**
   - (s)から始まるBIN16ビットデータ（上位バイトと下位バイト）を、(d)から始まるバイト単位のデータに分離して格納する様子を示す。
   - 上位バイト(1)、下位バイト(2)のデータがそれぞれ別々のバイトに分割される。

2. **例1: (n)=5の場合のデータ格納例**
   - (s)～(s)+2の下位8ビットまでのデータを(d)～(d)+4に格納。
   - (s)側の16ビットデータがそれぞれ上位バイトと下位バイトに分かれ、(d)側では上位バイトは00H、下位バイトに元のデータが格納されている。

3. **例2: D12～D14をD11～D16の下位8ビットに格納する場合**
   - D12～D14の16ビットデータをD11～D16の下位8ビットに分離して格納。
   - 上位8ビットは自動的に00Hが格納されている。

### 表
| 分離するデータが格納されているデバイス範囲 | 分離するデータを格納するデバイス範囲 |
| :-------------------------------------------- | :------------------------------------ |
| (s)～(s)+((n/2)-1)                             | (d)+0～(d)+(n)-1                      |

- 分離するデータが格納されているデバイス範囲と、分離したデータを格納するデバイス範囲が重複していても正常に処理される。
