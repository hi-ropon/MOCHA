### テキスト
#### 使用可能デバイス
オペランドごとに使用可能なビット、ワード、ダブルワード、間接指定、定数、その他のデバイスが一覧表で示されています。  
例えば、(s1)オペランドはワードデバイスに対応し、間接指定も可能ですが、ビットデバイスや定数は使用不可となっています。  
(U)や(d1)オペランドは多くのデバイスに対応していませんが、(s1)、(s2)、(s3)、(s4)、(d2)はそれぞれ異なる対応範囲を持っています。

#### コントロールデータ(d1)
オペランド(d1)の詳細説明があり、特に「使用用途設定エリア」のビットフィールドb0、b1、b2の意味が解説されています。  
- b0は指定対象がファイルかフォルダかを指定（0：ファイル指定、1：フォルダ指定）  
- b2は移動対象の指定方法（0：指定フォルダごと移動、1：指定フォルダ内のすべてのファイル・サブフォルダを移動）  
- b2が0の場合は指定フォルダごと移動し、フォルダ構成のまま移動される。

### 図
フォルダ構成の図があり、ルートフォルダから指定フォルダ(s2)と移動先フォルダ(s4)の関係を示しています。  
- 指定フォルダ(s2)内のフォルダやファイル（folder1、folder1a、user1.csv、user2.csvなど）が示されている。  
- 移動先フォルダ(s4)に対して、指定フォルダごと（フォルダ構成のまま）移動される様子が矢印で表現されている。  
- 命令実行後（移動後）は指定したフォルダごと削除（移動）されることが説明されています。
