### テキスト
#### 機能
(s3)で指定した入力値（BIN16ビット値）を、(s1)、(s2)で指定した不感帯の上下限範囲に基づき、(d)で指定したデバイスに格納する出力値を制御します。出力値は以下の条件により決定されます。

#### BAND命令の例
- BAND(P)命令の場合、演算結果が-32768～32767を超えるときの動作例が示されています。
- BAND(P)_U命令の場合、演算結果が0～65535を超えるときの動作例が示されています。

#### エラー
- エラーコード3405Hは、(s1)で指定した下限値が(s2)で指定した上限値より大きい場合に発生します。

### 表
#### 条件と出力値の対応表
| 条件 | 出力値に格納される値 |
| :--- | :--- |
| 不感帯下限値(s1) > 入力値(s3)のとき | 入力値(s3) - 不感帯下限値(s1) |
| 不感帯上限値(s2) < 入力値(s3)のとき | 入力値(s3) - 不感帯上限値(s2) |
| 不感帯下限値(s1) ≤ 入力値(s3) ≤ 不感帯上限値(s2)のとき | 0 |

#### エラーコード一覧
| エラーコード(SD0) | 内容 |
| :--- | :--- |
| 3405H | (s1)で指定した下限値が、(s2)で指定した上限値よりも大きいとき。 |

### 図
- 横軸に入力値(s3)、縦軸に出力値(d)をとったグラフで、不感帯の上下限(s1、s2)を境に出力値が変化する様子を示しています。
- 不感帯範囲内では出力値は0、それ以外は入力値から不感帯の上下限を引いた値が出力される直線的な関係を表しています。
