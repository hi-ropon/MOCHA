### テキスト
#### 注意事項
- 探索方法と探索回数範囲は、SM755（スケーリングデータチェック設定）のON/OFFにより異なる。
- SM755がOFFの場合は「逐次探索」、ONの場合は「二分探索」が行われる。
- 逐次探索の探索回数範囲は1回から65535回、二分探索は最大16回。
- スケーリング用変換データが昇順にソートされている場合、SM755の状態により探索方法が異なるため、処理速度も異なる。
- 探索回数が少ないほど処理速度は速くなる。

#### 探索速度に関する補足
- 逐次探索の処理速度が遅くなる場合：
  - 座標ポイント数が最大で、(s1)が座標ポイント1～15の間にある場合、逐次探索回数が15となり処理速度が遅くなる。
- 二分探索の処理速度が遅くなる場合：
  - 最大探索回数が16回のため、(s1)が座標ポイント17以降にある場合、二分探索回数が逐次探索回数より多くなり処理速度が遅くなる。

### 図
- 横軸に座標ポイント(s1)、縦軸に座標ポイント数をとったグラフ。
- 座標ポイント数は32767で固定。
- (1)の位置では逐次探索回数=1、二分探索回数=15。
- (2)の位置では逐次探索回数=32767、二分探索回数=15。
- 凡例により、逐次探索回数が二分探索回数以下の場合は逐次探索の方が速く、逆の場合は二分探索の方が速いことを示している。

### 表
#### エラーコード(SD0)
| エラーコード | 内容 |
| :--- | :--- |
| 3405H | スケーリング用変換データの(s1)より前のポイントのX座標データが昇順に設定されていない場合に発生。<br>ただし、SM755がONの時はこのエラーは検出されない。<br>(s1)で指定した入力値が設定したスケーリング用変換データの範囲外の場合。<br>(s2)のデバイスから座標ポイント数が1～65535の範囲外の場合も該当。 |
