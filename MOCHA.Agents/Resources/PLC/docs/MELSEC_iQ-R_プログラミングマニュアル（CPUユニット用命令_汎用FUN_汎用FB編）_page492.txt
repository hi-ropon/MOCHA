### テキスト
#### 機能概要
(s)で指定されたデバイス番号以降に格納されている文字列を、BIN32ビットデータに変換し、桁数(d1)とBINデータ(d2)に格納する方法について説明しています。文字列→BIN変換では、(s)で指定されたデバイス番号から00Hが格納されているデバイス番号までのデータを文字列として処理します。

#### 変換ルールと注意点
- (s)で指定する文字列の全文字数は2～13文字。
- 小数部の文字数は0～10文字。ただし「全桁数-3」以下にする必要あり。
- BIN値に変換できる文字列の範囲は30H～39H（数字）、符号と小数点は別途指定。
- 符号は正の数が20H、負の数が2DH。
- 小数点は2EH。
- (d1)には全桁数（符号・小数点含む）を格納し、(d1)+1には小数部の桁数を格納。
- (d2)には小数点を無視した文字列をBIN32ビットデータとして格納。

### 図
#### 文字列→BIN32変換のイメージ図
- 左側に(s)以降のデバイスに格納されたASCIIコードの文字列が示されている。
- 文字列の終端は00Hで示される。
- 文字列の各文字はASCIIコードで表され、符号や数字が並ぶ。
- 右側に(d1)に全桁数、(d1)+1に小数部桁数、(d2)に小数点を無視した整数値が格納される流れを矢印で示している。

#### 変換例：「-12345.678」の場合
- (s)以降に「-12345.678」の文字列がASCIIコードで格納されている表。
- それを基に(d1)に10（全桁数）、(d1)+1に3（小数部桁数）を格納。
- (d2)には小数点を無視した「-12345678」のBIN32ビットデータが格納されることを示す。

### 箇条書きの補足説明
- 符号付き数値の範囲は-2147483648～2147483647。
- 符号なしの場合は0～4294967295。
- 小数点以下の文字列は2EH以降で表現。
- BIN32ビットデータは小数点を無視した文字列を変換して格納。
