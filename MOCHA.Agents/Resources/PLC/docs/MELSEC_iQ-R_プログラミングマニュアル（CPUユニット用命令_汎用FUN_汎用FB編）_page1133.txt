### テキスト
#### DTOＤ(P)命令の概要
- (s2)で指定したデバイスから(n)×2点分のデータを、(U/H)で指定したユニットまたは自号機CPUユニット内のバッファメモリの(s1)で指定したアドレス以降に書き込みます。
- 書き込みはユニットへのダブルワードデータライトとして行われます。
- (s2)に定数が指定された場合は、同一データ((s2)で指定した値)を指定したバッファメモリアドレスから(n)×2ワード分に書き込みます。
- 命令実行時に対象ユニットの異常や処理中などでアクセスできない場合は無処理となります。

### 図
1. **ユニットへのダブルワードデータライト（デバイス指定時）**
   - 左側にCPUユニットのメモリ領域が(n)×2点分示され、右側にユニットバッファメモリの(n)×2ワード分の領域が示されている。
   - 矢印でCPUユニットからユニットバッファメモリへデータが書き込まれる様子を表現。

2. **自号機CPUユニットへのダブルワードデータライト（デバイス指定時）**
   - 左側にデバイス/ラベルメモリの(n)×2点分、右側に自号機(U/H)のCPUバッファメモリの(n)×2ワード分が示されている。
   - 矢印でデバイス/ラベルメモリから自号機CPUバッファメモリへデータが書き込まれる様子を表現。

3. **ユニットへのダブルワードデータライト（定数指定時）**
   - 左側にCPUユニットの定数70000が示され、右側のユニットバッファメモリの(s1)から(n)×2ワード分に70000が繰り返し書き込まれている様子を示す。
   - 同一データを複数ワードに書き込むイメージ。

4. **自号機CPUユニットへのダブルワードデータライト（定数指定時）**
   - 左側にデバイス/ラベルメモリの定数70000、右側に自号機CPUバッファメモリの(s1)から(n)×2ワード分に70000が繰り返し書き込まれている様子を示す。

### 注意事項
- 命令実行時に対象ユニットが異常や処理中でアクセスできない場合は無処理となる。
