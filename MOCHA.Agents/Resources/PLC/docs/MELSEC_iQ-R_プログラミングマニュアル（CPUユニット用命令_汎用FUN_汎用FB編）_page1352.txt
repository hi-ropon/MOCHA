### テキスト
#### 32.31 DINT型→INT型変換
「DINT_TO_INT(_E)」関数は、DINT型データをINT型データへ変換します。

##### 機能
- (s)に入力されたDINT型のデータをINT型のデータに変換し、(d)から出力します。
- (s)への入力値はDINT型のデータ値です。

##### 演算結果
1. EN/ENOなしファンクション  
   演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション  
   実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
|-------------|--------------|-----|
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、(d)から出力されたデータを使用しないようプログラム処理を行う必要があります。

##### エラー
- エラーコード 3401H  
  内容：(s)で設定した符号付きBIN32ビットデータが-32768～32767の範囲外のときに発生。

### 表
#### 設定データ（内容、種別、データ型）

| 引数 | 内容 | 種別 | データ型 |
|------|------|------|----------|
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | DINT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

### 図
- ラダー、FBD/LD、ST言語での関数使用例を示す簡単な回路図が掲載されています。
  - ラダー/FBD/LD：[EN/ENOなし]と[EN/ENO付き]の2種類のブロック図。
  - ST言語：[EN/ENOなし] d:=DINT_TO_INT(s); [EN/ENO付き] d:=DINT_TO_INT_E(EN,ENO,s);
