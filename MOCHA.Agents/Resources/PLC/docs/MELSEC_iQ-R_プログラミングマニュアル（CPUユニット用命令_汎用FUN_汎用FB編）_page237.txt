### テキスト
#### 機能
(d)で指定したBIN16ビットデータと、(s)で指定したBIN16ビットデータの減算を行い、その減算結果を(d)で指定したデバイスに格納します。

- 減算結果にアンダーフローが生じた場合、桁下げは無視されます。この場合、SM700（キャリフラグ）はONしません。

#### エラー
演算エラーはありません。

### 図
- 減算の基本例：
  - (d) 5678 (BIN) から (s) 1234 (BIN) を引くと、結果は (d) 4444 (BIN) となる。
- アンダーフロー発生時の例（-(P)の場合）：
  - (d) -12345 (BIN) から (s) 23456 (BIN) を引くと、結果は (d) 29735 (BIN) となる。
  - (d) 12345 (BIN) から (s) -23456 (BIN) を引くと、結果は (d) -29735 (BIN) となる。
- アンダーフロー発生時の例（-(P)_Uの場合）：
  - (d) 56789 (BIN) から (s) 56790 (BIN) を引くと、結果は (d) 65535 (BIN) となる。

各図は、b15からb0までの16ビットのビット列を示し、BIN16ビットデータの減算処理の具体的な数値例を視覚的に表しています。
