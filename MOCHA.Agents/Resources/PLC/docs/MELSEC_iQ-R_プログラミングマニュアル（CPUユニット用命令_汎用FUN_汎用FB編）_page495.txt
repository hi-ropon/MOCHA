### テキスト
#### 機能
- 指定されたデバイス番号(s)以降に、指定された文字数(n)分の16進アスキーデータをBIN値に変換し、指定デバイス番号(d)以降に格納する。
- 文字数(n)を設定すると、文字列の範囲(s)、BINデータ格納範囲(d)が自動決定される。
- ASCIIコードの範囲は30H～39H、41H～46H。
- 変換対象のASCIIデータと変換後のBINデータのデバイス範囲が重複しても正常に処理される。
- 文字数(n)が4の倍数でない場合、BIN値格納の余り部分は自動的に0Hで埋められる。
- 文字数(n)が「0」の場合は変換処理を行わない。

#### 例
- 文字数(n)が9の場合、9文字分のASCIIコードをBIN値に変換し、余りの3桁は0Hで埋める。
- 具体的なASCIIコードとBIN値の対応例を図で示している。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3401H              | (s)に16進数値の文字列以外の文字(30H～39H、41H～46H以外のASCIIコード)が設定されている。 |
| 3405H              | (n)に指定可能な範囲外のデータを入力したとき。指定文字数が0～16383以外。 |

### 図
- ASCIIコードの4文字分（例：ASCII 1～4）をBIN16形式の4桁のBIN値に変換するイメージ図。

- 文字数が4の倍数でない場合のBIN値格納の余り部分に0Hを自動格納する例。
