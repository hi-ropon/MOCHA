### テキスト
- 割込みプログラム中でのEI命令の動作は、EI命令実行前のDI（指定優先度以下の割込み禁止）命令の実行により変わる。
- DI命令実行後の割込みプログラム中のEI命令は実行可能。
- DI命令未実行の場合、割込みプログラム中のEI命令は実行不可（無処理）。
- 割込み用ポインタは1ステップを占有し、例としてI10（ステップ50）、X1C（ステップ51）、Y10（ステップ52）を示す。
- マスタコントロール中にEI命令、DI命令がある場合はMC命令の実行に関わらずEI命令、DI命令を実行する。

### 図（ラダー図）
- 左図：DI命令実行後に割込みプログラムでEI命令を実行した場合の動作。
  - メインルーチンプログラム（A）にてDI命令が実行される。
  - 割込みプログラム（B）でEI命令が実行可能。
- 右図：DI命令未実行で割込みプログラムでEI命令を実行した場合の動作。
  - メインルーチンプログラム（A）でDI命令未実行。
  - 割込みプログラム（B）でEI命令は実行不可（無処理）。

- 割込み用ポインタのラダー図例：
  - I10（ステップ50）、X1C（ステップ51）、Y10（ステップ52）を使用し、IRET命令で割込み終了を示す。
