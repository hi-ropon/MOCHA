### 表
#### 命令処理時間一覧
この表は各命令の処理時間をマイクロ秒（μs）単位で示しています。処理時間はCPUユニット別に分かれており、R00CPU、R01CPU、R02CPU、及びその他のCPUユニットごとに最小値と最大値が記載されています。

| 項目 | 内容 |
| :--- | :--- |
| 命令名 | 各命令の名称（例：NDIS、NUNI、WTOBなど） |
| 条件 | 命令実行時の条件やパラメータ（例：(n)=1、(n)=96、SM773=ON/OFFなど） |
| 処理時間（μs） | 各CPUユニットごとの処理時間の最小値と最大値 |

##### 主な特徴
- NDIS、NUNI命令は実行時にR00CPU、R01CPUで3.7～6.5μs、R02CPUで2.7～5.5μs、その他CPUユニットで2.2～5.0μsの処理時間。
- WTOB命令はパラメータnによって処理時間が変動し、n=1の場合はR00CPUで4.0～7.0μs、n=96の場合は14.5～17.5μsと大幅に増加。
- SMOV命令はSM773のON/OFF状態によって処理時間が異なり、OFF時はR00CPUで5.0～10.0μs、ON時は4.7～7.6μs。
- ビット操作命令（ROR、RCR、DRORなど）は処理時間が非常に短く、約0.28μs前後で一定。
- CJ命令はR00CPUで4.0～4.7μs、R02CPUで1.3～2.3μs、その他CPUユニットで1.0～2.0μsの処理時間。

この表はCPUの種類や命令の条件によって処理時間がどのように変化するかを詳細に示しており、システムの性能評価や最適化に役立つ情報となっています。
</contents>
