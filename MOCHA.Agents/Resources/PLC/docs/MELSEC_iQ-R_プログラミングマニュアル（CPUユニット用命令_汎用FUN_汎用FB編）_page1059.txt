### テキスト
#### 注意事項
完了デバイス(d1)+1の異常完了信号がONになり、(d2)の完了ステータスにエラーコードが格納される場合について説明している。以下のケースが該当する。

- データベースアクセス命令実行中に本命令を実行した場合
- (s1)で範囲外の識別番号が指定された場合
- (s1)で未オープンの識別番号を指定した場合
- ファームウェアバージョンが"28"より前のシーケンサCPUで、(s1)に"28"以降のファームウェアバージョンが構築または操作したデータベースを指定した場合
- (s2)で指定したテーブル名が存在しない場合
- (s2)で指定したテーブル名の文字数が32文字を超える場合
- (s3)で削除条件のサイズを範囲外の値に設定した場合
- データベースのDELETE処理に失敗した場合
- (s3)で設定する条件間演算値を範囲外の値に設定した場合
- (s3)+35で指定する条件間演算値が1または2で、(s3)+1に0を設定している場合
- (s3)+2で設定するフィールド名が未設定の場合
- (s3)+2～(s3)+□で指定したフィールド名に、範囲外のフィールド名を設定した場合

### 表
#### エラーコード一覧
| エラーコード (SD0) | 内容 |
| :----------------- | :--- |
| 2820H              | (s2)、(s3)、(d1)で指定した範囲が該当デバイス/ラベルを超える場合 |

オペランドの完了ステータスに格納されるエラーコードについては、1066ページの「データベースアクセス命令で発生するエラーコード」を参照するよう案内がある。
