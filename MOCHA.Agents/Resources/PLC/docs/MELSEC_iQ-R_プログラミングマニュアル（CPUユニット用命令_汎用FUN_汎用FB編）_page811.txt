### テキスト
#### SM705（変換桁数切換）がOFFの場合の動作
(d)～(d)+3には、10進アスキーデータが固定桁数（8桁）で格納されます。

- ASCIIコードの桁ごとの意味は以下の通りです。
  - ASCII 10^7：千万の位のアスキーコード
  - ASCII 10^6：百万の位のアスキーコード
  - ASCII 10^1：十の位のアスキーコード
  - ASCII 10^0：一の位のアスキーコード

- (1)の説明として、(d)+(4)には00Hが格納され、SM701（出力文字数切換）がOFFのときは00Hが格納され、ONのときは変化しません。

- (d)に格納する演算結果で、有効桁数の左側の00には20H（スペース）が格納されます（0サプレス）。例えば「00012098」の場合、「000」はそれぞれ20H（スペース）となり、「12098」が有効桁数となります。

#### 例
(s)に01234056を指定した場合の説明。

### 図
- ビット配置図：
  - (s)+1のビット範囲はb31～b16、(s)のビット範囲はb15～b0で、それぞれのビットが10のべき乗に対応。
- 格納イメージ：
  - (d)～(d)+3にASCIIコードが8桁分格納される様子を示す。
- 例のビット配置：
  - (s)+1に「0 1 2 3」、(s)に「4 0 5 6」がビット単位で格納されている図。

### 表
- ASCIIコードの桁ごとの対応表（b15～b0）：

| アドレス | b15～b8 | b7～b0 |
| :------- | :------ | :----- |
| (d)      | ASCII 10^6 | ASCII 10^7 |
| (d)+1    | ASCII 10^4 | ASCII 10^5 |
| (d)+2    | ASCII 10^2 | ASCII 10^3 |
| (d)+3    | ASCII 10^0 | ASCII 10^1 |
| (d)+4    | 00H       |          |

- 例の16進コード対応表：

| アドレス | b15～b8 | b7～b0 |
| :------- | :------ | :----- |
| (d)      | 31H (1) | 20H (SP) |
| (d)+1    | 33H (3) | 32H (2)  |
| (d)+2    | 30H (0) | 34H (4)  |
| (d)+3    | 36H (6) | 35H (5)  |
| (d)+4    | 00H     |          |

### 補足
- 20Hはスペースコードであり、0サプレス時に有効桁数の左側に挿入される。
- 00Hは文字列の終端を示すコードとして格納される。
