### テキスト
#### 機能
- (s1)、(s2)に入力されたTIME型データの乗算((s1)×(s2))を行い、演算結果を(d)からTIME型で出力する。

#### 演算処理
- (s1)への入力値はTIME型データ値。
- (s2)への入力値はINT型/DINT型/REAL型/LREAL型のいずれか。
- 演算結果にアンダーフローやオーバーフローが発生しても演算エラーにはならず、(d)に特定の値で出力される。
- MUL_TIME_Eの場合、出力変数ENOからTRUEが出力される。演算結果は64ビットデータだが、出力は上位32ビットを削除した時間型データで出力される。

#### 例
- 通常例：s1に「T#1d2h33m44s55ms」、s2に「2」を入力した場合、演算結果(d)は「T#2d5h7m28s110ms」となる。
- オーバーフロー例：s1に「T#24d20h31m23s647ms(7FFFFFFFH)」、s2に「2」を入力した場合、演算結果(d)は「T#-2ms(FFFFFFFEH)」となり、最上位ビットが1のため負の時間となる。
- アンダーフロー例：s1に「T#-24d20h31m23s648ms(80000000H)」、s2に「2」を入力した場合、演算結果(d)は「T#0ms(00000000H)」となり、最上位ビットが0のため正の時間となる。

#### 演算結果
1. EN/ENOなしファンクション  
   - 演算処理を実行し、(d)から演算出力値が出力される。
2. EN/ENO付きファンクション  
   - 実行条件ENと演算結果ENOの組み合わせにより、演算結果が変わる。  
   - ENがTRUEのときはENOもTRUEで演算出力値が(d)に出力される。  
   - ENがFALSEのときはENOがFALSEとなり、(d)の値は不定値となる。  
   - ENOがFALSEの場合は(d)のデータは不定値となるため、そのデータを使用しないようプログラムで処理する必要がある。
