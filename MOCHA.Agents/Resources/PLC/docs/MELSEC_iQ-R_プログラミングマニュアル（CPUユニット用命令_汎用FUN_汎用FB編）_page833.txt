### テキスト
#### 機能
(s1)で指定されたデバイス番号以降に格納されている文字列データに対し、(s2)で指定された位置から(s2)+1で指定された文字分のデータを、(d)で指定されたデバイス番号以降に格納します。

- 文字列の終端を示すNULLコード(00H)が文字列データの最後に自動付加されます。
- (s2)+1で指定する文字数が0の場合は処理を行いません。
- (s2)+1で指定する文字数が-1の場合は、(s1)で指定された最終文字データまでを(d)で指定されたデバイス以降に格納します。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 2820H | (s1)で指定したデバイス番号以降、デバイス/ラベルメモリの各設定エリアに00Hが存在しないとき。 |
| 3405H | (s1)の文字列が16383文字を超えるとき。<br>(s2)に指定可能な範囲外のデータを設定したとき。<br>・(s2)の値が0以下<br>・(s2)+1の値が有効値(-1, 0, 1以上)以外<br>・(s2)の値が(s1)の文字数を超えている<br>・(s2)と(s2)+1を加算した値が(s1)の文字数を超えている |

### 図
- 文字列「ABCDEFGHIJK」が16進数コードで表され、(s1)で指定された位置から(s2)で指定した5文字分を(d)で指定した位置に格納する例を示しています。
- 1つ目の図は(s2)+1が5の場合で、「EFGHI」が(d)の位置に格納される様子を示しています。
- 2つ目の図は(s2)+1が-1の場合で、(s1)の最終文字まで「EFGHIJK」が(d)の位置に格納される様子を示しています。
- 各文字は16進数のASCIIコードで表現され、NULLコード(00H)が文字列の終端を示しています。
