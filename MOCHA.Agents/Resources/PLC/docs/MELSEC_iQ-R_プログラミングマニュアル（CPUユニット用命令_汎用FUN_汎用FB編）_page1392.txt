### テキスト
#### 32.56 STRING型→INT型変換
STRING_TO_INT(_E)関数は、STRING型データをINT型データに変換します。

##### 対応CPU
RnCPU、RnENCPU、RnPCPU（プロセス・三重化・一般）、RnSPFCPU（安全・一般）、RnSfCPU（安全）

##### 使用例
- ラダー（FBD/LD）では、EN/ENOなしとEN/ENO付きの2種類の回路図が示されています。
- ST（構造化テキスト）では、EN/ENOなしの場合は `d:=STRING_TO_INT(s);`、EN/ENO付きの場合は `d:=STRING_TO_INT_E(EN,ENO,s);` と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | STRING(6) |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

#### 機能
##### 演算処理
- 入力されたSTRING型データ(s)をINT型データ(d)に変換して出力します。
- STRING型の各バイトは以下のようにASCIIコードで構成されており、4バイト目は文字列の終端を示す00Hです。

| 上位バイト | 下位バイト |
| --- | --- |
| 万の位のアスキーコード | 符号のアスキーコード |
| 百の位のアスキーコード | 千の位のアスキーコード |
| 一の位のアスキーコード | 十の位のアスキーコード |
| 00H（文字列の終端） |  |

- 入力値の範囲は以下の通りです。
  - アスキーコード：30H～39H、20H、2DH、00Hの範囲
  - STRING型のデータ値：-32768～32767の範囲

##### 演算結果
1. EN/ENOなしファンクション  
   演算処理を実行し、dから演算出力値が出力されます。

2. EN/ENO付きファンクション  
   実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | 演算結果 (d) |
| --- | --- | --- |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となります。その場合はdから出力されたデータを使用しないようプログラム処理を行う必要があります。
