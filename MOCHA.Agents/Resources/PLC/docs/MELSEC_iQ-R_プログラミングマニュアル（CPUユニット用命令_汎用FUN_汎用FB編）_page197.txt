本ページは、マスタコントロール命令に関する説明で構成されています。上部に使用可能デバイスの一覧表があり、その下にマスタコントロール命令の機能説明、エンジニアリングツール上の回路図と実際の動作回路の比較図が配置されています。さらに、MC命令の動作詳細を示す表があり、デバイスの状態とその動作内容がまとめられています。テキスト説明、表、回路図（図解）の3つのセクションで構成されています。

### テキスト
#### 使用可能デバイス
- オペランドごとに使用可能なビット、ワード、ダブルワード、間接指定、定数、その他のデバイスが表形式で示されています。
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、特定の安全デバイスおよび定数のみ使用可能であることが記載されています。

#### 機能
- マスタコントロール命令は回路の共通母線を開閉し、効率的な回路切換えプログラムを作成するための命令であると説明されています。
- マスタコントロールを使用した回路の例として、左側にエンジニアリングツール上の表示、右側に実際の動作回路が示されています。
- 図中の注釈として「X0がONのときのみ実行」と記載されています。

#### MC命令の動作
- MC命令の開始時に実行指令がONの場合、MC命令からMCR命令間の演算結果は命令（回路）どおりとなる。
- MC命令の実行指令がOFFの場合、MC命令からMCR命令間の演算結果は表に示された状態になる。
- MC命令がOFFでもMC命令からMCR命令間の命令は実行されるため、スキャンタイムは短縮されない。

### 表
#### デバイスの状態と動作内容

| デバイス                     | デバイスの状態説明                                                                                   |
|-----------------------------|----------------------------------------------------------------------------------------------------|
| 高速タイマ、低速タイマ       | カウント値は0となり、コイル、接点ともOFFになる                                                     |
| 高速積算タイマ、低速積算タイマ | コイルはOFFになるがカウント値、接点とも現在の状態を保持する                                         |
| カウンタ                     | 同上                                                                                               |
| OUT命令中のデバイス          | 強制的にOFFにする                                                                                   |
| SET命令、RST命令中のデバイス | 現在の状態を保持する                                                                                 |
| SFT(P)命令中のデバイス       | 同上                                                                                               |
| 基本命令、応用命令中のデバイス | 同上                                                                                               |

### 図
- 左図はエンジニアリングツール上のマスタコントロールを使用した回路表示で、X0、N1、M0などの接点やコイルが配置されている。
- 右図は実際の動作回路で、左図の回路を共通母線でまとめて効率的に制御している様子を示す。
- 両図の比較により、マスタコントロール命令による回路の効率的な切換え方法が視覚的に理解できる。
