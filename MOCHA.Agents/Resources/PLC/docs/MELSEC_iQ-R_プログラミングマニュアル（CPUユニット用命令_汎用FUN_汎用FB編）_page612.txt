### テキスト
#### RCL(P)命令の概要
- (d)で指定されたデバイスのBIN16ビットデータを、SM700（キャリフラグ）を含めて(n)ビット左へ回転させる命令です。
- SM700（キャリフラグ）は、RCL(P)命令の実行前の状態によりONまたはOFFになります。
- (d)にビットデバイスが指定された場合は、桁指定で指定されたデバイス範囲で回転します。回転するビット数は(n)を桁指定で指定された点数の余りで計算されます。
- (n)は0～15を指定し、16以上の値が指定された場合は(n)を16で割った余りの値で回転します。

#### エラー
- 演算エラーはありません。

### 図
- ビット列の左回転の動作を示しています。
- 最初のビット列（b15～b0）とSM700（キャリフラグ）の状態が示され、1ビット左回転するとビットが一つ左にずれ、キャリフラグの値が最下位ビットに入る様子が描かれています。
- さらに1ビット左回転すると、元のb15の値がキャリフラグに入り、キャリフラグの値がビット列の最下位ビットに入る動作が繰り返されます。
- (n)ビット左回転のイメージとして、ビット列が左にnビット回転し、キャリフラグの値がビット列の最下位に入ることを示しています。
