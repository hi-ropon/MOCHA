<overview>
このページは、プログラム制御におけるCALL(P)命令の使用に関する説明で構成されています。テキスト部分では、CALL(P)命令の引数に使用したデバイスをサブルーチンプログラムで使用してはいけない旨の注意が記載されています。表や表形式のデータはなく、図部分としてはラダー図とそれに対応するデバイスの値の変化を示す図解が含まれています。図はCALL命令の動作例を示し、サブルーチンプログラム実行前からRET命令実行後までのデバイスの値の変化を段階的に説明しています。
</overview>

<contents>
### テキスト
- CALL(P)命令の引数に使用したデバイスをサブルーチンプログラムで使用してはいけない。使用した場合は正常な演算ができない。

### 図
- ラダー図例:
  - メインプログラムP0で、M0がONのときCALL命令でP0のサブルーチンプログラムを呼び出し、引数にD0を指定。
  - サブルーチンプログラム内では、M0がONのとき、K100をK3で除算した結果をFD0に格納し、K100をD1に加算している。
  - 最後にRET命令でサブルーチンプログラムを終了。

- デバイス値の変化図:
  - サブルーチンプログラム実行前のD0～D3とFD0の値を示す。
  - CALL命令実行直後、D0～D3の値は変わらず、FD0にサブルーチンプログラムの引数値が格納される。
  - サブルーチンプログラム実行時、D1の値が110に変化し、FD0の値も変化。
  - RET命令実行後、D0は33に、D1は110のまま、D2とD3は変わらず。FD0は不定値になる。

- 注釈:
  1. サブルーチンプログラムの実行結果が格納される。
  2. ファンクションデバイスの値に置き換わる。
  3. D1はファンクションデバイスの値が反映されない。
  4. 不定値になる。

この内容は、CALL命令の引数に指定したデバイスをサブルーチンプログラム内で使用すると、正常な演算結果が得られないことを具体的なラダー図とデバイス値の変化例で示しています。
</contents>
