### テキスト
#### BIN16ビットデータ最大値検索
「MAX(P)(_U)」命令は、指定したデバイスからn点のBIN16ビットデータの中から最大値を検索する命令です。  
対応するCPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）となっています。  
ST命令では「ENO:=MAXP(EN,s,n,d);」「ENO:=MAXP_U(EN,s,n,d);」の形式で使用します。  
FBD/LD命令はST命令と異なり対応していません。汎用ファンクションのMAXを使用するよう案内があります。  

##### 実行条件
- MAX、MAX_U命令は立下りエッジで実行
- MAXP、MAXP_U命令は立上りエッジで実行

### 図
- ラダー図：s、d、nの3つの入力とENOの出力を持つ命令ブロックの形状が示されています。
- FBD/LD図：EN、s、nの入力とENO、dの出力を持つブロック図が示されています。

| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s) MAX(P) / MAX(P)_U | サーチするデータが格納されている先頭デバイス | － | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S*1 / ANY16_U*1 |
| (d) MAX(P) / MAX(P)_U | 最大値の検索結果を格納する先頭デバイス | － | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S_ARRAY(要素数:3) / ANY16_U_ARRAY(要素数:3) |
| (n) | サーチデータ数 | 0～65535 | 符号なしBIN16ビット | ANY16 |
| EN | 実行条件 | － | ビット | BOOL |
| ENO | 実行結果 | － | ビット | BOOL |

*1 ラベルで設定する場合は、動作に必要な領域が確保できる配列を定義し、その配列型ラベルの要素を指定する必要があります。
