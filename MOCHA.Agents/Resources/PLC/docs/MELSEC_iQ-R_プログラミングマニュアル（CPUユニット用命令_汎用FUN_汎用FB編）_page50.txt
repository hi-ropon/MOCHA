### テキスト
#### 1.4 命令処理時間の高速化 - サブセット処理
サブセット処理とは、サブセット処理が可能な命令において、各オペランドで指定したデバイスやラベルが特定の条件を満たす場合に、ステップ数の減少や命令の処理時間の高速化を実現する仕組みです。条件を満たさない場合は命令記号やオペランドの数は変わりません。

##### サブセット処理が可能な命令
各命令のサブセット処理の可否については、別途1557ページの基本ステップ数とサブセット処理の可否を参照するよう案内されています。

##### オペランドの条件
サブセット処理となるためのオペランドが満たすべき条件を示しています。

- ■デバイスを指定した場合  
  オペランドでデバイスを指定した場合の条件を表で示しています。

### 表
| オペランドのデータ型 | 条件*1 |
|----------------------|---------|
| ビットデータ | 以下のいずれかを満足：ユーザデバイス、CPUバッファメモリアクセスデバイスの自号機指定（U3Enへのインデックス修飾除く）、CPUバッファメモリアクセスデバイスの定周期通信エリアの他号機指定、ファイルレジスタ、ローカルデバイス、リフレッシュデータレジスタ |
| 符号付き16ビットデータ、符号なし16ビットデータ、符号付き32ビットデータ、符号なし32ビットデータ | 以下のいずれかを満足：ユーザデバイス、CPUバッファメモリアクセスデバイスの自号機指定（U3Enへのインデックス修飾除く）、CPUバッファメモリアクセスデバイスの定周期通信エリアの他号機指定、インデックスレジスタ、ファイルレジスタ、ローカルデバイス、リフレッシュデータレジスタ、定数（10進数、16進数） |
| 単精度実数 | 以下のいずれかを満足：ユーザデバイス、CPUバッファメモリアクセスデバイスの自号機指定（U3Enへのインデックス修飾除く）、CPUバッファメモリアクセスデバイスの定周期通信エリアの他号機指定、インデックスレジスタ、ファイルレジスタ、ローカルデバイス、リフレッシュデータレジスタ、定数（単精度実数） |

*注釈*  
*1 ビットNo.指定、桁指定、間接指定、インデックス修飾したデバイスも含む。  
*2 自号機のCPUバッファメモリアクセスデバイスにて、U3En¥G□、U3En¥G□Zn、U3En¥HG□、U3En¥HG□Znを使用した場合に該当。  
*3 他号機のCPUバッファメモリアクセスデバイスにて、U3En¥HG□、U3En¥HGDZnを使用した場合に該当。

- ■デバイスを割り付けたラベルを指定した場合  
  デバイス指定時と同様の条件となります。

- ■各ラベルエリアに割り付けたラベルを指定した場合  
  オペランドでラベルエリアやラッチラベルエリアに割り付けたラベルを指定した場合、サブセット処理が可能な命令であれば、オペランドのデータ型に関係なくサブセット処理となります。ビットNo.指定や桁指定の場合も含みます。
