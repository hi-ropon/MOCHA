### テキスト
#### ブロック終了 RST [BL□]
指定したブロックを単独で非活性化する命令です。

対応CPUは以下の通りで、バージョンによる制約があります。
- R00CPU、R01CPU、R02CPUは制約なし。
- RnCPU（R00CPU、R01CPU、R02CPU除く）、RnENCPUはファームウェアバージョン12以降で使用可能。エンジニアリングツールはバージョン1.015R以降対応。
- プロセスCPU（プロセスモード）はファームウェアバージョン03以降で使用可能。ツールは1.020W以降対応。
- プロセスCPU（二重化モード）はファームウェアバージョン18以降で使用可能。ツールは1.050C以降対応。

##### 機能
- 指定したブロックを単独で非活性化する。
- 活性中のステップがある場合はすべて非活性化し、コイル出力もOFFにする。
- SFC用情報デバイスのブロック起動終了ビットを設定している場合、そのビットデバイスはONからOFFになる。
- 命令実行時に指定ブロックがすでに非活性状態の場合は命令を無処理（NOP相当）として継続。
- プロセスCPU（二重化モード）ではバックアップモードの待機系で実行しても無処理となる。

### 図
- **ラダー図**  
  指定ブロック(d)のリセットを示すラダー図が掲載されています。

- **ST（構造化テキスト）**  
  `ENO := RST(EN, d);` という形式で記述。

- **FBD/LD**  
  EN（入力）、ENO（出力）、d（指定ブロック）を持つブロック図。
