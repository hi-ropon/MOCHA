### テキスト
#### 機能
(s)で指定された符号付きBIN32ビットデータを、符号付きBIN16ビットデータに変換し、(d)で指定されたデバイスに格納します。  
プログラム例では、M0がONしたときにD0～D1に格納された符号付きBIN32ビットデータを符号付きBIN16ビットデータに変換し、D100へ格納します。

### 図
- プログラム例の回路図  
  SM402の接点とM0の接点があり、M0がONするとDMOVP命令でD0に16進数HFFFF9876のデータを格納し、DINT2INT命令でD0の32ビットデータを16ビットに変換してD100に格納する流れを示しています。  
- ビット構成の説明  
  - D1, D0に格納された符号付きBIN32ビットデータは、上位16ビット（b31～b16）がFFFFH、下位16ビット（b15～b0）が9876Hで、符号付き16ビットデータとして-26506を表しています。  
  - 変換後のD100には下位16ビットの9876H（-26506）が格納されます。
