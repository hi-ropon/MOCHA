### テキスト
#### 機能
- (s)で指定されたBIN16ビットデータを16進数で表現した数値をアスキーコードに変換し、(d)で指定されたデバイス番号以降に格納する。
- SM705（変換桁数切換）の状態により、(d)に格納される16進アスキーデータの形式が異なる。

#### 動作の概要
- SM705がOFFの場合とONの場合の動作をそれぞれ説明。
- SM705に対応しているCPUユニットのファームウェアバージョンについては別途マニュアル参照。
- SM705に対応していないCPUユニットでは、SM705をONにしてもOFFの場合と同様の動作となる。

### 表
| SM705の状態 | (d)の格納形式 | 参照先 |
| :--- | :--- | :--- |
| OFF | 固定桁数（4桁）で格納 | 793ページ SM705(変換桁数切換)がOFFの場合の動作 |
| ON | (s)の値によって各桁を前詰めで格納 | 793ページ SM705(変換桁数切換)がONの場合の動作 |

### 図（動作例）
- (s)にBIN16ビットデータ「0ABCH」が格納されている状態でBINHA(P)命令を実行した場合の動作を示す。
- SM705がOFFの場合：
  - (d)+0に「41H」「30H」、(d)+1に「43H」「42H」が格納される。
  - これは「0ABC」の4桁がアスキーデータに変換されている状態。
- SM705がONの場合：
  - (d)+0に「42H」「41H」、(d)+1に「00H」「43H」が格納される。
  - 「0ABC」の先頭の0を省略した「ABC」がアスキーデータに変換され、終端に00Hが格納されている。

### 補足
- SM705がOFFの場合は固定桁数での格納となり、常に4桁のアスキーデータに変換される。
- SM705がONの場合は前詰めで格納され、先頭の0は省略されるが、終端に00Hが付加される。
