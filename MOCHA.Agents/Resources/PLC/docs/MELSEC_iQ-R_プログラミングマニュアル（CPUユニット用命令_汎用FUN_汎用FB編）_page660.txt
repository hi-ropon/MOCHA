### テキスト
#### ファイルの最後に追加
- 実行/完了タイプはCSV形式変換書き込み（10進数、符号付き16ビットデータ）
- 使用用途設定エリアは0H
- ファイル位置はFFFFFFFFH（ファイルの最後に追加）
- 列数指定は3H
- データタイプ指定はワード
- 書き込み先頭デバイスはD7
- 要求書き込みデータ数は8H

#### Point（注意点）
- 「要求書き込みデータ数」は「列数指定」の整数倍に設定する必要がある。整数倍でない場合、列数がバラバラになる。
- 最後のデータの後ろには必ず改行コードが入るため、使用用途設定エリアが0の場合は新しい行の先頭から追加される。
- ファイルの最後に追加する場合、「列数指定」を前回の書き込み時から変更すると列数がずれる。

### 図（CSV形式変換書き込みデータの配置と対応関係）
- 左側にCSV形式のデータ配置を示す表があり、1行目から5行目までのデータが列ごとに区切られている。各データの後にはCR（キャリッジリターン）とLF（ラインフィード）が付加されている。
- 右側にデバイス格納データ（書き込みデータ）の対応表があり、K6～K14の値がD0～D15のデバイスに対応している。
- 矢印でCSVデータの各値がどのデバイスに格納されるかを示している。
- Nはデータ数を示し、D7が書き込み先頭デバイスとして指定されている。

(1)は今回の開始位置を示し、(2)はデバイス格納データ（書き込みデータ）を表す。

---

以上より、本ページはCSV形式のファイル書き込みにおけるデータの配置方法と注意点を説明している。
