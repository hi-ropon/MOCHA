### テキスト
#### 機能
- XCALL命令はサブルーチンプログラムの実行と非実行処理を行う命令である。
- サブルーチンプログラムの実行は、各コイル命令を条件接点のON/OFF状態より演算する。
- 非実行処理は、各コイル命令を条件接点がOFF状態と同一の処理を行う。
- 非実行処理後の各コイル命令の演算結果は条件接点のON/OFFに関係なく、表に示す通りとなる。

### 表
#### 演算で使用するデバイスと演算結果（デバイスの状態）

| 演算で使用するデバイス                         | 演算結果（デバイスの状態）           |
|----------------------------------------------|------------------------------------|
| 高速タイマ、低速タイマ                         | 0となる                            |
| 高速積算タイマ、低速積算タイマ、カウンタ       | 現在の状態を保持する                |
| OUT命令中のデバイス                            | 強制的にOFFとなる                   |
| SET命令、RST命令中のデバイス、SFT(P)命令中のデバイス、基本/応用命令中のデバイス | 現在の状態を保持する                |
| PLS命令、パルス化命令(DP)                      | 条件接点OFFと同一処理となる        |

### 図
- ラダー図：X0の条件でXCALL命令が実行され、P1のサブルーチンプログラムが呼び出されている。
- タイミングチャート：
  1. X0の立上り（OFF→ON）でP1のサブルーチンプログラムを実行。
  2. X0のON中はP1のサブルーチンプログラムを実行（ただしX0の立上りは含まない）。
  3. X0の立下り（ON→OFF）でP1のサブルーチンプログラムの非実行処理を行う。

### 注釈
- (1) X0の立上りでP1のサブルーチンプログラムを実行する。
- (2) X0がONの間はP1のサブルーチンプログラムを実行し続けるが、立上りは含まれない。
- (3) X0の立下りでP1のサブルーチンプログラムの非実行処理を行う。
