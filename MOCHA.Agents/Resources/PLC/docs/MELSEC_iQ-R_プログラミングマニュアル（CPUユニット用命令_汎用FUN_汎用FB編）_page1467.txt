### テキスト
#### NE(_E)関数の概要
NE(_E)関数は入力値のデータ比較結果を出力します。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）など複数の種類に対応しています。

#### 注意事項
- バージョンが1.035M以降のエンジニアリングツールで対応しており、それ以前のバージョンでは演算子として記述します。
- 安全プログラムではREAL型、LREAL型、STRING型データを指定しないこと。指定した場合はエラーとなります。

### 図
- ラダー、FBD/LD形式の回路図例
  - EN/ENOなしの場合：s1、s2の2つの入力とdの出力がある比較ブロック。
  - EN/ENO付きの場合：EN（実行条件）、ENO（出力状態）、s1、s2の入力、dの出力がある比較ブロック。
- ST形式の記述例
  - EN/ENOなし：d := NE(s1, s2);
  - EN/ENO付き：d := NE_E(EN, ENO, s1, s2);

### 表
#### 設定データ（内容、種別、データ型）

| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s1, s2 | 入力 | 入力変数 | ANY_ELEMENTARY*¹ |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力(TRUE:真値、FALSE:偽値) | 出力変数 | BOOL |

*¹ ANY_ELEMENTARYは基本データ型を示す。  
* 安全プログラムではREAL型、LREAL型、STRING型データを指定しないこと。指定した場合はエラーとなる。
