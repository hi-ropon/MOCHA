### テキスト
#### 機能
(d)で指定されたデバイスのBIN32ビットデータと、(s)で指定されたデバイスのBIN32ビットデータの否定排他的論理和（XNR）を求め、その結果を(d)で指定されたデバイスに格納する機能について説明しています。

- ビットデバイスの場合、桁指定による点数以降のビットデバイスは0として演算されます。

#### エラー
演算エラーはありません。

### 図
ビット配置の例として、(d)と(s)で指定されたBIN32ビットデータのビット列が示されています。

- (d)のビット列はb31からb0までの32ビットで構成されており、具体的なビット値が示されています。
- (s)のビット列も同様に32ビットで示されており、(d)のビット列と比較されています。
- これらのビット列に対してXNR演算を行い、結果が(d)のビット列に格納される様子が矢印で示されています。
- 演算結果のビット列も具体的なビット値で示されています。

この図は、BIN32ビットデータの否定排他的論理和演算の動作をビット単位で視覚的に理解できるようにしています。
