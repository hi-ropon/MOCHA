このページは主にテキストと図で構成されています。テキスト部分では、指定されたデバイスのBIN32ビットデータに対する論理演算の機能説明とエラーに関する記述があります。図部分では、ビットごとの排他的論理和（XOR）演算の具体例が示されています。表やグラフは含まれていません。
</overview>

<contents>
### テキスト
#### 機能
- (s1)で指定されたデバイスのBIN32ビットデータと、(s2)で指定されたデバイスのBIN32ビットデータのビットごとの排他的論理和（XOR）を求め、その結果を(d)で指定されたデバイスに格納する。
- ビットデバイスの場合、桁指定による点数以降のビットデバイスは0として演算する。

#### エラー
- 演算エラーはありません。

### 図
- 図は3つのBIN32ビットデータのビット配列を示している。
  - (s1)のビット配列：b31からb0までのビットが並び、例として「1111...00001111」となっている。
  - (s2)のビット配列：同様にb31からb0までのビットが並び、「1111...1010100」となっている。
  - (d)のビット配列：(s1)と(s2)のXOR演算結果が格納されており、「0000...10100111」となっている。
- 各ビットの位置が示され、XOR演算の流れが矢印で表現されている。

</contents>
