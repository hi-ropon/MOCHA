### テキスト
#### 機能
(d)で指定されたBIN32ビットデータと、(s)で指定されたBIN32ビットデータの減算を行い、その減算結果を(d)で指定したデバイスに格納します。

- 減算結果にアンダーフローが発生した場合、桁上げは無視されます。この場合、SM700（キャリフラグ）はONしません。

#### エラー
演算エラーはありません。

### 図
- BIN32ビットデータの減算例が3つ示されています。

1. 正常な減算例：
   - (d)+1と(d)のビット範囲はb31からb0まで。
   - 567890 (BIN) から 123456 (BIN) を減算し、結果 444434 (BIN) を得る。

2. アンダーフローが発生した場合の例（[D-(P)の場合]）：
   - 1234567890 (BIN) から -987654321 (BIN) を減算し、結果 -2072745085 (BIN) となる。
   - また、-1234567890 (BIN) から 987654321 (BIN) を減算し、結果 2072745085 (BIN) となる。

3. [D-(P)_Uの場合]の例：
   - 3456789012 (BIN) から 3060399406 (BIN) を減算し、結果 39389606 (BIN) となる。

各図はビット範囲の指定と数値の減算過程を示し、減算結果がどのように格納されるかを視覚的に説明しています。
