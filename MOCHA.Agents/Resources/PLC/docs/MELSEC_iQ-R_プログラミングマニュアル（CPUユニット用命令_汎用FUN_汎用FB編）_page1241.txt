### テキスト
- 入出力データエリアで「ライト」となっているデータは、ユーザープログラムで書き込む必要がある。逆に「リード」データはプログラムで読み出して使用する。
- 「リード/ライト禁止」や「リード」となっているデータに書き込みを行うと正常な演算ができなくなるため、絶対に行わないこと。ただし、初期状態から制御を開始する場合はプログラムでデータクリアが必要。
- 使用ループ分のデバイス点数が指定した最終デバイス番号を超えるとエラーとなり処理されない。
- 手動モードで手動操作量(MV_MAN)の出力を行う場合は、S(P).PIDCONT命令を毎スキャン実行する必要がある。実行しないとバンプレス機能が働かない。
- S(P).PIDCONT命令は、測定値(PV)の取り込み用A/D変換ユニットと操作量(MV)出力用D/A変換ユニットが正常なときのみ実行されるよう、各ユニットのREADY信号でインターロックを行う必要がある。
- 各ユニット異常時は正常なPV取り込みやMV出力ができず、PID演算も正常に行えない。

#### 図
- S.PIDCONT命令のタイミングを示すタイムチャート
  - (1) 制御指令
  - (2) A/D変換ユニットのREADY信号
  - (3) D/A変換ユニットのREADY信号
  - 命令はこれらのREADY信号を確認しながら実行される。
