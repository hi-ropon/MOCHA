### テキスト
#### 32.27 INT型→STRING型変換
INT_TO_STRING(_E)関数は、INT型データをSTRING型データへ変換します。

対応CPUはRnCPU、RnENCPU（プロセス、三重化、一般、安全）で、RnSFCCPU（一般、安全）には対応していません。

#### 使用例（ラダー、FBD/LD）
- EN/ENOなしの場合：d = INT_TO_STRING(s);
- EN/ENO付きの場合：d = INT_TO_STRING_E(EN, ENO, s);

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | INT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | STRING(6) |

#### 機能
##### 演算処理
- 入力されたINT型データ(s)をSTRING型データ(d)に変換して出力します。
- 変換後のSTRINGは4ワード（6バイト）で構成され、上位バイトに万位、百位、一の位のアスキーコード、下位バイトに符号や千位、十位のアスキーコードが格納されます。
- 符号は、正の値の場合は20H（スペース）、負の値の場合は2DH（マイナス）が格納されます。
- 有効桁数が少ない場合は、上位桁に20H（スペース）を格納します。
- 文字列の最後（4ワード目）には、SM701（出力文字数切換）がOFFの場合、00Hが格納されます。

#### 例
-123を入力した場合の変換例が示されています。  
上位バイトには20H（スペース）、31H（'1'）、33H（'3'）が入り、下位バイトには2DH（'-'）、20H（スペース）、32H（'2'）が入り、最後に00Hが格納されます。
