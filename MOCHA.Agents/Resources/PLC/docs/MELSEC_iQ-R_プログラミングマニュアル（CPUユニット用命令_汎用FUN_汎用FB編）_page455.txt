### テキスト
#### 機能
(s)で指定された符号なしBIN32ビットデータを、符号なしBIN16ビットデータに変換し、(d)で指定されたデバイスに格納します。

#### プログラム例の説明
M0がONしたとき、D0～D1に格納された符号なしBIN32ビットデータを符号なしBIN16ビットデータに変換し、D100へ格納します。

#### エラー
演算エラーはありません。

### 図の説明
図は、SM402のM0がONの状態を示し、D0とD1に格納された32ビットデータ（4567Hと89ABH）が、(s)の指定により符号なしBIN32ビットデータとして扱われています。このデータは、UDINT2UINT命令により、(d)の指定であるD100に符号なしBIN16ビットデータ（89ABH）として格納されます。

- b31～b0は32ビットのビット位置を示し、D1が上位16ビット（4567H）、D0が下位16ビット（89ABH）を表します。
- D100には下位16ビットのデータ（89ABH）が格納されます。
- 変換の流れとして、32ビットデータから下位16ビットを取り出して16ビットデータに変換し、指定デバイスに格納することを示しています。
