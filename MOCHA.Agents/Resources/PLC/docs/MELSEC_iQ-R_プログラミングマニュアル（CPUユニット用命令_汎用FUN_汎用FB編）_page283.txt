### テキスト
#### 機能
- (s1)で指定したデバイスから(n)点分のBIN32ビットデータと、(s2)で指定したデバイスから(n)点分のBIN32ビットデータ、または定数の加算を行い、加算結果を(d)で指定したデバイス以降に格納する。
- ブロック加算は32ビット単位で行う。

#### 例示
##### (s2)にデバイスを指定した場合（符号付き指定時）
- (s1)と(s2)の各ビットデータを加算し、結果を(d)に格納する。
- 図では、(s1)と(s2)の各ビットの値（例：-30000、40000、-50000など）を示し、加算結果が(d)に格納される様子をビット単位で表現。

##### (s2)に定数を指定した場合（符号付き指定時）
- (s1)のビットデータに対し、(s2)に指定した定数（例：50000）を加算し、結果を(d)に格納。
- 図で具体的な数値例を示し、加算の流れを説明。

#### 注意事項
- (s1)または(s2)が(d)と同一デバイス（完全一致）で指定された場合は演算可能。
- ただし、(s1)または(s2)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲が部分的に重複するとエラーとなる。

#### 例示（デバイスの一致例）
- (s2)と(d)から4点分のデバイスが完全一致する場合は演算可能（図で示す）。
- 部分的に一致する場合は演算エラーとなる（図で示す）。

#### その他の注意点
- (n)で指定した値が0の場合は無処理となる。
- 演算結果にオーバーフローが生じた場合、SM700（キャリフラグ）はONしない。

### 表
| 符号付きを指定した場合 | 符号なしを指定した場合 |
|-------------------------|-------------------------|
| K2147483647 (H7FFFFFFF) + K2 (H00000002) → K-2147483647 (H80000001) | K4294967295 (FFFFFFFF) + K1 (H00000001) → K0 (H00000000) |
| K-2147483647 (H80000001) + K-2 (HFFFFFFFE) → K2147483647 (H7FFFFFFF) | なし |

- 符号付き指定時はオーバーフロー時に値が反転する例を示す。
- 符号なし指定時はオーバーフロー時に0に戻る例を示す。

### 図
- ビット単位のデータ構造を示す図が複数あり、各ビットの値や加算の流れを視覚的に説明。
- デバイスの完全一致・部分一致の違いを示す図があり、演算可能かエラーかを判別できるようになっている。
