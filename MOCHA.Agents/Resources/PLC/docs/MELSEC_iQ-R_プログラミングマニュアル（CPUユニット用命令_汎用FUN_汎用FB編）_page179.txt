<overview>
本ページは「機能」「エラー」「Point」の3つのセクションで構成されています。  
テキスト部分では、実行指令がONになった際のデバイスの状態変化について説明しています。  
表部分では、ビットデバイスとワードデバイスのビット指定に関するデバイスの状態を示しています。  
図部分では、SET命令とRST命令によるデバイスのON/OFF動作のタイミングを示す回路図とタイミングチャートが掲載されています。  
</overview>

<contents>
### テキスト
#### 機能
- 実行指令がONになると、指定デバイスは以下の状態になります。

#### 表
| デバイス | デバイスの状態 |
| :--- | :--- |
| ビットデバイス | コイル、接点をONします。 |
| ワードデバイスのビット指定 | 指定ビットを1にします。 |

- ONにしたデバイスは、実行指令がOFFになってもONのまま保持されます。  
- SET命令でONにしたデバイスは、RST命令でOFFにできます。  
- 実行指令がOFFの場合、デバイスの状態は変化しません。

#### 図
- 回路図には、X5入力がONでSET命令によりY10がONになる動作、X7入力がONでRST命令によりY10がOFFになる動作が示されています。  
- タイミングチャートでは、X5、X7のON/OFFに応じてY10の状態が変化する様子が矢印で表現されています。

### テキスト
#### エラー
- 演算エラーはありません。

### テキスト
#### Point
- デバイスにXを使用する場合は、実入力で使用していないデバイス番号を使うこと。  
- 実入力デバイスと同一番号を使うと、実入力のデータがSET命令で指定した入力Xに上書きされるため注意が必要です。
</contents>
