### テキスト
#### BIN16ビットデータテーブルソート2 (SORTTBL2(_U))
- R00CPU、R01CPU、R02CPUはファームウェアバージョンが"08"以降で使用可能。エンジニアリングツールはバージョン1.050C以降で使用可能。
- RnCPU（R00CPU、R01CPU、R02CPUを除く）、RnENCPUはファームウェアバージョンが"40"以降で使用可能。エンジニアリングツールはバージョン1.050C以降で使用可能。
- (s)で指定された行方向に連続する((n1)×(n2))点のBIN16ビットデータテーブル（並び替え元）を、(n3)列のデータを基準にデータ行を昇順/降順で並び替え、(d)で指定した((n1)×(n2))点のBIN16ビットデータテーブル（並び替え後）に格納する。

#### Point
- 列方向に連続したデバイスでテーブルを構成する場合は、737ページのSORTTBL(_U)命令を使用する。

### 図
- 左図は並び替え前のBIN16ビットデータテーブルを示し、(s)で示された元データの各セルに数値が入っている。
- 右図は並び替え後のBIN16ビットデータテーブルを示し、(d)で示された格納先に並び替えたデータが入っている。
- 並び替えは(n3)列のデータを基準に行方向に行われていることがわかる。
- ラダー図は命令の入力パラメータ(s, n1, n2, n3, d)と出力(ENO)を示す。
- FBD/LD図は同様にEN、s、n1、n2、n3、dの入力とENOの出力を示す。
