### テキスト
#### ファイルの最後の行の続きから追加
ファイルの最後の行の続きからデータを追加する場合の条件や設定内容を示しています。

- 実行/完了タイプはCSV形式変換書き込み（10進数、符号付き16ビットデータ）
- 使用用途設定エリアは「1H」
- ファイル位置は「FFFFFFFFH」（ファイルの最後に追加）
- データタイプは「ワード」
- 書込み先頭デバイスは「D7」
- 要求書込みデータ数は「2H」

#### 注釈
- (1)は今回の開始位置を示す
- (2)はデバイス格納データ（書込みデータ）を示す
- Nはデータ数を示す
- 使用用途設定エリアで「1」を設定した場合、ファイル末尾の改行コードがカンマに変換され、最後の行の続きからデータを追加できる

### 図
- 左側にCSV形式の2行分のデータが示されている。1行目は「1,2,3,4 CR LF」、2行目は「5,6,7,8 CR LF」となっている。
- 2行目の「7」と「8」の部分が矢印で示され、右側のデバイスD7、D8、D9に対応している。
- デバイスD7からD9にデータが書き込まれることを表している。
- 矢印の説明として、(1)が今回の開始位置、(2)がデバイス格納データ、Nがデータ数であることが記載されている。

この図は、CSV形式のファイルの最後の行の続きからデータを追加する際のデータ配置とデバイスへの書込みの関係を視覚的に示しています。
