### テキスト
#### ロングタイマ、ロング積算タイマデバイス使用時の動作
ロングタイマ、ロング積算タイマは、扱うデータが現在値の幅（32ビット）を超える場合に、現在値の領域だけでなく前回値、接点、コイルの領域も使用して動作します。

現在値の一括転送にBMOV命令を使用する場合、現在値のみの一括転送はできないため、現在値、接点、コイルを一括転送し、転送後のデータで現在値のみを使用してください。

現在値の一括転送にDMOV命令を使用する場合は、現在値のみの転送をFOR～NEXT命令で繰り返し、現在値を転送してください。

#### 例
タイマデバイスの現在値を一括転送する場合、BMOV命令を使用するときは現在値のみを一括転送します。

### 表
#### デバイス構成

| デバイス | 構成内容 |
| --- | --- |
| タイマ(T)、積算タイマ(ST)、カウンタ(C) | 現在値（T0、T1、T2など）、接点、コイル |
| ロングタイマ(LT)、ロング積算タイマ(LST) | 現在値（LT0、LT1）、FRT前回値、接点、コイル |
| ロングカウンタ(LC) | 現在値（LC0、LC1）、接点、コイル |

### 図
- BMOVP命令を用いたラダー図例：M0接点がONのとき、BMOVP命令でT0からD0へ4ワード分のデータを転送する。
- データ転送イメージ：T0～T3の現在値がD0～D3に転送される様子を示す。
