### テキスト
#### 設定データの内容、範囲、データ型
- オペランドごとに内容、範囲、データ型、データ型（ラベル）が説明されています。
- (s)：入力の先頭デバイス、データ型はビット、ラベルはANY_BOOL。
- (d1)：出力の先頭デバイス、データ型はビット、ラベルはANY_BOOL。
- (d2)：マトリクス入力データを格納する先頭デバイス、データ型はビット、ラベルはANY_BOOL。
- (n)：入力列数、範囲は2～8、データ型は符号なしBIN16ビット、ラベルはANY16。
- EN：実行条件、データ型はビット、ラベルはBOOL。
- ENO：実行結果、データ型はビット、ラベルはBOOL。

注釈として、ラベル設定時の配列型ラベルの要素指定についての注意書きがあります。

### 表
#### 使用可能デバイス一覧
- オペランドごとに使用可能なデバイスの種類がビット、ワード、ダブルワード、間接指定、定数、その他のカテゴリで示されています。
- (s)はXのみ使用可能。
- (d1)はYのみ使用可能。
- (d2)はX、Yともに使用不可。
- (n)はX、Y、ワード、ダブルワード、定数の一部が使用可能。

具体的なデバイス例として、ビットはX, Y, M, L, SM, F, B, SB, FX, FY、ワードはT, ST, C, D, W, SD, SW, FD, R, ZR, RDなどが挙げられています。

注釈として、Xのみ使用可能、Yのみ使用可能の注意書きがあります。
