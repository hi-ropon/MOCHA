### テキスト
#### 機能
(s)で指定された文字列の長さを検出し、(d)で指定されたデバイス番号以降に格納します。  
(s)で指定されたデバイス番号から、00Hが格納されているデバイス番号までのデータを文字列として処理します。

#### 例
(s)以降に「ABCDEFGH」が格納されている場合、各文字は16進数で対応するコード（例：42Hは'B'、41Hは'A'）としてデバイスに格納されます。  
文字列の終端は00Hで示され、(d)には文字列の長さ（この例では9）が格納されます。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 2820H              | (s)で指定したデバイス番号以降のデバイス/ラベルメモリの各設定エリア範囲内に00Hが設定されていないとき。 |
| 3405H              | (s)の文字列が16383文字を超えるとき。 |

### 図
- 文字列の格納イメージ図：  
  (s)から始まるデバイスに文字列が2文字ずつ16ビット単位で格納され、00Hが文字列の終端を示す。  
  (d)には文字列の長さが格納される。

- 例の図：  
  「ABCDEFGH」が(s)以降のデバイスに16進数コードで格納されている様子を示し、(d)に文字列長9が格納されている。
