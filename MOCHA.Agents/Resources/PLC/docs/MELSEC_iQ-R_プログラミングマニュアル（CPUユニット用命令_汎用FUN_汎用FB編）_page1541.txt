### 表
#### 命令処理時間一覧表
この表は各命令名ごとに、条件別の処理時間（単位はマイクロ秒）を示しています。処理時間はCPUユニット別に最小値と最大値が記載されており、以下のCPUユニットに分けられています。

- R00CPU、R01CPU
- R02CPU
- 左記以外のCPUユニット

##### 表の構成
| 項目 | 内容 |
| :--- | :--- |
| 命令名 | 対象の命令名が記載されています。 |
| 条件 | 命令に対する特定の条件やパラメータがある場合に記載されています。例：関数の引数(s)、(n)など。 |
| 処理時間(μs) | CPUユニットごとに最小値と最大値が示されています。 |

##### 代表的な命令例と処理時間
- TAND命令：R00CPU、R01CPUで最小4.3μs、最大15.4μs。R02CPUで最小3.3μs、最大14.4μs。
- EXP命令：(s)=-10の場合、R00CPU、R01CPUで最小2.9μs、最大6.0μs。R02CPUで最小1.9μs、最大5.0μs。
- EMAX命令：(n)=1の場合、R00CPU、R01CPUで最小4.0μs、最大6.6μs。R02CPUで最小3.0μs、最大5.6μs。
- POWD命令：(s1)=1.23E+5、(s2)=3.45E+0の場合、R00CPU、R01CPUで最小5.9μs、最大20.8μs。R02CPUで最小4.9μs、最大19.8μs。

##### 条件付き命令の例
- EXP、EXPD、LOG、LOGDなどは引数(s)の値によって処理時間が異なります。
- EMAX、EDMAX、EMIN、EDMINは引数(n)の値によって処理時間が変動します。

##### その他
- ZPUSH、ZPOP命令はZ、LZ全範囲やデフォルト設定の点数によって処理時間が異なります。
- LIMIT命令は条件なしで処理時間が示されています。

この表は命令ごとの処理性能を比較・把握するためのものであり、CPUユニットや条件による処理時間の違いを詳細に示しています。
