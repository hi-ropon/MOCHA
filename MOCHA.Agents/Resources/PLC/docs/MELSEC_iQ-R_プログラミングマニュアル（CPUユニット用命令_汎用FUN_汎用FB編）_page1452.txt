### テキスト
#### 機能：演算処理
1. AND(_E)
- (s1)～(s28)に入力されたBOOL型/WORD型/DWORD型データをビットごとに論理積（AND）を行い、演算結果を(d)から(s)と同一のデータ型で出力します。

##### 例（データ型がWORD型の場合）
- (s1)と(s2)のビット列をAND演算し、結果を(d)に出力する。
- 図示されたビット列で、1と0のビットごとにANDを行い、結果が示されている。

2. OR(_E)
- (s1)～(s28)に入力されたBOOL型/WORD型/DWORD型データをビットごとに論理和（OR）を行い、演算結果を(d)から(s)と同一のデータ型で出力します。

##### 例（データ型がWORD型の場合）
- (s1)と(s2)のビット列をOR演算し、結果を(d)に出力する。
- 図示されたビット列で、1と0のビットごとにORを行い、結果が示されている。

3. XOR(_E)
- (s1)～(s28)に入力されたBOOL型/WORD型/DWORD型データをビットごとに排他的論理和（XOR）を行い、演算結果を(d)から(s)と同一のデータ型で出力します。
- (s)が3つ以上ある場合は、(s1)と(s2)をXORした結果に対して、(s3)をXORし、さらに(s4)があればその結果に対して(s4)をXORし、以降(s5)、(s6)と続く数だけXORしていきます。

##### 例（データ型がWORD型の場合）
- (s1)と(s2)のビット列をXOR演算し、結果を(d)に出力する。

##### 例（データ型がBOOL型の場合）
- (s1)～(s5)までのBOOL値を順にXOR演算するフロー図が示されている。
- (1) sの数が3つの場合、(2) 4つの場合、(3) 5つの場合のXOR演算の流れが矢印付きで説明されている。
- (4) 以降のsの数だけXORしていくことを示す。

### 図（ビット列の説明）
- AND、OR、XORそれぞれの演算について、WORD型のビット列が上下に2つ並び、その間に演算記号（AND、OR、XOR）があり、下に結果のビット列が示されている。
- BOOL型の場合は、TRUE/FALSEの値を用いたXOR演算の流れを矢印で示した図がある。

### 表
- 特に表形式のデータはなく、ビット列の図解とテキスト説明で構成されている。
