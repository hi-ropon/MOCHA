### 表
#### 命令処理時間一覧表
この表は各命令の処理時間を示しており、以下の項目で構成されています。

- **命令名**: 各命令の名称が記載されています。例として「E- (s1) (s2) (d)」「ED+ (s) (d)」「ED- (s) (d)」「E*」「E/」などがあります。
- **条件**: 命令実行時の条件が記載されています。例えば、(s1)=0、(s2)=0、(d)=0、(s)=2^1023などの具体的な条件が示されています。
- **処理時間(μs)**: CPUユニット別に処理時間の最小値と最大値が示されています。
  - R00CPU、R01CPU
  - R02CPU
  - 左記以外のCPUユニット

処理時間はマイクロ秒単位で記載されており、命令や条件によって大きく異なります。例えば、単純な命令では0.0098μs程度の処理時間から、複雑な命令や条件付き命令では最大で14.2μs程度まで幅があります。

#### 処理時間の特徴
- 一部の命令（例：E-、E*）は非常に短い処理時間（0.0098μs程度）で処理されます。
- 条件付き命令や数値変換命令（例：ED+、ED-、EVAL、EREXPなど）は処理時間が長く、最大で10μs以上かかる場合があります。
- CPUユニットによって処理時間に差があり、R02CPUは一般的にR00CPU、R01CPUよりも短い処理時間を示す傾向があります。
- 数値変換命令（INT2FLT、UINT2FLT、DINT2FLTなど）はほぼ同じ処理時間で、約0.00686μsから0.02744μsの範囲です。

この表は命令の性能評価や最適化の参考資料として利用されることが想定されます。
