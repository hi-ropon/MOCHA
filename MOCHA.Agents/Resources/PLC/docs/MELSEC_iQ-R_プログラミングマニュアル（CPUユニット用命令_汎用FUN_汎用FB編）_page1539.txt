### 表
#### 命令処理時間一覧表
- 表は「命令名」「条件」「処理時間（μs）」の3つの大きな項目で構成されています。
- 処理時間はCPUユニット別に「R00CPU、R01CPU」「R02CPU」「左記以外のCPUユニット」の3区分に分かれており、それぞれ最小値と最大値が記載されています。

##### 主な内容
- 命令名には「ORE<>」「ORE>」「ORE<=」などの比較命令や、「LDED=」「ANDED=」「ORED=」などの論理演算命令が含まれています。
- 条件は「無実行時」「実行時:導通時/非導通時」などがあり、命令の実行状態に応じた処理時間が示されています。
- 処理時間は命令やCPUユニットによって異なり、例えば「ORE<>」はR00CPUで約0.18816μs、「LDED=」は3.4～6.0μsの範囲で変動しています。
- 一部命令（ECMP、EDCMP、EZCP、EDZCPなど）は条件欄が「—」となっており、処理時間のみ記載されています。
- 「E+ (s) (d)」「E+ (s1) (s2) (d)」「E- (s) (d)」などの命令は条件に具体的な数値（s=0, d=0など）が示され、処理時間が約0.31360μsで一定しています。

##### 表の構造
| 命令名 | 条件 | R00CPU、R01CPU（最小・最大） | R02CPU（最小・最大） | 左記以外のCPUユニット（最小・最大） |
| :--- | :--- | :--- | :--- | :--- |
| 例: ORE<> | 無実行時 | 0.18816 | 0.02352 | 0.00588 |
| 例: LDED= | 導通時/非導通時 | 3.400～6.000 | 2.400～5.000 | 1.900～4.500 |

- 各命令の処理時間はCPUユニットの種類と実行条件によって異なるため、詳細な性能評価や最適化に利用可能です。
