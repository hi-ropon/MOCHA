<overview>
このページは、ファイルへのデータ書き込み方法について説明しています。テキスト部分では、書き込み時の条件や指定項目が箇条書きで示されています。図は、CSV形式のデータがどのようにデバイスに格納されるかを示すフロー図で、1行のデータが複数の列に分割され、それぞれがデバイスのメモリに対応している様子を表しています。表はありません。
</overview>

<contents>
### テキスト
- データを追加する場合の手順について説明しています。
- 書き込みを行うファイルを指定し、既存ファイルがあっても削除して再度新規作成することを明記。
- 実行/完了タイプはCSV形式変換書き込み（10進数、符号付き16ビットデータ）。
- ファイル位置は00000000H（ファイルの先頭から）。
- 列数指定は4H。
- データタイプはワード。
- 書き込み先デバイスはD0。
- 要求書き込みデータ数は6H。

### 図
- (1)の開始行から、1行目は「1,2,3,4 CR LF」、2行目は「5,6 CR LF」というCSV形式のデータが示されている。
- 1行目の各列（1列～4列）がそれぞれ「1」「2」「3」「4」と区切られ、CR（キャリッジリターン）とLF（ラインフィード）が続く。
- 2行目は「5」「6」と区切られ、同様にCR LFが続く。
- (2)のデバイス格納データ（書き込みデータ）では、K6～K12のキーコードがD0～D13のデバイスに対応している。
- 矢印で、CSVの各列のデータがどのデバイスに書き込まれるかが示されている。
- Nはデータ数を表す。

### 注釈
- (1)は開始行を示す。
- (2)はデバイス格納データ（書き込みデータ）を示す。
- Nはデータ数を示す。
</contents>
