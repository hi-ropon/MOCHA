### テキスト
#### 32.15 DWORD型→INT型変換
DWORD型データをINT型データへ変換する関数「DWORD_TO_INT(_E)」について説明しています。

##### 対応CPU
- RnCPU、RnENCPU、RnPCPU（プロセス）、RnPCPU（三重化）、RnPSFCPU（一般）、RnPSFCPU（安全）、RnSFCPU（一般）、RnSFCPU（安全）

##### 機能概要
- 入力されたDWORD型データをINT型データに変換し、出力します。
- 入力値の上位16ビットは捨てられます。

### 図
- ラダー、FBD/LDの関数ブロック図が2種類示されています。
  - EN/ENOなし：入力(s)と出力(d)のみのシンプルなブロック
  - EN/ENO付き：実行条件入力(EN)、実行状態出力(ENO)、入力(s)、出力(d)のブロック
- ビット列の変換イメージ図
  - DWORD型の16進数「BC614EH」（2進数表記付き）からINT型の10進数「24910」へ変換される様子を示しています。
  - 上位16ビットは捨てられ、下位16ビットのみがINT型として出力されることを視覚的に説明。

### 表
#### 設定データ（引数の内容、種別、データ型）
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | DWORD |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

#### 演算結果（EN/ENO付きファンクションの動作）
| 実行条件 EN | 演算結果 ENO | 演算結果 (d) |
| :--- | :--- | :--- |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOがFALSEの場合、(d)から出力されるデータは不定値となるため、そのデータを使用しないようプログラムで処理する必要がある。

### 機能詳細
- (s)に入力されたDWORD型データをINT型データに変換し、(d)から出力する。
- 入力値の上位16ビットは捨てられる。
- (s)への入力値はDWORD型のデータ値である。

### 演算処理のポイント
- EN/ENOなしファンクションは単純に変換処理を行い、(d)から出力。
- EN/ENO付きファンクションは実行条件に応じて演算を行い、正常時は変換結果を出力、停止時は不定値を出力する。
