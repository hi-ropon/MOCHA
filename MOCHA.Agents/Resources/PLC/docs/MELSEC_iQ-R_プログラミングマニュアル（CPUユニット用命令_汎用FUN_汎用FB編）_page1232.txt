### テキスト
#### その他の機能
PID制御命令によるPID制御で、バンプレス切換えと操作量上下限リミッタ制御を自動的に行う機能について説明しています。

##### バンプレス切換え
バンプレスとは、モード切換え（手動⇔自動）時に操作量（MV）を連続的に制御させる機能です。モード切換えを行うと、自動モードの操作量エリアと手動モードの操作量エリア間でデータを転送します。
- 手動モードから自動モードへの切換え時：手動モードの操作量を自動モードの操作量エリアに転送
- 自動モードから手動モードへの切換え時：自動モードの操作量を手動モードの操作量エリアに転送
モード切換えは入出力データエリアで行います。

**Point**
PID制御の自動モードと手動モードの違いは以下の通りです。
- 自動モード：PID制御命令でPID演算を行い算出した操作量により制御対象を制御
- 手動モード：PID演算を行わず算出した操作量により制御対象を制御。手動モードに設定されたループはサンプリング周期ごとに測定値（PV）を設定値エリアに格納

##### 操作量上下限リミッタ制御
操作量上下限リミッタ制御は、PID演算で算出した操作量の上限または下限を制限する機能です。この機能は自動モード時のみ有効で、手動モード時は実行されません。
操作量上限値（MVHL）、操作量下限値（MVLL）を設定することで、PID演算で算出した操作量を下限値から上限値の範囲内に制限します。

**動作イメージ**
- EV（制御量）に対して、MVHL（上限値）とMVLL（下限値）で制限をかける。
- MV_AUTOはリミッタ制御がかからなかった場合の操作量。

**設定範囲とデフォルト値**
- 操作量上限値（MVHL）、操作量下限値（MVLL）は各ループに対して-50～2050の範囲で設定可能
- デフォルト値は操作量上限値：2000、操作量下限値：0
- 上限値が下限値より小さい設定はエラーとなる

### 図
