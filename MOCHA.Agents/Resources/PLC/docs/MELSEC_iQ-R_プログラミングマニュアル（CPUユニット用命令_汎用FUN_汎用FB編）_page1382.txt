### テキスト
#### 32.47 TIME型→BOOL型変換
TIME型データをBOOL型データへ変換する関数「TIME_TO_BOOL(_E)」について説明しています。

##### 対応CPU
RnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）

##### 機能概要
TIME型データをBOOL型データに変換します。

##### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | TIME |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | BOOL |

##### 演算処理
- 入力されたTIME型データ(s)をBOOL型データ(d)に変換して出力します。
- 具体例として、TIMEが0msの場合はFALSE、20分34秒567msの場合はTRUEを出力します。

##### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、dから演算出力値が出力されます。

2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
| :--- | :--- | :--- |
| TRUE (演算実行) | TRUE | 演算出力値 |
| FALSE (演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となるため、その場合はdのデータを使用しないようプログラム処理を行う必要があります。

##### エラー
演算エラーはありません。

### 図
- ラダー、FBD/LDでの関数ブロック図  
　EN/ENOなしとEN/ENO付きの2種類のブロック図が示されています。  
　EN（実行条件）、s（TIME入力）、d（BOOL出力）、ENO（出力状態）が表現されています。

- ST言語での記述例  
　EN/ENOなし：d:=TIME_TO_BOOL(s);  
　EN/ENO付き：d:=TIME_TO_BOOL_E(EN,ENO,s);

- TIME型からBOOL型への変換イメージ図  
　TIMEが0msの場合はFALSE、20分34秒567msの場合はTRUEに変換されることを示しています。
