<overview>
本ページは表形式のデータで構成されており、命令ごとの処理時間に関する詳細な情報が記載されています。テキスト部分はほとんどなく、表が中心です。表は命令名、条件、各CPUユニット（R00CPU、R01CPU、R02CPU、その他CPUユニット）ごとの処理時間（最小値・最大値）を示しています。グラフや図は含まれていません。

### 表
#### 命令処理時間一覧
命令ごとに、条件別の処理時間（単位はマイクロ秒）がCPUユニット別に示されています。

| 項目 | 内容 |
| :--- | :--- |
| 命令名 | 各種命令の名称（例：SUM、DSUM、BONなど） |
| 条件 | 命令実行時の条件（例：(s)=0、(s)=FFFFH、(n)=1、(n)=96など） |
| 処理時間（μs） | 各CPUユニットごとの処理時間の最小値・最大値 |
| CPUユニット | R00CPU、R01CPU、R02CPU、左記以外のCPUユニット |

- R00CPU、R01CPUは同じ列にまとめられており、それぞれの最小・最大処理時間が記載されています。
- R02CPUも同様に最小・最大処理時間が示されています。
- その他のCPUユニットについても最小・最大処理時間が記載されています。

#### 主な命令と処理時間の例
- **SUM命令**  
  条件(s)=0または(s)=FFFFHの場合、R00CPU/R01CPUで最小5.0μs、最大5.5μs、R02CPUで最小1.6μs、最大2.2μs、その他CPUユニットで最小1.3μs、最大1.9μs。

- **DSUM命令**  
  条件(s)=0の場合、R00CPU/R01CPUで最小7.6μs、最大8.0μs、R02CPUで最小2.0μs、最大2.7μs、その他CPUユニットで最小1.7μs、最大2.4μs。

- **MAX命令**  
  条件(n)=1の場合、R00CPU/R01CPUで最小3.9μs、最大6.5μs、R02CPUで最小2.9μs、最大5.5μs、その他CPUユニットで最小2.4μs、最大5.0μs。  
  条件(n)=96の場合は処理時間が大幅に増加。

- **SORT命令群（SORTD、SORTD_U、DSORTDなど）**  
  条件(n)=1、(s1)=1など複数条件で処理時間が示されており、データ量や条件により処理時間が変動。

- **WSUM命令**  
  条件(n)=1の場合、R00CPU/R01CPUで最小3.7μs、最大4.1μs、R02CPUで最小1.3μs、最大2.2μs、その他CPUユニットで最小0.9μs、最大1.8μs。  
  条件(n)=96の場合は大幅に処理時間が増加。

#### 備考
- 条件は主にパラメータ(s)や(n)の値で指定されており、これにより処理時間が変化することがわかる。
- 各CPUユニットの処理性能の違いが数値から読み取れる。
- 表の最下部に「付録 付1 命令処理時間」と記載されており、付録資料の一部であることが示されている。
