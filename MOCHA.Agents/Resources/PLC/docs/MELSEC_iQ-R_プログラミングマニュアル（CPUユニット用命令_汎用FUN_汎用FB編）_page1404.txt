### テキスト
#### 32.62 ビット配列→DINT型変換
「BITARR_TO_DINT(_E)」はビット配列から指定のビット数をDINT型データに変換する機能です。対応CPUはRnCPU、RnENCPU、プロセスCPU、二重化CPU、一般CPU、安全CPUなど複数あります。

##### 機能
- 演算処理：
  - 入力されたビット配列要素を先頭として、指定されたビット数(n)分をANY32型データに変換し、出力(d)します。
  - 指定ビット数以上の出力ビットは0に設定されます。

- 演算結果：
  1. EN/ENOなしファンクション：演算処理を実行し、(d)から演算出力値が出力されます。
  2. EN/ENO付きファンクション：実行条件と演算結果は表の通りです。

##### エラー
- 演算エラーはありません。

### 図
- ラダー、FBD/LD言語での使用例：
  - EN/ENOなし：s（ビット配列）、n（ビット数）、d（出力）を持つシンプルなブロック図
  - EN/ENO付き：EN（実行条件）、ENO（出力状態）、s、n、dの入出力を持つブロック図
- ST言語での使用例：
  - EN/ENOなし：d := BITARR_TO_DINT(s, n)
  - EN/ENO付き：d := BITARR_TO_DINT_E(EN, ENO, s, n)

### 表
#### 設定データ（引数一覧）
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(BitArr) | 入力(要素指定は変数も可) | 入力変数 | BOOL配列要素 |
| n | 4,8,12,16,20,24,28,32の定数のみ指定可能 | 入力変数 | INT |
| ENO | 出力状態(TRUE:正常実行、FALSE:異常または停止) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY32 |

#### 演算結果（実行条件と演算結果の対応）
| 実行条件 | 演算結果 |
| :--- | :--- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、その場合は(d)のデータを使用しないようプログラムで処理してください。
