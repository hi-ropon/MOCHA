### テキスト
#### 機能
(s3)で指定した入力値（BIN32ビット値）に対し、(s1)または(s2)で指定したバイアス値を付加し、(d)で指定したデバイス番号に格納します。バイアス値は以下の条件により制御されます。

#### エラー
演算エラーはありません。

### 表
| 条件 | 出力値に格納される値 |
| :--- | :--- |
| 入力値((s3), (s3)+1)<0のとき | 入力値((s3), (s3)+1)+負のバイアス値(s1)、(s1)+1 |
| 入力値((s3), (s3)+1)=0のとき | 0 |
| 入力値((s3), (s3)+1)>0のとき | 入力値((s3), (s3)+1)+正のバイアス値(s2)、(s2)+1 |

### グラフ
座標軸上に、横軸が(s3)+1、縦軸が(d)+1、(s2)+1、(s1)+1を示す点が配置されています。0を中心に、負のバイアス値(s1)が左下方向、正のバイアス値(s2)が左上方向に示されており、入力値の符号に応じてバイアス値が加算される様子を視覚的に表現しています。

### 例
- DZONE(P)命令の場合、演算結果が-2147483648～2147483647を超えるときの具体例が示されています。例えば、(s1)、(s1)+1が-1000で、(s3)、(s3)+1が-2147483648のとき、出力値は-2147483648+(-1000)=-80000000H-FFFFFFC18H=7FFFFC18H=2147482648となります。
- DZONE(P)_U命令の場合、演算結果が0～4294967295を超えるときの例が示されています。例えば、(s2)、(s2)+1が1000で、(s3)、(s3)+1が4294967295のとき、出力値は4294967295+1000=FFFFFFFFH-00003E8H=000003E7H=999となります。
- DZONE(P)_U命令では、(s1)、(s1)+1はダミーデータ扱いとなり使用しません。
