### テキスト
#### 設定データの内容、範囲、データ型
- オペランドごとに、内容、範囲、データ型（およびデータ型ラベル）が定義されている。
- 例として、(s)はデータテーブルが格納されている先頭デバイスで、符号付きBIN16ビットや符号なしBIN16ビットのデータ型がある。
- (n1)はデータ（行）数で1～32の範囲、(n2)はデータ（列）数で1～6の範囲。
- (d)は演算結果を格納する先頭デバイス。
- (n3)は並び替えの基準となるデータ（列）の列番号。
- EN、ENOは実行条件と実行結果でビット型。

#### 使用可能デバイス
- オペランドごとにビット、ワード、ダブルワード、間接指定、定数、その他の対応可否を表で示す。
- 例えば、(n1)や(n2)はビット、ワード、ダブルワード、定数に対応しているが、(s)や(d)は一部対応していない。

#### 機能説明
- (s)で指定された列方向に連続する((n1)×(n2))点のデータテーブル（並び替え元）を、(n3)列のデータを基準に昇順または降順で並び替え、(d)で指定した((n1)×(n2))点のデータテーブル（並び替え後）に格納する。
- データテーブル構成の例として、(n1)=3、(n2)=4の場合の行・列ごとのデバイス指定が示されている。
- 並び替えは命令実行開始時のSM774（データテーブルソート命令実行順）のON/OFF状態で昇順・降順を設定。
- 実行指令ONでSD774のビット2がONとなり並び替え開始。完了後に完了フラグがONになる。
- SD774のビット2がON中はSORTTBL(_U)命令は実行不可。
