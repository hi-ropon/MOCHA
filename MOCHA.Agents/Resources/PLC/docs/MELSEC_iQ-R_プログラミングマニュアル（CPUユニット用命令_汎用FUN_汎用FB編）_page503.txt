### テキスト
#### 機能
- (s)の2^(n)ビットのデータから1になっているビットに対応したBIN値を(d)に格納する機能について説明しています。
- (n)は1～8の範囲で指定可能で、(n)=0の場合は無処理となり、(d)で指定されたデバイスの内容は変わりません。
- ビットデバイスは1ビット、ワードデバイスは16ビットとして扱われます。
- 複数のビットが1のときは、上位のビット位置で処理されます。

図解では、ビット位置0～7のうち、ビット2が1の場合、対応するBIN値「110（二進数）=6（十進数）」が(d)に格納される例が示されています。

### 表
#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3401H              | (n)が0～8以外のとき。<br>(s)から2^(n)ビット分のデータがすべて0のとき。 |

この表は、指定範囲外の(n)値や、対象ビットデータがすべて0の場合に発生するエラーコードとその内容を示しています。
