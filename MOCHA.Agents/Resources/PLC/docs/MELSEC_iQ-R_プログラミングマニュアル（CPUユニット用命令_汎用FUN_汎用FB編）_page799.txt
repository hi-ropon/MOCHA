### テキスト
#### BIN16ビットデータ→文字列変換
STR(P)(_U)命令は、BIN16ビットデータを指定した位置に小数点を付加して文字列に変換する命令です。

#### 実行条件
- STR、STR_U命令は立ち下がりエッジで実行
- STRP、STRP_U命令は立ち上がりエッジで実行

#### 設定データ
##### 内容、範囲、データ型
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| --- | --- | --- | --- | --- |
| (s1) STR(P) / STR(P)_U | 変換する数値の桁数が格納されている先頭デバイス | なし | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S_ARRAY(要素数:2) / ANY16_U_ARRAY(要素数:2) |
| (s2) STR(P) / STR(P)_U | 変換するBINデータ | -32768～32767 / 0～65535 | 符号付きBIN16ビット / 符号なしBIN16ビット | ANY16_S / ANY16_U |
| (d) | 変換後の文字列を格納する先頭デバイス | なし | 文字列 | ANYSTRING_SINGLE |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |

### 図
- ラダー図：s1、s2、dの3つのオペランドを持つ命令ブロックの構造を示す。
- FBD/LD図：EN入力、s1、s2入力、ENO出力、d出力を持つ命令ブロックの構造を示す。
