### テキスト
#### 機能
- (d)で指定したデバイスのBIN16ビットデータを右へ(n)ビット分シフトする。
- 最上位から(n)ビット分は0になる。
- SM700では、シフト範囲から一つ右の1ビットが格納される。

#### 例1：(n)=6の場合
- 16ビットのビット列が右に6ビットシフトされる様子を示す。
- 元のビット列とシフト後のビット列が図で示されている。
- SM700の場合はシフト範囲外の1ビットが別に格納される。

#### 例2：(n)=4の場合（ビットデバイス指定時）
- Y1B～Y10のビット列が右に4ビットシフトされる様子を示す。
- 同様にシフト前後のビット列が図で示され、SM700の特性も記載。

#### 注意点
- 実際にシフトするビット数は(n)を桁指定で指定した点数で割った余りになる。
  - 例：n=15、点数=8の場合、15÷8=1余り7なので7ビットシフト。
- nは0～15の範囲で指定し、16以上の場合は16で割った余りの値でシフト。
  - 例：n=18の場合、18÷16=1余り2なので2ビットシフト。

### エラー
- 演算エラーは発生しない。

### 図（ビットシフトの例）
- 16ビットのビット列が右に指定ビット数分シフトされる様子をビットごとに示している。
- シフト後の空いた上位ビットは0で埋められる。
- SM700の場合はシフト範囲外の1ビットが別に格納されることを示す。
- ビットデバイス指定時も同様の動作を示す図がある。
