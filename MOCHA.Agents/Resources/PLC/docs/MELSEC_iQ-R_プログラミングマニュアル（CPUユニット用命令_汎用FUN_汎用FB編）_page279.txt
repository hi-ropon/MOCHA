### テキスト
#### 機能
- (s1)で指定したデバイスから(n)点分のBIN16ビットデータと、(s2)で指定したデバイスから(n)点分のBIN16ビットデータ、または定数の加算を行い、その加算結果を(d)で指定したデバイス以降に格納する。
- ブロック加算は16ビット単位で行う。

#### 演算結果のオーバーフローについて
- オーバーフローが発生した場合でも、SM700（キャリフラグ）はONしない。

### 図（例）
#### (s2)にデバイスを指定した場合（符号付き指定時）
- (s1)のデバイスに格納された16ビットBINデータの配列と、(s2)のデバイスに格納された16ビットBINデータの配列を加算。
- 加算結果は(d)のデバイスに格納される。
- 各データはb15からb0までの16ビットで表現されている。
- 例として、(s1)の1234、4567、-2000などの値と、(s2)の4000、1234、-1234などの値を加算し、結果を(d)に格納する様子を示す。

#### (s2)に定数を指定した場合（符号付き指定時）
- (s1)のデバイスに格納された16ビットBINデータの配列と、(s2)に指定した単一の定数（例：4321）を加算。
- 加算結果は(d)のデバイスに格納される。
- 例として、(s1)の1234、4567、-2000などの値に定数4321を加算し、結果を(d)に格納する様子を示す。

#### オーバーフロー時の例
- 符号付き指定の場合：
  - 例：K32767(H7FFF) + K2(H0002) = K-32767(H8001)
  - 例：K-32767(H8001) + K-2(HFFFE) = K32767(H7FFF)
- 符号なし指定の場合：
  - 例：K65535(HFFFF) + K1(H0001) = K0(H0000)
