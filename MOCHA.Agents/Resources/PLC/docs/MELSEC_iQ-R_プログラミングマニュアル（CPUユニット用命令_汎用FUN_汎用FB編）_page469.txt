### テキスト
#### 機能
(s)で指定されたデバイスから、(n)点分のBCDデータ（0～9999）をBIN16ビットに変換し、(d)で指定されたデバイス以降に格納します。

### 図
#### BCDデータとBINデータのビット配置
- 上段の図は、(s)で指定されたデバイスのBCDデータのビット配置を示しています。各ビットは1000、2000、4000、8000、1000、200、40、8、2、1の重みを持つBCD形式のビット列です。  
- 下段の図は、(d)で指定されたデバイスに格納されるBIN16ビットデータのビット配置を示しています。各ビットは8192、4096、2048、1024、512、256、128、64、32、16、8、4、2、1の重みを持つ2進数形式のビット列です。  
- それぞれの図は、(s)から(s)+(n)-1までのBCDデータが、(d)から(d)+(n)-1までのBINデータに変換されて格納される流れを表しています。

### 表
#### エラーコード（SD0）
| エラーコード | 内容 |
| :--- | :--- |
| 2821H | (s)、(d)のデバイスが重複しているとき。 |
| 3401H | (s)のデバイスから(n)点分のデータが0～9999以外のとき。 |
