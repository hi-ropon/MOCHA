### テキスト
#### 機能
- (s1)で指定したデバイスから(n)点分のBIN16ビットデータと、(s2)で指定したデバイスから(n)点分のBIN16ビットデータを比較し、(d)で指定したデバイス以降に演算結果を格納する。
- 比較条件が成立した場合は(d)の該当デバイスがON、成立しない場合はOFFになる。
- 比較演算は16ビット単位で行う。
- (s1)には定数を指定可能。

#### エラー
- エラーコード2821H：(s1)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲が重複している場合、または(s2)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲が重複している場合に発生。

### 図
- 比較演算のイメージ図が2つある。
  1. (s1)の複数点のBIN16ビットデータと(s2)の複数点のBIN16ビットデータを比較し、(d)にON/OFFの結果を格納する流れを示す。
  2. (s1)に定数を指定した場合の比較演算の流れを示す。
- 各デバイスのビットデータが縦に並び、比較演算の結果が対応する(d)のデバイスにON(1)またはOFF(0)で格納される様子を表現。

### 表
#### 比較演算命令の命令記号と条件、比較演算結果

| BKCMP<(P)(_U), BKCMP_LT(P)(_U)  | (s1)<(s2)    |              |
| BKCMP>=(P)(_U), BKCMP_GE(P)(_U) | (s1)≥(s2)    |              |
| BKCMP=(P)(_U), BKCMP_EQ(P)(_U)  | (s1)≠(s2)    | OFF(0)       |
| BKCMP<>(P)(_U), BKCMP_NE(P)(_U) | (s1)=(s2)    |              |
| BKCMP>(P)(_U), BKCMP_GT(P)(_U)  | (s1)≤(s2)    |              |

- (d)から(n)点に格納される比較演算結果がすべてON(1)のとき、SM704（ブロック比較信号）がONになる。
