### テキスト
#### 機能
(s1)、(s2)、(s3)に入力されたBOOL型/INT型/DINT型/WORD型/DWORD型/REAL型/LREAL型/STRING型/TIME型データに従い、(d)から(s1)、(s2)、(s3)と同一のデータ型で出力します。  
- (s2)の入力値が(s3)の入力値の場合、(d)から(s3)の入力値を出力。  
- (s2)の入力値が(s1)の入力値の場合、(d)から(s1)の入力値を出力。  
- (s1)の入力値が(s2)の入力値<(s3)の入力値の場合、(d)から(s2)の入力値を出力。

#### 例（INT型の場合）
- (s1)、(s2)、(s3)への入力値はBOOL型/INT型/DINT型/WORD型/DWORD型/REAL型/LREAL型/STRING型/TIME型のデータ値。  
- 文字列比較条件は以下の通り。  
  - 一致の条件：すべての文字列が一致した場合。  
  - 大きい文字列の条件：異なる文字列の場合は文字コードが大きい文字列、文字列長が異なる場合は長い文字列。  
  - 小さい文字列の条件：異なる文字列の場合は文字コードが小さい文字列、文字列長が異なる場合は短い文字列。

#### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、(d)から演算出力値が出力される。  
2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の表の通り。

| 実行条件 | 演算結果 |  
| --- | --- |  
| EN | ENO | (d) |  
| TRUE（演算実行） | TRUE | 演算出力値 |  
| FALSE（演算停止） | FALSE*1 | 不定値 |  

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、(d)のデータを使用しないようプログラム処理を行う必要がある。

#### エラー
- (s1)、(s2)、(s3)がINT型/WORD型の場合

| エラーコード(SD0) | 内容 |  
| --- | --- |  
| 3405H | (s1)で指定した下限リミット値が、(s2)で指定した上限リミット値よりも大きいとき。 |

- (s1)、(s2)、(s3)がDINT型/DWORD型/TIME型の場合

| エラーコード(SD0) | 内容 |  
| --- | --- |  
| 3405H | (s1)で指定した下限リミット値が、(s2)で指定した上限リミット値よりも大きいとき。 |

### 図
- INT型データの例として、(s1)=-100、(s2)=120、(s3)=100の入力値がLIMITブロックに入り、(d)から100が出力されるブロック図。  
- グラフは(d)を縦軸、(s2)を横軸にとり、(s1)、(s2)、(s3)の入力値に応じた出力(d)の関係を示している。
