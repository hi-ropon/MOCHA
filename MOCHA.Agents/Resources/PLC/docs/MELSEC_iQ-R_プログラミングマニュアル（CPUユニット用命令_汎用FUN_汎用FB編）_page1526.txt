本ページは命令ごとの処理時間に関する表で構成されています。テキスト部分はほとんどなく、表がメインです。表は命令名、条件、各CPUユニット（R00CPU、R01CPU、R02CPU、その他CPUユニット）ごとの処理時間（最小・最大）を示しています。グラフや図は含まれていません。

### 表
#### 命令処理時間一覧
命令ごとに、条件別の処理時間（単位：マイクロ秒）がCPUユニット別に示されています。

| 項目 | 内容 |
| :--- | :--- |
| 命令名 | 命令の名称（例：OUT(LC)、SET、RSTなど） |
| 条件 | 命令の実行条件（例：無実行時、実行時、導通時など） |
| 処理時間(μs) | CPUユニット別の処理時間。R00CPU、R01CPU、R02CPU、左記以外のCPUユニットに分かれ、最小値と最大値が記載されている場合もある。 |

#### 主な内容説明
- OUT(LC)命令は無実行時と実行時（カウント時/タイムアップ後）で処理時間がほぼ同じで、R00CPUで約0.38μs、R02CPUで約0.048μs、その他CPUで約0.012μs。
- SET命令は無実行時でR00CPU約0.062μs、R02CPU約0.008μs、その他CPUで約0.002μs。
- SET(F)命令は無実行時は短いが、実行時は大幅に増加し、R00CPUで最大約83.2μs、R02CPUで最大約82.2μs、その他CPUで最大約81.7μs。
- RST命令は無実行時と実行時で処理時間が異なり、RST(F)命令の実行時はR00CPUで最大約30.8μs、R02CPUで最大約29.8μs、その他CPUで最大約29.3μs。
- DELTA命令は無実行時は短いが、実行時はR00CPUで最大約7.8μs、R02CPUで最大約6.8μs、その他CPUで最大約6.3μs。
- SFT命令も同様に無実行時は短く、実行時は最大でR00CPU約4.8μs、R02CPU約3.8μs、その他CPU約3.3μs。
- PHASE命令は導通時と非導通時でほぼ同じ処理時間で、R00CPU約0.6μs、R02CPU約0.1μs、その他CPU約0.06μs。
- NOP命令は非常に短く、R00CPUで約0.03μs、R02CPUで約0.004μs、その他CPUで約0.001μs。
- LD系命令（LD=、LD<>、LD>など）は導通時/非導通時で処理時間がほぼ同じで、R00CPU約0.19μs、R02CPU約0.024μs、その他CPU約0.006μs。
- AND系命令も無実行時と実行時（導通時/非導通時）で処理時間が異なり、無実行時は約0.13μs、実行時は約0.19μs。

#### 特記事項
- FEND命令の処理時間は別マニュアル（MELSEC iQ-R CPUユニットユーザーズマニュアル（応用編））を参照するよう指示があります。
- STOP命令は処理時間の記載なし（―）となっています。

以上の表は、各命令のCPU負荷や処理速度を把握するための重要なデータであり、PLCプログラムの最適化や性能評価に利用されます。
