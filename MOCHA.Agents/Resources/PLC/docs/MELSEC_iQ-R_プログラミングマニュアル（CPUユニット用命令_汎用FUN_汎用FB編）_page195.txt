### テキスト
#### 機能
■ビットデバイスの場合  
(d)で指定したデバイスの1つ前のデバイスのON/OFF状態を(d)で指定のデバイスにシフトし、(d)で指定したデバイスの1つ前のデバイスはOFFになる。

**例**  
SFTP命令でM11が指定された場合、SFTP命令実行時にM10のON/OFFをM11にシフトし、M10をOFFにする。  
- シフトする先頭のデバイスはSET命令でONにする必要がある。  
- 連続でSFT(P)命令を用いる場合は、デバイス番号の大きいものからプログラムを作成する。

■ワードデバイスのビット指定の場合  
(d)で指定したビットの1つ前のビットの1/0状態を(d)で指定のビットにシフトし、(d)で指定したビットの1つ前のビットは0になる。

**例**  
SFT(P)命令でD0.5（D0のビット5）が指定された場合、SFT(P)命令実行時にD0のb4の1/0をb5にシフトし、b4を0にする。

#### エラー
演算エラーはありません。

### 図
- ビットデバイスのシフト動作を示すラダー図：M0がONのとき、X2がONでSET命令によりM10がONになる。SFTP命令でM14からM11まで順にシフトされる様子を示す。  
- ビット状態の変化表：M15からM8までの9ビットの状態が7段階で変化し、シフトの動きを矢印で示している。  
- ワードデバイスのビット指定シフトのビット列変化図：D0のビット15からビット0までの状態を示し、シフト実行前後でb4のビットがb5にシフトされ、b4が0になることを示している。
