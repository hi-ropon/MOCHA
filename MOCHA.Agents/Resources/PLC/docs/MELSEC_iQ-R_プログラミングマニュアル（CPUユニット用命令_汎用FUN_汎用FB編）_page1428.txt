### テキスト
#### 34.1 加算
ADD(_E)関数は、入力値の和((s1)+(s2)+...+(s28))を出力します。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、三重化）、RnPSFCPU（一般、安全）、RnSFCPU（一般、安全）です。

ラダーやFBD/LDではEN/ENOなし、EN/ENO付きの2種類のブロックがあり、ST言語ではEN/ENOなしの場合は「d:=ADD(s1,s2);」、EN/ENO付きの場合は「d:=ADD_E(EN,ENO,s1,s2);」と記述します。

入力変数は2～28個まで変更可能です。

##### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s1(IN1)～s28(IN28) | 入力 | 入力変数 | ANY_NUM*1 |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY_NUM*1 |

*1 安全プログラムではREAL型/LREAL型データを指定しないこと。指定するとエラーになる。

##### 機能
- (s1)～(s28)に入力されたINT型/DINT型/WORD型/DWORD型/REAL型/LREAL型データの加算を行い、演算結果を(d)から(s)と同一のデータ型で出力します。

##### 例（データ型がINT型の場合）
1234 + 5678 = 6912 と加算されます。

- 入力値はINT型/DINT型/WORD型/DWORD型/REAL型/LREAL型のデータ値です。
- 演算結果にアンダーフロー/オーバーフローが生じた場合、(d)に以下のように出力されます。

| データ型 | 内容 |
| :--- | :--- |
| INT型 | アンダーフローおよびオーバーフローが生じても演算エラーにはならず、ADD_Eの場合はENOからTRUEが出力されます。<br>【例1】<br>32767+2=-32767 (7FFFH)+(0002H)=(8001H)<br>最上位ビットが1になるため負の値になる。<br>【例2】<br>-32767+(-2)=-32766 (8000H)+(FFFEH)=(7FFEH)<br>最上位ビットが0になるため正の値になる。 |
