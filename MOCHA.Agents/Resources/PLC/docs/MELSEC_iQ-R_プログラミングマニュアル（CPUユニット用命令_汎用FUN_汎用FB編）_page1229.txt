### テキスト
#### PID制御用データの概要
PID演算における基準値を設定するためのデータです。PIDCONT命令でPID演算を開始する前に、PIDINIT命令でCPUユニット内部に登録します。ワードデバイスの任意の番号に設定可能ですが、使用ループ分のすべてのデータを連続したデバイス番号に設定する必要があります。PID制御データは全ループ共通で設定するデータと、各ループに設定するデータの2種類があります。

### 表
#### PID制御用データの種類と内容

| データの種類       | 項目               | 内容                                                                                     | 設定範囲                  | 設定データが指定範囲外のときの処理                                                                                  |
|--------------------|--------------------|------------------------------------------------------------------------------------------|---------------------------|---------------------------------------------------------------------------------------------------------------------|
| 共通設定データ     | 使用ループ数       | PID演算を実行させるループ数の設定。                                                      | 1～32                     | エラーとなり、全ループPID演算を実行しません。                                                                        |
|                    | 1スキャンの実行ループ数 | サンプリング周期に達したループが複数あったとき、1回のPID演算で何ループを実行させるかの設定。 | 1～32                     |                                                                                                                     |
| 各ループの設定データ | 演算式選択         | PID演算式の正動作か逆動作かの選択。                                                     | 0:正動作 1:逆動作         | エラーとなり、該当ループのPID演算を実行しません。                                                                    |
|                    | サンプリング周期(Ts) | PID演算を行う周期の設定。                                                                | 1～6000（単位：10ms）     |                                                                                                                     |
|                    | 比例定数(Kp)       | PID演算の比例の比。                                                                      | 1～10000（単位：0.01）    |                                                                                                                     |
|                    | 積分定数(Ti)       | 積分動作の効果の大きさを表す定数。大きくすると操作量の変化がゆるやかになる。             | 1～32767（単位：100ms）   |                                                                                                                     |
|                    | 微分定数(Td)       | 微分動作の効果の大きさを表す定数。大きくすると制御対象のわずかな変化で大きな操作量変化に。 | 0～30000（単位：10ms）    |                                                                                                                     |
|                    | フィルタ係数(α)    | 測定値に対してフィルタをかける程度の設定。0に近いほどフィルタは効かなくなる。             | 1～100                    |                                                                                                                     |
|                    | 操作量下限値(MVLL) | 自動モード時の操作量下限値。設定範囲外は-50または-32768～32767に変換。                   | -50～2050*1 / -32768～32767*2 | "PIDリミット制限あり"設定時は操作量下限値(MVLL)または操作量上限値(MVHL)の値が範囲外のとき、下記の値に変換される。-50未満は-50、2050超は2050になる。 |
|                    | 操作量上限値(MVHL) | 自動モード時の操作量上限値。設定範囲外は2050または-32768～32767に変換。                 | -50～2050*1 / -32768～32767*2 | 同上                                                                                                                |
|                    | 操作量変化率リミット値(ΔMVL) | 前回と今回の操作量変化量の制限値。制限値超過時はアラーム用デバイスのb1が1になる。制限なしの場合は制限しない。 | 0～2000*1 / 0～32767*2    | "PIDリミット制限あり"設定時は制限値未満は0、2000超は2000に変換。                                                  |
|                    | 測定値変化率リミット値(ΔPVL) | 前回と今回の測定値変化量の制限値。制限値超過時はアラーム用デバイスのb0が1になる。制限なしの場合は制限しない。 | 0～2000*1 / 0～32767*2    | 同上                                                                                                                |
|                    | 微分ゲイン(KD)*3  | 微分動作に対して時間幅（動作遅れ）を持たせる設定。値が大きいほど時間幅は小さくなり完全微分に近づく。 | 0～32767（単位：0.01）    | エラーとなり、該当ループのPID演算を実行しません。                                                                    |

*1 "PIDリミット制限あり"設定時の範囲  
*2 "PIDリミット制限なし"設定時の範囲  
*3 不完全微分時のみ設定可能

---

以上の内容により、PID制御用データは共通設定とループ毎の詳細設定に分かれ、各パラメータの意味と設定範囲、範囲外の処理が明確に示されています。
