このページは主にテキストと図で構成されており、BIN16ビットデータの加算機能について説明しています。テキスト部分では機能の概要とエラーの有無について記載されており、図部分では具体的な加算例が示されています。表は含まれていません。

### テキスト
#### 機能
(s1)で指定したBIN16ビットデータと、(s2)で指定したBIN16ビットデータの加算を行い、その加算結果を(d)で指定したデバイスに格納します。

- 加算結果にオーバーフローが生じた場合、桁上げは無視されます。この場合、SM700（キャリフラグ）はONしません。

#### エラー
演算エラーはありません。

### 図
- 加算例1（符号なし加算）  
  (s1) 5678 (BIN) + (s2) 1234 (BIN) = (d) 6912 (BIN)

- 加算例2（符号あり加算、オーバーフローなし）  
  (s1) -12345 (BIN) + (s2) -23456 (BIN) = (d) 29735 (BIN)

- 加算例3（符号あり加算、結果が負数）  
  (s1) 12345 (BIN) + (s2) 23456 (BIN) = (d) -29735 (BIN)

- 加算例4（符号なし加算、オーバーフロー発生）  
  (s1) 56789 (BIN) + (s2) 12345 (BIN) = (d) 3598 (BIN)

各図は、ビット位置b15からb0までの16ビットデータの加算過程を示しており、(s1)、(s2)が入力データ、(d)が加算結果を表しています。
</contents>
