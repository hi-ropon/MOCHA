### テキスト
#### 機能
(s1)で指定されたデバイスのBIN32ビットデータと、(s2)で指定されたデバイスのBIN32ビットデータの否定排他的論理和（XNR）を求め、その結果を(d)で指定されたデバイスに格納します。

- ビットデバイスの場合、桁指定による点数以降のビットデバイスは0として演算します。

#### エラー
演算エラーはありません。

### 図
図は3つのBIN32ビットデータのビット配列を示しています。

- (s1)のビット配列はb31からb0までの32ビットで、各ビットの状態が0または1で示されています。
- (s2)のビット配列も同様に32ビットで示されており、(s1)と異なるビットパターンを持っています。
- (d)のビット配列は、(s1)と(s2)のビットごとの否定排他的論理和（XNR）演算の結果を示しています。

ビット配列の上部にはビット番号（b31、b16、b15、b0）が示され、ビットの並びと演算の対応関係がわかりやすく表現されています。
</contents>
