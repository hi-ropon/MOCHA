### テキスト
#### DWORD型→STRING型変換
DWORD型データをSTRING型データへ変換する関数「DWORD_TO_STRING(_E)」について説明しています。

#### 関数概要
- 対応CPU：RnCPU、RnENCPU（プロセス、三重化、一般、安全）に対応
- 機能：DWORD型の入力データをSTRING型に変換して出力します。

#### 引数の内容、種別、データ型
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力データ | 入力変数 | DWORD |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力データ | 出力変数 | STRING(8) |

#### 機能詳細
- 演算処理：
  - 入力されたDWORD型データ(s)をSTRING型データ(d)に変換して出力します。
  - 入力値はDWORD型の数値。
  - 文字列の最後には、SM701（出力文字数切換）がOFFの場合、00H（NULL文字）を格納します。

- 演算結果：
  1. EN/ENOなしファンクション
     - 演算処理を実行し、dから演算出力値が出力される。
  2. EN/ENO付きファンクション
     - 実行条件と演算結果は以下の通り。

| 実行条件 | 演算結果 |
| :--- | :--- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、その場合は(d)のデータを使用しないようプログラムで処理してください。

#### エラー
- 演算エラーはありません。

### 図
- ラダー/FBD/LDでの関数ブロックの形状を示す図
  - EN/ENOなしの場合：入力s、出力dのブロック
  - EN/ENO付きの場合：入力EN、s、出力ENO、dのブロック

- DWORD型からSTRING型への変換イメージ図
  - 入力(s)のDWORD値（例：0H、12345678H）をSTRING型の文字列（例：0、12345678）に変換して出力(d)する様子を矢印で示している。
