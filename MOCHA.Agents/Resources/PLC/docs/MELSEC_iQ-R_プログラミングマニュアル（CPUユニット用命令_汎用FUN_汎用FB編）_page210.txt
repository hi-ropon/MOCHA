### テキスト
#### 5.9 無処理命令（NOP）
無処理命令（NOP）は、プログラムのデバッグ用にスペースを設ける場合などに使用されます。

##### 対応CPU
- RnCPU、RnENCPU（プロセス）、RnPCPU（二重化）、RnPCPU（一般）、RnSFPCPU（安全）、RnSFPCPU（一般）、RnSFCPU（安全）

##### ラダー
- ST：対応していません。

##### FBD/LD
- 対応していません。

##### 実行条件
| 命令 | 実行条件 |
| :--- | :--- |
| NOP  | 常時実行 |

##### 機能
- 無処理の命令で、それまでの演算に何も影響を与えません。
- NOP命令は以下の場合に使用します。
  - プログラムのデバッグ用にスペースを設ける。
  - ステップ数を変えずに命令を削除する（NOP命令に書き換えます）。
  - 一時的に命令を削除する。

##### Point
- NOP命令の挿入・削除については「GX Works3オペレーティングマニュアル」を参照してください。

##### エラー
- 演算エラーはありません。
