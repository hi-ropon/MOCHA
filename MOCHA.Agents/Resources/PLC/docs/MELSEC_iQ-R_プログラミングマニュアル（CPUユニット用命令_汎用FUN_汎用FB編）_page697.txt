### テキスト
#### 機能
(s3)で指定した入力値（BIN16ビット値）に対し、(s1)または(s2)で指定したバイアス値を付加し、(d)で指定したデバイス番号に格納します。バイアス値は以下の条件で制御されます。

- 入力値(s3) < 0 のとき：入力値(s3) + 負のバイアス値(s1)
- 入力値(s3) = 0 のとき：0
- 入力値(s3) > 0 のとき：入力値(s3) + 正のバイアス値(s2)

#### 例
- ZONE(P)命令の場合、演算結果が-32768～32767を超えるときは、例として(s1)が-100、(s3)が-32768のとき、出力値は -32768 + (-100) = 8000H - FF9CH = 7F9CH = 32668 となる。
- ZONE(P)_U命令の場合、演算結果が0～65535を超えるときは、例として(s2)が100、(s3)が65535のとき、出力値は 65535 + 100 = FFFFH - 0064H = 0063H = 99 となる。
- ZONE(P)_U命令の場合、(s1)はダミーデータ扱いとして使用しない。

#### エラー
演算エラーはありません。

### 表
| 条件               | 出力値に格納される値                     |
|--------------------|------------------------------------------|
| 入力値(s3) < 0 のとき | 入力値(s3) + 負のバイアス値(s1)          |
| 入力値(s3) = 0 のとき | 0                                        |
| 入力値(s3) > 0 のとき | 入力値(s3) + 正のバイアス値(s2)          |

### 図（グラフ）
- 横軸が入力値(s3)、縦軸が出力値(d)を示す座標軸。
- 原点0を中心に、負のバイアス値(s1)は原点の下側に、正のバイアス値(s2)は原点の上側に位置している。
- 入力値が負のときはs1のバイアスが加算され、正のときはs2のバイアスが加算されることを示している。
