<overview>
このページは、プログラム実行制御命令の一つであるEI命令とDI命令の優先度割込みに関する説明で構成されています。ページ全体はテキスト説明とラダー図（グラフ的表現）で構成されており、EI命令の実行時の割込み許可範囲や禁止範囲を示すラダー図が中心です。テキスト部分では、EI命令とDI命令の動作条件や制限事項、割込み優先度の設定に関する注意点が記載されています。
</overview>

<contents>
### テキスト
- EI命令の実行により、対となるDI命令で禁止した優先度の割込みを許可する。ただし、DI（割込み禁止）命令のみで割込み禁止にしている場合は、EI命令を1回実行するのみで全ての優先度の割込みを許可する。
- DI（指定優先度以下の割込み禁止）命令の多重実行時、指定する割込み禁止優先度が現在禁止している割込み禁止優先度以下の場合は、現在の割込み禁止優先度は変更されない。
- DI命令のネスティングは16重まで可能。
- 割込みポイントの割込み優先度はパラメータにより設定可能（MELSEC iQ-R CPUユニットユーザーズマニュアル応用編参照）。
- 割込み禁止となっている優先度は、SD758（割込み禁止優先度設定値）で確認可能。
- 割込みプログラム中にDI命令を実行し割込み禁止優先度が変更された場合は、SD758の値も変更される。

### 図（ラダー図）
- ラダー図はEI命令とDI命令の組み合わせによる割込み許可区間を示す。
- 「A」はシーケンスプログラムを示す。
- (1) すべての優先度の割込み許可区間：EI命令が実行されている状態。
- (2) 優先度6～8の割込み禁止区間（優先度1～5の割込み許可区間）：DI K6命令が実行されている状態。
- (3) 優先度5～8の割込み禁止区間（優先度1～4の割込み許可区間）：DI K5命令が実行されている状態。
- (4) 優先度6～8の割込み禁止区間（優先度1～5の割込み許可区間）：EI命令が実行されている状態。
- (5) すべての優先度の割込み許可区間：EI命令が実行されている状態。
- (6) [DI K5]に対応するEI命令。
- (7) [DI K6]に対応するEI命令。

この図はEI命令の実行により、DI命令で禁止された優先度の割込みがどの範囲で許可されるかを視覚的に示している。
