### テキスト
#### SM705（変換桁数切換）がONの場合の(s)の設定方法
(s)には任意の桁数（00H（NULLコード）まで）の10進アスキーデータを設定します。ただし、最大桁数が4桁の場合は00H（NULLコード）の設定は不要です。(s)の設定方法を以下に示します。

- (s)～(s)+1で指定するアスキーデータは0～9999の範囲内です。
- (s)+2以降のデータは無視されます。
- 各位に設定するアスキーコードは30H～39Hの範囲内です。
- 各位に設定するアスキーコードが20Hのときは30Hとして処理されます。00Hのときは10進アスキーデータの終端として処理されます。

#### ASCIIコードの説明
- ASCII 10³: 千の位のアスキーコード
- ASCII 10²: 百の位のアスキーコード
- ASCII 10¹: 十の位のアスキーコード
- ASCII 10⁰: 一の位のアスキーコード

### 表
#### (s)に設定する値と(s)～(s)+1の内容

| (s)に設定する値 | (s)～(s)+1の内容 |
| :--- | :--- |
| 0～9 | (s)+0の上位バイトに00Hを設定。<br> (s)+1以降は無視される。<br> ビット配置: (s)に00H、(s)+1にASCII 10⁰ |
| 10～99 | (s)+1の下位バイトに00Hを設定。<br> (s)+1の上位バイト以降は無視される。<br> ビット配置: (s)にASCII 10⁰、(s)+1に00H |
| 100～999 | (s)+1の上位バイトに00Hを設定。<br> (s)+2以降は無視される。<br> ビット配置: (s)にASCII 10¹とASCII 10²、(s)+1に00HとASCII 10⁰ |
| 1000～9999 | (s)+2以降は無視される。<br> ビット配置: (s)にASCII 10²とASCII 10³、(s)+1にASCII 10⁰とASCII 10¹ |

### 図（ビット配置の説明）
- 0～9の場合、(s)の上位バイトに00Hを設定し、(s)+1にASCII 10⁰（1の位のASCIIコード）を設定。
- 10～99の場合、(s)+1の下位バイトに00Hを設定し、(s)にASCII 10⁰を設定。
- 100～999の場合、(s)+1の上位バイトに00Hを設定し、(s)にASCII 10¹（十の位）とASCII 10²（百の位）を設定。
- 1000～9999の場合、(s)にASCII 10²（百の位）とASCII 10³（千の位）、(s)+1にASCII 10⁰（一の位）とASCII 10¹（十の位）を設定。

### エラー
| エラーコード (SD0) | 内容 |
| :--- | :--- |
| 3401H | (s)に変換できない不正なデータを入力したとき。<br>・データ中に0～9以外の文字がある。 |
