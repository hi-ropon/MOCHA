### テキスト
REAL型のデータ値をSTRING型の文字列データに変換する方法について説明しています。

- (s)への入力値はREAL型のデータ値である。
- 変換後の文字列データは(d)のように出力される。
  - 整数部は1桁、小数部は5桁、指数部は2桁で固定される（全13桁）。
  - 2バイト目、4バイト目、10バイト目にはそれぞれ自動的に「20H」（スペース）、「2EH」（.）、「45H」（E）が格納される。
- 符号データ（整数部）には、入力値が正のときは「20H」（スペース）、負のときは「2DH」（-）が格納される。
- 小数部の6桁目以降は四捨五入される。
- 有効桁数が少ない場合、小数部には「30H」（0）が格納される。
- 指数部の符号データは、指数が正のとき「2BH」（+）、負のとき「2DH」（-）が格納される。
- 指数部が1桁の場合、指数部十の位に「30H」（0）が格納される。
- 文字列の最後（7ワード目）には自動的に00Hが格納される。

### 図（変換例の説明）
1. 例1: 入力値 -12.3456（REAL型）
   - 変換後の文字列は13桁で構成される。
   - 整数部は「1」、小数部は「23456」、指数部は「+01」。
   - 2バイト目はスペース（20H）、4バイト目は「.」（2EH）、10バイト目は「E」（45H）。
   - 符号は負のため「-」（2DH）が整数部に格納される。
   - 小数部6桁目以降は四捨五入される。

2. 例2: 入力値 -12.345678（REAL型）
   - 小数部は5桁で「34567」まで表示し、6桁目以降は四捨五入される。
   - 有効桁数が少ない場合は小数部に「0」が補填される。

3. 例3: 入力値 -12.34（REAL型）
   - 小数部が少ない場合、小数部の残りは「0」で埋められる。

4. 例4: 入力値 -12.3456（REAL型）
   - 指数部が2桁で「+01」となる。
   - 指数部の符号は「+」で格納される。
   - 文字列の最後には自動的に00Hが格納される。

以上のように、REAL型の数値を13桁の固定フォーマットの文字列に変換し、符号や小数部、指数部の扱いを詳細に規定しています。
