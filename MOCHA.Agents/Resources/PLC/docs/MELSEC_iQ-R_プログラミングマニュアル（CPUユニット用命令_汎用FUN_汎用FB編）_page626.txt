### テキスト
#### 機能
(d)で指定されたデータテーブルの(n)番目のデータを削除し、(s)で指定されたデバイスに格納します。FDEL(P)命令実行後は、データテーブルの(n)+1番目以降のデータが1個ずつ前詰めされます。
- Nはデータ数を示します。
- 例として、(n)=3の場合は(d)+3が対象となり、(s)に0が格納されます。
- データ数(d)には0～65535の値が格納可能です。

### 図
- 左側の図は、データテーブル(d)の4番目から6番目までのデータが示されており、(n)=3のとき(d)+3のデータ（4444）が削除対象であることを示しています。
- 右側の図は、削除後のデータテーブルの状態を示し、(d)+4の位置に0が入り、(s)に削除されたデータ4444が格納されていることを表しています。
- 矢印でデータの移動と格納先が示されています。

### 表
#### エラー
| エラーコード (SD0) | 内容 |
| :----------------- | :--- |
| 3405H              | (n)の値が0でFDEL(P)命令を実行したとき。 |
|                    | (d)の値が0でFDEL(P)命令を実行したとき。 |

エラーコード3405Hは、(n)または(d)の値が0の場合にFDEL(P)命令を実行すると発生します。
