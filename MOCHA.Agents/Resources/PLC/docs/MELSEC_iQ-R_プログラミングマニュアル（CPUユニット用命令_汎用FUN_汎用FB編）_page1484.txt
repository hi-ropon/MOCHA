### テキスト
#### 39.8 文字列の検索
FIND(_E)関数は、文字列を検索して検索結果を出力します。

対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス・三重化・一般・安全）など複数のCPUに対応しています。

#### 使用例（ラダー、FBD/LD、ST）
- ラダー/FBD/LDでは、EN/ENOなしとEN/ENO付きの2種類の回路図が示されています。
- ST言語では、EN/ENOなしの場合は `d:=FIND(s1,s2);`、EN/ENO付きの場合は `d:=FIND_E(EN,ENO,s1,s2);` と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s1(IN1), s2(IN2) | 入力文字列 | 入力変数 | ANYSTRING_SINGLE |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力結果 | 出力変数 | INT |

#### 機能
- (s1)に入力された文字列の最初から、(s2)に入力された文字列を検索し、検索結果を(d)から出力します。
- 検索結果は、最初に検索した文字列の先頭文字位置を出力します。
- (s1)の文字列から(s2)の文字列が検索できなかった場合は、0を出力します。
- (s1)、(s2)への入力値はSTRING型のデータ値で0～255バイトの範囲内です。

### 図・表の説明
- ラダー/FBD/LDの図は、EN/ENOなしとEN/ENO付きの2種類の回路図を示し、s1、s2の入力とdの出力の関係を表しています。
- ST言語の例は、EN/ENOなしとEN/ENO付きの関数呼び出し構文を示しています。
- 設定データの表は、引数名、内容、種別、データ型をまとめています。
- 文字列検索の動作説明図は、(s1)の文字列「ABCD1234567」と(s2)の文字列「1234」を16進数のバイト列で示し、検索対象のバイト位置を特定している様子を表しています。
- 検索結果(d)は整数値5として出力されていることを示しています。
