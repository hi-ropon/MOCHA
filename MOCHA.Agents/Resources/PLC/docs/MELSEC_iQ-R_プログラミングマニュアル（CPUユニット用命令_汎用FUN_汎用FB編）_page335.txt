### テキスト
#### 機能
- (s1)で指定されたデバイスから(n)点分の内容と、(s2)で指定されたデバイスから(n)点分の内容の否定排他的論理和（XNR）を求め、その結果を(d)で指定されたデバイス以降に格納する。
- (s2)には-32768～32767の符号付きBIN16ビットの定数を指定可能。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 2821H | (s1)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s1)と(d)に同一デバイスを指定した場合は除く。<br>また、(s2)から(n)点分のデバイス範囲と(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s2)と(d)に同一デバイスを指定した場合は除く。 |

### 図（ビット列の説明）
- (s1)と(s2)で指定されたビット列の各ビットをXNR演算し、その結果を(d)以降に格納する動作を示す。
- 上段の図は(s1)と(s2)がそれぞれ複数行のビット列で表されており、対応するビットごとにXNR演算を行う。
- 下段の図は(s2)に符号付きBIN16ビットの定数が指定された場合の例で、同様に(s1)のビット列と定数のビット列のXNR演算結果を(d)以降に格納する様子を示している。
- 各ビット列はb15からb0までの16ビットで構成されている。
