### テキスト
#### 数値変換の例
(s)で指定された文字列において、符号と最初の0以外の数値との間に「20H（スペース）」または「30H（0）」が存在する場合、これらを無視してBIN値に変換するルールが説明されています。

##### 例1
符号と最初の0以外の数値との間に20Hが存在する場合（符号付き指定）
- 数値「-6543.21」のうち、20H（スペース）を無視してBIN32形式に変換。
- 全桁数12、小数部桁数2で変換される。

##### 例2
符号と最初の0以外の数値との間に30Hが存在する場合
- 数値「0.000054321」のうち、30H（0）を無視してBIN32形式に変換。
- 全桁数11、小数部桁数8で変換される。

### 表
#### エラーコード（SD0）一覧

| エラーコード | 内容 |
| :--- | :--- |
| 2820H | (s)で指定したデバイス番号から最終デバイス番号までの間に00Hが設定されていない場合。 |
| 3401H | (s)に変換できない不正なデータ入力時。<br>・文字列の文字数が2～13以外<br>・小数部文字数が0～10以外<br>・全文字数と小数部文字数の関係が規定範囲外<br>・DVAL(P)命令使用時に20H、2DH以外のASCIIコードが設定されている<br>・DVAL(P)_U命令使用時に20H以外のASCIIコードが設定されている<br>・各数字の桁に30H～39H、および2EH(小数点)以外のASCIIコードが設定されている<br>・小数点が複数設定されている |
| 3405H | (s)で指定する文字列の文字数が16,383文字を超える場合。 |

- 3401Hのエラーは、BIN値変換時に各命令で変換可能な範囲を超えた場合にも発生します。
