### テキスト
#### 機能
(s)で指定したデバイスの間接アドレスを、(d)で指定したデバイスに格納します。具体的には、(d)+0、(d)+1のデバイスに格納されたアドレスを使い、プログラムでデバイスの間接アドレス指定を行う際に利用します。

- (s)にはビットデバイスの桁指定はできず、ワードデバイスのビット指定もできません。
- デバイスの間接指定については、別途「MELSEC iQ-R CPUユニットユーザーズマニュアル（応用編）」を参照してください。
- ラベルの間接指定は対応していませんが、(d)にラベルを指定して使用する場合は表の方法で対応可能です。

#### エラー
演算エラーはありません。

### 図
- ラダープログラム例：
  - ADRSET W100 D100：W100のアドレスをD100、D101に格納
  - MOV K1234 @D100：D100、D101に格納されたアドレスを使い、W100に1234を書き込む
- デバイスアドレスの関係図：
  - D100、D101にW100のアドレスが格納されている様子
  - W100に値1234が書き込まれる流れを示す

### 表
| ラベル | 内容 |
| :--- | :--- |
| デバイスを指定したグローバルラベルの場合 | 間接アドレスとして使用する場合、グローバルラベルに指定したデバイスの間接指定を使用。<br>■STの例<br>ADRSET(TRUE, intVar, gvAddr); // intVarの間接アドレスをgvAddrへ読み出す。<br>INC(TRUE, @D0); // gvAddrへ指定したデバイスD0の間接指定で使用する。 |
