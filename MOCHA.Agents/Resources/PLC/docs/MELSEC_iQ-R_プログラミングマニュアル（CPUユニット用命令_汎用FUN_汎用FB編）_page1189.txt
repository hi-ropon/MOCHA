### テキスト
「Read（コマンド：0401H）」はワード単位でのデータ読出しを行うコマンドであり、送信する要求データと正常終了時の応答データの構造を示しています。

#### 例：D100～D102の値を読み出す場合
- 要求データと応答データの具体例を示し、どのようなデータが送受信されるかを説明しています。

### 図（要求データ）
- 要求データは複数のバイトで構成されており、各バイトの意味が示されています。
- コマンドコード（01H, 04H）
- サブコマンド（00H, 00H）
- 先頭デバイス番号（64H, 00H, 00H）
- デバイスコード（A8H）
- デバイス点数（03H, 00H）
- これらのバイトはメモリ上のアドレス（例：(s2)+2 〜 (s2)+6）に対応しています。

### 図（応答データ）
- 応答データは読み出したデバイスの値を示し、D100、D101、D102の3つのワードが含まれています。
- 各ワードは2バイトで構成され、バイトの並び順とアドレス（例：(d1)+2 〜 (d1)+4）が示されています。
- 具体的な値として、D100は34H 12H、D101は02H 00H、D102はEFH 1DHとなっています。

以上により、ワード単位読出しコマンドの送信データ構造と受信データ構造が具体的に理解できる内容となっています。
