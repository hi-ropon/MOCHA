### テキスト
#### 機能
- (s)で指定されたデバイスのBIN32ビットデータと同一のデータを、(d)で指定されたデバイスへ(n)点分転送する機能について説明しています。
- (s)に桁指定がある場合、転送されるデータは桁指定分のみとなります。
- 例として、(s)にK5Y0が指定された場合、(s)のワードデバイスの下位20ビット（5桁分）が対象となることが示されています。
- (d)に桁指定がある場合は、(d)で指定された桁指定分のデータを転送します。こちらも(d)にK5Y0が指定された場合、(s)のデバイスの下位20ビットが対象となります。
- (s)、(d)の両方で桁指定がある場合でも、桁数に関係なく(d)で指定された桁指定分のデータを転送します。
- (n)で指定された値が0の場合は無処理となります。

#### エラー
- 演算エラーはありません。

### 図（グラフ）
- 最初の図は、(s)で指定されたデバイスのBIN32ビットデータ（例：1234567H）が、(d)で指定された複数のデバイスアドレスに順次転送される様子を示しています。b31からb0までのビット構成が示され、(d)+1、(d)+2などのアドレスに対応しています。
- 次の図は、(s)に桁指定がある場合の転送イメージで、(s)のワードデバイスの下位20ビット（5桁分）が転送対象となり、無視されるビットや格納されるビットの範囲が示されています。
- さらに、(d)に桁指定がある場合の転送イメージが示されており、(d)で指定された20ビット分のデータが転送される様子が矢印で表現されています。
- 各図にはビットの範囲やアドレスの関係が詳細に示されており、転送対象のビット範囲や無視される部分が明確に区別されています。
