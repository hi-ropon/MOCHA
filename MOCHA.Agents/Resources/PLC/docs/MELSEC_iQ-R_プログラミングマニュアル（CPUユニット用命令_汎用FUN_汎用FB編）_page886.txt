### テキスト
#### 機能
- (s)で指定した符号なしBIN32ビットデータを単精度実数に変換し、(d)へ格納する。
- 単精度実数は32ビットで処理されるため、2進数表現時の有効桁数は24ビット、10進数表現時は約7桁となる。
- 整数値が0～16777215（24ビットのBIN値）の範囲を超えると、変換値に誤差が生じる。
- 変換結果は整数値の上位ビットから25ビット目を0にし、26ビット目以降は切り捨てられる。

#### エラー
- 演算エラーは発生しない。

### 図
- ラダー図：SM402の接点とM0の接点を使い、DMOVP命令でK123456をD0に移動し、UDINT2FLT命令でD0の値をD100に単精度実数として格納する流れを示す。
- ビット列の変換例：
  - 1つ目の例では、b31～b0のビット列が示され、上位ビット25～31は0にされ、26ビット目以降が切り捨てられた結果、10進数で222030032となる。
  - 2つ目の例では、別のビット列が示され、同様に上位ビット25～31は0にされ、26ビット目以降が切り捨てられた結果、10進数で372588928となる。
- (1)は切り捨てられるビット、(2)は0が挿入されるビットを示す。
