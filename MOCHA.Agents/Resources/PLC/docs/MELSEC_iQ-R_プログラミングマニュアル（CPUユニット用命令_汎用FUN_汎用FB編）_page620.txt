### テキスト
#### 機能
(d)で指定されたテーブルの最後に格納したデータを(s)で指定されたデバイスに格納します。FPOP(P)命令実行後は、FPOP(P)命令で読み出したデータを格納していたデバイスが0となります。
- (d)に格納されている値が0のときは、FPOP(P)命令を実行しないようにインタロックをとる必要があります。
- データ数(d)には0～65535が格納されます。

#### 注意事項
- Nはデータ数を示します。
- (1)の位置に0が格納されます。

### 図
- 左図：(d)から(d)+7までのデータテーブルの例。7個のデータが格納されており、最後の(d)+7には0が格納されている。
- 右図：(d)から(d)+7までのデータテーブルの例。6個のデータが格納されており、最後の(d)+7には0が格納されている。さらに(s)のデバイスには-4321が格納されている。

### 表
#### エラー
| エラーコード (SD0) | 内容                                   |
|--------------------|--------------------------------------|
| 3405H              | (d)の値が0でFPOP(P)命令を実行したとき。 |
