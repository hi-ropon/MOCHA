### テキスト
#### 機能
(s1)で指定されたデバイスから(n)点分の内容と、(s2)で指定されたデバイスから(n)点分の内容の論理和を求め、その結果を(d)で指定されたデバイス以降に格納します。

また、(s2)には-32768～32767（符号付きBIN16ビット）の定数を指定することも可能です。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 2821H | (s1)から(n)点分のデバイス範囲と、(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s1)と(d)に同一デバイスを指定した場合は除く。<br>(s2)から(n)点分のデバイス範囲と、(d)から(n)点分のデバイス範囲の一部が重複している場合。ただし、(s2)と(d)に同一デバイスを指定した場合は除く。 |

### 図（動作イメージ）
1. 論理和演算の動作例（デバイス範囲指定）
- (s1)と(s2)で指定されたデバイスのビット列がそれぞれ示されている。
- それらのビット列の論理和をとり、(d)で指定されたデバイスに結果が格納される様子をビット単位で示している。
- 各ビットは0または1で表され、複数行にわたり(n)点分のビット列が並んでいる。

2. 定数指定時の動作例
- (s2)に符号付きBIN16ビットの定数が指定された場合の例。
- (s1)のビット列と定数のビット列の論理和をとり、(d)に格納する動作を示している。
- こちらもビット単位での論理和の結果が図示されている。

以上により、論理和演算命令の動作とエラー条件が理解できる内容となっています。
