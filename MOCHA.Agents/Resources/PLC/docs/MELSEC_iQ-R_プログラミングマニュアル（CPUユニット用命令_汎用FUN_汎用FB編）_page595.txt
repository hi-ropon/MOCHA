### テキスト
#### ファンクションデバイスのデータサイズ
- FX、FYのビットデバイスは1点、ワードデバイスのビット指定時は1ビットのデータサイズ。
- FDのビットデバイスの桁指定時は4ワード、ワードデバイスは4ワードのデータサイズ。
- ビットデバイスの桁指定で指定されたデバイス番号が16の倍数でなくてもエラーにならない。
- FDの上位2ワードは0となる。

#### 注意事項
- FBD/LDプログラムはサブルーチンプログラムとして作成できない。
- FBD/LDでは引数を持つサブルーチンプログラムの実行はできない。

### 図
#### 指定されたデバイスの内容例
- X0が入力となり、EFCall命令で"ABC"という文字列とP0、M0、D0、D30のデバイスが指定されている。
- (1) M0を占有しFXへ転送、(2) D0～D3を占有しFD1へ転送、(3) D30～D33を占有しFD2へ転送する流れを示す。
- EFCall(P)命令は(s1)～(s5)まで使用可能。
- サブルーチンプログラムで使用するファンクションデバイス数とEFCall(P)命令の引数は同数にする必要がある。
- EFCall(P)命令のネスティングは16重まで可能で、CALL(P)、FCALL(P)、ECALL(P)、EFCall(P)、XCALL命令の合計で16重まで。

| 3360H | 17重目のネスティングが実行されたとき。 |

| 3381H | EFCall(P)命令実行後、RET命令実行前にEND命令、FEND命令、GOEND命令、STOP命令を実行したとき。 |
| 3382H | EFCall(P)命令実行前にRET命令を実行したとき。 |
