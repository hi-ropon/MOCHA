### テキスト
#### 機能概要
(s1)で指定されたデバイス番号以降に格納されている文字列データに対し、(s2)+1で指定された文字数分のデータを、(d)で指定されたデバイス番号以降に格納されている文字列データの(s2)で指定された位置以降に格納します。

- 文字列の終端はNULLコード（00H）が自動付加されます。
- (s2)+1で指定する文字数が0の場合は処理を行いません。
- (s2)+1で指定した文字数が(d)で指定された文字列データの最終文字を超えた場合は、(d)の最終文字までデータを格納します。
- (s2)+1で指定する文字数が-1の場合は、(s1)で指定された最終文字データまでを(d)で指定されたデバイス以降に格納します。

### 表・図の説明
#### 文字列データの格納例
- 表は16ビット単位（b15〜b0）でデバイス番号ごとに文字コード（16進数）と対応する文字を示しています。
- (s1)は元の文字列データの開始デバイス番号、(s2)は(d)の文字列データの左からの位置、(s2)+1は(s1)の文字列データの文字数を示します。
- 実行前後のデータ配置を比較し、どのように文字列が(d)の位置以降に挿入・置換されるかを示しています。

##### 例1
- (s1)=デバイス番号、(s2)=3、(s2)+1=6の場合
- (d)の文字列"ABCDEFGHI"の3文字目以降に"s1"の文字列"012345678"の6文字分が挿入され、結果は"AB012345I"となる。

##### 例2
- (s2)=5、(s2)+1=8の場合
- (d)の文字列"ABCDEFGHI"の5文字目以降に"s1"の文字列"012345678"の8文字分が挿入され、結果は"ABCD01234"となる。ただし、35H(5)〜37H(7)は格納されない。

##### 例3
- (s2)=2、(s2)+1=-1の場合
- (d)の文字列"ABCDEFGHIJK"の2文字目以降に"s1"の文字列"012345"の全文字が挿入され、結果は"A012345HIJK"となる。

### ポイント
- 文字列の終端は必ずNULLコード(00H)で示される。
- 文字数指定が0の場合は処理なし。
- 文字数指定が-1の場合は、(s1)の文字列の終端まで全て格納。
- 文字数指定が(d)の文字列長を超える場合は(d)の終端まで格納。
