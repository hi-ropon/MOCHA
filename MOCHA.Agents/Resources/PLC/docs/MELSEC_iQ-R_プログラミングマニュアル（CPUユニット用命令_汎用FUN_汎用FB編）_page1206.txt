<overview>
このページは、ファイル転送機能用のFTPGET命令に関する説明で構成されています。主にテキスト部分で命令の動作やパラメータの説明があり、図が1点掲載されています。図はファイルパスの文字列構造を示すもので、テキストの補足説明として機能しています。表はなく、グラフもありません。
</overview>

<contents>
### テキスト
- SP.FTPGET命令で処理するファイルの総数は、(s1)+2の「処理ファイル総数」に格納され、処理が完了したファイルの数が(s1)+3の「処理完了ファイル数」に格納される。
- (s2)にはFTPサーバの転送元ファイルが格納されているフォルダパス、ファイル名（拡張子を含む）をUnicode文字列で指定する。ファイルパスの長さは最大255文字。
- ファイルパスはFTPサーバのホームディレクトリからの相対パスで指定し、区切り文字には半角の「￥」または「/」を使用する。
- ただし、FTPサーバによっては「￥」を区切り文字として使用できない場合がある。

### 図
- ファイルパスの文字列構造を示す図。
- 例として「… "￥database1￥recipe￥recipe_db.txt" …」という文字列が示されている。
- 文字列は最大255文字までで、(s2)が1文字目、(s2)+1が2文字目、(s2)+2が3文字目…と続き、最後は0000Hで文字列の終端を示す。
- フォルダパスやファイル名の区切り文字は半角の「￥」または「/」を使用することが注記されている。

</contents>
