### テキスト
#### 制御用データ
PID演算命令の制御用データの内容について説明しています。

##### サンプリングタイム：(s3)
- 設定範囲は1～32767ミリ秒(ms)。
- PID演算を行うための周期(ms)を設定します。
- オートチューニング（リミットサイクル法）時は、シーケンサの演算周期よりサンプリングタイムが短くならないように設定。
- オートチューニング（ステップ応答法）時は1000ms以上に設定。

##### 最大誤差について
- サンプリングタイム(Ts)の最大誤差は、-(1演算周期+1ms)～+(1演算周期)となる。
- サンプリングタイム(Ts)が小さい場合、最大誤差の変動が問題になることがあるため、コンスタントスキャンを設定して実行するか、タイマ割込みルーチン内にプログラムすることを推奨。
- シーケンサの1演算周期よりもサンプリングタイムが短い場合、PID演算エラー（11A6H）が発生するが、サンプリングタイムを演算周期としてPID演算が実行される。
- その場合、PID演算命令をタイマ割込み内で使用し、PID演算命令の実行直前に(s3)+7をクリアしてから使用すること。

### ラダー図
- X0がONのとき、
  - MOVP命令でK0をD107にセット（(s3)+7をリセット）
  - PID命令でD0、D1、D100、D150を使用してPID演算を実行

### 注釈
1. (s3)+7をリセットするのは初回の割込みルーチン実行時で、内部処理用レジスタをパルス化指令でクリアするため。
2. PID演算命令を実行する。
