### テキスト
#### 機能
(s)で指定された符号付きBIN16ビットデータを、符号付きBIN32ビットデータに変換し、(d)で指定されたデバイスに格納します。  
プログラム例では、M0がONしたとき、D0に格納された符号付きBIN16ビットデータを符号付きBIN32ビットデータに変換し、D100～D101へ格納します。

#### エラー
演算エラーはありません。

### 図
#### プログラム例のラダー図
- SM402がONのとき、M0がONすると以下の動作を行う。
- MOVP命令でH0B000の値をD0にセット。
- INT2DINT命令でD0の符号付きBIN16ビットデータを符号付きBIN32ビットデータに変換し、D100に格納。

#### ビット構成の説明
- (s)のD0は16ビットで、b15～b0の範囲を持ち、値はB000H（-20480）となっている。
- (d)のD100、D101は32ビットで、b31～b16はFFFFH（変換前の最上位ビットの値で埋める）、b15～b0はB000H（下位16ビットに元の値を格納）となっている。

#### 注釈
1. 変換前の最上位ビットの値で埋める。  
2. 下位16ビットへ格納。
