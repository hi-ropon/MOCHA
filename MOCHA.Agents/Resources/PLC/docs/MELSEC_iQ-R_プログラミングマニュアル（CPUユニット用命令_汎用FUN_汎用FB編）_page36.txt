### テキスト
#### ビットデータ
ビットデータは、接点やコイルなどを1ビット単位で扱うデータです。

#### データサイズとデータの範囲
- ビットデータのデータサイズは「1ビット」
- 値の範囲は「0, 1」

#### ビットデバイス/ラベルにおけるビットデータの扱い
ビットデバイスやラベル1点に対して、1ビットのデータを扱うことが可能。

#### ワードデバイスにおけるビットデータの扱い
- ワードデバイスはビットNo.指定を行うことで、指定ビットNo.のビットデータを扱える。
- ビット指定の表記は「ワードデバイス番号.ビットNo.」の形式。
- ビットNo.は16進数で0～Fの範囲で指定可能。
- 例として、D0のビット5(b5)は「D0.5」、ビット10(b10)は「D0.A」と表記。
- ワードデバイスのビット指定が可能なデバイスは以下の通り。

### 表1：ワードデバイスのビット指定が可能なデバイス
| 項目 | デバイス |
| :--- | :--- |
| ワードデバイスのビット指定が可能なデバイス | ・データレジスタ(D)<br>・リンクレジスタ(W, リロ￥W)<br>・リンク特殊レジスタ(SW, リロ￥SW)<br>・ファンクションレジスタ(FD)<br>・特殊レジスタ(SD)<br>・ユニットアクセスデバイス(Uロ￥G)<br>・CPUバッファメモリアクセスデバイス(U3Eロ￥G、U3Eロ￥HG)<br>・ファイルレジスタ(RZ, ZR)<br>・ユニットリフレッシュ用レジスタ(RD) |

#### 安全デバイスのビット指定
- SIL2プロセスCPUおよび安全CPUの安全プログラムで使用する安全デバイスのビットNo.指定も16進数0～Fの範囲で指定可能。
- 例として、SA￥D0のビット5(b5)は「SA￥D0.5」、ビット10(b10)は「SA￥D0.A」と表記。
- 安全デバイスのビット指定が可能なデバイスは以下の通り。

### 表2：安全デバイスのビット指定が可能なデバイス
| 項目 | デバイス |
| :--- | :--- |
| ワードデバイスのビット指定が可能なデバイス | ・安全データレジスタ(SA￥D)<br>・安全リンクレジスタ(SA￥W)<br>・安全特殊レジスタ(SA￥SD) |
