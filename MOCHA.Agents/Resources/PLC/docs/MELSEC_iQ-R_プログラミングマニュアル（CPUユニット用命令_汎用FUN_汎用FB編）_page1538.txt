### 表
#### 命令処理時間一覧表
この表は各命令の処理時間をマイクロ秒（μs）単位で示しています。処理時間はCPUユニット別に分かれており、R00CPU・R01CPU、R02CPU、そしてそれ以外のCPUユニットの3種類に分類されています。それぞれのCPUユニットについて、最小処理時間と最大処理時間が記載されています。

| 項目 | 内容 |
| :--- | :--- |
| 命令名 | 各命令の名称が記載されています。例：STR_U、DSTR、BCDDDAなど。 |
| 条件 | 命令実行時の条件が記載されています。例：特定の文字数、特定の値(s=1など)、一致条件など。条件がない場合は「—」と記載。 |
| 処理時間(μs) | CPUユニットごとに最小値と最大値が示されています。 |

#### 主な特徴
- 文字列関連命令（WS2SJIS、SJIS2WS、RIGHT、LEFTなど）は、処理する文字数によって処理時間が大きく変動します。例えば、文字数1の場合と96の場合で処理時間に大きな差があります。
- 条件付き命令（BCDDDA、DBCDDAなど）は条件値によって処理時間が異なります。
- 論理演算命令（LDE=、ANDE=、ORE=など）は、導通時・非導通時や実行時・無実行時で処理時間が細かく区別されていますが、処理時間の差は非常に小さいです。
- 処理時間はCPUユニットによって異なり、R02CPUが比較的高速である傾向があります。

#### 具体例
- STR_U命令はR00CPUで4.4～8.8μs、R02CPUで3.4～7.8μs、その他CPUで2.9～7.3μsの処理時間。
- WS2SJIS命令は文字数1でR00CPUが4.8～11.8μs、文字数96で54.0～60.7μsと大幅に増加。
- INSTR命令は一致なしの場合、R00CPUで9.0～16.5μs、R02CPUで8.0～15.5μs、その他CPUで7.5～15.0μs。

この表は命令ごとの処理性能評価や最適化の参考資料として利用されることが想定されます。
