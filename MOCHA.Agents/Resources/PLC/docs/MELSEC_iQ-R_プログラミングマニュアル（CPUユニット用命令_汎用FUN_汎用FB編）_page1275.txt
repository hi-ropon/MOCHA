### テキスト
#### 機能概要
- ステップ活性状態をBIN32ビット単位で一括読み出し可能。
- ブロック指定なしの場合は、以下のブロックのステップ活性状態を読み出す。
  - シーケンスプログラム中で命令を実行した場合：ブロック0
  - SFCプログラム（動作出力内）で命令を実行した場合：命令を実行したブロック（自ブロック）
- 読み出した結果は指定デバイスに格納され、ステップが活性状態なら1、非活性状態なら0が格納される。
- 読み出すデータ内で欠番のステップNo.に対応するビットは0が格納される。

#### 注意点
- ブロック指定なしで読み出すデータ範囲がブロック内の最大ステップNo.を超えた場合、不定なデータが格納される。
- ブロック指定ありで読み出すデータ範囲がブロック内のステップ数を超えた場合、余りのビットは0が格納される。
- ブロック指定ありで読み出すデータの先頭が存在しないブロックやステップの場合、読み出すビットデータは0になる。
- SFCプログラムが存在しない状態で実行した場合も読み出すビットデータは0になる。

### 図・表の説明
- ビット列の図解では、b15からb0までのビットが横に並び、各ビットがステップNo.に対応。
- (s)は読み出し元のビット列、(d)は読み出し先のデバイスのビット列を示す。
- 例として、指定ブロックのS5、S8、S18、S29が存在しない場合は、そのビット位置に0が格納されることを示す。
- ブロックの最終ステップNo.がS26の場合、S0からS31をD0からD1に読み出す例では、S27以降のビットは不定（？）として示されている。
- ブロック指定ありの場合は、余分なビットは0で埋められることを示す。

### まとめ
本命令はSFCプログラムのステップ活性状態をビット単位で効率的に読み出すためのものであり、読み出し範囲やブロック指定の有無によって格納されるデータの内容や不定値の扱いが異なることを理解する必要がある。
