### テキスト
#### 同一BIN16ビットデータブロック転送(16ビット)
FMOV(P)命令は、BIN16ビットデータを指定されたデバイスから(n)点(0～65535)に転送します。

#### 注意事項
- ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定してください。
- バージョンが1.030G以降のエンジニアリングツールでは、INT型/WORD型以外のデータ型も使用可能です。ただし、オプション設定の命令引数のデータ型チェックを「する」にすると、使用できるデータ型はINT型/WORD型(ANY16の範囲)のみとなります。
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、上記のデータ型の安全デバイスおよび安全ラベルのみ使用可能です。

### 図
#### ラダー図
- FMOV(P)命令のラダー表現が示されている。
- 入力パラメータは(s)、(d)、(n)で、これらを指定して転送を行う。

#### FBD/LD図
- FMOV(P)命令のFBD/LD表現が示されている。
- ブロック内にEN、ENO、s、d、nの端子がある。

### 表
#### 実行条件
| 命令   | 実行条件                   |
|--------|----------------------------|
| FMOV   | 立下りエッジなしのパルス     |
| FMOVP  | 立上りエッジのパルス         |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容                                   | 範囲           | データ型           | データ型(ラベル) |
|------------|--------------------------------------|----------------|--------------------|------------------|
| (s)        | 転送するデータまたは転送するデータが格納されているデバイス | -32768～32767  | 符号付きBIN16ビット | ANY16            |
| (d)        | 転送先の先頭デバイス                   | なし           | 符号付きBIN16ビット | ANY16*1          |
| (n)        | 転送数                               | 0～65535       | 符号なしBIN16ビット | ANY16            |
| EN         | 実行条件                             | なし           | ビット             | BOOL             |
| ENO        | 実行結果                             | なし           | ビット             | BOOL             |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要がある。
