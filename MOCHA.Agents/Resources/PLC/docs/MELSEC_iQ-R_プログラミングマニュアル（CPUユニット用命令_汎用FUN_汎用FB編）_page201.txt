### テキスト
#### フェーズ処理開始
「フェーズ処理開始」命令（PHASE）は、特定のCPUやファームウェアバージョンで使用可能であることが記載されています。フェーズ処理を開始する命令であり、ラダー図ではシンプルな接点記号で表現されています。

#### 実行条件
- 命令：PHASE
- 実行条件：常時実行

#### 機能
- (s)で指定したデバイスがOFF→ON、ON→ONの場合、接続したフェーズ処理の演算結果は命令（回路）どおりとなる。
- (s)で指定したデバイスがON→OFFの場合、接続したフェーズ処理の演算結果は以下の通り。
- (s)で指定したデバイスがOFF→OFFの場合、接続したフェーズ処理は無処理となる。

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |

*1 配列型ラベルの場合、配列の添字にラベルまたはデバイスは使用不可。

#### 使用可能デバイス
| オペランド | ビット | ワード | ダブルワード | 間接指定 | 定数 | その他 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| (s) | X, Y, M, L, SM, F, B, SB, FX, FY (○*1*3) | T, ST, C, D, W, SD, SW, FD, R, ZR, RD (－*2*3) | U□￥G□, J□￥□, U3E□￥(H)G□ (－) | Z, LT, LST, LZ (－) | K, H (－) | E, $ (－) |

*1 Fは使用不可。  
*2 T, ST, C, R, ZRは使用不可。  
*3 インデックス修飾は使用不可。

#### 演算で使用するデバイスと演算結果（デバイスの状態）
| 演算で使用するデバイス | 演算結果（デバイスの状態） |
| :--- | :--- |
| 高速タイマ、低速タイマ | カウント値は0となり、コイル、接点ともOFFになる。 |
| 高速積算タイマ、低速積算タイマ | コイルはOFFになるがカウント値、接点とも現在の状態を保持する。 |
| カウンタ | － |
| OUT命令中のデバイス | すべてOFFにする。 |
| SET/RST命令中のデバイス、SFT命令中のデバイス、基本/応用命令中のデバイス | 現在の状態を保持する。 |
