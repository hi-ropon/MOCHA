### 表
#### 命令名ごとの基本ステップ数とサブセット可否一覧
- 表は3列構成で、左から「命令名」「基本ステップ数」「サブセット可否」となっています。
- 「命令名」には各種命令の名称が記載されています。
- 「基本ステップ数」には各命令の基本的なステップ数が数字で示されています。一部の命令は「2+ファイル名の文字数」など、文字数に依存する表記もあります。
- 「サブセット可否」には「〇」または「—」で可否が示されており、「〇」はサブセット処理が可能、「—」は不可を意味しています。

#### 表の例
| 命令名 | 基本ステップ数 | サブセット可否 |
| :--- | :---: | :---: |
| DTO | 5 | 〇 |
| DTOP | 6 | 〇 |
| TYPERD | 3 | — |
| SP.SOCOPEN | 5 | — |
| PSTOP | 2+ファイル名の文字数 | — |

- 命令名はタイマー関連（DTO、DTOPなど）、ソケット通信関連（SP.SOCOPEN、SP.SOCLOSEなど）、PID制御関連（PID、S.PIDINITなど）など多岐にわたります。
- 基本ステップ数は命令の処理に必要なステップの目安を示しており、命令の複雑さや処理内容の違いを反映しています。
- サブセット可否は命令がサブセット処理に対応しているかどうかを示し、制御プログラムの設計時に重要な情報となります。
</contents>
