### テキスト
「データベースアクセス命令の完了ステータスに格納されるエラーコードの一覧を示します。」という説明文が冒頭にあります。

### 表
#### データベースアクセス命令で発生するエラーコード一覧

| エラーコード | エラー内容 | 処置方法 |
|--------------|------------|----------|
| 0103H | CPU内部異常 | 電源OFF→ON、もしくはリセットを行い、再度実行できるか確認する。 |
| 0104H | 実行中のデータベースアクセス命令がある。 | データベースアクセス命令が完了しない間に別の命令を実行していないか確認する。 |
| 0105H | 指定したパスがデータベースフォルダではない。データベースが壊れている。 | パスのフォルダにデータベースファイルが存在するか確認し、適切なパスに修正する。 |
| 0106H | データベース内部のファイルをオープンできない。異なるファームウェアバージョンで構築されたデータベースを指定している。 | DBIMPORT(P)命令でデータベースを再構築する。 |
| 0107H | 使用できないドライブ番号が指定された。 | ドライブ番号が2になっているか確認し、2以外なら修正する。 |
| 0108H | データベースの削除に失敗した。 | 他の機能でアクセスしていないか確認し、DBIMPORT(P)命令で再構築する。 |
| 07D1H | 指定されたテーブルが存在しない。 | データベースに指定したテーブルがあるか確認する。 |
| 0940H | テーブル名に使用できない文字が含まれている。 | テーブル名は数字以外の文字から始め、使用できない文字を含めないようにする。 |
| 0941H | テーブル名が重複している。 | Unicodeテキストファイルのテーブル名設定で大小文字の区別をせず、同じ文字を指定していないか確認する。 |
| 0942H | 指定されたテーブルが存在しない。 | テーブルの存在を確認する。 |
| 0945H | 指定されたテーブルが他の機能によって使用中である。 | 他の機能での使用が終わった後、再度命令を実行する。 |
| 0946H | 指定されたテーブルが他の機能によって使用中である。 | 同上。 |
| 094AH | データベース名を指定していない。 | 正しいデータベース名を指定する。 |
| 094BH | 指定されたフォルダにデータベースが格納されていない。 | フォルダ名やフォルダ内のデータベース関連ファイルを確認する。 |
| 094CH | フィールド名に使用できない文字が含まれている。 | フィールド名に使用できない文字を含めないようにする。 |
| 094DH | フィールド名を重複指定している。 | Unicodeテキストファイルや命令の引数で重複しているフィールド名がないか確認し修正する。 |
| 094EH | 存在しないフィールドを指定している。 | テーブル内に指定したフィールドが存在するか確認する。 |
| 0950H | インデックスに使用できない文字が含まれている。 | インデックスに使用できない文字を含めないようにする。 |
| 0951H | 主キー、またはNOT NULL制約を付けているフィールドに空のデータ(NULL値)を設定しようとした。 | テーブル構造を確認し、NULL許可の有無を確認する。 |
| 0952H | 主キー制約が付いているフィールドに同一の値を追加しようとした。 | 追加しようとしたレコードの設定値が既に登録レコードと重複していないか確認する。 |
| 0953H | インデックスを設定したフィールドに重複データを繰り返し設定した場合、メモリ不足になる。 | テーブル定義を変更し、再構築する。 |
| 0954H | 設定値をデータ型に適した形式に自動変換できない。 | 設定値の書式が仕様に従っているか確認する。 |
| 0959H | インポート時、データ型がテーブル定義と異なっている。 | Unicodeテキストでインポートする際、設定値がフィールドのデータ型に適した値か確認する。 |
| 095CH | 文字列で大小比較を行うなど、比較する値のデータ型が間違っている。 | 比較する値を修正する。 |
| 095DH | データベースアクセス命令の引数で指定する設定値のデータ型変換に失敗したため、フィールドに設定値を格納できない。 | 設定値がデータ型の仕様に適合しているか確認する。 |
| 095EH | 一つのフィールドにインデックスを重複して割り当てようとした。 | 主キーや外部キー設定フィールドにインデックスを重複して割り当てていないか確認する。 |
| 095FH | インデックス名が重複している。 | 各インデックス名が重複していないか確認する。 |
| 096CH | キー制約として外部キーが指定されたフィールドに追加する設定値が参照先設定値に含まれていない。 | 参照先の設定値を確認する。 |
| 096EH | 他のテーブルによって参照されているため、レコードを削除できない。 | 削除するレコードの設定値に外部キー参照が含まれていないか確認する。 |
| 0970H | レコードを追加・削除しようとしたが、参照するためのテーブルがない。 | 参照先のテーブルがあるか確認する。 |
| 0971H | 参照先のテーブルに登録されていない設定値を追加しようとした。 | 参照先のテーブルの設定値を確認する。 |

以上のように、エラーコードごとに発生原因と対処方法が具体的に記載されています。
