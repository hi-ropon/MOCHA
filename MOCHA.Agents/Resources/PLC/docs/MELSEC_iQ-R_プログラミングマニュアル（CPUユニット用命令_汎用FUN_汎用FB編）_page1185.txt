### テキスト・表

#### 使用可能デバイス
- オペランドごとに、ビット、ワード、ダブルワード、間接指定、定数、その他の使用可否を示す表。
- ビット列はX, Y, M, L, SM, F, B, SB, FX, FYなどのデバイス名。
- ワードはT, ST, C, D, W, SD, SW, FD, R, ZR, RDなど。
- ダブルワードはZ、LT, LST, LCなど。
- 間接指定はK, H, E, $。
- その他(U)は特定の記号で示されている。
- 注釈として「T, ST, C, FDは使用できません」との記載あり。

#### コントロールデータ（オペランド：s1）
- 各デバイス項目ごとに内容、設定範囲、セット側の情報を詳細に記載。
- +0 実行・異常時完了タイプ
  - ビット0: 到達確認なし（自局から要求伝文を送信したことで完了）
  - ビット7: 異常完了時のデータセット状態指定（0:セットしない、1:セットする）
- +1 完了ステータス
  - 命令完了時の状態格納（0:正常、0以外:異常エラーコード）
- +2 自局使用チャンネル
  - チャンネル指定（1:シリアル番号なし、2～9:シリアル番号付き）
- +3 相手機器IPアドレス（第3、4オクテット）
- +4 相手機器IPアドレス（第1、2オクテット）
- +5 相手機器ポート番号（1～65534）
- +6 要求先ネットワーク番号（固定0000H）
- +7 要求先局番（固定00FFH）
- +8 要求先ユニットI/O番号
  - 管理CPUやマルチCPU各号機、制御系CPU、待機系CPU、A系CPU、B系CPUの番号指定
- +9 要求先マルチドロップ局番（固定0000H）

各項目はユーザまたはシステム側で設定可能な範囲が示されている。
