### テキスト
#### 機能
(s)で指定されたデバイスから(n)点のBIN16ビットデータを(d)で指定されたデバイスから(n)点分、一括転送する命令の説明です。

- 転送元と転送先のデバイスが重複していても転送可能。
- 転送はデバイス番号の小さい方から大きい方へ順に行われる。
- (s)がワードデバイスで(d)がビットデバイスの場合、ワードデバイスはビットデバイスのビット指定で指定されたビット数が対象となる。

#### 例
(d)にK1Y30が指定された場合、(s)で指定されたワードデバイスの下位4ビットが対象となる。

- (s)、(d)が両方ビットデバイスの場合は、必ず(s)、(d)の桁数を同一にする必要がある。
- (s)、(d)にリンクダイレクトデバイス、ユニットアクセスデバイス、CPUバッファメモリアクセスデバイスを使用する場合は、(s)か(d)の片方のみ指定可能。ただし、I/O No.指定にインデックス修飾を指定していない自号機CPUユニットのCPUバッファメモリアクセスデバイス(U3E0￥G□)は、(s)、(d)の両方で指定可能。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3420H              | (s)、(d)の両方でリンクダイレクトデバイス、ユニットアクセスデバイス、CPUバッファメモリアクセスデバイスを指定した場合。ただし、I/O No.指定にインデックス修飾を指定していない自号機CPUユニットのCPUバッファメモリアクセスデバイス(U3E0￥G□)指定時は除く。 |

### 図
- ビットデータ転送のイメージ図：  
  (s)から(n)点のBIN16ビットデータを(d)に転送する様子を示す。  
  b15からb0までのビットが並び、複数のアドレスにまたがるデータの転送を矢印で表現。

- 例の図：  
  (d)にK1Y30が指定された場合、(s)のワードデバイスD100～D102の下位4ビットが(d)のY30～Y3Bに転送される様子を示す。  
  各ビットの位置（b15～b0、b4～b0）や転送範囲(n)が矢印で示されている。
