### テキスト
#### 32.12 WORD型→STRING型変換
関数「WORD_TO_STRING(_E)」はWORD型データをSTRING型データに変換します。対応CPUはRnCPU、RnENCPU、RnPCPU（三重化・一般）、RnSPFCPU（安全・一般）です。

#### 使用例
- ラダー、FBD、LDではEN/ENOなしとEN/ENO付きの2種類の関数ブロックが示されています。
- ST言語ではEN/ENOなしの場合は「d:=WORD_TO_STRING(s);」、EN/ENO付きの場合は「d:=WORD_TO_STRING_E(EN,ENO,s);」と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | WORD |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | STRING(4) |

#### 機能
- 演算処理として、入力されたWORD型データ(s)をSTRING型データに変換し、出力(d)します。
- 入力値はWORD型のデータ値であり、文字列の最後にはSM701（出力文字数切換）がOFFの場合、00Hを格納します。

#### 演算結果
1. EN/ENOなしファンクション  
   演算処理を実行し、dから演算出力値が出力されます。

2. EN/ENO付きファンクション  
   実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |  |
| --- | --- | --- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となるため、その場合はdから出力されたデータを使用しないようプログラムで処理してください。

#### エラー
演算エラーはありません。

### 図・イメージ
- ラダー/FBD/LDの関数ブロック図は、EN/ENOなしとEN/ENO付きの2種類のブロックが示されており、入力sと出力dの関係が視覚的に表現されています。
- ST言語の記述例も示されています。
- 演算処理のイメージ図では、WORD型の16進数データ（例：0H、1567H）がSTRING型の数値（例：0、1567）に変換される様子が矢印で示されています。
