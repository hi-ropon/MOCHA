このページは主に3つのセクションで構成されています。上部に「使用可能デバイス」に関する表が2つあり、オペランドごとに使用可能なビット、ワード、ダブルワード、間接指定、定数などのデバイス種類が一覧化されています。中段には「機能」説明があり、BIN16ビットデータの乗算処理の動作概要と例がテキストと簡単な図で示されています。下部には「エラー」についての簡潔な説明があります。図や表はすべてテキスト情報の補足として機能しています。

### テキスト
#### 使用可能デバイス
- オペランドごとに使用可能なデバイスの種類が表形式で示されています。
- ビットデバイス、ワードデバイス、ダブルワードデバイス、間接指定、定数、その他のカテゴリに分かれています。
- それぞれのオペランド（(s1)、(s2)、(d)）に対して、使用可能なデバイスが○で示されています。
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、特定の安全デバイスおよび定数のみ使用可能であることが補足されています。

#### 機能
- (s1)で指定したBIN16ビットデータと、(s2)で指定したBIN16ビットデータの乗算を行い、その結果を(d)で指定したデバイスに格納します。
- (d)がビットデバイスの場合は、下位のビットから指定されます。
- 乗算のビット構成例が図で示されており、(s1)と(s2)はそれぞれ16ビット（b15～b0）、(d)は32ビット（b31～b0）として表現されています。
- 具体例として、(d)がビットデバイスの場合の乗算結果のビット指定例が3つ挙げられています。
  - K1：下位の4ビット（b0～b3）
  - K4：下位の16ビット（b0～b15）
  - K8：下位の32ビット（b0～b31）

#### エラー
- 演算エラーはありません。
