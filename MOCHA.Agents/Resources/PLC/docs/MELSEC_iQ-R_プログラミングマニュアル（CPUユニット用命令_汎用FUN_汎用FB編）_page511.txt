### テキスト
#### 機能
(s1)で指定されたデバイス番号以降に格納しているデータの各ビットを、(s2)で指定されたビット分ずつ分離し、(d)で指定されたデバイス番号以降に格納します。

- (s2)で指定する分離ビット数は1～16ビットの範囲内で指定可能。
- (s2)による指定デバイス番号から、「0」を格納しているデバイス番号までを分離ビット数として処理。
- 分離するデータのデバイス範囲((s1)～(s1)の終了範囲)と、分離したデータを格納するデバイス範囲((d)～(d)の終了範囲)は重複しないようにする必要がある。重複すると正しい演算結果が得られない場合がある。
- (s1)、(s2)、(d)で指定するデバイス番号は重複させてはいけない。

#### 注意事項
- (s2)～(s2)+6は指定ビット分
- (1)は設定終了指定
- (2)は無視される

### 図
ビット分離のイメージ図が示されている。

- (s1)のデバイスのビット列（例：16ビットや15ビットなど）から、(s2)で指定されたビット数ごとに分割されている。
- 分割されたビットは(d)のデバイスに順次格納される。
- 具体例として、(s2)で6ビット、8ビット、4ビットなどのビット数が指定されている様子が示されている。
- 矢印でビットの分割と格納先の対応がわかりやすく表現されている。
