### テキスト
#### 35.2 nビット右シフト（SHR(_E)）
入力値をnビット分右へシフトして出力する命令です。  
対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス/二重化/一般/安全）で、一部のCPUでは使用不可です。

#### 機能
- 入力された16ビットまたは32ビットデータ（WORD型/DWORD型）をnビット右シフトし、出力します。  
- シフトするビット数nは入力値で指定します。  
- 出力データの型は入力データと同じです。  
- 最上位からnビット分は0に置き換えられます。

### 図
- ラダー回路（FBD/LD）では、EN（実行条件）、s（入力データ）、n（シフトビット数）、d（出力データ）を持つブロックが示されています。  
- ST言語ではENの有無により2種類の記述があり、d:=SHR(s,n) または d:=SHR_E(EN,ENO,s,n) の形で表現されます。  
- 例として、16ビットデータ「270FH」を8ビット右シフトした場合のビット列変化が示されています。  
  - 元データ（270FH）のビット列は「0001 0001 1100 1111」  
  - 右シフト後（27H）は「0000 0000 0010 0111」となり、上位8ビットは0になります。

### 表
#### 設定データ（引数一覧）

| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件（TRUE:実行、FALSE:停止） | 入力変数 | BOOL |
| s(IN) | 入力データ | 入力変数 | ANY_BIT |
| n(N) | シフトビット数指定 | 入力変数 | ANY_BIT |
| ENO | 出力状態（TRUE:正常、FALSE:異常） | 出力変数 | BOOL |
| d | 出力データ | 出力変数 | ANY_BIT |

#### 入力値の範囲
- (s)のデータが16ビット（WORD型）の場合、nの入力値は0～15の範囲。  
  - nの入力値の下位4ビット分のデータが使用される。  
  - 例：入力値6の場合は6、22の場合も6（22の下位4ビットは6）  
- (s)のデータが32ビット（DWORD型）の場合、nの入力値は0～310の範囲。  
  - nの入力値の下位5ビット分のデータが使用される。  
  - 例：入力値6の場合は6、22の場合は22。
