### テキスト
#### SM705（変換桁数切換）がONの場合の動作
(d)には、有効桁数の左側の0を省略した桁数（最大8桁）の10進アスキーデータが格納されます。
(s)の値と(d)に格納される値の例を示しています。

- 桁数が最大桁数（8桁）未満の場合、SM701（出力文字数切換）のON/OFFに関わらず、文字列の終端に00Hを格納します。文字列の終端が下位バイトの場合、上位バイトにも00Hを格納します。
- 桁数が最大桁数（8桁）の場合、SM701がOFFのときは(d)+4に00Hを格納し、ONのときは(d)+4は変化しません。

### 表
#### (s)の値と(d)～(d)+3の内容対応表

| (s)の値 | (d)～(d)+3の内容 |
| :--- | :--- |
| 0H～9H | (d)の上位バイトに00Hが格納され、(d)+1以降は変化しない。 |
| 10H～99H | (d)+1に00Hが格納され、(d)+2以降は変化しない。 |
| 1000000H～9999999H | (d)+3の上位バイトに00Hが格納される。具体的なASCIIコードの配置は表内図参照。 |
| 10000000H～9999999H | SM701がOFFのとき(d)+4に00Hが格納される。ONのときは変化しない。 |

### 図（表内のビット配置説明）
- ASCIIコードは10進の桁ごとに分割され、b15～b0のビットに対応して格納される。
- 例として、(d)～(d)+3の各バイトにASCII 10^0～10^7のコードが格納される様子が示されている。
- SM701のON/OFFによる格納の違いも図で説明されている。

### 補足説明
- ASCII 10^7は千万の位のアスキーコード、10^6は百万の位のアスキーコード、10^1は十の位、10^0は一の位のアスキーコードを示す。
