### テキスト
#### 32.68 ワードラベルの指定ビットコピー
「CPY_BIT_OF_INT(_E)」は、ワードラベルの指定したビットを別のワードラベルの指定ビットにコピーする機能です。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）です。

##### ラダー図
- EN/ENOなし：入力s、n1、n2を受け取り、出力dへコピーするシンプルなブロック。
- EN/ENO付き：EN（実行条件）とENO（出力状態）を持ち、同様の入力と出力を持つブロック。

##### ST言語
- EN/ENOなし：d := CPY_BIT_OF_INT(s, n1, n2);
- EN/ENO付き：d := CPY_BIT_OF_INT_E(EN, ENO, s, n1, n2);

※FBD/LDでは対応しておらず、ラベルのビット指定を使用する必要があります。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s | 入力 | 入力変数 | INT |
| n1 | 入力変数のビット指定(0～15の定数のみ指定可能) | 入力変数 | INT |
| n2 | 出力変数のビット指定(0～15の定数のみ指定可能) | 入力変数 | INT |
| ENO | 出力状態(TRUE:正常実行、FALSE:異常または停止) | 出力変数 | BOOL |
| d | 入出力 | 入出力変数 | INT |

#### 機能
- 演算処理：dのn2ビット目に、sのn1ビット目の値をコピーする。
- 演算結果：
  1. EN/ENOなしファンクション：演算処理を実行し、dから演算出力値が出力される。
  2. EN/ENO付きファンクション：実行条件に応じて演算結果が変わる。

| 実行条件 EN | 演算結果 ENO | (d) |
| --- | --- | --- |
| TRUE (演算実行) | TRUE | 演算出力値 |
| FALSE (演算停止) | FALSE | 不定値 |

※ ENOがFALSEの場合、dから出力されるデータは不定値となるため、その場合はdのデータを使用しないようプログラムで処理する必要がある。

### 図
- ラダー図（EN/ENOなし）：s（入力）、n1、n2（ビット指定）、d（出力）を持つシンプルなブロック。
- ラダー図（EN/ENO付き）：EN（実行条件）、ENO（出力状態）、s、n1、n2、dを持つブロック。
