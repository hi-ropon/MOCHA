### テキスト
#### 1 命令概要
##### 1.1 命令の構成
シーケンサで使用できる命令は、命令部とオペランド部に分かれます。
- 命令部：命令の機能を示す部分
- オペランド部：命令で使用するデータを示す部分

オペランド部は「ソースデータ」「ディスティネーションデータ」「数値データ」に分類されます。

#### ソース(s)
ソースは演算で使用するデータです。命令で指定するラベルやデバイスにより以下の種類があります。

| 種類       | 内容                                                                                      |
|------------|-------------------------------------------------------------------------------------------|
| 定数       | 演算で使用する数値を指定。プログラム作成時に設定し、実行中は変更不可。可変データの場合はインデックス修飾が必要。 |
| デバイス/ラベル | 演算で使用するデータの格納先であるデバイスまたはラベルを指定。実行前に指定デバイスやラベルにデータを格納しておく必要がある。実行中は格納データを変更可能。 |

※インデックス修飾については別マニュアル参照。

#### ディスティネーション(d)
ディスティネーションは演算後のデータが格納される場所です。ただし命令によっては演算前に格納が必要な場合もあります。

#### 例：BIN16ビットデータの加算命令の場合
- 図1：(+)命令の構成で、s（ソース）とd（ディスティネーション）が示されている。
  - (1) 演算実行前に使用するデータを格納
  - (2) 演算結果のみ格納

ディスティネーションには必ずデータを格納するためのラベルやデバイスを設定する必要があります。

### 図
- BIN16ビットデータの加算命令の命令構成図
  - 左図：+ 命令にs（ソース）とd（ディスティネーション）がある
  - 右図：+ 命令にs1、s2（複数のソース）とdがある
