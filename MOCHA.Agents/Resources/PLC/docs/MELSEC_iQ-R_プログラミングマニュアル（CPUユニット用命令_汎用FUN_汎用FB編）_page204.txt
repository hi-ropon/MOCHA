### テキスト
#### 注意事項
- PHASECHG命令はフェーズ処理内で実行する必要がある。フェーズ処理外で実行すると、最後にON実行したPHASE命令の指定デバイスをOFFにする。ただし、フェーズの移行（指定デバイスのON）は行われる。
- PHASE命令をON実行後、PHASECHG命令を実行するまでの間にRUN中の回路ブロック変更やシーケンサへの書き込みがあった場合は例外。
- 直前のPHASE命令で指定したデバイスと同じデバイスをPHASECHG命令で指定しないこと。指定するとフェーズは終了せず、移行もしない。
- PHASE命令で使用していないデバイスをPHASECHG命令で指定しないこと。指定するとフェーズは終了するが移行はしない。指定したデバイスはONになる。
- デバイスに「x」を使用する場合は、実入力で使用していないデバイス番号を使うこと。実入力デバイスと同一番号を使うと、実入力データがPHASECHG命令で指定した入力xに上書きされる。

#### エラー
- 演算エラーはない。

### 図
1. PHASE命令とPHASECHG命令で同じデバイスを指定した場合のイメージ図  
   - PHASE命令でM0をONにし、PHASECHG命令で同じM0を指定すると、フェーズは終了せず移行もしないことを示す。
2. PHASE命令で使用していないデバイス（例：M100）をPHASECHG命令で指定した場合のイメージ図  
   - フェーズは終了するが移行はしない。指定したデバイスはONになることを示す。

以上の内容はPHASE命令とPHASECHG命令の正しい使用方法と注意点を説明している。
</contents>
