### テキスト
#### 同一デバイスのPLF命令を使用した場合
PLF命令は、実行指令がONからOFFに変わる時に指定デバイスをONにします。ON→OFF以外（OFF→ON、ON→ON）の場合は指定デバイスをOFFにします。
同一デバイスのPLF命令を1スキャンに複数回実行した場合、各PLF命令の実行指令がON→OFFのときに指定デバイスをONにし、それ以外のときは指定デバイスをOFFにします。
そのため、同一デバイスのPLF命令を1スキャンに複数回実行した場合、PLF命令でONしたデバイスが1スキャン中にONしないことがあります。

### 図
#### ラダー図とタイミングチャート
- ラダー図にはX0、X1の入力と、それぞれに対応するPLF命令でM0を制御する回路が示されています。
- タイミングチャートはX0、X1、M0のON/OFF状態を時間軸で表現しています。
- X0とX1のON/OFFタイミングが異なる場合、指定デバイスM0が1スキャン中にONしない例を示しています。

##### タイミングチャートの説明
1. X0がON→OFFのため、M0はONになる。
2. X1がON→OFF以外のため、M0はOFFになる。
3. X0がON→OFF以外のため、M0はOFFのまま。
4. X1がON→OFF以外のため、M0はOFFのまま。

このように、複数のPLF命令が同一デバイスを制御する場合、ON/OFFタイミングの違いにより期待通りにデバイスがONにならないことがあることを示しています。
