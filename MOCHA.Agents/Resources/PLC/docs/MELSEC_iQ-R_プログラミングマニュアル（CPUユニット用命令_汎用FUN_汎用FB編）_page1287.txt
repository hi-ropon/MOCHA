### テキスト
#### 機能
- 指定ブロックを、停止していたステップから実行を再開させる機能について説明しています。
- 動作保持状態で停止した動作保持ステップ（移行チェックあり/なし）では、動作保持状態で実行を再開します。
- SFC用情報デバイスのブロック停止再開ビットが設定されている場合、そのビットデバイスをOFFにします。
- SM325（ブロック停止時の出力モード）の状態により、ブロック再開後のPLS命令や立上り時のみ実行する命令の動作が異なることを説明しています。
- プロセスCPU（二重化モード）の場合、バックアップモードの待機系で実行しても無処理となることを記載しています。

#### 注意事項
- コイル保持ステップは、保持中にSM325がOFFの時に停止すると非活性となり、保持状態を再開できません。
- SM325がONの時に停止した場合は再開可能です。

### 表
#### SM325の状態とPLS命令、立上り時のみ実行する命令の動作

| SM325の状態               | PLS命令、立上り時のみ実行する命令の動作 |
|--------------------------|------------------------------------------|
| ON時（コイル出力保持）   | 再実行しません。                          |
| OFF時（コイル出力OFF）   | 再実行します。                            |

#### エラーコード(SD0)一覧

| エラーコード | 内容                                                                                   |
|--------------|----------------------------------------------------------------------------------------|
| 31A1H        | 指定ブロックが存在しないとき。SFCプログラムが存在しない（スキャン実行タイプのSFCプログラムが存在しない）、またはSM321（SFCプログラムの起動/停止）がOFFに設定して命令を実行したとき。 |
| 31A2H        | 指定ブロックNo.が指定可能なブロックNo.の範囲外のとき。                               |
