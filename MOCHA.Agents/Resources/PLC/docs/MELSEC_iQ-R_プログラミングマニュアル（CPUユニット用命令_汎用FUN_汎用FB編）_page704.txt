### テキスト
#### 機能
(s2)で指定したスケーリング用変換データ（32ビットデータ単位）に対し、(s1)で指定した入力値からスケーリングを行い、(d)で指定されたデバイス番号に演算結果を格納します。スケーリング変換は、(s2)で指定したデバイス以降に格納したスケーリング用変換データに基づいて行います。

- 演算結果が整数値にならない場合は、小数点以下1桁目を四捨五入します。
- スケーリング用変換データのX座標データは昇順に設定してください。
- (s1)はスケーリング用変換データ範囲内((s2)、(s2)+1のデバイス値)で設定してください。
- 複数のポイントが同一X座標を指定している場合は、ポイントNo.が最も大きいポイントのY座標の値を出力します。
- (s2)で指定するスケーリング用変換データの座標ポイント数は1～4294967295にしてください。

#### 注意事項
- 探索方法と探索回数範囲は、SM755（スケーリングデータチェック設定）のON/OFFにより異なります。
- スケーリング用変換データが昇順にソートされている場合、SM755の状態により探索方法が異なり、処理速度も変わります。探索回数が少ないほど処理速度は速くなります。

### 表
#### 設定項目（nは(s2)で指定する座標ポイント数を示す）
| 座標ポイント数 | デバイス割付 |
| :--- | :--- |
| (s2)+1, (s2) |  |
| ポイント1 | X座標: (s2)+3, (s2)+2<br>Y座標: (s2)+5, (s2)+4 |
| ポイント2 | X座標: (s2)+7, (s2)+6<br>Y座標: (s2)+9, (s2)+8 |
| ... |  |
| ポイントn | X座標: (s2)+4n-1, (s2)+4n-2<br>Y座標: (s2)+4n+1, (s2)+4n |

#### SM755による探索方法と探索回数範囲
| SM755 | 探索方法 | 探索回数範囲 |
| :--- | :--- | :--- |
| OFF | 逐次探索 | 1回数 ≤ 4294967295 |
| ON | 二分探索 | 1回数 ≤ 32 |

### 図・グラフ
#### 演算範囲図
- 横軸X、縦軸Yの座標系で、ポイント1からポイントnまでの座標点を結んだ折れ線グラフ。
- 演算可能範囲は(s1)の範囲内で示され、演算エラーは範囲外を指す。
- (d)はポイントの位置を示す。

#### 探索回数と処理速度の関係グラフ
- 横軸に(s1)、縦軸に座標ポイント数をとった折れ線グラフ。
- 逐次探索回数は最大32767、二分探索回数は最大15。
- (1) 逐次探索回数 < 二分探索回数の場合、逐次探索の処理速度が速い。
- (2) 逐次探索回数 > 二分探索回数の場合、二分探索の処理速度が速い。

以上より、スケーリング用変換データの設定方法、探索方法の違いによる処理速度の変化、及び演算範囲のイメージが理解できます。
