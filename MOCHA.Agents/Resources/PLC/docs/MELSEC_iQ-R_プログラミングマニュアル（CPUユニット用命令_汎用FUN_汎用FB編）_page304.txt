### テキスト
#### 16ビットブロックデータ論理積 BKAND(P)
指定した2つのBIN16ビットデータブロックの論理積を行う命令について説明しています。

- **命令名**: BKAND, BKANDP
- **対応CPU**: RnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）
- **説明**: 2つの16ビットビットデータブロックの論理積を実行する。

#### ラダー図とFBD/LD図
- ラダー図では、s1, s2, d, n の4つのオペランドが示されている。
- FBD/LD図では、EN（実行条件）、ENO（実行結果）、s1, s2, n, d の入出力端子が描かれている。

### 表
#### 実行条件
| 命令   | 実行条件の波形例               |
|--------|-------------------------------|
| BKAND  | 立下りエッジ                   |
| BKANDP | 立上りエッジ                   |

#### 設定データ：内容、範囲、データ型
| オペランド | 内容                                               | 範囲           | データ型           | データ型（ラベル） |
|------------|----------------------------------------------------|----------------|--------------------|--------------------|
| (s1)       | 論理積を行うデータまたは、論理積を行うデータが格納されている先頭デバイス | －             | 符号付きBIN16ビット | ANY16*1            |
| (s2)       | 論理積を行うデータまたは、データが格納されている先頭デバイス           | -32768～32767  | 符号付きBIN16ビット | ANY16*1            |
| (d)        | 論理積の結果を格納する先頭デバイス                                   | －             | 符号付きBIN16ビット | ANY16*1            |
| (n)        | 演算データ個数                                               | 0～65535       | 符号なしBIN16ビット | ANY16              |
| EN         | 実行条件                                                   | －             | ビット             | BOOL               |
| ENO        | 実行結果                                                   | －             | ビット             | BOOL               |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要がある。
