### テキスト
#### 注意事項
- CPY_BIT_OF_INT(_E)をSTで使用する場合は、戻り値を変数に代入するプログラムを作成する必要がある。
  - 例: `g_int2 := CPY_BIT_OF_INT(g_int1,5,3);`
  - 例: `g_bool1 := GET_BIT_OF_INT(g_int2,3);`
- CPY_BIT_OF_INT(_E)の戻り値は入出力を兼ねているため、他の命令やファンクション、ファンクションブロックの入力引数に直接代入しないこと。
  - 例として、GET_BIT_OF_INTの第一引数にCPY_BIT_OF_INTを直接使うと不定値になる。

#### エラー
- 演算エラーはない。

#### Point
- ラベルのビット指定を使うと、CPY_BIT_OF_INTと同じ動作のプログラムを簡潔に作成できる。
- 例として、CPY_BIT_OF_INTを使った場合と同様に、`g_int2`のビット3（b3）を`g_int1`のビット5（b5）の値に変更する方法を示す。

### 図（言語化説明）
- **ラダーの場合**: ラダー図で`g_int1.5`から`g_int2.3`へビット指定で値をコピーする線が引かれている。
- **STの場合**: `g_int2.3 := g_int1.5;`という1行のコードでビット指定コピーを表現。
- **FBD/LDの場合**: ファンクションブロック図で`g_int1.5`から`g_int2.3`へビット指定コピーを示すブロックが配置されている。

以上より、ビット単位のコピー操作を複数のプログラミング言語・表現方法で簡潔に実装できることを説明している。
