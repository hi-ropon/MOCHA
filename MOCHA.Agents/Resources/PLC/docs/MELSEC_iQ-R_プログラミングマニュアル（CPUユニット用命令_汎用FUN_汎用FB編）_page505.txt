### テキスト
#### 機能
- (s)の下位4ビットで指定された0～Fのデータを7セグメント表示データにデコードし、(d)に格納する。
- ビットデバイスの場合、(d)は7セグメント表示データを格納する先頭デバイスを示す。
- ワードデバイスの場合、(d)は格納するデバイス番号を示す。

#### 例
- **ビットデバイスの場合**  
  Y48～Y4Fは次のデータが出力されるまで変化しない。  
  図示では、SEG K7 K2Y48の命令があり、Y48～Y4Fのビット列（例：00100111）が示されている。

- **ワードデバイスの場合**  
  上位8ビットは必ず0、下位8ビットに7セグメント表示データを格納する。  
  図示では、SEG K7 D8の命令があり、b15～b0のビット列（例：0000000000010111）が示されている。

#### 注意事項
- 7セグメントデコード表は以下の表に示す。

### 図
- 7セグメントの構成図は、7つのセグメント（b0～b6）がどの位置に対応するかを示している。
- ビット配置はb7～b0で示され、b7は常に0。
- ビットパターンは4ビットで16進数0～Fを表し、それに対応した7セグメントの点灯パターンが(d)に格納される。

### エラー
- 演算エラーはありません。
