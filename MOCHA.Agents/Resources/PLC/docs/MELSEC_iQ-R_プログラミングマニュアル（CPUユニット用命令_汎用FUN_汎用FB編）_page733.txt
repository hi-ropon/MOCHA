### テキスト
#### BIN16ビットデータソート（SORTD(_U)）
指定した(n)点分のBIN16ビットデータを昇順または降順にソート（並び替え）します。

#### 実行条件
- SORTD、SORTD_U命令は立ち上がりエッジで実行されます。

### 図
- **ラダー図**  
  ラダー回路では、(s1)、(n)、(s2)、(d1)、(d2)の各オペランドが並び、ENO出力が示されています。  
  ST言語では、`ENO:=SORTD(EN,s1,n,s2,d1,d2);` または `ENO:=SORTD_U(EN,s1,n,s2,d1,d2);` と記述します。

- **FBD/LD図**  
  ブロック図では、EN入力、s1、n、s2の入力とENO、d1、d2の出力端子が示されています。

### 表
#### 設定データ：内容、範囲、データ型

| オペランド | 内容                         | 範囲       | データ型           | データ型(ラベル)               |
|-------------|------------------------------|------------|--------------------|-------------------------------|
| (s1)        | ソートするテーブルの先頭デバイス | —          | 符号付きBIN16ビット | ANY16_S*1                     |
|             |                              |            | 符号なしBIN16ビット | ANY16_U*1                     |
| (n)         | ソートデータ数               | 0～65535   | 符号なしBIN16ビット | ANY16                         |
| (s2)        | 1回の実行で比較するデータ数 | 0～65535   | 符号なしBIN16ビット | ANY16                         |
| (d1)        | ソート完了でONさせるビットデバイス番号 | —          | ビット               | ANY_BOOL                      |
| (d2)        | システム使用デバイス         | —          | 符号付きBIN16ビット | ANY16_ARRAY（要素数：2）       |
| EN          | 実行条件                     | —          | ビット               | BOOL                         |
| ENO         | 実行結果                     | —          | ビット               | BOOL                         |

*1 ラベルで設定する場合は、動作に必要な領域が確保できる配列を定義し、その配列型ラベルの要素を指定する必要があります。

- T、C、STは使用できません。

### ポイント
- SORTDは符号付きBIN16ビットデータのソート、SORTD_Uは符号なしBIN16ビットデータのソートを行います。
- ソート完了時に指定したビットデバイス(d1)がONになります。
- (s1)はソート対象のテーブル先頭デバイス、(n)はソートデータ数、(s2)は比較するデータ数を指定します。
- (d2)はシステム使用デバイスで、ANY16_ARRAY型の配列で2要素必要です。
