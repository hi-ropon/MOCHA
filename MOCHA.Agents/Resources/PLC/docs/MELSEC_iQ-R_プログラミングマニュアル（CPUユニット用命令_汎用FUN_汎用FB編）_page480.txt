### テキスト
#### 機能概要
- (s)で指定されたデバイス番号以降に格納されている16進アスキーデータをBIN32ビットデータに変換し、(d)で指定されたデバイス番号に格納する。
- SM705の変換桁数切換（ON/OFF）状態によって、(s)に設定する16進アスキーデータの設定方法が異なる。
- SM705に対応しているCPUユニットのファームウェアバージョンについては別マニュアルを参照。
- SM705非対応CPUユニットでは、ONにしてもOFFの場合と同様の動作となる。

#### SM705(変換桁数切換)がOFFの場合の(s)の設定方法
- (s)から(s)+3に16進アスキーデータを8桁固定で設定する。
- ASCIIコードは各桁ごとに設定し、範囲は30H～39Hおよび41H～46H。
- (s)～(s)+3で指定するアスキーデータは00000000H～FFFFFFFFHの範囲内。
- (s)+4以降のデータは無視される。

### 表
| SM705*1の状態 | (s)の設定方法 | 参照先 |
| :--- | :--- | :--- |
| OFF | 固定桁数(8桁)で設定 | 480ページ SM705(変換桁数切換)がOFFの場合の(s)の設定方法 |
| ON | 任意の桁数(最大8桁)で設定 | 481ページ SM705(変換桁数切換)がONの場合の(s)の設定方法 |

*1 SM705対応CPUユニットのファームウェアバージョンについては別途参照。

### 図
#### 16進アスキーデータのビット配置とBIN32ビットデータへの変換
- (s)～(s)+3の各デバイスにASCIIコードが格納されている。
- 各ASCIIコードは2桁の16進数で表され、b15～b0のビットに対応。
- 4つのASCIIコードを組み合わせてBIN32ビットデータに変換。
- BIN32は(d)+1と(d)の2つのデバイスに分割して格納される。
- 上位16ビットと下位16ビットに分けて格納。

#### 例
- (s)に5CB807E1Hが指定された場合のビット配置例を示す。
- (s)～(s)+3の各ビットにASCIIコード（16進数）が割り当てられ、BIN32データとして(d)+1に5CB8H、(d)に07E1Hが格納される。
