### テキスト
#### BIN16ビットデータテーブルソート
- 対応CPUは、R00CPU、R01CPU、R02CPUはファームウェアバージョンが"08"以降、RnENCPUは"40"以降で使用可能。
- エンジニアリングツールのバージョンは1.050C以降が必要。
- コマンドSORTTBL(_U)は、(n1)×(n2)点のBIN16ビットデータテーブルを、(n3)列のデータを基準に昇順または降順で並び替え、並び替え後のテーブルに格納する。

#### Point
- 行方向に連続したデバイスでテーブルを構成する場合は、指定の命令を使用すること。

### 図
#### BIN16ビットデータテーブルの並び替えイメージ
- 左図は並び替え前のテーブルで、(n1)×(n2)の点に対し、(s)で示された元データが配置されている。
- 赤枠で囲まれた列（(s)+5列）を基準に昇順/降順で並び替えを行う。
- 右図は並び替え後のテーブルで、(d)で示された並び替え後のデータが格納されている。
- 並び替えは(n3)列方向に対して行われている。

#### ラダー図
- ラダー命令ブロックにて、パラメータ(s)(n1)(n2)(d)(n3)を入力し、SORTTBL命令を実行する形を示す。

#### FBD/LD図
- ブロック図でEN（イネーブル）、s（元データアドレス）、n1、n2、n3（テーブルサイズや列数）、d（出力先アドレス）、ENO（イネーブル出力）を示す。

### 表
| 命令       | 実行条件 |
|------------|----------|
| SORTTBL    |          |
| SORTTBL_U  |          |

- 実行条件の詳細は記載なし。

### ST言語例
- ENO:=SORTTBL(EN,s,n1,n2,n3,d);
- ENO:=SORTTBL_U(EN,s,n1,n2,n3,d);

以上より、BIN16ビットデータテーブルの特定列を基準に昇順/降順で並び替えを行う命令の使い方とイメージが示されています。
