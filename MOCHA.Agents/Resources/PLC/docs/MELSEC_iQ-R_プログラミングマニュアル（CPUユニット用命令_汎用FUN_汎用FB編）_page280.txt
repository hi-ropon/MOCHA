### テキスト
#### BIN16ビットブロックデータ減算
指定した2つのBIN16ビットデータブロックを減算する命令について説明しています。

#### 実行条件
- BK-、BK-_U命令は立ち上がりエッジで実行される。
- BK-P、BK-P_U命令は立ち上がりエッジで実行される。

#### 注意事項
- バージョン"1.035M"以降のエンジニアリングツールで対応している。

### 図
#### ラダー図
- (s1)、(s2)、(d)、(n)の4つのオペランドがあり、これらを用いて減算処理を行う構造を示す。

#### FBD/LD図
- EN（入力イネーブル）、s1、s2、n（減算データ個数）を入力し、d（演算結果）とENO（実行結果）を出力するブロック図。

### 表
#### 設定データ（内容、範囲、データ型）
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s1) BK-(P) | 減算されるデータまたは、減算されるデータが格納されている先頭デバイス | -32768～32767 | 符号付きBIN16ビット | ANY16_S*1 |
| (s1) BK-(P)_U | 同上（符号なしBIN16ビット） | 0～65535 | 符号なしBIN16ビット | ANY16_U*1 |
| (s2) BK-(P) | 減算データまたは、減算データが格納されている先頭デバイス | -32768～32767 | 符号付きBIN16ビット | ANY16_S*1 |
| (s2) BK-(P)_U | 同上（符号なしBIN16ビット） | 0～65535 | 符号なしBIN16ビット | ANY16_U*1 |
| (d) BK-(P) | 演算結果を格納する先頭デバイス | — | 符号付きBIN16ビット | ANY16_S*1 |
| (d) BK-(P)_U | 同上（符号なしBIN16ビット） | — | 符号なしBIN16ビット | ANY16_U*1 |
| (n) | 減算データ個数 | 0～65535 | 符号なしBIN16ビット | ANY16 |
| EN | 実行条件 | — | ビット | BOOL |
| ENO | 実行結果 | — | ビット | BOOL |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要がある。
