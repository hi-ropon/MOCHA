### テキスト
#### バイナリ読出し時の注意点
- 対象ファイルの拡張子を省略した場合、拡張子は「.BIN」となります。
- 存在しないファイルが指定された場合はエラーとなります。
- 既存のファイルの大きさよりも大きいファイル位置が指定された場合は、0点での読出しとなり、正常完了します。

#### バイナリ読出しの例（BIN16ビットデータの場合）
バイナリ読出し時のデータ構造と読み込みの流れを示しています。

### 図の説明
- **コントロールデータ**（(d1)+0～(d1)+7）
  - (d1)+0: 実行/完了タイプ（例：0000H）
  - (d1)+1: 完了ステータス（例：-）
  - (d1)+2: 要求読出しデータ数（例：K3）
  - (d1)+3: 最大読出しデータ数（例：K0）
  - (d1)+4: K1（詳細不明）
  - (d1)+5: ファイル位置
  - (d1)+6: 列数指定（例：K0）
  - (d1)+7: データタイプ指定（例：K0）

- **データ格納デバイス**（(d2)+0～(d2)+3）
  - (d2)+0: K3（データ数）
  - (d2)+1～(d2)+3: 読出しデータ（16ビット単位で2バイトずつ）
    - (d2)+1: 33H 22H
    - (d2)+2: 55H 44H
    - (d2)+3: 77H 66H

- **ファイルデータ（バイト単位）**
  - 00H, 11H, 22H, 33H, 44H, 55H, 66H, 77H, 88H, 99H, AAH
  - ファイルデータのバイト列から、16ビット単位でデータ格納デバイスに読み込まれている様子を矢印で示しています。

この図は、ファイルのバイトデータがどのように16ビット単位のデータとしてデバイスに読み込まれるかを視覚的に表現しています。
