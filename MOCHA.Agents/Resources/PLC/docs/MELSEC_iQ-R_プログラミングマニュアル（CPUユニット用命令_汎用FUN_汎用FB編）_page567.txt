<overview>
このページは、プログラム制御におけるDI（割込み禁止）命令とDI（指定優先度以下の割込み禁止）命令の実行順序に関する説明と、それに伴うスキャン実行タイププログラムの動作タイミングを示す図が含まれています。さらに、これらの命令に関連するエラーコードとその内容をまとめた表が掲載されています。構成は、上部に命令実行のタイミングを示す図、中段に説明テキスト、下部にエラーコードの表という構成です。
</overview>

### テキスト・図
- DI（指定優先度以下の割込み禁止）命令とDI（割込み禁止）命令を実行した場合の動作順序について、2つのパターンのタイミング図が示されています。
- 図中の「A」はスキャン実行タイププログラムを表し、(1)、(2)、(3)はそれぞれ割込み許可区間や禁止区間の優先度範囲を示しています。
- 1つ目の図は「DI(指定優先度以下の割込み禁止)命令→DI(割込み禁止)命令」の順で実行した場合の動作を示し、3つの区間（すべての優先度の割込み許可区間、優先度2～8の割込み禁止区間、優先度1～8の割込み禁止区間）に分けて説明しています。
- 2つ目の図は「DI(割込み禁止)命令→DI(指定優先度以下の割込み禁止)命令」の順で実行した場合の動作を示し、2つの区間（すべての優先度の割込み許可区間、優先度1～8の割込み禁止区間）に分けて説明しています。

### 表：エラーコード
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3362H              | DI(割込み禁止)命令、DI(指定優先度以下の割込み禁止)命令によるネスティングが16重を超えた場合。 |
| 3405H              | (s)で指定した優先度が範囲外であるとき。 |

- エラーコード3362Hは、DI命令によるネスティング（入れ子状態）が16回を超えた場合に発生します。
- エラーコード3405Hは、指定した優先度が許容範囲外の場合に発生します。
