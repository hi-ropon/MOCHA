本ページは、マルチCPUシステムにおけるユニット形名の読み出し方法について説明しています。ページ構成は以下の通りです。
- テキスト部分：マルチCPUシステム構成時のユニット形名の読み出しに関する説明と注意点。
- 図部分：ユニット形名を保持している場合、保持していない場合、空きスロットの場合のユニット構成と対応するデータ格納例を示す3つの図。
- 表部分：エラーコード（SD0）とその内容をまとめた表。

<contents>
### テキスト
- マルチCPUシステム構成時に、他号機のCPUユニットが管理しているユニットを指定しても、ユニット形名は読み出せないことを説明しています。
- プログラム例として、M0がONしたときに、入出力番号が0020Hに装着したユニットのユニット形名をD0以降に格納する方法を示しています。

### 図
#### 形名を保持しているユニット（例：RJ71GP-SX）
- ユニット構成は「PW（電源）」「CPU」「RJ71 GP21-SX」。
- 先頭入出力番号は0020H。
- D0～D9の各データに16進数でユニット形名の文字コードが格納されている。
- D0は0を格納し、ユニットが形名を保持していることを示す。
- D1～D4に「4AH(J)」「52H(R)」「31H(1)」「37H(7)」など、ユニット形名の文字コードが格納されている。
- 17文字に満たない場合は残りの文字に00Hを格納し、18文字目にも00Hを格納する。

#### 形名を保持していないユニット（例：RX40）
- ユニット構成は「PW」「CPU」「RX40」。
- 先頭入出力番号は0020H。
- D0に1を格納し、ユニット種別と点数を組み合わせた文字列を格納したことを示す。
- D1～D4に「4EH(N)」「49H(I)」「55H(U)」「50H(P)」など、ユニット種別と点数を組み合わせた文字列が格納されている。
- 17文字に満たない場合は残りの文字に00Hを格納し、18文字目にも00Hを格納する。

#### 空きスロットの場合
- ユニット構成は「PW」「CPU」「空き」。
- 先頭入出力番号は0020H。
- D0に-1を格納し、形名を格納していないことを示す。
- D1～D9はすべて00Hを格納。

### 表
#### エラーコード（SD0）と内容
| エラーコード | 内容 |
| :--- | :--- |
| 2800H | (H)の指定が0～FFH、3E0～3E3H以外のとき。 |
| 2810H | 読出し対象ユニットが故障などの原因で交信できないとき。 |
