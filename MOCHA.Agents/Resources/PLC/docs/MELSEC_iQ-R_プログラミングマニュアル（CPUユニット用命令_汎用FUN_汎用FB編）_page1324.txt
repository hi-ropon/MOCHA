### テキスト
#### WORD型→DINT型変換
関数「WORD_TO_DINT(_E)」はWORD型データをDINT型データへ変換します。

対応CPUはRnCPU、RnECPU（プロセス、三重化、一般、安全）で、RnSFCPU（一般、安全）には対応していません。

#### 使用例
- ラダー回路（FBD/LD）では、EN/ENOなしとEN/ENO付きの2種類の回路図が示されています。
- ST言語では、EN/ENOなしの場合は「d:=WORD_TO_DINT(s);」、EN/ENO付きの場合は「d:=WORD_TO_DINT_E(EN,ENO,s);」と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | WORD |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | DINT |

#### 機能
- (s)に入力されたWORD型のデータをDINT型のデータに変換して(d)から出力します。
- データ変換後、上位16ビットは0となります。
- (s)への入力値はWORD型のデータ値です。

#### 演算処理のイメージ
WORD型の16ビットデータ（例：5678H）がDINT型の32ビットデータ（例：22136）に変換され、上位16ビットは0で埋められます。

#### 演算結果
1. EN/ENOなしファンクション  
　演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション  
　実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |
| --- | --- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、その場合は(d)から出力されたデータを使用しないようプログラムで処理してください。
