### テキスト
#### BCD8桁減算
指定した2つのBCD8桁データを減算する命令について説明しています。

- 対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（一般、安全）で、RnSFCPU（一般、安全）には対応していません。
- ラダー、ST、FBD/LDのうち、STとFBD/LDは対応していません（3オペランドの場合は268ページ参照）。

#### 実行条件
| 命令 | 実行条件 |
| --- | --- |
| DB- | 立ち上がりエッジ |
| DB-P | 立ち下がりエッジ |

#### 機能
(d)で指定したBCD8桁データから(s)で指定したBCD8桁データを減算し、その結果を(d)で指定したデバイスに格納します。

- 減算例として、(d)+1、(d)、(s)+1、(s)、(d)+1のビット配置が示され、(s)の下位4桁に0が入ることが説明されています。
- 減算結果にアンダーフローが発生した場合は、結果が99999999となり、SM700（キャリフラグ）はONしません。

| オペランド | 内容 | 範囲 | データ型 | データ型（ラベル） |

### 図
- ラダー回路の簡易図：2つのオペランド(s)と(d)を指定して減算を行うイメージ。
- 減算例：
  - (d)+1、(d)、(s)+1、(s)、(d)+1の8桁BCDデータのビット配置を示し、(s)の下位4桁に0が入ることを説明。
  - アンダーフロー発生時の例として、12345678 - 12345679 = 99999999となることを示し、キャリフラグはONしないことを明記。
