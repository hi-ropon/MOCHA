### テキスト
#### BOOL型→INT型変換（BOOL_TO_INT(_E)）
BOOL型データをINT型データへ変換する関数について説明しています。

##### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件（TRUE:実行、FALSE:停止） | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | BOOL |
| ENO | 出力状態（TRUE:正常実行、FALSE:異常または停止） | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

##### 機能
- 演算処理  
  入力されたBOOL型データ(s)をINT型データに変換し、dから出力します。  
  入力値がFALSEの場合はINT型の0を出力し、TRUEの場合はINT型の1を出力します。  
  sへの入力値はBOOL型のデータ値です。

- 演算結果  
  1. EN/ENOなしファンクション  
  演算処理を実行し、dから演算出力値が出力されます。  
  2. EN/ENO付きファンクション  
  実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
| :--- | :--- | :--- |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となります。その場合は(d)から出力されたデータを使用しないようプログラム処理を行う必要があります。

##### エラー
演算エラーはありません。

### 図
- ラダー、FBD/LD言語での関数ブロック図  
  EN/ENOなしとEN/ENO付きの2種類のブロック図が示されています。  
  EN/ENOなしは単純にs入力とd出力のブロック、EN/ENO付きはEN入力、ENO出力が追加されています。

- ST言語での記述例  
  EN/ENOなし: d := BOOL_TO_INT(s);  
  EN/ENO付き: d := BOOL_TO_INT_E(EN, ENO, s);
