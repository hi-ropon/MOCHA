<overview>
本ページは、SM705の変換桁数切換機能に関する説明で構成されています。テキスト部分では機能の概要と動作の説明があり、表部分ではSM705の状態ごとの(d)の格納形式が示されています。さらに、動作例としてBINDA(P)命令実行時のビットデータの格納状態を示す図があり、OFF/ONの場合の違いを視覚的に説明しています。
</overview>

<contents>
### テキスト
#### 機能
- (s)で指定されたBIN16ビットデータを10進数で表現した数値をアスキーコードに変換し、(d)で指定されたデバイス番号以降に格納する。
- SM705（変換桁数切換）の状態によって、(d)に格納される10進アスキーデータの形式が異なる。

#### 動作の概要
- SM705がOFFの場合とONの場合の動作をそれぞれ説明。
- 例として、(s)に数値「-12」が格納されている状態でBINDA(P)命令を実行した場合の動作を示す。

### 表
| SM705*1の状態 | (d)の格納形式 | 参照先 |
| :--- | :--- | :--- |
| OFF | 固定桁数（符号＋5桁）で格納 | 784ページ SM705(変換桁数切換)がOFFの場合の動作 |
| ON | (s)の値によって各桁を前詰めで格納 | 784ページ SM705(変換桁数切換)がONの場合の動作 |

*1 SM705に対応しているCPUユニットのファームウェアバージョンについては別途参照が必要。対応していないCPUユニットではON/OFFに関わらずOFFと同様の動作となる。

### 図（動作例の説明）
- (s)に「-12」が格納されている状態でBINDA(P)命令を実行した場合の(d)の格納状態を示す。
- SM705: OFFの場合
  - (d)+0に「20H」「2DH」（符号'-'）が格納される。
  - (d)+1に「20H」「20H」
  - (d)+2に「32H」「31H」
  - 1文字目は符号で、数値部分は右詰めで格納される。数値部分が5桁に満たない場合、上位桁は20H（スペース）で埋められる。
- SM705: ONの場合
  - (d)+0に「31H」「2DH」
  - (d)+1に「00H」「32H」
  - (d)+2はNULLで埋められる。
  - 数値部分は前詰めで格納され、5桁に満たない場合は終端に00Hが格納される。

### 補足
- SM705がOFFの場合は固定桁数で、1文字目は符号、正の数の場合はスペースとなる。
- SM705がONの場合は前詰めで格納され、数値部分が5桁に満たない場合は終端に00Hが入る。
