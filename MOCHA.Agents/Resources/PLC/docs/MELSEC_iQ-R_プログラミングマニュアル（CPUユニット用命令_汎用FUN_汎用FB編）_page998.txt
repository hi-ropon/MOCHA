### テキスト
#### 機能
- ブロックNo.を意識せずに、(s2)で指定されたデバイスに格納されている下位8ビットの内容を、(s1)で指定された通しバイトNo.のファイルレジスタに書き込む。
- (s2)で指定されたデバイスの上位8ビットのデータは無視される。
- 通しバイトNo.に対応するファイルレジスタ番号の対応関係が示されている。

#### 例
- (s1)=12340の場合、ZR11170の下位8ビットに書き込む。
- (s1)=43257の場合、ZR21628の上位8ビットに書き込む。

#### エラー
| エラーコード(SD0) | 内容 |
|-------------------|------|
| 2820H             | 指定可能な範囲を超えたデバイス番号（通しバイトNo.）を指定したとき。 |

### 図
- ファイルレジスタの構造図：ZR0からZR32768までのエリアがブロックNo.0、No.1、No.2に分かれていることを示す。
- 書き込み先指定の説明図：通しバイトNo.が指定されると対応するファイルレジスタの下位8ビットにデータが書き込まれ、上位8ビットは無視されることを示す。
- ファイルレジスタ番号と対応する通しバイトNo.の関係図：奇数番号指定時と偶数番号指定時のデータ配置を示す。
- 具体例のビット配置図：
  - (s1)=12340の場合、ZR6170の下位8ビットにデータを書き込み、上位8ビットは無視される。
  - (s1)=43257の場合、ZR21628の上位8ビットにデータを書き込み、下位8ビットは無視される。

### 表
- エラーコード表として、2820Hのエラーコードとその内容が記載されている。
