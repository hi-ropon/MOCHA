### テキスト
#### 使用可能デバイス
- オペランドごとに使用可能なビット、ワード、ダブルワード、間接指定、定数の種類が表形式で示されている。
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、特定の安全デバイスおよび定数のみ使用可能。

#### 機能
- (s)で指定されたデバイスのBIN16ビットデータと同一のデータを、(d)で指定されたデバイスへ(n)点分転送する。
- (s)がワードデバイスで(d)がビットデバイスの場合、ワードデバイスはビットデバイスの桁指定で指定されたビット数が対象となる。
- 例として、(d)にK1Y30が指定された場合、(s)で指定されたワードデバイスの下位4ビットが対象となる。
- (s)が桁指定したビットデバイスで(d)がワードデバイスの場合、ワードデバイスはビットデバイスの桁指定で指定されたビット数が対象となる。
- 例として、(s)にK1X20が指定された場合、(d)のワードデバイスの下位4ビットに転送される。
- (s)、(d)が両方ともビットデバイスの場合は、(s)、(d)の桁数を同一にする必要がある。

#### エラー
- 演算エラーは発生しない。

### 図
#### ビットデバイスからワードデバイスへの転送例
- (s)のビットデバイス3456Hのデータを(d)の連続したワードデバイスに(n)点分転送するイメージ図。
- (d)+1、(d)+2…と連続して同じデータ3456Hが転送される。

#### ワードデバイスからビットデバイスへの転送例
- (s)にK1Y30が指定された場合、(s)のワードデバイスD100の下位4ビットが(d)のビットデバイスの指定された範囲に転送される。
- (s)にK1X20が指定された場合、(d)のワードデバイスの下位4ビットに転送される様子を示す。
- 転送時のビット配置や無視されるビット、格納されるビットの説明付き。
