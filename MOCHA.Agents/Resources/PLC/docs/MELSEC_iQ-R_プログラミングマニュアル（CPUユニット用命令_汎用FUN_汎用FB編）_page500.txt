### テキスト
#### 8→256ビットデコード（DECO(P)）
指定されたデバイスの下位(n)ビットをデコードします。

#### 実行条件
- DECO命令は立ち下がりエッジで実行
- DECOP命令は立ち上がりエッジで実行

### 図
#### ラダー図
- ラダー回路の接点として、(s)、(d)、(n)の3つの入力があり、ST命令として
  ENO:=DECO(EN,s,n,d);
  ENO:=DECOP(EN,s,n,d);
  の形で使用される。

#### FBD/LD図
- ブロック図形状は四角形で、左側にEN、s、nの入力端子、右側にENO、dの出力端子がある。

### 表

| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s) | デコードデータまたは、デコードデータが格納されているデバイス番号 | なし | ビット/符号付きBIN16ビット | ANY_ELEMENTARY |
| (d) | デコード結果を格納するデバイス | なし | ビット/ワード | ANY_ELEMENTARY |
| (n) | 有効ビット長 | 1～8 | 符号なしBIN16ビット | ANY16 |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |
