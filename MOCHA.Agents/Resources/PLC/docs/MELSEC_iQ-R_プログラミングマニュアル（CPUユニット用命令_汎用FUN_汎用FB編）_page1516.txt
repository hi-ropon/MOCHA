### テキスト
#### 43.6 オンディレイタイマ (TON(_E))
- プロセスCPU（冗長モード）およびSIL2プロセスCPUの両系実行プログラムで使用した場合、バックアップモードの待機系では動作しない旨の注意書きがあります。
- 指定された時間の後に信号をONにするタイマ機能です。

#### 機能
##### 演算処理
1. 出力
- 入力sがONになると、nで設定された時間経過後にd1がONになる。
- d2にはd1がONしてからの遅延経過時間がセットされる。
- sがOFFになるとd1はOFFになり、遅延経過時間もリセットされる。
- 経過時間のカウントにはロングタイマを使用。

2. 遅延時間の設定
- nの有効設定範囲はT#1ms～T#2147483ms。
- エンジニアリングツールのタイマ時限設定を変更することで有効範囲が変わる場合がある。
- nの設定値はdがOFF→ON（立上り）となったときの値が使用される。
- sがONの時にnの値を変更した場合、変更値が有効になるのはsの次回立上り時。

### 図
- ラダー図（EN/ENOなし、EN/ENO付き）とST言語でのタイマ使用例が示されている。
  - ラダー図はs（時間計測）、n（遅延時間設定）、d1（出力）、d2（経過時間）を示す。
  - ST言語ではインスタンス名とパラメータ（IN:=s, PT:=n, Q:=d1, ET:=d2）を指定。

### 表
#### 設定データ（引数一覧）
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 時間計測 | 入力変数 | BOOL |
| n(PT) | 遅延時間設定値 | 入力変数 | TIME |
| ENO | 出力状態(TRUE:正常、FALSE:異常または停止) | 出力変数 | BOOL |
| d1(Q) | 出力 | 出力変数 | BOOL |
| d2(ET) | 経過時間 | 出力変数 | TIME |

#### 遅延時間設定の最小値・最大値
| 最小値 | 最大値 |
| :--- | :--- |
| タイマ時限のロングタイマ設定値[ms]と同じ。ただし1ms未満の場合は1ms。 | 遅延時間設定値は時間型(32ビット値)のため最大値は時間型の範囲に収まる値。<br>例：<br>・0.001msの場合はT#1ms～T#2147483ms<br>・1000msの場合はT#1000ms～T#2147483000ms |
