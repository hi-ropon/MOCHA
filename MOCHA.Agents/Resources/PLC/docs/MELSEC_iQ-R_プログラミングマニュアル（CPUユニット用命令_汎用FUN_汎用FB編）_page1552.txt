### 表
#### 命令処理時間一覧表
命令名ごとに、処理条件とCPUユニット別の処理時間（単位はマイクロ秒）が記載されています。CPUユニットはR00CPU、R01CPU、R02CPU、及びそれ以外のCPUユニットに分類され、それぞれ最小値と最大値が示されています。

- **命令名**: 例としてFROMD、DFROMD、TOD、DTOD、TYPERD、UNIINFRD、S.RTREAD、S.RTWRITE、LOGTRG、LOGTRGR、DATATRG、SP.SOCOPEN、SP.SOCCLOSEなど。
- **条件**: バッファメモリの読み出し・書き込み回数(n=1, n=320, n=500, n=1000など)、CPUバッファメモリへの読み出し・書き込み、自動機能の有無、TCP/UDP通信状態などが記載。
- **処理時間**: 各CPUユニットごとに最小値と最大値が示されている。例えば、FROMD命令のバッファメモリ読み出し(n=1)では、R00CPU・R01CPUで最小8.4μs、最大8.9μs、R02CPUで最小4.5μs、最大6.4μs、その他CPUユニットで最小4.0μs、最大5.7μsとなっている。

#### 特記事項
- UNIINFRD命令の処理時間はプロセスCPU使用時の値も併記されている。
- DATATRGは収集対象デバイス点数が32点であることが記載されている。
- SP.SOCOPENとSP.SOCCLOSEはTCPおよびUDPの各種状態（Active、Unpassive、Fullpassive、自機から実行、相手機器から実行）で処理時間が示されている。

全体として、命令の種類や条件により処理時間が大きく異なることがわかり、CPUユニットごとの性能差も明示されています。
</contents>
