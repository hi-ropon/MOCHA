### テキスト
#### 機能
- (s1)、(s2)に入力されたTIME型データの加算((s1)+(s2))を行い、演算結果を(d)からTIME型で出力する。

#### 演算処理
- (s1)、(s2)への入力値はTIME型のデータ値である。
- 演算結果にアンダーフローやオーバーフローが発生しても演算エラーにはならず、(d)にそのまま出力される。
- ADD_TIME_Eの場合、出力変数ENOからTRUEが出力される。

#### 例
- 通常の加算例：
  - (s1)に「T#1d2h33m44s55ms」（1日2時間33分44秒55ミリ秒）
  - (s2)に「T#2ms」（2ミリ秒）
  - 演算結果(d)は「T#1d2h33m44s57ms」となる。

- オーバーフロー例：
  - (s1)に「T#24d20h31m23s647m」（7FFFFFFFH）
  - (s2)に「T#2ms」（00000002H）
  - 演算結果(d)は「T#-24d20h31m23s647ms」（80000001H）となり、最上位ビットが1になるため負の時間となる。

- アンダーフロー例：
  - (s1)に「T#-24d20h31m23s648ms」（80000000H）
  - (s2)に「T#-2ms」（FFFFFFFFH）
  - 演算結果(d)は「T#24d20h31m23s646ms」（7FFFFFFH）となり、最上位ビットが0になるため正の時間となる。

#### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、(d)から演算出力値が出力される。

2. EN/ENO付きファンクション
- 実行条件と演算結果は以下の表の通り。

### 表
| 実行条件 | 演算結果 |  |
| :--- | :--- | :--- |
| EN | ENO | (d) |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となるため、そのデータを使用しないようプログラムで処理する必要がある。

#### エラー
- 演算エラーは発生しない。
