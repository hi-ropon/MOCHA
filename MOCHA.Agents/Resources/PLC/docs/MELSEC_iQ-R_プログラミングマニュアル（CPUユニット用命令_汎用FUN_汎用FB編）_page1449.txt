### テキスト
#### nビット右ローテーション（ROR(_E)）
入力値をnビット分右へ回転して出力する命令です。対応CPUはRnCPU、RnENCPU、RnPCPU（二重化・一般・安全）、RnPSFCPU（一般・安全）です。

ラダー図はEN/ENOなしとEN/ENO付きの2種類があり、ST言語ではEN/ENOなしの場合は「d:=ROR(s,n);」、EN/ENO付きの場合は「d:=ROR_E(EN,ENO,s,n);」と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力データ | 入力変数 | ANY_BIT |
| n(N) | シフトビット数指定 | 入力変数 | ANY_BIT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力データ | 出力変数 | ANY_BIT |

#### 機能
- 入力された16ビットまたは32ビットデータ（WORD型/DWORD型）をnビット右へ回転（ローテーション）し、同じデータ型で出力します。
- 右回転するビット数nは入力値に依存します。

#### 例
入力データが16ビットWORD型で、nへの入力値が3の場合、3ビット分右へ回転します。

#### 入力値の範囲
- (s)のデータが16ビットデータ(WORD型)の場合：
  - nの入力値は0～15の範囲内
  - nへの入力値の下位4ビット分のデータが使用される
  - 例：入力値6の場合は6、220の場合も6として扱う
- (s)のデータが32ビットデータ(DWORD型)の場合：
  - nの入力値は0～31の範囲内
  - nへの入力値の下位5ビット分のデータが使用される
  - 例：入力値6の場合は6、22の場合は22として扱う

### 図・表の説明
- ラダー図：EN/ENOなしはs,nを入力しdを出力。EN/ENO付きはEN,ENO,s,nを扱う。
- ST言語表記：EN/ENOなしとありの2パターン。
- ビット回転イメージ：WORD/DWORDのビット列が右へnビット分回転する様子を矢印で示す。
- 具体例のビット展開図：16進数270FHのビット列が3ビット右回転しE4E1Hになる過程をビット単位で示す。
- 表：引数の内容・種別・データ型を一覧化。入力値の範囲と使用されるビット数の違いを16ビット/32ビットで比較。
