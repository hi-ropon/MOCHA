### テキスト
#### SM705（変換桁数切換）がONの場合の動作
- (d)には10進アスキーデータが前詰めで格納される。
- (s)の値と(d)～(d)+2に格納される値の例を示す。

#### 演算桁数に関する注意事項
- 演算桁数が最大桁数（符号+10桁）未満の場合、SM701（出力文字数切換）のON/OFFに関わらず、文字列の終端に00Hを格納する。文字列の終端が下位バイトの場合、上位バイトにも00Hを格納。
- 演算桁数が最大桁数の場合、SM701がOFFのときは(d)+5の上位バイトに00Hを格納。
- SM701がONのときは(d)+5の上位バイトに20H（スペース）を格納。

#### エラー
- 演算エラーはない。

### 表・図
#### (s)の値と(d)～(d)+2の内容の対応表
- 左側の表は、(s)の値が「0」または「正の数（数値部1桁）」の場合の内容を示す。
  - (d)の上位バイトに00Hが格納され、(d)+1以降は変化しない。
  - ビット配置図では、b15～b0のビットにASCIIコード（ASCII 10^0）が格納されている。

- 右側の表は、(s)の値が「正の数（数値部2桁）」または「負の数（数値部1桁）」の場合の内容を示す。
  - (d)+1に00Hが格納され、(d)+2以降は変化しない。
  - ビット配置図では、ASCIIコードの10^0、10^1、2DH（マイナス記号）がビットに対応。

#### 大きい桁数の数値例
- 正の数（数値部9桁、10桁）および負の数（数値部8桁、10桁）について、(d)～(d)+5にASCIIコードが順に格納される。
- 負の数（数値部10桁）の場合、(d)の上位バイトにSM701のON/OFFに応じて00Hまたは20H（スペース）が格納される。
- ASCIIコードの説明：
  - ASCII 10^9：十億の位のアスキーコード
  - ASCII 10^8：一億の位のアスキーコード
  - ASCII 10^1：十の位のアスキーコード
  - ASCII 10^0：一の位のアスキーコード

このページは、SM705の変換桁数切換がONの場合のデータ格納方法を詳細に示しており、特に10進アスキーデータのビット配置と格納位置の具体例を表と図で説明しています。また、演算桁数の最大値に関する注意点とエラーの有無についても記載しています。
