テキスト部分では、デバイスのビット状態のチェックと結果の格納方法について説明しています。  
図はビットの状態を示すビットマップの例を表しており、ビットのON/OFF判定の仕組みを視覚的に示しています。

### テキスト
#### 機能
- (s)で指定したデバイスの(n)ビットの状態がONかOFFかをチェックし、その結果を(d)で指定したデバイスへ格納します。
- 結果がONであれば(d)にONを、OFFであれば(d)にOFFを格納します。

### 図
- ビットマップの例が示されており、b31からb0までの32ビットが並んでいます。
- 各ビットの状態（0または1）が示されており、(n)で指定したビット位置の状態を読み取ります。
- 例として(n)=5のビットは0（OFF）、(n)=29のビットは1（ON）であることが示されています。
- (s)+1, (s)はビットの範囲を示し、(d)は結果を格納するデバイスを指しています。

### 表
#### エラー
| エラーコード (SD0) | 内容                         |
|--------------------|------------------------------|
| 3405H              | (n)に指定した値が下記範囲外のとき。0～31 |

- 指定したビット番号(n)が0～31の範囲外の場合にエラーコード3405Hが発生します。
</contents>
