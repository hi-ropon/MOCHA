### 表
#### 使用可能デバイス一覧
- オペランドごとに使用可能なビット、ワード、ダブルワード、間接指定、定数、その他のデバイスが表形式で示されている。
- (s1)、(s2)、(d)の3種類のオペランドに対して、それぞれ使用可能なデバイスが○で示されている。
- SIL2プロセスCPUおよび安全CPUの安全プログラムでは、特定の安全デバイスおよび定数のみ使用可能であることが別表で示されている。

### テキスト
#### 機能
- (s1)で指定したBIN32ビットデータと、(s2)で指定したBIN32ビットデータの乗算を行い、その結果を(d)で指定したデバイスに格納する。
- (d)がビットデバイスの場合、乗算結果の下位32ビットまでが対象で、上位32ビットを指定することはできない。
- 上位32ビットのデータが必要な場合は、一度ワードデバイスにデータを格納し、ワードデバイスの(d)+2、(d)+3のデータを指定ビットデバイスに転送する必要がある。

#### 例
- (d)がビットデバイスの場合の乗算結果のビット割り当て例
  - K1: 下位4ビット (b0～b3)
  - K4: 下位16ビット (b0～b15)
  - K8: 下位32ビット (b0～b31)

#### エラー
- 演算エラーは発生しない。

### 図
- BIN32ビットデータの乗算イメージ図
  - (s1)+1、(s1)、(s2)+1、(s2)のビット範囲を示し、乗算結果が(d)+3、(d)+2、(d)+1、(d)に格納される様子をビット単位で表現。
  - 乗算結果の数値例（70109427840）も記載されている。
