### テキスト
#### 浮動小数点数データの整数部・小数部の桁数指定
- 全桁数から符号、小数点、小数部を除いた桁数が単精度実数データの整数部より大きい場合、符号と整数部の間に20H（スペース）を格納する。
- (s1)で指定できる32ビット浮動小数点型実数データの整数部桁数は1～16桁。
- 変換した文字列の最後には自動的に00Hを格納。

#### 指数形式
- (s2)に1を指定すると指数形式となる。
- (s2)+1は全桁数、(s2)+2は小数部桁数を示す。
- 変換後の文字列データは(d)以降のデバイス番号に格納される。
- 整数部の符号には単精度実数データが正のとき20H（スペース）、負のとき2DH（-）を格納。
- 整数部は1桁固定で、整数部と符号の間に20H（スペース）を格納。
- (s2)+1で指定できる全桁数は、小数部桁数が0のときは桁数（最大24）≥2となり、0以外のときは桁数（最大24）≥(小数部桁数+7)。
- (s2)+2で指定できる小数部桁数は0～7桁。ただし、小数部桁数≤(全桁数-7)となるように設定。
- 小数部桁数の範囲に単精度実数データの小数部が収まらない場合は、下位小数部が四捨五入される。
