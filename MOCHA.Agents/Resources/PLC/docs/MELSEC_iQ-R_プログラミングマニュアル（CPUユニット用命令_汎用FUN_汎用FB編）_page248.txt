### テキスト
#### BIN16ビット乗算命令 *(P)(_U)
指定した2つのBIN16ビットデータを乗算する命令です。対応CPUはRnCPU、RnENCPU（プロセス、三重化、一般、安全）です。  
バージョン1.035M以降のエンジニアリングツールで対応しています。

#### 実行条件
命令には4種類あり、実行条件は以下の通りです。  
- * および *_U：立ち上がりエッジで実行  
- *P および *P_U：立ち下がりエッジで実行

### 図
#### ラダー図
- ラダー図では、(s1)、(s2)の2つの入力と(d)の出力を持つ命令ブロックが示されています。

#### FBD/LD図
- FBD/LD図では、EN（実行条件）、ENO（実行結果）、s1、s2（乗算対象データ）、d（演算結果格納先）を持つ命令ブロックが示されています。  
- ブロック内にはMULTI、MULTIP、MULTI_U、MULTIP_Uのいずれかが入ります。

### 表
#### 実行条件一覧

| 命令   | 実行条件               |
|--------|------------------------|
| *      | 立ち上がりエッジ       |
| *_U    | 立ち上がりエッジ       |
| *P     | 立ち下がりエッジ       |
| *P_U   | 立ち下がりエッジ       |

| オペランド | 内容                                   | 範囲           | データ型           | データ型(ラベル) |
|------------|--------------------------------------|----------------|--------------------|------------------|
| (s1)       | 乗算されるデータまたは乗算されるデータが格納されているデバイス | -32768～32767  | 符号付きBIN16ビット | ANY16_S          |
|            |                                      | 0～65535       | 符号なしBIN16ビット | ANY16_U          |
| (s2)       | 乗算データまたは乗算データが格納されているデバイス           | -32768～32767  | 符号付きBIN16ビット | ANY16_S          |
|            |                                      | 0～65535       | 符号なしBIN16ビット | ANY16_U          |
| (d)        | 演算結果を格納する先頭デバイス                             | —              | 符号付きBIN32ビット | ANY32_S          |
|            |                                      |                | 符号なしBIN32ビット | ANY32_U          |
