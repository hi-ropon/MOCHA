### テキスト
#### BIN16ビット単位スケーリング(ポイント別座標データ)
スケーリング用変換データ（16ビットデータ単位）に対し、指定した入力値でスケーリング（ポイント別座標データ）を行う命令について説明しています。

#### 実行条件
- SCL、SCL_U命令は立ち下がりエッジで実行
- SCLP、SCLP_U命令は立ち上がりエッジで実行

### 図
- **ラダー図**  
  (s1)、(s2)、(d)の3つのオペランドを持つ命令のラダー表現が示されています。  
  ENO:=SCL(EN,s1,s2,d); ENO:=SCLP(EN,s1,s2,d); ENO:=SCL_U(EN,s1,s2,d); ENO:=SCLP_U(EN,s1,s2,d)の形式で記述。

- **FBD/LD図**  
  EN、s1、s2を入力、dを出力とする命令ブロックの図が示されています。

| (s2)       | スケーリング用変換データが格納されている先頭デバイス | なし         | 符号付きBIN16ビット*1 | ANY16_S*2 (SCL(P))<br>ANY16_U*2 (SCL(P)_U) |

*1 (s2)の座標ポイント数は符号なしBIN16ビットとなる。  
*2 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する。
