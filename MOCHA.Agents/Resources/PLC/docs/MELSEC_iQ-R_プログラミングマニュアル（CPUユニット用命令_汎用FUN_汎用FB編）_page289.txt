### テキスト
#### 機能
- (d)で指定されたデバイス（BIN16ビットデータ）を+1する動作について説明しています。
- 図により、b15からb0までのビットで構成される16ビットデータが+1される様子を示しています。
- (d)で指定されたデバイスの内容が32767のときにINC(P)命令を実行すると、-32768が(d)で指定されたデバイスに格納される（符号付きの場合）。
- (d)で指定されたデバイスの内容が65535のときにINC(P)_U命令を実行すると、0が(d)で指定されたデバイスに格納される（符号なしの場合）。

#### エラー
- 演算エラーは発生しないと明記されています。

### 図
- 16ビットビットデータのビット構成を示す図で、b15からb0までのビットが並び、5678（BIN）が+1されて5679（BIN）になる様子を表現しています。
