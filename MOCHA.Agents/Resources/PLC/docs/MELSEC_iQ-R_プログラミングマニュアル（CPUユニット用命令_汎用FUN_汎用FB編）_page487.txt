### テキスト
#### SM705（変換桁数切換）がONの場合の(s)の設定方法
(s)には任意の桁数（00HのNULLコードまで）の10進アスキーデータを設定します。ただし最大桁数が8桁の場合は00Hの設定は不要です。
(s)の設定方法を以下に示します。

- (s)～(s)+3で指定するアスキーデータは0～99999999の範囲内です。
- (s)+4以降のデータは無視されます。
- 各位に設定するアスキーコードは30H～39Hの範囲内です。
- 各位に設定するアスキーコードが20Hのときは30Hとして処理されます。00Hのときは10進アスキーデータの終端として処理されます。

#### エラー
| エラーコード(SD0) | 内容 |
|-------------------|------|
| 3401H             | (s)に変換できない不正なデータを入力したとき。<br>・データ中に0～9以外の文字がある。 |

### 表・図
#### (s)に設定する値と(s)～(s)+3の内容対応表

| (s)に設定する値 | (s)～(s)+3の内容 |
|-----------------|------------------|
| 0～9            | (s)の上位バイトに00Hを設定。<br>(s)+1以降は無視。<br>（図）b15～b0のビット配置：(s)に00HとASCII 10^0を設定。 |
| 10～99          | (s)+1の下位バイトに00Hを設定。<br>(s)+1の上位バイト以降は無視。<br>（図）b15～b0のビット配置：ASCII 10^0とASCII 10^1を設定。 |
| 1000000～99999999 | (s)+3の上位バイトに00Hを設定。<br>(s)+4以降は無視。<br>（図）b15～b0のビット配置：ASCII 10^5～10^0を4バイトに分けて設定。 |
| 10000000～999999999 | (s)+4以降は無視。<br>（図）b15～b0のビット配置：ASCII 10^7～10^0を4バイトに分けて設定。 |

#### ASCIIコードの説明
- ASCII 10^7: 千万の位のアスキーコード
- ASCII 10^6: 百万の位のアスキーコード
- ASCII 10^5: 十万の位のアスキーコード
- ASCII 10^4: 万の位のアスキーコード
- ASCII 10^3: 千の位のアスキーコード
- ASCII 10^2: 百の位のアスキーコード
- ASCII 10^1: 十の位のアスキーコード
- ASCII 10^0: 一の位のアスキーコード

### ポイント
- (s)～(s)+3の4バイトで最大8桁の10進アスキーデータを表現。
- 00Hは10進アスキーデータの終端コードとして扱われる。
- 20Hのコードは30Hとして処理される。
