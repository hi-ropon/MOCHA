### テキスト
#### 機能
(s)で指定された符号付きBIN32ビットデータを、符号なしBIN16ビットデータに変換し、(d)で指定されたデバイスに格納する機能について説明しています。

具体的には、プログラム例でM0がONしたとき、D0～D1に格納された符号付きBIN32ビットデータを符号なしBIN16ビットデータに変換し、D100へ格納します。

#### エラー
演算エラーはありません。

### 図
プログラム例のフローチャートとデータのビット構成を示しています。

- SM402の接点がONになると、DMOVP命令でH87654321（16進数）のデータがD0に格納されます。
- 次にDINT2UINT命令で、D0の符号付きBIN32ビットデータを符号なしBIN16ビットデータに変換し、D100に格納します。

ビット構成の説明：
- D1、D0に格納された符号付きBIN32ビットデータは、上位16ビット（b31～b16）が8765H、下位16ビット（b15～b0）が4321Hで、符号付き32ビット整数として-2023406815を表します。
- 変換後のD100には下位16ビットの4321H（10進数で17185）が符号なし16ビットデータとして格納されます。

(1)は「下位16ビットへ格納」を示しています。
