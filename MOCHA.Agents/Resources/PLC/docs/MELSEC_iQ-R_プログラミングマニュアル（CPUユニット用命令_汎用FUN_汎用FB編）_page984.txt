### テキスト
#### 機能
- インデックスレジスタの内容を、(d)で指定したデバイスまたは配列型ラベルの要素以降に退避する。
- 退避すると退避回数(d)が+1される。
- インデックスレジスタは24ワード分退避される。インデックスレジスタが0点の場合はロングインデックスレジスタを12点分退避。
- データ復帰にはZPOP(P)命令を使用。ZPUSH(P)命令とZPOP(P)命令は対で使用し、入れ子構造でスタックとして利用可能。
- 入れ子構造はZPUSH(P)命令の(d)で指定した領域を、再度ZPUSH(P)命令の(d)に指定することで作成可能。実行ごとに退避回数が+1される。
- 別領域を指定した場合は指定した別領域にインデックスレジスタの内容を退避。
- 入れ子構造の場合、ZPUSH(P)命令実行ごとに退避データが追加されるため、使用回数分の領域を確保する必要あり。
- (d)以降に使用される領域の構成は図の通り。

#### 図の説明
- 図は(d)から始まるメモリ領域の構成を示す。
- (d)+0は退避回数を示す。
- (d)+1から(d)+24までは1ネスティング目（24ワード分）の退避領域。
- (d)+25以降は2ネスティング目の退避領域。
- 各ネスティング目はZ0、Z1などのインデックスレジスタの内容が格納される。
