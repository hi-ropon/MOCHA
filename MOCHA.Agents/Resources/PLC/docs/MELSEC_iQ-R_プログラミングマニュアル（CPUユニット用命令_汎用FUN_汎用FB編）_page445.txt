### テキスト
#### 機能
(s)で指定された符号なしBIN16ビットデータを、符号なしBIN32ビットデータに変換し、(d)で指定されたデバイスに格納する機能について説明しています。具体的には、M0がONしたとき、D0に格納された符号なしBIN16ビットデータを符号なしBIN32ビットデータに変換し、D100～D101へ格納する例が示されています。

#### エラー
演算エラーはありません。

### 図
プログラム例のラダー図とビットデータの構造を示しています。

- ラダー図では、SM402の接点とM0の接点があり、M0がONするとMOVP命令でH0F000をD0に格納し、その後UINT2UDINT命令でD0の16ビットデータを32ビットデータに変換してD100に格納しています。
- ビットデータ構造の説明では、
  - (s)の16ビットデータD0はb15～b0の範囲で「F000H」（10進数61440）を保持。
  - (d)の32ビットデータD101,D100はb31～b0の範囲で、上位16ビット（b31～b16）に0を格納し、下位16ビット（b15～b0）にD0の値「F000H」を格納しています。
- (1)は上位16ビットに0を格納することを示し、(2)は下位16ビットに元の16ビットデータを格納することを示しています。
