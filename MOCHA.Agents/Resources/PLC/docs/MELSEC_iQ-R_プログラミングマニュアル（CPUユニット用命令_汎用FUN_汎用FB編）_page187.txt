### テキスト
#### 機能
- 実行指令のOFF→ON時に指定デバイスをONにし、それ以外の状態変化（ON→ON、ON→OFF、OFF→OFF）の場合はOFFにしません。
- 1スキャン中に(d)で指定されたデバイスのPLS命令が1つの場合、指定デバイスは1スキャンONします。
- 同一デバイスに対してPLS命令を1スキャンに複数回実行した場合の動作は後述のタイミングチャートを参照。

#### 動作説明
- PLS命令実行後にRUN→STOPにし、再度RUNにしてもPLS命令は実行されません。
- 実行指令にラッチリレー(LR)を指定し、LRがON状態で電源をOFF→ONにすると、実行指令は1スキャン目にOFF→ONとなり、PLS命令を実行して指定デバイスがONします。電源ON後1スキャン目でONしたデバイスは次のPLS命令（立上り出力）でOFFされます。
- PLS命令は命令実行後の次の命令実行時にOFF処理を行います。ただし、SIL2プロセスCPUおよび安全CPUの安全プログラムでは、PLS命令は安全サイクル処理の安全プログラムが動作し、命令が実行されるまで指定のデバイス/ラベルが安全サイクル処理ONとなります。
- PLS命令の引数に一般/安全共有ラベルを使用する場合、安全サイクル処理の割り込みタイミングにより、該当ラベルを使用している安全プログラムまたは一般プログラムでON検出できないことや複数周期にわたりON検出することがあります。

#### Point
- PLS命令をCJ命令でジャンプさせたり、実行したサブルーチンプログラムをCALL(P)命令でコールしなかった場合、(d)で指定のデバイスが1スキャン以上ONになることがあるため注意が必要。

#### エラー
- 演算エラーはありません。

### 図（タイミングチャート）
1. **1つ目のタイミングチャート**
   - X5入力がOFFからONに変わると、PLS命令によりM0がONになる。
   - X5がOFFになるとM0もOFFになる。
   - ON期間は1スキャン（Sc）分で示されている。

2. **2つ目のタイミングチャート**
   - X0入力のON/OFFに伴うPLS命令の動作を詳細に示す。
   - PLS M0が1スキャンONし、その後CPUユニットの演算停止時間、RUN/STOP/RESETスイッチの状態変化により命令の実行状態が変わる様子を示す。
   - 各状態変化は番号(1)～(4)で説明されている。
   - (1) PLS M0の1スキャン
   - (2) CPUユニットの演算停止時間
   - (3) CPUユニットのRUN/STOP/RESETスイッチがRUN→STOPに切り替わる
   - (4) CPUユニットのRUN/STOP/RESETスイッチがSTOP→RUNに切り替わる

以上より、PLS命令の動作タイミングと注意点が詳細に説明されています。
