### テキスト
#### ユニットアクセス命令の処理選択
- SD775のビットb0～b4、b6、b13、b15で実行/非実行を選択し、SM775をONにすることで処理を選択可能。
- 各ビットの意味は以下の通り。
  1. I/Oリフレッシュ、マルチCPUシステムのグループ外の入出力取込み
  2. CC-Linkユニットのリンクリフレッシュ
  3. CC-Link IEコントローラネットワークユニット、MELSECNET/Hユニットのリンクリフレッシュ
  4. インテリジェント機能ユニット
  5. マルチCPUシステムのCPUバッファメモリを使用したリフレッシュ（END時）
  6. CC-Link IEフィールドネットワークユニットのリンクリフレッシュ
  7. CC-Link IEフィールドネットワークBasicのリンクリフレッシュ
  8. デバイス/ラベルアクセスサービス処理（エンジニアリングツール、GOT、その他外部機器との交信）

- シンプルモーションユニット(RD77GF)のリンクリフレッシュも含む。
- COM(P)命令実行時に指定されたリフレッシュ処理を行う。

#### プログラム例
- SM402がONのとき、M0がONするとCC-Link IEフィールドネットワークユニットのリンクリフレッシュを実行する例を示す。

#### 注意事項
- COM(P)命令はプログラム中で何回でも使用可能。ただし、SD775で選択された処理の時間分だけプログラムのスキャンタイムが延びる。
- COM(P)命令実行中は割込み許可状態となるため、割込みプログラムでリフレッシュデータを使用するとデータの泣き別れが発生する可能性がある。
- EthernetポートにEthernet接続時、COM(P)命令でデバイス/ラベルアクセスサービス処理を行うと処理時間が延びる可能性がある。

#### Point
- COM(P)命令は割込みプログラムでは使用できない。

#### エラー
- 演算エラーはなし。

### 図

- COM(P)命令の処理フロー図：COM(P)命令実行時に指定した処理が行われ、END命令で終了する流れを示す。
