このページは主にテキストと図で構成されており、BIN32ビットデータの減算機能について説明しています。テキスト部分では機能の概要とエラーの有無について記載されており、図部分ではBIN32ビットデータの減算の具体例が示されています。表は含まれていません。

### テキスト
#### 機能
(s1)で指定したBIN32ビットデータと、(s2)で指定したBIN32ビットデータの減算を行い、その結果を(d)で指定したデバイスに格納します。

- 減算結果にアンダーフローが生じた場合、桁上げは無視されます。この場合、SM700（キャリフラグ）はONしません。

#### エラー
演算エラーはありません。

### 図
- BIN32ビットデータの減算例が3パターン示されています。

1. 正の数同士の減算例  
   (s1)に567890、(s2)に123456を指定し、減算結果444434を(d)に格納。

2. 負の数を含む減算例（D-(P)の場合）  
   - (s1)に1234567890、(s2)に-987654321を指定し、結果-2072745085を(d)に格納。  
   - (s1)に-1234567890、(s2)に987654321を指定し、結果2072745085を(d)に格納。

3. D-(P)_Uの場合  
   (s1)に3456789012、(s2)に3060399406を指定し、結果396389606を(d)に格納。

各図では、ビット位置（b31～b0）とデータの2進数表現が示され、どのビットがどのデータに対応しているかが視覚的に説明されています。
