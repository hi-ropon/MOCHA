### テキスト
#### ユニットからの1ワード/2ワードデータリード(16ビット指定)
FROM(P)、DFROM(P)命令は、指定されたユニットや他号機CPUユニット内のバッファメモリアドレスからデータを読み出す命令です。
- FROM(P): nワードのデータを読み出す。
- DFROM(P): n×2ワードのデータを読み出す。

#### ラダー、ST、FBD/LDでの使用例
- ラダー: (U/H)、(s)、(d)、(n)の各パラメータを指定する形で記述。
- ST: ENO:=FROM(EN,U/H,s,n,d); などの形式で記述。
- FBD/LD: EN、U/H、s、nを入力し、dを出力するブロック図。

### 図
- ラダー図: ユニット指定(U/H)、読み出し開始アドレス(s)、データ格納先(d)、読み出しデータ数(n)の入力を示す。
- FBD/LD図: EN入力、U/H、s、n入力、d出力のブロック図。
- 実行条件タイミング図: FROM/DFROM命令は立下りで実行、FROMP/DFROMP命令は立上りで実行されることを示す。

### 表
#### 実行条件
| 命令       | 実行条件           |
|------------|--------------------|
| FROM       | 立下り             |
| DFROM      | 立下り             |
| FROMP      | 立上り             |
| DFROMP     | 立上り             |

#### 設定データ（内容、範囲、データ型）
| オペランド | 内容                                               | 範囲          | データ型           | データ型(ラベル) |
|------------|----------------------------------------------------|---------------|--------------------|------------------|
| (U/H)      | ユニットおよびCPUユニットの先頭入出力番号(16進数4桁の上3桁) | 0H～FFFH、3E0H～3E3H | 符号なしBIN16ビット | ANY16            |
| (s)        | 読み出すデータが格納されているバッファメモリまたはCPUユニットの先頭アドレス | 0～65535      | 符号なしBIN16ビット | ANY16            |
| (d)        | 読み出したデータを格納する先頭デバイス                  | —             | 符号付きBIN16ビット (FROM(P)) 符号付きBIN32ビット (DFROM(P)) | ANY16*1 (FROM(P)) ANY32*1 (DFROM(P)) |
| (n)        | 読出しデータ数                                       | 0～65535      | 符号なしBIN16ビット | ANY16            |
| EN         | 実行条件                                           | —             | ビット               | BOOL             |
| ENO        | 実行結果                                           | —             | ビット               | BOOL             |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要がある。
