### テキスト
#### BIN32ビットデータ最小値検索
「DMIN(P)(_U)」命令は、指定したデバイスからn点のBIN32ビットデータの中から最小値を検索する命令です。

##### ■説明
- DMIN命令、DMIN_U命令はST言語、FBD/LD言語では対応していません。汎用ファンクションのMINを使用してください。

##### ■実行条件
| 命令       | 実行条件           |
|------------|--------------------|
| DMIN       | 立ち上がりエッジ    |
| DMIN_U     | 立ち上がりエッジ    |
| DMINP      | ハイレベル         |
| DMINP_U    | ハイレベル         |

##### ■注意事項
- ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定してください。
- 使用するプログラム言語にかかわらず、デバイスで指定します。ラベルは指定しないでください。

### 図
#### ラダー図
- 入力：(s)サーチデータ先頭デバイス、(d)最小値格納先先頭デバイス、(n)サーチデータ数
- 出力：ENO（実行結果）
- 命令は「ENO:=DMINP(EN,s,n,d);」または「ENO:=DMINP_U(EN,s,n,d);」の形式で記述

#### FBD/LD図
- ブロック図でEN（イネーブル）、ENO（イネーブル出力）、s（サーチデータ先頭）、n（サーチ数）、d（最小値格納先）を示す

| (d)        | 最小値の検索結果を格納する先頭デバイス       | —          | 符号付きBIN32ビット / 符号なしBIN32ビット | —*2 (ANY32_S_ARRAY) / —*2 (ANY32_U_ARRAY) |

### まとめ
本命令は、指定した範囲のBIN32ビットデータの中から最小値を検索し、その結果を指定デバイスに格納します。実行条件や使用可能なデバイスの種類が詳細に示されており、ラダー図やFBD/LD図で命令の構造も理解できます。
