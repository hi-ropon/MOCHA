<overview>
本ページは主にテキストと表で構成されており、STRING型データの小数点型式および指数型式への変換方法について説明しています。具体的には、STRING型データのバイト配列の構造を示す表が2つあり、それぞれ小数点型式と指数型式の例を示しています。また、変換結果としてREAL型の数値が示されています。さらに、符号の扱いについてもテキストで解説されています。グラフや図はなく、表とテキスト中心の構成です。
</overview>

<contents>
### テキスト
- STRING型データは、小数点型式と指数型式のどちらにも変換可能であることを説明しています。
- STRING型データは符号、小数点、指数部を除いて6桁が有効で、7桁目以降はカットされて変換されることが記載されています。
- 小数点型式では符号に2BH(+)を指定するか符号を省略すると正の値として変換され、2DH(-)を指定すると負の値として変換されます。
- 指数型式でも指数部の符号に2BH(+)を指定するか符号を省略すると正の値として変換され、2DH(-)を指定すると負の値として変換されます。

### 表（小数点型式の場合）
| オフセット | 上位バイト | 下位バイト | 内容 |
| :--- | :--- | :--- | :--- |
| +0 | 31H(1) | 2DH(-) | 符号「-」を示す |
| +1 | 33H(3) | 2EH(.) | 小数点「.」を示す |
| +2 | 30H(0) | 35H(5) | 数字「0」「5」 |
| +3 | 34H(4) | 33H(3) | 数字「4」「3」 |
| +4 | 00H |  | 終端 |
- このバイト列は「-1.35034」という文字列を表し、REAL型の値「-1.35034」に変換される。

### 表（指数型式の場合）
| オフセット | 上位バイト | 下位バイト | 内容 |
| :--- | :--- | :--- | :--- |
| +0 | 31H(1) | 2DH(-) | 符号「-」を示す |
| +1 | 33H(3) | 2EH(.) | 小数点「.」を示す |
| +2 | 30H(0) | 35H(5) | 数字「0」「5」 |
| +3 | 34H(4) | 33H(3) | 数字「4」「3」 |
| +4 | 2DH(-) | 45H(E) | 指数部の符号「-」と「E」 |
| +5 | 30H(0) | 31H(1) | 指数部の数字「0」「1」 |
| +6 | 00H |  | 終端 |
- このバイト列は「-1.35034E-10」という文字列を表し、REAL型の値「-1.35034E-10」に変換される。

### 変換例
- 小数点型式のSTRINGデータ「-1.35034」はREAL型の「-1.35034」に変換される。
- 指数型式のSTRINGデータ「-1.35034E-10」はREAL型の「-1.35034E-10」に変換される。

### 符号の扱い
- 小数点型式では符号に2BH(+)を指定するか符号を省略すると正の値として変換し、2DH(-)を指定すると負の値として変換する。
- 指数型式でも指数部の符号に2BH(+)を指定するか符号を省略すると正の値として変換し、2DH(-)を指定すると負の値として変換する。
