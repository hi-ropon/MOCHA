### テキスト
#### 注意事項
以下の場合に、(d1)+1の異常完了信号がONになり、(d2)の完了ステータスにエラーコードが格納されます。

- データベースアクセス命令実行中にDBIMPORT(P)命令を実行した場合
- 指定したUnicodeテキストファイルが存在しない場合
- Unicodeテキストファイルのフィールド名行で規定したフィールド数とレコード行のフィールド数が一致しない場合
- Unicodeテキストファイルでテーブル定義開始タグや終了タグがない場合
- Unicodeテキストファイルのキー制約で範囲外の値が設定された場合
- Unicodeテキストファイルのデータベース名、テーブル名、フィールド名が32文字を超えた場合
- Unicodeテキストファイルのテーブル数、フィールド数が最大数を超えた場合
- Unicodeテキストファイルのデータタイプで規定外の型が指定された場合
- Unicodeテキストファイル内のレコード数が最大件数を超えた場合（ファームウェアバージョン"28"より前のシーケンサCPUの場合）
- データベースへのアクセスに失敗した場合
- データベース名に指定不可能な文字が含まれている場合
- Unicodeテキストファイルで指定したデータベース名と、(s)で指定したフォルダパスの文字数（ドライブパス文字含む）の合計が128文字を超えた場合
- オープンしているデータベースが存在する状態でDBIMPORT(P)命令を実行した場合
- コメント行の文字数が最大文字数を超えた場合
- 指定したUnicodeテキストファイルのフォルダ名にASCIIコードで表現可能な文字(0020H～007EH)以外を指定した場合
- DBIMPORT(P)命令実行時に、Unicodeテキストファイルのデータベース名がSDメモリカード内に存在しており、そのデータベースが他の機能によって操作されている場合

Unicodeテキストの書式が原因で異常を検出した場合、DBIMPORT(P)命令は(d1)+1の異常完了信号をONにし、SD760、SD761(DBIMPORT命令Unicodeテキストファイル異常箇所)に異常を検出したUnicodeテキストの行番号を格納します。

#### エラーコード一覧
| エラーコード (SD0) | 内容 |
|--------------------|------|
