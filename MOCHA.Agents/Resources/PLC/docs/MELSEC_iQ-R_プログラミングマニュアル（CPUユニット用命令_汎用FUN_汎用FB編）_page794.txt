### テキスト
#### 注意事項
SM705（変換桁数切換）がONのとき、(d)には演算結果を有効桁数分だけ格納します。そのため、BINHA(P)命令を連続して実行し、演算結果を同一デバイスに格納する場合、(d)の一部に前回の演算結果が上書きされずに残る可能性があります。

#### 例
(s)が「2A6F」のときにBINHA(P)命令を実行し、さらに(s)が「9」のときにBINHA(P)命令を実行する場合の例を示しています。

1. 「2A6F」を文字列に変換
2. 「9」を文字列に変換
3. (d)+1に前回の変換結果の一部が残ったままになる

#### 回避方法
格納エリア(d)+0～(d)+1をすべてクリアしてからBINHA(P)命令を実行するようにプログラムを作成してください。

### 図（説明用イラスト）
1. (s)に「2A6F」があり、BINHA(P)命令実行後の(d)+0～(d)+2の格納状態を示す。
   - (d)+0に「41H (A)」「32H (2)」
   - (d)+1に「46H (F)」「36H (6)」
   - (d)+2に「00H」

2. (s)に「9」があり、BINHA(P)命令実行後の(d)+0～(d)+2の格納状態を示す。
   - (d)+0に「00H」「39H (9)」
   - (d)+1に「46H (F)」「36H (6)」 ← 前回の結果の一部が残っている
   - (d)+2に「00H」

3. 回避策として、(d)+0～(d)+1をクリアした後にBINHA(P)命令を実行した場合の格納状態。
   - (d)+0～(d)+1がすべて「00H」にクリアされている。
   - その後、(s)に「9」を設定し、BINHA(P)命令実行後は
     - (d)+0に「00H」「39H (9)」
     - (d)+1に「00H」「00H」
     - (d)+2に「00H」

このように、格納エリアを事前にクリアすることで、前回の演算結果が残る問題を回避できます。
