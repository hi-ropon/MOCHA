### テキスト
#### 使用可能ブロック数
他号機CPUユニットアクセス命令は、システムエリアを16ワードで構成する「ブロック」を最小単位として使用します。  
他号機CPUユニットアクセス命令で使用可能なブロック数について説明しています。

#### 読出し/書込みが可能な最大データ点数
命令で読出し/書込み可能な最大データ点数は、マルチCPUシステム構成のCPUユニット台数により決まります。

#### 命令で使用するブロック数
命令で使用するブロック数は、書込み/読出しデータ点数によって決まります。  
読出し時と書込み時で計算式が異なり、それぞれの計算式と例が示されています。

---

### 表
#### CPUユニット台数ごとの最大ブロック数

| CPUユニット台数 | 最大ブロック数 |
| :--- | :--- |
| 2台構成 | 599ブロック |
| 3台構成 | 299ブロック |
| 4台構成 | 199ブロック |

#### CPUユニット台数ごとの読出し/書込み可能な最大データ点数

| CPUユニット台数 | 読出し可能な最大データ点数 | 書込み可能な最大データ点数 |
| :--- | :--- | :--- |
| 2台 | 8192点 | 8192点 |
| 3台 | 4096点 | 4096点 |
| 4台 | 2048点 | 2048点 |

#### 命令で使用するブロック数の計算

| 読出し/書込み | ブロック数 | 例 |
| :--- | :--- | :--- |
| 読出し時 | 命令で使用するブロック数 = (21 + 読出しデータ点数) ÷ 16 | 読出しデータ点数が100点の場合、(21 + 100) ÷ 16 = 7[ブロック] |
| 書込み時 | 命令で使用するブロック数 = (19 + 書込みデータ点数) ÷ 16 | 書込みデータ点数が100点の場合、(19 + 100) ÷ 16 = 7[ブロック] |

### 図
3台のCPUユニットによるマルチCPUシステムのブロック数の関係を示す図です。  
- 1号機、2号機、3号機の各システムエリアが示され、それぞれの間で送信エリアと受信エリアが存在します。  
- 各送信エリア・受信エリアは299ブロックで構成されており、1号機から2号機・3号機へ、2号機から1号機・3号機へ、3号機から1号機・2号機へ送受信が行われる構造を表しています。  
- 矢印で送信・受信の方向が示され、各エリアのブロック数が299ブロックであることが明記されています。  
- この図により、3台構成時の各CPUユニット間の通信領域のブロック割り当てが視覚的に理解できます。
