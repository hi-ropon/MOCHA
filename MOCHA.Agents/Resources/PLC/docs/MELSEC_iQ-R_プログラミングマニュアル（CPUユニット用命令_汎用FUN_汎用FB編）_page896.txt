### テキスト
#### 機能
- (s)で指定されたデバイス番号以降に格納されている文字列を単精度実数データに変換し、(d)で指定されたデバイスに格納する。
- 指定する文字列は小数点形式、指数形式のどちらでも単精度実数に変換可能。

#### 小数点形式の場合
- (s)で指定した文字列が小数点形式の場合の変換処理の流れを説明。
- 文字列中の20H（スペース）は無視される。
- 文字列は最大24文字まで設定可能。20H（スペース）、30H（0）も1文字としてカウント。

### 表・図
#### 文字列のバイト配置例
- 1文字目のアスキーコードがb15～b8b7に、2文字目がb8b7～b0に格納されている。
- 文字列の終端は00Hで示す。

#### 小数点形式の変換例
- 例1: 「-1.07812」という文字列の16進コードが示され、単精度実数に変換される様子を図示。
- 例2: 20H（スペース）が含まれる場合の処理例。スペースは無視され、7桁目以降は切り捨てられる。
- 例3: 符号の扱いについて、2BH(+)を指定すると正の値、2DH(-)を指定すると負の値として変換。
- 例4: 数値の間に20H（スペース）や30H（0）が存在する場合は無視して変換。

### ポイント
- 文字列中の符号、小数点、指数部を除き6桁が有効。
- 7桁目以降は切り捨て。
- 20H（スペース）は無視されるが、30H（0）は1文字としてカウントされる。
- 符号の指定により正負の値を判別。
