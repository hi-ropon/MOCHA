### テキスト
#### 機能
- 指定ブロックを一時停止状態にします。
- SFC用情報デバイスのブロック停止再開ビットを設定している場合、そのビットデバイスもONになります。
- SFC用情報デバイスの停止時モードビットの設定が即停止か移行後停止かで、命令実行後に各ステップが停止するタイミングが異なります。

#### 注意事項
- コイル保持ステップは、保持中でSM325がOFFの時に停止した場合、非活性となり保持状態を再開できません。
- SM325がONの時に停止した場合は再開可能です。

#### エラー
- 指定ブロックが存在しない場合や、SFCプログラムが存在しない場合、または指定ブロックNo.が範囲外の場合にエラーコードが発生します。

### 表
#### 停止時モードビットの設定と停止するタイミング
| 停止時モードビットの設定 | 停止するタイミング |
| :----------------------- | :----------------- |
| 即停止                   | 命令実行直後に、すべてのステップが停止します。 |
| 移行後停止               | 命令実行後もステップは動作し続け、移行条件成立後、移行先のステップで停止します。 |

#### SM325の状態とOUT命令によるコイル出力状態
| SM325の状態 | OUT命令によるコイル出力状態 |
| :---------- | :-------------------------- |
| ON時        | コイル出力は、停止中も停止直前の状態を保持します。 |
| OFF時       | コイル出力は、停止時にOFFします。 |

#### エラーコード（SD0）と内容
| エラーコード | 内容 |
| :----------- | :--- |
| 31A1H        | 指定ブロックが存在しないとき。SFCプログラムが存在しない、またはSM321のSFCプログラム起動/停止がOFFに設定されている場合に命令を実行したとき。 |
| 31A2H        | 指定ブロックNo.が指定可能なブロックNo.の範囲外のとき。 |

### ポイント
- ブロック停止/再開時の動作は、SM325とSFC用情報デバイスの停止時モードビットの設定、ステップの保持/非保持の組合せによって決まる。
- 詳細は「MELSEC iQ-R プログラミングマニュアル（プログラム設計編）」を参照すること。
