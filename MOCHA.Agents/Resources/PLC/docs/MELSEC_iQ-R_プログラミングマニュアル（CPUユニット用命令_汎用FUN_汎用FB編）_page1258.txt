### テキスト
#### 書込み/読出し可能デバイス
他号機CPUユニットアクセス命令により、他号機CPUユニットへの書込み/読出し可能なデバイスについて説明しています。

#### デバイスの指定方法と書込み/読出し可能範囲
他号機CPUユニットのデバイス指定は文字列で行い、例えば「D200」のように直接指定します。文字列指定により、他号機のデバイス全範囲への書込み/読出しが可能です。例として、自号機のデータレジスタが12K点、他号機のデータレジスタが16K点の場合、他号機の先頭から16K点分の書込み/読出しが可能です。

### 表
#### 書込み/読出し可能デバイス一覧

| 分類               | 種類           | デバイス名*1                   | 相手先デバイス設定可否 | 設定時の条件                                                                                   |
|--------------------|----------------|-------------------------------|------------------------|----------------------------------------------------------------------------------------------|
| ユーザデバイス システムデバイス | ビットデバイス   | X, Y, M, L, B, F, SB, SM      | 可                     | ・16ビット(4桁)の桁指定であること。<br>・開始ビットデバイスが16(10H)の倍数であること。       |
|                    | ワードデバイス | T, ST, C, D, W, SW, SD        | 可                     | なし                                                                                         |
| ファイルレジスタ     | ワードデバイス | R, ZR                         | 可                     | なし                                                                                         |
| 間接指定             | —              | —                             | 否                     | なし                                                                                         |

*1 文字列指定によるインデックスレジスタを使ったインデックス修飾（Z、ZZ表現）が可能。例として「K4M0Z0」でZ=0の場合、M0+16=M16となり、K4M16で他号機にアクセス可能。

### 図
#### デバイス指定例
- 他号機CPUユニットの書込みデバイス「D200」を直接指定する例を示す。
- X0の接点がONのとき、MP.DDWR命令でH3E1、D0、D100、"D200"、M0の順に処理されるイメージ。
- 自号機CPUユニットと他号機CPUユニットのデータレジスタ範囲（12K点と16K点）を示し、他号機の16K点分の書込み/読出しが可能であることを図示。

### Point
- デバイスNo.の上位に「0」を付けても付けていない場合と同じデバイスとして扱う。例：「D1」と「D0001」は同じD1として処理。
- 大文字・小文字の違いは無視し、同一デバイスとして扱う。例：「D1」と「d1」は同じD1として処理。
- 他号機CPUユニットに存在しないデバイスを指定すると異常完了となるため注意が必要。
