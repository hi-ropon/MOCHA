### テキスト
#### DBUPDATE(P)命令の使用例
ID=0002HのレコードのOutputを35に更新する場合の例を示しています。

##### 注釈
1. 1個目の条件のデータサイズはWORD型（1ワード）です。
2. 2個目の条件はないため0になります。
3. 更新条件（1個目）のフィールド名を指定します。
4. 条件番号は「=」のため、K1を設定します。
5. 条件間演算値はないため、K0を設定します。
6. 判定値はK2を設定します。

#### 製品生産実績テーブル（proReport）

| 0002H | "製品名2" | 35     |
| 0003H | "製品名3" | 38     |
| ...   | ...       | ...    |

### 図
#### データベース構造イメージ
- 上記の表がデータベースとして管理されていることを示す円筒形の図。

#### DBUPDATE命令プログラム構成
- 命令の構成は以下の通りです。
  - D0: DBUPDATE命令コード
  - "proReport": テーブル名
  - D100: フィールド数（0001H）
  - D200: 1レコードのサイズ（WORD型1ワード、0001H）
  - D300～D336: 更新条件の指定
    - D300: 条件のデータサイズ（0001H）
    - D304: 条件のフィールド名（0000H）
    - D333～D336: 条件番号（0000H）、K1、K0、K2の設定
  - D202: 設定値（K35）を指定し、Outputを35に更新

- 矢印とテキストで各パラメータの意味が示されており、ID=0002Hの生産個数を35に更新する流れを視覚的に説明しています。
