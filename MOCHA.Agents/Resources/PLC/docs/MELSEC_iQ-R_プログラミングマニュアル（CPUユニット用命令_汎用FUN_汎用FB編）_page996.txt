### テキスト
#### 機能
- ブロックNo.を意識せず、(s)で指定された通しバイトNo.のファイルレジスタ内容を読み出し、(d)で指定されたデバイスの下位8ビットに格納する。
- (d)で指定された上位8ビットは00Hになる。
- 通しバイトNo.に対応するファイルレジスタ番号の対応表が示されている。
- 例として、(s)=23560の場合はZR11780の下位8ビットのデータを読み出す。
- 例として、(s)=43257の場合はZR21628の上位8ビットのデータを読み出す。

#### エラー
| エラーコード(SD0) | 内容 |
|-------------------|------|
| 2820H             | 指定可能な範囲を超えたデバイス番号(通しバイトNo.)を指定したとき。 |

### 図
- ファイルレジスタの構造図：ブロックNo.0、No.1、No.2のエリアが示されている。
- 通しバイトNo.から読み出し先指定への流れを示す矢印。
- 読み出しデータのビット構成図：b15～b0のうち、下位8ビットが読み出し内容、上位8ビットは00H。
- 通しバイトNo.に対応するファイルレジスタ番号の表（ZR0～ZR2503までの例）。
- 例示図：(s)=23560指定時のZR11780の下位8ビット読み出しの流れ。
- 例示図：(s)=43257指定時のZR21628の上位8ビット読み出しの流れ。
