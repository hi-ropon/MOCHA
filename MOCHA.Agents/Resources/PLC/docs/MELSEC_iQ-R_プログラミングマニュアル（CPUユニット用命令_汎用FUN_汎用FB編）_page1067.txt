### 表
#### エラーコード一覧と処置方法
| エラーコード | エラー内容 | 処置方法 |
| :--- | :--- | :--- |
| 0972H | 外部キーで参照するフィールドのデータ型が一致しない。 | 両者でフィールドのデータ型が一致しているか確認する。 |
| 0974H | 外部キーで参照するテーブルがない。 | 正しいテーブル名を指定する。 |
| 0976H | 外部キーで参照するテーブルのフィールドがない。 | 正しいフィールド名を指定する。 |
| 0984H | トランザクションが既に実行されている。 | トランザクションを多重実行していないか確認する。 |
| 09C2H | 使用しているCPUユニットでは非対応のデータベースに対し、操作を行おうとした。 | - ファームウェアバージョンが"28"より前のシーケンサCPUの場合、DBEXPORT(P)命令実行後に取得したUnicodeテキストファイルを使用し、DBIMPORT(P)命令を実行する。<br>- ファームウェアバージョンが"28"以降のシーケンサCPUの場合、DBEXPORT(P)命令実行後に取得したUnicodeテキストファイルを使用し、DBIMPORT(P)命令を実行する。 |
| 09C3H | ファームウェアバージョンが"28"より前のシーケンサCPUのDBIMPORT(P)命令で作成したデータベースをファームウェアバージョンが"28"以降のシーケンサCPUで操作した後、ファームウェアバージョンが"28"より前のシーケンサCPUで操作した。 | - エラーとなったデータベースは、ファームウェアバージョンが"28"以降のシーケンサCPUで操作する。<br>- ファームウェアバージョンが"28"より前のシーケンサCPUで操作した場合、DBEXPORT(P)命令を実行し、出力されたUnicodeテキストファイルをファームウェアバージョンが"28"より前のシーケンサCPUでDBIMPORT(P)命令を実行してから操作する。 |
| 09D7H | BOOL型のデータを指定していない。 | BOOL型データを設定する。 |
| 09F9H | レコード追加時、存在しないフィールドを指定した。 | テーブル構成を確認する。 |
| 0A0AH | 禁止されたフィールド名を指定した。 | 別のフィールド名を指定する。 |
| 0A16H | 主キーを設定しているフィールドに空の値を設定した状態でレコードを追加しようとした。 | Unicodeテキストで主キーを設定したフィールドの設定値で空の値が設定されていないか確認する。 |
| 0A17H | 主キーを設定しているフィールドに同一の値が含まれた状態でレコードを追加しようとした。 | Unicodeテキストで主キーを設定したフィールドの設定値で同一の値が設定されていないか確認する。 |
| 0A2BH | 複数個の主キーを設定しようとした。 | Unicodeテキストのキー設定にて、複数個の主キーを設定していないか確認する。 |
| 0A2FH | 参照先のテーブルに主キーが存在しない。 | Unicodeテキストでインポートしようとした設定値がフィールドのデータ型に適した値か確認する。 |
| 0A30H | 外部キーを設定したフィールドの設定値が主キーを設定したフィールドの設定値に含まれていない。 | 外部キーとして追加しようとしたフィールドの設定値が、主キーを設定したフィールドの設定値に含まれているか確認する。 |
| 0A31H | 外部キーによって参照されているフィールドの設定値を変更もしくは削除しようとした。 | 各テーブルの参照関係を確認する。 |
| 0A50H | 同一テーブルにインデックス名を重複して割り当てようとした。 | Unicodeテキストの内容を確認し、インデックス名が重複しないようにする。 |
| 0A5DH | 結合するフィールドのデータ型が異なる。 | 結合するフィールドのデータ型があっているか確認する。 |
| 0A5FH | 同一テーブルにフィールド名を重複指定している。 | 同一テーブルでフィールド名が重複していないか確認し、フィールド名を修正する。 |
| 0AAAH | 符号なし整数データ型のフィールドに負の値を指定した。 | フィールドのデータ型が、負の値を受け付けない型でないかを確認する。 |
| 0AB1H | 整数型のデータ型に文字列を指定しようとした。 | フィールドのデータ型が整数型の場合、文字列を指定していないか確認する。 |
| 0AB9H | 設定値が未設定のフィールドに対してインデックスを割り当てようとした。 | インデックスを指定するフィールドのレコードに設定値がすべて設定されているか確認する。 |
| 0AECH | テーブルのオープンに失敗した。 | 電源OFF→ON、もしくはリセットを行い、オープンできるか確認する。 |
| 1000H | インポート時に指定したUnicodeテキストファイルがない。 | フォルダパスで示したファイルがUnicodeテキストファイルか確認し、間違っていれば適切なフォルダパスに修正する。 |
| 1002H | 割り当て可能な識別番号がない。 | オープンできるデータベースは最大4個であるため、4個におさまるようにオープンするデータベース数を調整する。 |
| 1003H | オープンしていない識別番号を指定した。 | DBOPEN(P)命令であらかじめデータベースをオープンし、識別番号を取得する。 |
| 1004H | 命令で指定する識別番号が範囲外である。 | DBOPEN(P)命令で取得した識別番号を指定する。 |
| 1007H | 命令で指定するフィールド数を指定していない。 | 引数のフィールド数を1以上に設定する。 |
| 1008H | ・命令で指定するフィールド数が範囲外である。<br>・命令で指定するフィールド名が、範囲外のフィールド名である。 | ・引数のフィールド数を128以内に設定する。<br>・引数のフィールド名を、指定可能範囲内(128番目以内)のフィールド名に設定する。 |

### テキスト
- 各エラーコードに対応したエラー内容の説明と、具体的な処置方法が詳細に記載されている。
- 特にファームウェアバージョンによる動作の違いや、Unicodeテキストファイルの取り扱いに関する注意点が明記されている。
- エラーの多くはデータベースのフィールド設定やインデックス設定、データ型の不一致に起因しているため、設定内容の確認と修正が主な対処法となっている。
