### テキスト
#### 演算処理
(s)に入力されたBCD型（WORD型またはDWORD型）のデータを、DINT型のデータに変換して(d)から出力します。

- (s)にWORD型を指定した場合、9999H（BCD(WORD)）が9999（DINT）に変換されることを示しています。
- (s)にDWORD型を指定した場合、99999999H（BCD(DWORD)）が99999999（DINT）に変換されることを示しています。
- (s)への入力値はWORD型データ値で0H～9999H（各桁は0～9の範囲）、DWORD型データ値で0H～99999999H（各桁は0～9の範囲）の範囲内である必要があります。
- (s)にはWORD型、DWORD型が指定可能で、BOOL型は指定できません。

#### 演算結果
1. EN/ENOなしファンクション  
演算処理を実行し、(d)から演算出力値が出力されます。

2. EN/ENO付きファンクション  
実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |  
| :--- | :--- |  
| EN | ENO | (d) |  
| TRUE（演算実行） | TRUE | 演算出力値 |  
| FALSE（演算停止） | FALSE*1 | 不定値 |  

*1 ENOからFALSEが出力された場合、(d)から出力されるデータは不定値となります。その場合は(d)から出力されたデータを使用しないようなプログラムの処理を行う必要があります。

### 図
- BCD(WORD)の9999HがDINTの9999に変換される過程をビット単位で示しています。BCDの各4ビットが対応する10進数の桁を表し、DINTのビット配列に変換される様子が示されています。
- 同様にBCD(DWORD)の99999999HがDINTの99999999に変換される過程もビット単位で示されています。
- どちらの場合も、BCDの各桁は0～9の範囲であることが前提で、変換後のDINTは対応する10進数値となります。

### 表
#### 実行条件と演算結果

| 実行条件 | 演算結果 | 備考 |
| :--- | :--- | :--- |
| EN | ENO | (d) |
| TRUE（演算実行） | TRUE | 演算出力値が(d)から出力される |
| FALSE（演算停止） | FALSE | (d)から出力されるデータは不定値となるため使用不可 |

* ENOがFALSEの場合は(d)のデータは不定値となり、プログラム側での適切な処理が必要。
