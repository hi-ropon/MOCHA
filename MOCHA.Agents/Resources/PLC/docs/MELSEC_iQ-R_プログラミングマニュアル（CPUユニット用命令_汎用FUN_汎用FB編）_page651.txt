### テキスト
#### タイプ1 コントロールデータ(d1)
タイプ1のコントロールデータ(d1)について、各オペランドのデバイスごとに項目、内容、設定範囲、セット側が表形式で説明されています。

##### ■実行/完了タイプ (+0)
実行タイプを指定します。  
- 00**H: バイナリ書込み  
  - 0000H: BIN16ビットデータ  
  - 0001H: BIN32ビットデータ  
- 01**H: CSV形式変換書込み  
  - 0100H: 10進数(符号付き16ビットデータ)  
  - 0101H: 10進数(符号なし16ビットデータ)  
  - 0110H: 10進数(符号付き32ビットデータ)  
  - 0111H: 10進数(符号なし32ビットデータ)  
  - 0120H: 16進数(16ビットデータ)  
  - 0121H: 16進数(32ビットデータ)  
  - 0130H: 文字列(アスキー文字)  
  - 0140H: 浮動小数点実数(単精度実数)  
  - 0141H: 浮動小数点実数(倍精度実数)  

##### ■完了ステータス (+1)
命令完了時に完了ステータスを格納。  
- 0000H: 正常完了  
- 0000H以外: 異常完了(エラーコード)  

##### ■書込み結果データ数 (+2)
指定データに対して実際に書き込んだデータ数が入る。単位は(d1)+7のデータタイプ指定による。

##### ■使用用途設定エリア (+3)
b15～b0のビット指定あり。  
b0は書込み開始位置設定で、バイナリ書込み時は必ず0を指定。CSV形式変換書込みやファイルの最後に追加指定時は書込み開始位置を指定。  
- 0: ファイルの最後に追加  
- 1: ファイルの最後の改行コードをカンマに変換して追加  

##### ■ファイル位置 (+4 +5)
(d1)+0でバイナリ書込み指定時:  
- 00000000H: ファイルの先頭から  
- 00000001H～FFFFFFFFH: 指定位置から  
- FFFFFFFFH: ファイルの最後に追加  

CSV形式変換書込み指定時:  
- 00000000H～FFFFFFFFH: ファイルの先頭から  
- FFFFFFFFH: ファイルの最後に追加  

##### ■列数指定 (+6)
バイナリ書込み指定時は必ず0を指定。CSV形式変換書込み時は書込み列数を指定。  
- 0: 列なし（1行）  
- 0以外: 指定数の列  

##### ■データタイプ指定 (+7)
- 0: ワード  
- 1: 偶数バイト  
- 2: (d1)+0で指定したデータ型の単位  
- 3: 奇数バイト  

ワード、偶数バイト、奇数バイトは特定のBIN16ビットデータや10進数16ビットデータ指定時に使用可能。

### 注釈・補足
- 奇数バイト指定は特定機種のみ使用可能。対応ファームウェアバージョンに差異あり。  
- データタイプ指定で偶数バイトや奇数バイトを指定した場合、要求書込みデータ数の設定範囲が制限される。  
- 書込み開始位置設定は、CSV形式変換書込みかつファイル位置指定で「ファイルの最後に追加」を指定した場合のみ有効。  
- 対応機種やファームウェアバージョンにより使用可否や仕様が異なるため注意が必要。
