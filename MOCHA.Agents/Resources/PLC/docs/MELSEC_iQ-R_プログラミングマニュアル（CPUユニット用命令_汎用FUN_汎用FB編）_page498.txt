### テキスト
#### BIN16ビットデータ2の補数(符号反転)
BIN16ビットデバイスの符号を反転する命令です。

#### 実行条件
- NEG命令は立ち上がりエッジで実行されます。
- NEGP命令は立ち下がりエッジで実行されます。

#### 機能
(d)で指定されたBIN16ビットデバイスの符号を反転し、同じ(d)のデバイスに格納します。主に正負の符号を反転する際に使用します。

#### エラー
演算エラーはありません。

### 図
- ラダー図：NEG命令のラダー回路シンボルが示されており、(d)のデバイスに対して符号反転を行うことを表しています。
- ST言語：ENO:=NEG(EN,d); ENO:=NEGP(EN,d); の記述があり、実行条件付きの符号反転命令であることを示しています。
- FBD/LD図：EN入力、ENO出力、dデータ入力のブロック図が示されています。
- 符号変換イメージ：16ビットのビット列の例を用いて、実行前のビットパターン、符号変換の過程、実行後のビットパターンをビット単位で示しています。例えば、実行前のビット列「1010101010101010」（-21846）に対し、符号反転を行い「0101010101010110」（21846）となる様子が図示されています。
