### テキスト
#### マルチプレクサ（MUX(_E)）
複数の入力値のうち一つを出力する機能を持つ。

#### 設定データ
##### 内容、種別、データ型
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| n(K) | 出力値選択 | 入力変数 | ANY_INT |
| s1(IN0)～s28(IN27) | 入力 | 入力変数 | ANY*1 |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY*1 |

*1 安全プログラムではREAL型/LREAL型/STRING型データを指定しないこと。指定するとエラーになる。

#### 機能
##### 演算処理
- (n)へ入力された値に従い、(s1)～(s28)へ入力された値のいずれかを、(d)から(s)と同一のデータ型で出力する。
- (n)の入力値が0の場合、(s1)へ入力された値を(d)から出力する。
- (n)の入力値が(n)-1の場合、(sn)へ入力された値を(d)から出力する。

##### 例（データ型がINT型の場合）
- nの値に応じて、s1～s28のいずれかのINT値をMUXが選択し、dから出力する。
- nにsのピン数の範囲外を入力した場合、dから不定値が出力される（演算エラーにはならない）。MUX_Eの場合はENOからFALSEが出力される。
- nの入力値はINT型/DINT型/WORD型/DWORD型のデータ値で0～27の範囲内（sのピン数の範囲内）。
- sの入力値はBOOL型/INT型/DINT型/WORD型/DWORD型/REAL型/LREAL型/STRING型/TIME型/構造体型/配列型のデータ値。

### 図
- ラダー回路図（EN/ENOなし、EN/ENO付き）: MUXの入力端子（n、s1、s2）と出力端子（d）、EN/ENO端子の配置を示す。
- ST言語記述例: EN/ENOなしの場合は「d:=MUX(n,s1,s2);」、EN/ENO付きの場合は「d:=MUX_E(EN,ENO,n,s1,s2);」と記述。
- 演算例ブロック図（INT型の場合）: nの入力値に応じてs1～s28のINT値からMUXが選択し、dに出力する様子を示す。
