#### ファイルレジスタ使用時の制約事項
リフレッシュデバイスにファイルレジスタを指定した場合の注意点を示しています。

##### プログラム名と同一のファイルレジスタを指定した場合
- パラメータでプログラム名と同じファイルレジスタを指定すると、正しくリフレッシュできません。
- プログラム設定で最後の番号に設定したプログラム名と同じファイルレジスタにリフレッシュされます。
- リフレッシュデータの読み書きを行う場合は、QDRSET命令を使い、該当ファイルレジスタに切り換えてから指定する必要があります。

##### QDRSET命令でファイル名やドライブNo.を変更した場合
- QDRSET命令でファイルレジスタのファイル名やドライブNo.を変更すると、リフレッシュ直前の設定ファイルにリンクしてリフレッシュされます。
- リフレッシュデータの読み書きを行う場合は、リフレッシュ直前の設定ファイルを指定してください。

##### RSET命令でブロックNo.を切り換えた場合
- RSET命令でブロックNo.を切り換えた場合は以下に注意が必要です。
  - 切り換えたブロックNo.のファイルレジスタ(R)にリフレッシュされる。
  - リフレッシュ直前のブロックNo.のファイルレジスタ(R)にリフレッシュされる。
- リフレッシュデータの読み書きを行う場合は、リフレッシュ直前のブロックNo.を指定してください。
