#### Point
- CPUバッファメモリの読出し/書込み可能エリアのリフレッシュエリアについて、リフレッシュ設定しない場合は読出し/書込み指定可能エリアとして使用可能。
- リフレッシュ設定した場合でも、リフレッシュ送信範囲以降は読出し/書込み指定可能エリアとして使用可能。
- CPUバッファメモリへのデータ書込みは、CPUバッファメモリアクセスデバイスが使用可能。
- TOD(P)命令、DTOD(P)命令では、64Kを超えるバッファメモリアドレスにデータを書き込むことが可能。

### 図
- CPUバッファメモリのアドレス範囲を示す図
  - 縦軸にアドレス範囲を示し、上部に「システムエリア」、その下に「リフレッシュエリア」があり、リフレッシュエリアの下から下端までが「読出し/書込み指定可能エリア」として示されている。
  - アドレスは0から524287までの範囲で示されている。
