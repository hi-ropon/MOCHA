### テキスト
#### 16ビットブロックデータ排他的論理和（BKXOR(P)）
指定した2つのBIN16ビットデータブロックの排他的論理和（XOR）を行う命令です。

#### 実行条件
- BKXOR命令は立ち下がりエッジで実行されます。
- BKXORP命令は立ち上がりエッジで実行されます。

#### 設定データの内容、範囲、データ型
| オペランド | 内容 | 範囲 | データ型 | データ型（ラベル） |
| --- | --- | --- | --- | --- |
| (s1) | 論理演算データが格納されている先頭デバイス | なし | 符号付きBIN16ビット | ANY16*1 |
| (s2) | 論理演算データまたは論理演算データが格納されている先頭デバイス | -32768～32767 | 符号付きBIN16ビット | ANY16*1 |
| (d) | 演算結果を格納する先頭デバイス | なし | 符号付きBIN16ビット | ANY16*1 |
| (n) | 演算データ個数 | 0～65535 | 符号なしBIN16ビット | ANY16 |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |

*1 ラベルで設定する場合は、動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定する必要があります。

### 図
- ラダー図：s1、s2、d、nの各パラメータを持つBKXOR命令のラダー表現
- ST言語：ENO:=BKXOR(EN,s1,s2,n,d); または ENO:=BKXORP(EN,s1,s2,n,d);
- FBD/LD図：EN入力、ENO出力、s1、s2、d、nの各入力を持つ命令ブロック

### 表
#### 実行条件
| 命令 | 実行条件 |
| --- | --- |
| BKXOR | 立ち下がりエッジ |
| BKXORP | 立ち上がりエッジ |

#### 使用可能デバイス
| オペランド | ビット | ワード | ダブルワード | 間接指定 | 定数 | その他 |
| --- | --- | --- | --- | --- | --- | --- |
| (s1)*1 | - | ○（T,ST,C,D,W,SD,SW,FD,R,ZR,RD） | - | - | ○ | - |
| (s2)*1 | - | ○（T,ST,C,D,W,SD,SW,FD,R,ZR,RD） | - | - | ○ | - |
| (d)*1 | - | ○（T,ST,C,D,W,SD,SW,FD,R,ZR,RD） | - | - | - | - |
| (n) | ○（X,Y,M,L,SM,F,B,SB,FX,FY） | ○（J口￥口,U口￥G口,J口￥口,U3E口￥(H)G口） | ○（Z） | ○（LT,LST,LZ,LC） | - | - |

*1 (s1)と(d)または(s2)と(d)に同一デバイス番号を指定可能。
