### テキスト
#### 同一デバイスのOUT命令、SET/RST命令、PLS/PLF命令使用時の動作
同一デバイスを使用したOUT命令、SET/RST命令、PLS/PLF命令を1スキャンに複数実行した場合の動作について説明しています。

**Point**
- SIL2プロセスCPUおよび安全CPUの安全プログラムで同一デバイスを使用した場合、1安全サイクル処理に複数実行した動作は以下のように読み替えます。
  - 「スキャン」→「安全サイクル処理」
  - 「X0」→「SA¥X0」、 「X1」→「SA¥X1」、 「M0」→「SA¥M0」
  - 図中X0の立上りの「END」→「安全サイクル処理開始」
  - 図中X0の立下りの「END」→「安全サイクル処理終了」
- PLF命令ではX0の立上り/立下りにかかわらず、「END」は「安全サイクル処理開始」と読み替えます。

#### 同一デバイスのOUT命令の場合
- 1スキャン中に同一デバイスのOUT命令を複数回実行しないことが推奨されます。
- 複数回実行した場合は、各OUT命令実行時に演算結果により指定デバイスがON/OFFされます。
- 各OUT命令の実行時にON/OFFが決まるため、1スキャン中にON/OFFが繰り返されることがあります。
- 例として、入力X0とX1で同一の内部リレーM0をON/OFFする回路を示し、その動作を説明しています。

### 図
- ラダー図：X0とX1の入力がそれぞれM0をON/OFFする回路を示しています。
- タイミングチャート：
  - X0、X1、M0のON/OFF状態を時間軸で示し、OUT命令の実行タイミングと状態変化を矢印で表現。
  - (1) X0がONのためM0がONになる。
  - (2) X1がOFFのためM0がOFFになる。
  - (3) X0がOFFのためM0はOFFのまま。
  - (4) X1がONのためM0がONになる。
- OUT命令で出力(Y)を指定すると、1スキャンの最後に実行したOUT命令のON/OFF状態が出力されることを示しています。
