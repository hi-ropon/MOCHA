本ページは、命令におけるビットデバイスの桁指定方法と、ワードデバイスおよびラベルにおける32ビットデータの扱いについて説明しています。構成は以下の通りです。
- テキスト部分：桁指定の説明、ワードデバイス/ラベルの32ビットデータの扱いに関する解説
- 表部分：なし
- 図部分：命令のソース側・ディスティネーション側に桁指定した場合の回路例とビット配置の図解
</overview>

<contents>
### テキスト
#### ソース(s)側に桁指定したビットデバイスを指定した場合
命令のソース側に桁指定したビットデバイスを指定すると、ディスティネーション側のワードデバイスには、ソース側で桁指定したビットより上位のビットに0が格納されます。

#### ディスティネーション(d)側に桁指定したビットデバイスを指定した場合
命令のディスティネーション側に桁指定がある場合、桁指定による点数がディスティネーション側の対象となり、桁指定による点数より上位のビットデバイスは変化しません。

#### ワードデバイス/ラベルにおける32ビットデータの扱い
- ワードデバイスは2点で32ビットデータとして扱います。ただし以下のデバイスは1点で32ビットデータを扱えます。
  - ロングタイマ（LT）
  - ロング積算タイマ（LST）
  - ロングカウンタ（LC）
  - ロングインデックスレジスタ（LZ）
- ダブルワード型ラベルは1点で32ビットデータとして扱います。

### 図
#### ソース側に桁指定したビットデバイスを指定した場合の回路例と処理
- 回路例：X10がONのとき、DMOV命令でK1X0のビット指定された値をD0に転送
- 処理：D0の下位16ビット（b0～b15）にX3,X2,X1,X0のビットが対応し、上位ビットは0で埋められる

#### ディスティネーション側に桁指定したビットデバイスを指定した場合の回路例と処理
- 回路例：X10がONのとき、DMOV命令でD0のワードデータをK5M10に転送
- 処理：D0のビット列がM25～M10に対応し、M41～M26の上位ビットは変化しない

(注記) 上位ビットは変化しないことが明記されている。
