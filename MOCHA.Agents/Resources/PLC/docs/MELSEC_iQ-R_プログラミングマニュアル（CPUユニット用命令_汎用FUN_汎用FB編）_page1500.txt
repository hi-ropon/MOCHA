### テキスト
#### 42 エッジ検出ファンクションブロック
##### 42.1 立ち上りエッジ検出
「R_TRIG(_E)」は信号の立ち上がりを検出してパルス信号を出力するファンクションブロックです。対応CPUはRnCPU、RnENCPU、RnPCPU（プロセス、二重化）、RnPSFCPU（安全、一般）となっています。

### 図
- ラダー、FBD/LD言語での表記例
  - ENなしの場合は、s入力とd出力のシンプルなブロック図
  - EN/ENO付きの場合は、EN入力、ENO出力が追加されたブロック図
- ST言語での表記例
  - EN/ENOなし: インスタンス名(CLk:=s, Q:=d);
  - EN/ENO付き: インスタンス名(EN:=en, ENO:=eno, CLK:=s, Q:=d);

### 表
#### 設定データ：内容、種別、データ型
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(CLK) | 立ち上がりエッジ検出入力 | 入力変数 | BOOL |
| ENO | 出力状態(TRUE:正常、FALSE:異常または停止) | 出力変数 | BOOL |
| d(Q) | 出力 | 出力変数 | BOOL |

以上の内容により、R_TRIGファンクションブロックは入力信号の立ち上がりを検出し、パルス信号を出力する動作を行うことが理解できます。
