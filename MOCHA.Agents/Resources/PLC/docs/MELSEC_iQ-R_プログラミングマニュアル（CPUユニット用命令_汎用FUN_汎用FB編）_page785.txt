### テキスト
#### エラー
演算エラーはありません。

#### 注意事項
SM705（変換桁数切換）がONのとき、(d)には演算結果を有効桁数分だけ格納する仕様である。そのため、BINDA(P)(_U)命令を連続して実行し、演算結果を同一デバイスに格納する場合、(d)の一部に前回の演算結果が上書きされずに残る可能性がある。

##### 例
(s)が「-12345」のときにBINDA(P)命令を実行し、続けて(s)が「-67」のときにBINDA(P)命令を実行する場合、以下のような現象が起きる。

1. 「-12345」を文字列に変換し、(d)+0～(d)+3に格納する。
2. 「-67」を文字列に変換し、(d)+0～(d)+3に格納するが、(d)+2以降に前回の変換結果の一部が残ったままになる。

これを回避するには、(d)+0～(d)+2の格納エリアをすべてクリアしてからBINDA(P)(_U)命令を実行するプログラムを作成する必要がある。

### 図（例示）
1. (s) = -12345 のときのBINDA(P)命令実行例
- (d)+0: 31H (1), 2DH (-)
- (d)+1: 33H (3), 32H (2)
- (d)+2: 35H (5), 34H (4)
- (d)+3: 00H

2. (s) = -67 のときのBINDA(P)命令実行例（前回の結果が一部残存）
- (d)+0: 36H (6), 2DH (-)
- (d)+1: 00H, 37H (7)
- (d)+2: 35H (5), 34H (4) ←前回の残存部分
- (d)+3: 00H

3. 回避策として、(d)+0～(d)+2をクリアしてから実行した場合
- (d)+0～(d)+2: 全て00Hでクリア
- (s) = -67 のBINDA(P)命令実行後の格納状態
  - (d)+0: 36H (6), 2DH (-)
  - (d)+1: 00H, 37H (7)
  - (d)+2: 00H, 00H
  - (d)+3: 00H

このように、クリア処理を挟むことで前回の変換結果の残存を防止できる。
