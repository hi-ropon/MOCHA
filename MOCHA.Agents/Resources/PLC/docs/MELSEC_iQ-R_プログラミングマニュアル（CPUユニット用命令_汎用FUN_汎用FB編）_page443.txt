### テキスト
#### 機能
- (s)で指定された符号なしBIN16ビットデータを、符号付きBIN32ビットデータに変換し、(d)で指定されたデバイスに格納する。
- プログラム例では、M0がONしたとき、D0に格納された符号なしBIN16ビットデータを符号付きBIN32ビットデータに変換し、D100～D101へ格納する。

#### エラー
- 演算エラーはありません。

### 図
- ラダー図には、SM402、M0、MOVP、UINT2DINT命令が示されている。
  - SM402がONのとき、MOVP命令でD0に0を格納。
  - M0がONのとき、UINT2DINT命令でD0の符号なしBIN16ビットデータを符号付きBIN32ビットデータに変換し、D100に格納。
- ビット構成図では、
  - (s)のD0は16ビット（b15～b0）で、値は0xD000H（53248）。
  - (d)のD100、D101は32ビット（b31～b0）で、上位16ビット（D101）が0x0000H、下位16ビット（D100）が0xD000H（53248）となっている。
- 注釈として、
  1. 0を格納（上位16ビット）
  2. 下位16ビットへ格納（元の16ビットデータ）

以上により、符号なし16ビットデータを符号付き32ビットデータに変換し、上位16ビットに0をセットして格納する動作を示している。
