### テキスト
#### RCR(P)命令の説明
- (d)で指定されたデバイスのBIN16ビットデータを、SM700（キャリフラグ）を含めて(n)ビット右へ回転させる命令です。
- SM700（キャリフラグ）は、RCR(P)命令の実行前の状態によりONまたはOFFになります。

- (d)にビットデバイスが指定された場合は、桁指定で指定されたデバイス範囲で回転します。実際に回転するビット数は、(n)÷(桁指定で指定された点数)の余りとなります。
  - 例：n=15、桁指定点数=12ビットの場合、15÷12=1余り3となり、3ビット右へ回転します。
- (n)は0～15を指定可能で、16以上の値が指定された場合は(n)÷16の余りの値で回転します。
  - 例：n=18のとき、18÷16=1余り2となり、2ビット右へ回転します。

#### エラー
- 演算エラーはありません。

### 図
- ビット列の変化を示す図が4段階で示されています。
  - 最初のビット列はb15からb0までの16ビットが0または1で表現されており、SM700の値は0。
  - 1ビット右ローテーション後のビット列とSM700の値が変化。
  - さらに1ビット右ローテーション後のビット列とSM700の値。
  - (n)ビット右ローテーション後のビット列とSM700の値。
- 矢印でビットの右回転の流れを示し、キャリフラグ（SM700）の値が回転に伴いどのように変わるかを視覚的に説明しています。
