### テキスト
#### 非実行処理についての説明
非実行処理は、各コイル命令を条件設定がOFF状態と同じ処理を行うことを意味します。

#### 図の説明
図は時間軸上にA、Bという実行プログラム名を示し、3つのポイントでの動作を示しています。
1. "A"を指定してPOFF命令を実行
2. "A"が非実行処理状態
3. "A"を待機タイププログラムへ切り換え

#### 非実行処理後の各コイル命令の演算結果
条件接点のON/OFFに関係なく以下のようになります。
- OUT命令: 強制的にOFF
- SET命令、RST命令、SFT命令、基本命令、応用命令: 状態を保持
- PLS命令、PLS化命令(□P): 条件接点OFFと同一処理
- OUT T命令: 低速/高速タイマの現在値は0
- OUT ST命令、OUT C命令: 現在値は保持
