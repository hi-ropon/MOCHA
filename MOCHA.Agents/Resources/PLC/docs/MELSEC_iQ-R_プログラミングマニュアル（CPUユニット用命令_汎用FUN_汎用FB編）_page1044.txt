### テキスト
#### 更新条件
- オペランドは(s5)。
- デバイスごとに「条件1個目のデータサイズ」「条件2個目のデータサイズ」「更新条件」の3つの項目がある。
- 条件1個目のデータサイズは1～125の範囲でユーザが設定。
- 条件2個目のデータサイズは0～125の範囲で設定可能。0は条件なし、それ以外はデータサイズを示す。
- 更新条件は判定値のデータ型により(s5)の最終アドレスが変化する。

##### 更新条件の詳細
- フィールド名(1)：32文字固定のUnicode文字列で設定。32文字未満の場合は前詰めし、0000Hでパディング。
- 条件番号(2)：更新条件を示す番号。1～8までの判定条件がある。
  - 1: 判定値と等しい
  - 2: 判定値以外
  - 3: 判定値より小さい
  - 4: 判定値より大きい
  - 5: 判定値以下
  - 6: 判定値以上
  - 7: NULL（値が設定されていない）
  - 8: not NULL（値が設定されている）
- 条件間演算値(3)：隣接条件との論理演算値。0=なし、1=AND、2=OR。条件1個の場合は0に設定し、条件2個目は無視。
- 判定値(4)：更新条件の判定に使う値。条件番号が7または8の場合は判定値の領域をデータサイズ分用意。

- (s5)の設定フォーマットは、データ型がWORDの場合、フィールド名、条件番号、条件間演算値、判定値を更新条件数分設定。判定値のデータサイズは各フィールドのデータ型サイズに従う。

### 表
| デバイス | 項目 | 内容 | 設定範囲 | セット側 |
| :--- | :--- | :--- | :--- | :--- |
| +0 | 条件1個目のデータサイズ | 1個目の更新条件のデータサイズを設定 | 1～125 | ユーザ |
| +1 | 条件2個目のデータサイズ | 2個目の更新条件のデータサイズを設定。0は条件なし、それ以外はデータサイズ | 0～125 | ユーザ |
| +2～+□ | 更新条件 | 更新条件を設定。判定値のデータ型により(s5)の最終アドレスが変化 | － | ユーザ |

### 図
- (s5)のビット配置図
  - b15～b0までのビット単位でフィールドが区切られている。
  - (s5)は1個目の更新条件のデータサイズ。
  - (s5)+1は2個目の更新条件のデータサイズ。
  - (s5)+2～(s5)+33はフィールド名（32文字分）。
  - (s5)+34は条件番号。
  - (s5)+35は条件間演算値。
  - (s5)+36は判定値。
  - (s5)+37～(s5)+68は2個目のフィールド名。
  - (s5)+69は条件番号。
  - (s5)+70は条件間演算値（0固定）。
  - (s5)+71は判定値。

- 矢印で1個目と2個目のフィールドに関する条件の範囲が示されている。
