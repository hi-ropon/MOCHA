### テキスト
#### 桁指定の範囲
桁指定ごとの16ビットデータの範囲を示しています。

| 桁指定 | 10進表記 | 16進表記 |
|--------|----------|----------|
| K1     | 0～15    | 0H～FH   |
| K2     | 0～255   | 00H～FFH |
| K3     | 0～4095  | 000H～FFFH |
| K4     | 符号付き16ビットデータ：-32768～32767<br>符号なし16ビットデータ：0～65535 | 0000H～FFFFH |

##### 例
X0を桁指定した場合の対象点数は以下の通りです。
- K1X0 → X0～X3の4点が対象
- K2X0 → X0～X7の8点が対象
- K3X0 → X0～XBの12点が対象
- K4X0 → X0～XFの16点が対象

### 図
- 対象点数のイメージ図：XFからX0までのビットが並び、K1X0、K2X0、K3X0、K4X0の範囲を矢印で示している。

#### ソース(s)側に桁指定したビットデバイスを指定した場合
- 命令のソース側に桁指定したビットデバイスを指定すると、ディスティネーション側のワードデバイスには、ソース側で桁指定したビットより上位のビットに0が格納される。

- 回路例：X10がONのとき、MOV命令でK1X0のビットをD0に転送する例
- 処理図：D0の16ビットのうち、b3～b0にK1X0のX3～X0のビットが入り、それより上位のビットは0になる。

#### ディスティネーション(d)側に桁指定したビットデバイスを指定した場合
- 命令のディスティネーション側に桁指定がある場合、桁指定による点数がディスティネーション側の対象となる。
- 桁指定による点数より上位のビットデバイスは変化しない。

- 回路例：X10がONのとき、MOV命令でD0のデータをK2M100に転送する例
- 処理図：D0のビットb15～b0のうち、K2M100のM115～M100に対応するビットが転送されるが、上位ビットは変化しない。

(注記) (1): 変化しません。
