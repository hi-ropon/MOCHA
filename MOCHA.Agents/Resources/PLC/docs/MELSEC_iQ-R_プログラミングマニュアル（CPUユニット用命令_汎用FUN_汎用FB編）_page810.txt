### テキスト
#### 機能
- (s)で指定されたBCD8桁データの各桁の数値をアスキーコードに変換し、(d)で指定されたデバイス番号以降に格納する。
- SM705（変換桁数切換）の状態により、(d)に格納される10進アスキーデータの形式が異なる。

#### SM705の状態と動作概要
- SM705がOFFの場合は固定桁数（8桁）で格納。
- SM705がONの場合は(s)の値によって各桁を前詰めで格納。

#### 動作の概要
SM705がOFFとONの場合の動作を例示。

### 表
| SM705の状態 | (d)の格納形式 | 参照先 |
| :--- | :--- | :--- |
| OFF | 固定桁数（8桁）で格納 | 811ページ SM705(変換桁数切換)がOFFの場合の動作 |
| ON | (s)の値によって各桁を前詰めで格納 | 812ページ SM705(変換桁数切換)がONの場合の動作 |

※SM705に対応しているCPUユニットのファームウェアバージョンについては別途参照が必要。

### 図（動作例）
- 例として、(s)にBCD8桁データ「00012345」が格納されている状態でDBCDDA(P)命令を実行した場合の動作を示す。

#### SM705: OFFの場合
- (d)+0～(d)+3に16進数でアスキーコードが固定桁数で格納される。
- 「00012345」の0は20H（スペース）に変換されて格納される。

#### SM705: ONの場合
- (d)+0～(d)+3に16進数でアスキーコードが前詰めで格納される。
- 「00012345」の先頭の0を省略し「12345」がアスキーデータに変換されて格納され、終端に00Hが格納される。

### 補足
- SM705がOFFの場合は固定桁数での格納となる。
- SM705がONの場合は前詰めで格納されるため、先頭の0は省略される。
