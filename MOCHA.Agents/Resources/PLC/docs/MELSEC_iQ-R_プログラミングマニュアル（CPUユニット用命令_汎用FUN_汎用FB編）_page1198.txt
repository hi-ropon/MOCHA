### テキスト
#### 注意事項
- ファイル転送中にケーブル断やCPUユニットの電源断・リセットが発生した場合、不要なファイル（一時ファイルや不完全なファイル）は削除し、再度実行すること。
- 転送完了後の転送元ファイル削除設定（(s1)+0のビット2）を1（削除）に設定する場合は注意が必要。
- ファイルパスワード機能で転送元ファイルがアクセス制限されている場合、SP.FTPPUT命令が異常完了することがある。ただし、削除設定を0にし、ファイルパスワードが「書込み禁止」の場合はSP.FTPPUT命令を実行可能。
- (s1)+0のビット3（ファイル転送時の一時ファイル作成設定）を1（作成する）に設定すると、転送先に12文字の一時ファイル（FTPCLI_I.TMP）が作成されるため、転送先フォルダパスと合わせたパス長がFTPサーバの最大パス長を超えないように注意。
- サポートされていない文字は「_」に変換される。これはファイル名やフォルダ名に含まれる場合も同様で、ワイルドカード指定時や転送元CPUユニットに存在するファイル名にサポートされていない文字が含まれる場合も同様。
- プロセスCPU（二重化モード）ではファイル転送中の系切替直後にFTPサーバとの旧制御系コネクションが切断され、新制御系でFTPサーバと接続できない場合がある。この場合はFTPサーバが接続可能になるまでSP.FTPPUT命令を再実行する必要がある。

### 表
| 項目 | 内容 |
| :--- | :--- |
| CPUユニットのファイルを転送する場合 | CPUユニットの動作に必要なファイルであってもCPUユニットからファイルが削除される。削除した場合の動作は保証されない。 |
| 転送ファイル名をワイルドカード指定して転送する場合 | 意図せずに必要なファイルが削除されることがある。 |
