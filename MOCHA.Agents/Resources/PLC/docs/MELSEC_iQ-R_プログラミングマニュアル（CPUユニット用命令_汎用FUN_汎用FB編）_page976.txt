### テキスト
#### 機能
- (s)から(n)点分の倍精度実数のブロックデータに対して、最大値を検索し、(d)で指定したデバイスへ格納する。
- 最初に最大値を検出したデータを、(s)から何点目かを(d)+4で指定したデバイスへ格納し、最大値と同じ値の個数を(d)+5で指定したデバイスへ格納する。
- 検索結果（位置）の値については、(s)で指定したブロックデータの先頭を1点目として扱う。
- (n)で指定した値が0の場合は無処理となる。

#### エラー
| エラーコード (SD0) | 内容 |
|--------------------|------|
| 3402H              | (s)で指定したブロックデータに、倍精度実数以外の値が存在する場合。 |

### 図（説明用イメージ）
- 左側の表は、(s)から始まるブロックデータの例を示している。数値は1.2345、123.45、-1.2345、-12.345、-123.45の5点。
- 右側の表は、(d)から始まるデバイスに格納される結果を示す。
  - (d)、(d)+1、(d)+2、(d)+3には最大値123.45が格納される。
  - (d)+4には最大値が最初に検出された位置（2）が格納される。
  - (d)+5には最大値と同じ値の個数（1）が格納される。

この図は、最大値の検索とその位置および個数の格納方法を視覚的に説明している。
