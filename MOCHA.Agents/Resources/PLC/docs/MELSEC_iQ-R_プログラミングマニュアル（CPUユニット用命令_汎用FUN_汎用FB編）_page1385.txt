### テキスト
#### 32.50 TIME型→INT型変換
TIME型データをINT型データへ変換する関数「TIME_TO_INT(_E)」について説明しています。

##### 機能
・入力されたTIME型データ(s)をINT型データ(d)に変換して出力します。
・TIME型の上位16ビット（1ワード分）は変換時に切り捨てられます。

##### 演算処理
・sへの入力値はTIME型データ値（例：T#1s234ms）。
・dからINT型の数値（例：1234）が出力されます。

##### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、dから演算出力値が出力される。

2. EN/ENO付きファンクション
- 実行条件ENと演算結果ENOの組み合わせにより動作が変わる。
- ENがTRUE（演算実行）の場合、ENOもTRUEとなりdに演算出力値が出力される。
- ENがFALSE（演算停止）の場合、ENOはFALSEとなりdの値は不定値となる。
- ENOがFALSEの場合はdのデータを使用しないようプログラムで処理する必要がある。

##### エラー
・演算エラーは発生しない。

### 図
・ラダー、FBD/LDの関数ブロック図
  - EN/ENOなし：s（TIME型入力）、d（INT型出力）のみのシンプルなブロック
  - EN/ENO付き：EN（実行条件）、ENO（出力状態）付きのブロック
・ST（構造化テキスト）での使用例
  - EN/ENOなし：d:=TIME_TO_INT(s);
  - EN/ENO付き：d:=TIME_TO_INT_E(EN,ENO,s);

### 表
#### 設定データ（引数一覧）
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | TIME |
| ENO | 出力状態(TRUE:正常実行、FALSE:異常または停止) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

#### 演算結果（EN/ENO付きファンクション）
| 実行条件 EN | 演算結果 ENO | (d) |
| :--- | :--- | :--- |
| TRUE（演算実行） | TRUE | 演算出力値 |
| FALSE（演算停止） | FALSE | 不定値 |
