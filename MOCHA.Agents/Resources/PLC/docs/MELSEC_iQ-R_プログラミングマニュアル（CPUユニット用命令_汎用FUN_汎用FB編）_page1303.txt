<overview>
本ページは、待機系CPUユニットと制御系CPUユニット間のデータ書き込み動作を示すタイミングチャートを中心に構成されています。ページ全体は2つの例で構成されており、1つは正常完了の場合、もう1つは制御系のデバイス/ラベルが範囲外で異常完了した場合の動作を示しています。テキストは例の説明と各信号の名称、状態変化を示し、図はタイミングチャート形式で信号のON/OFFやデータの変化を視覚的に表現しています。表やグラフはなく、図解中心の技術説明ページです。
</overview>

<contents>
### テキスト
#### 例：正常完了の場合
- 待機系CPUユニットと制御系CPUユニット間のシーケンススキャンの動作を示す。
- CONTWRP命令のON/OFF切り替えにより、完了デバイス(d2)がONになり、1スキャンの間ON状態を維持した後OFFになる。
- 完了時の状態表示デバイス(d2)+1はOFFのまま。
- 書込みデータ格納デバイス(s2)に3000がセットされる。
- 制御系CPUユニットのシーケンススキャンでは、転送先デバイス(d1)に3000が書き込まれる。

#### 例：制御系のデバイス/ラベルが範囲外で異常完了した場合（完了ステータス(s1)+1=5）
- 待機系CPUユニットの動作は正常完了の場合と同様にCONTWRP命令のON/OFF切り替えがある。
- 完了デバイス(d2)と完了時の状態表示デバイス(d2)+1が1スキャンの間ONになる。
- 書込みデータ格納デバイス(s2)に3000がセットされる。
- 完了ステータス(s1)+1に5がセットされることで異常完了を示す。
- 制御系CPUユニット側では「制御系異常」として処理される。

### 図（タイミングチャート）
- 縦軸に各信号名（シーケンススキャン、CONTWRP命令、完了デバイス(d2)、完了時の状態表示デバイス(d2)+1、書込みデータ格納デバイス(s2)、転送先デバイス(d1)、完了ステータス(s1)+1）を配置。
- 横軸は時間軸で、信号のON/OFFや数値の変化を波形や数値で示す。
- 正常完了例では、CONTWRP命令がONになると完了デバイス(d2)がONになり、書込みデータが転送される様子を示す。
- 異常完了例では、完了デバイス(d2)と完了時の状態表示デバイス(d2)+1がONになり、完了ステータス(s1)+1に異常値5がセットされる様子を示す。
- 制御系CPUユニットの転送先デバイス(d1)は正常完了例のみで3000がセットされている。

</contents>
