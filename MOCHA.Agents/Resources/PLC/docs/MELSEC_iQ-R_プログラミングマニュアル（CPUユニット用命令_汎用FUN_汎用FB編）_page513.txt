### テキスト
#### 機能
(s1)で指定されたデバイス番号以降に格納しているデータの各ビットを、(s2)で指定されたビット分ずつ結合し、(d)で指定されたデバイス番号以降に格納します。

- (s2)で指定する結合ビット数は1～16ビットの範囲内で指定可能です。
- (s2)による指定デバイス番号から、「0」を格納しているデバイス番号までを結合ビット数として処理します。
- 結合するデータのデバイス範囲((s1)～(s1)の終了範囲)と、結合したデータを格納するデバイス範囲((d)～(d)の終了範囲)は重複しないようにしてください。重複した場合、正しい演算結果が得られないことがあります。
- (s1)、(s2)、(d)で指定するデバイス番号は重複させないでください。

#### 注意事項
- (s2)～(s2)+6の指定ビット分は無視されます。
- 設定終了指定は(1)で示されています。

### 図
- 複数のビット配置図があり、(s1)、(s2)、(d)の各デバイスのビットがどのように結合されるかを示しています。
- 例えば、(s1)の各ビット列が(s2)で指定されたビット数ごとに区切られ、(d)のデバイスに結合されて格納される様子が矢印で示されています。
- ビットの位置（b0～b15など）が明記されており、どのビットがどの位置に結合されるかが視覚的に理解できます。

### 表
#### エラーコード(SD0)と内容

| エラーコード | 内容 |
| :--- | :--- |
| 2821H | (s1)、(s2)のデバイスが重複しているとき。<br>(s1)、(d)のデバイスが重複しているとき。<br>(s2)、(d)のデバイスが重複しているとき。 |
| 3401H | (s2)に変換できない不正なデータを入力したとき。<br>・指定した結合ビット数指定が1～16ビットの範囲内で設定されていない。<br>・指定したデバイスから、デバイス/ラベルメモリの各設定エリアを超えるまでの間に、0が設定されていない。 |
