### テキスト
#### BIN16ビット単位スケーリング(X/Y別座標データ)
スケーリング用変換データ（16ビットデータ単位）に対し、指定した入力値でスケーリング（X/Y別座標データ）を行う命令について説明しています。

#### 実行条件
- SCL2、SCL2_U命令は立ち下がりエッジで実行
- SCL2P、SCL2P_U命令は立ち上がりエッジで実行

#### 設定データ
##### 内容、範囲、データ型
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s1) SCL2(P) | スケーリングを行う入力値または入力値が格納されているデバイス | -32768～32767 | 符号付きBIN16ビット | ANY16_S |
| SCL2(P)_U | 0～65535 | 符号なしBIN16ビット | ANY16_U |
| (s2) SCL2(P) | スケーリング用変換データが格納されている先頭デバイス | なし | 符号付きBIN16ビット*1 | ANY16_S*2 |
| SCL2(P)_U | なし | 符号なしBIN16ビット*1 | ANY16_U*2 |
| (d) SCL2(P) | スケーリングにより制御した出力値を格納するデバイス | なし | 符号付きBIN16ビット | ANY16_S |
| SCL2(P)_U | なし | 符号なしBIN16ビット | ANY16_U |
| EN | 実行条件 | なし | ビット | BOOL |
| ENO | 実行結果 | なし | ビット | BOOL |

*1 (s2)の座標ポイント数は符号なしBIN16ビットとなる  
*2 ラベルで設定する場合は動作に必要な領域が確保できるよう配列を定義し、その配列型ラベルの要素を指定すること

### 図
- ラダー図：s1、s2、dの3つの入力・出力を持つ命令ブロックのイメージ
- FBD/LD図：EN（入力）、ENO（出力）、s1、s2、dのポートを持つ命令ブロックの図
