### テキスト
#### 機能
- (d)で指定したデバイスのBIN16ビットデータを左へ(n)ビット分シフトする。
- 最下位から(n)ビット分は0になる。
- SM700では、シフト範囲から一つ左の1ビットが格納される。
- (d)にビットデバイスを指定した場合は、桁指定で指定したデバイス範囲で左シフトする。
- 実際にシフトするビット数は、(n)を桁指定で指定された点数の余りで計算される。
  - 例： (n)=15、桁指定点数=8ビットの場合、15-8=1余り7のため7ビットシフト。
  - (n)は0～15を指定可能。16以上の場合は(n)÷16の余りでシフト数を決定。
  - 例： (n)=18の場合、18÷16=1余り2のため2ビット左シフト。

#### エラー
- 演算エラーはなし。

### 図
#### 例1：(n)=8の場合
- 16ビットのビット列b15～b0が示されている。
- 元のビット列は「1111001100011111」。
- SM700では左に8ビットシフトし、左端の1ビットが格納されるため、結果は「0001111100000000」となる。
- 最下位8ビットは0になる。

#### 例2：(n)=5の場合（ビットデバイス指定時）
- X17～X10のビット列が示されている。
- 元のビット列は「0011010011」。
- SM700では左に5ビットシフトし、左端の1ビットが格納されるため、結果は「1011000000」となる。
- 最下位5ビットは0になる。

以上の内容から、データシフト命令は指定ビット数分左にシフトし、下位ビットは0クリア、SM700ではシフト範囲外の左隣のビットを格納する仕様であることがわかる。
