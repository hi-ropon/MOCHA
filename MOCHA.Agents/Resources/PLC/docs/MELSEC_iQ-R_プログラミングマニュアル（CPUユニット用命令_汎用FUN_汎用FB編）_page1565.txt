### 表
#### 命令名ごとの基本ステップ数とサブセット処理の可否一覧
- 表は3つの列で構成されています。
  1. 命令名：各命令の名称がアルファベットと数字の組み合わせで記載されています。
  2. 基本ステップ数：各命令の基本的なステップ数が数字で示されています（例：2、3、4、5など）。
  3. サブセット可否：「〇」または「―」でサブセット処理が可能かどうかを示しています。「〇」は可能、「―」は不可を意味します。

- 命令名の例：
  - DBL2UINTP（基本ステップ数4、サブセット不可）
  - INT2UINT（基本ステップ数3、サブセット可能）
  - DINT2INT（基本ステップ数2、サブセット可能）
  - BKBCD（基本ステップ数4、サブセット不可）

- サブセット処理が可能な命令は多く、基本ステップ数は2～4ステップが多い傾向です。
- サブセット不可の命令は主にDBL系やBKBCD系など特定の命令群に集中しています。
