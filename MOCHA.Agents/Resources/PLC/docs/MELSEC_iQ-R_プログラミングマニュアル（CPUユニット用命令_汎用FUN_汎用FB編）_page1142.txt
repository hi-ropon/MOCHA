### テキスト
#### ユニットアクセス命令の説明
- ユニットの脱着や二重化増設ベース構成時のユニットアクセスに関する注意事項が記載されています。
- 32点以上占有するユニットの場合、特定の情報が格納される仕組みについて説明。
- RnENCPU（ネットワーク部）の入出力番号指定時の特例についても触れています。

#### 読み出し結果のビット解説
- RJ71EN71ユニットのユニット情報を読み出す例を示しています。
- M0がONのときに、入出力番号が0040H以降のユニット情報をD0以降に格納するプログラム例をラダー図で示しています。

### 図（プログラム例）
- M0接点がONのとき、UNIINFRD命令でK4（H）、D0（d）、K2（n）を指定し、ユニット情報を読み出すラダー図。
- ユニット情報はD0～D3に格納される。

### ビットフィールドの説明

| ビット | 意味 |
| --- | --- |
| b15 | ユニットが装着されている（1） |

### 注釈
- 各ビットの意味が番号付きで詳細に説明されており、ユニットの状態や種類、エラー情報などを判別可能。
- D0、D1に情報が格納されるため、D2、D3は基本的に0となることが明記されている。
