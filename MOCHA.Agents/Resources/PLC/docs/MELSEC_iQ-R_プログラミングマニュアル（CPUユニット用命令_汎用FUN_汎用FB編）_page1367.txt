### テキスト
#### 32.40 REAL型→INT型変換
REAL_TO_INT(_E)関数はREAL型データをINT型データへ変換します。

対応CPUはRnCPU、RnENCPU、RnPCPU（二重化・一般・安全）で、RnSFCPU（一部）は非対応です。

#### 設定データ
引数の内容、種別、データ型は以下の通りです。

| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | REAL |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | INT |

#### 機能
##### 演算処理
- 入力されたREAL型データ(s)をINT型データに変換し、出力(d)します。
- 入力値の範囲は-32768～32767です。
- 変換後のデータはREAL型の小数点以下1桁目を四捨五入した値になります。

##### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、dから演算出力値が出力されます。

2. EN/ENO付きファンクション
- 実行条件と演算結果は以下の通りです。

| 実行条件 | 演算結果 |
| --- | --- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOがFALSEの場合、(d)から出力されるデータは不定値となるため、そのデータを使用しないようプログラム処理を行う必要があります。

### 図
- ラダー、FBD/LDでは、s入力とd出力の接続図が示されており、EN/ENO付きとなしの2パターンが記載されています。
- ST言語では、EN/ENOなしの場合は「d:=REAL_TO_INT(s);」、EN/ENO付きの場合は「d:=REAL_TO_INT_E(EN,ENO,s);」と記述します。
