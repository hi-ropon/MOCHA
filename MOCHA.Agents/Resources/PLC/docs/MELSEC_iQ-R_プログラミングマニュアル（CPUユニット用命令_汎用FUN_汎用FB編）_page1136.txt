### テキスト
#### 機能
- (H)で指定したスロットのユニット形名を読み出し、(d)で指定したデバイス以降へ格納する。
- 対象ユニットは以下の通り：
  - CPUユニット
  - 入力ユニット
  - 出力ユニット
  - 入出力混合ユニット
  - インテリジェント機能ユニット
- (H)には読み出し対象ユニットの先頭入出力番号を、16進数4桁で表したときの上3桁で指定する。
- 2スロット占有ユニットの場合は、各ユニットのマニュアルに記載の入出力占有点数を参照してスロット指定を行う。
- CPUユニットの先頭入出力番号は表に示す通り。

#### 格納値について
- (d)+0には命令実行結果を格納。
- (d)+1～(d)+9にはユニット形名を格納。
- (d)に格納する値の条件と内容は以下の通り。

### 図
- ユニット構成図が示されており、スロットごとにPW、CPU、RX10ユニットなどが並ぶ。
- 各ユニットの先頭入出力番号が16進数で示されている（例：0000H、0010H、0020H、0030H、0040H、0060H）。
- (1)の矢印で、読み出すユニットの先頭入出力番号はK4またはH4で指定することが示されている。

### 表
#### CPUユニットの先頭入出力番号
| CPUユニット | 先頭入出力番号 |
| :--- | :--- |
| 1号機 | 3E0H |
| 2号機 | 3E1H |
| 3号機 | 3E2H |
| 4号機 | 3E3H |

#### (d)に格納する値の条件と内容
| 条件 | (d)+0 | (d)+1～(d)+9 |
| :--- | :--- | :--- |
| 読出し対象ユニットが形名を保持している場合 | 0 | ユニットが保持している形名 |
| 読出し対象ユニットが形名を保持していない場合 | 1 | ユニット種別と点数を組み合わせた文字列 |
| 読出し対象スロットが空きスロットの場合 | -1 | 0000H |
| 読出し対象ユニットがオンライン交換中の場合 | 空欄 | 空欄 |
| (H)で指定した入出力番号がユニットの先頭番号でない場合 | 空欄 | 空欄 |
| 二重化増設ベース構成の待機系から増設ベースユニット上のユニットを指定した場合 | 空欄 | 空欄 |
| 読出し対象ユニットが再起動中の場合 | 空欄 | 空欄 |
