### テキスト
#### 32.61 ビット配列→INT型変換
ビット配列から指定のビット数をINT型データへ変換する機能について説明しています。

##### 機能概要
ビット配列の先頭から指定されたビット数をANY16型のデータに変換し、出力します。指定ビット数以上の出力ビットには0が設定されます。

##### 演算処理
- (s)に入力されたビット配列要素を先頭として(n)で指定されたビット数分をANY16型のデータに変換し(d)から出力。
- 指定ビット数以上の出力ビットは0になる。

##### 演算結果
1. EN/ENOなしファンクション：演算処理を実行し(d)から演算出力値が出力される。
2. EN/ENO付きファンクション：実行条件と演算結果は表の通り。
   - ENがTRUEのとき、ENOはTRUE、(d)に演算出力値
   - ENがFALSEのとき、ENOはFALSE、(d)は不定値（この場合(d)のデータは使用しない処理を推奨）

##### エラー
演算エラーはありません。

### 図
- ラダー/FBD/LD言語での使用例
  - EN/ENOなし：s（ビット配列）、n（ビット数）、d（出力）を持つブロック図
  - EN/ENO付き：EN（実行条件）、ENO（出力状態）、s、n、dを持つブロック図
- ST言語での使用例
  - EN/ENOなし：d := BITARR_TO_INT(s,n);
  - EN/ENO付き：d := BITARR_TO_INT_E(EN,ENO,s,n);

### 表
#### 設定データ（引数、内容、種別、データ型）
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(BitArr) | 入力(要素指定は変数も可) | 入力変数 | BOOL配列要素 |
| n | 4,8,12,16の定数のみ指定可能 | 入力変数 | INT |
| ENO | 出力状態(TRUE:正常実行、FALSE:異常または停止) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | ANY16 |

#### 実行条件と演算結果の対応表
| 実行条件 | 演算結果 |
| :--- | :--- |
| EN | ENO | (d) |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE | 不定値 |

*注記：ENOがFALSEの場合、(d)のデータは不定値となるため使用しない処理を推奨。
