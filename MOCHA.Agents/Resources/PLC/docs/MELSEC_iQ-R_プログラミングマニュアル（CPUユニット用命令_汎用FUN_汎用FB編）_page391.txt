### テキスト
#### 機能
- (d)で指定したデバイスから(n1)ダブルワード分のデータ範囲内で、左へ(n2)ダブルワード分シフトする。
- 最下位から(n2)ダブルワード分は0Hになる。
- (n1)または(n2)で指定した値が0のときは無処理となる。
- (n1)≦(n2)の場合は、(d)で指定したデバイスから(n1)ダブルワード分のデータはすべて0Hとなる。

#### エラー
- 演算エラーはありません。

### 図（例示）
- (n1)=9、(n2)=4の場合のデータシフトの動作を示す。
- 上段に(d)から始まるデバイスアドレスの位置を示し、下段に対応する16進数データが並ぶ。
- 矢印で示された範囲のデータが左へ4ダブルワード分シフトされ、シフト後のデータは下段に示されている。
- シフトされた分の下位4ダブルワードは0Hで埋められている。

### 表（データ配列）
| アドレス | データ（16進数） |
| :------- | :--------------- |
| (d)+17   | 30F1EH           |
| (d)+16   | 10000H           |
| (d)+15   | 30F1FFH          |
| (d)+14   | 1E100H           |
| (d)+13   | 1FFFFFH          |
| (d)+12   | 1020304H         |
| (d)+11   | 7FFFFH           |
| (d)+10   | 5101FH           |
| (d)+9    | 7FF2AH           |

- シフト後は、上記のデータが左へ4ダブルワード分移動し、下位4ダブルワードは0Hで埋められる。
