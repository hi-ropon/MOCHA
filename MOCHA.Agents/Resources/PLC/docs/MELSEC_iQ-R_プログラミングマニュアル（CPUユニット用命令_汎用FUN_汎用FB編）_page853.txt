### テキスト
#### 単精度実数帯域比較出力（EZCP(P)）
- 対応CPUはR00CPU、R01CPU、R02CPUに制約なし。RnCPU（R00CPU、R01CPU、R02CPU除く）やRnENCPUはファームウェアバージョン17以降で使用可能。エンジニアリングツールはバージョン1.020W以降が必要。
- 下限値(s1)と上限値(s2)の単精度実数の帯域に対し、比較データ(s3)の単精度実数で比較し、その結果に応じて(d)、(d)+1、(d)+2のいずれかがONになる。

#### 実行条件
- EZCP命令は立ち上がりエッジで実行
- EZCPP命令は立ち下がりエッジで実行

### 図
- ラダー図：3つの入力(s1, s2, s3)と出力(d)を持つ命令ブロックの形状を示す。
- ST言語：ENO:= EZCP(EN, s1, s2, s3, d); ENO:= EZCPP(EN, s1, s2, s3, d);
- FBD/LD図：EN、s1、s2、s3を入力、ENO、dを出力とする命令ブロックの図。

### 表
#### 設定データ（内容、範囲、データ型）
| オペランド | 内容 | 範囲 | データ型 | データ型(ラベル) |
| :--- | :--- | :--- | :--- | :--- |
| (s1) | 下限値または下限値を格納する先頭デバイス | 0, 2^-126 ≤ |s1| < 2^128 | 単精度実数 | ANYREAL_32 |
| (s2) | 上限値または上限値を格納する先頭デバイス | 0, 2^-126 ≤ |s2| < 2^128 | 単精度実数 | ANYREAL_32 |
| (s3) | 比較データまたは比較データを格納する先頭デバイス | 0, 2^-126 ≤ |s3| < 2^128 | 単精度実数 | ANYREAL_32 |
| (d) | 比較結果が格納される先頭デバイス | － | ビット | ANYBIT_ARRAY（要素数3） |
| EN | 実行条件 | － | ビット | BOOL |
| ENO | 実行結果 | － | ビット | BOOL |

#### 使用可能デバイス
- (s1), (s2), (s3)はビットデバイス不可、ワードデバイスの一部、ダブルワードのLT,LST,LC、間接指定可、定数は不可
- (d)はビットデバイスの一部、ワードデバイスの一部（T,ST,Cは不可）、間接指定可、定数不可
- 表内の記号「○」は使用可能、「－」は不可を示す。
