### テキスト
#### 使用可能デバイス
マルチCPUシステムで使用可能なデバイスの種類を示しています。ビット、ワード、ダブルワード、間接指定、定数、その他のカテゴリに分けて、各オペランド（U/H、s1、s2、d1、d2）で使用可能かどうかを○や―で表しています。

#### コントロールデータ
オペランド(s1)に関するコントロールデータの説明です。
- +0 完了ステータス：完了時のステータスを格納し、正常完了は0000H、異常完了はエラーコードを示します。
- +1 読出しデータ点数：読出しデータ点数をワード単位で設定し、設定範囲は1～8192（マルチCPUシステム構成時の最大設定範囲）。

#### 機能
- マルチCPUシステム構成時に、自号機CPUユニットで指定したデバイス(d1)以降のデータを、他号機CPUユニット(U/H)の指定したデバイス(d1)以降に、(s1)+1で指定した読出しデータ点数分格納します。
- D(P).DDRD命令、M(P).DDRD命令の動作概要を示しています。

### 図
- マルチCPUシステム構成時のデータ格納イメージ図：自号機CPUユニット（読出し要求元）から他号機CPUユニット（読出し対象先）へ、d1以降のデータを(s1)+1分転送する様子を矢印で示しています。
- DP.DDRD命令の動作概要シーケンス図：命令実行から要求信号の送信、転送、応答信号の受信、受付処理、完了デバイス(d2)のON/OFF状態変化までのタイミングを示しています。CPU間周期通信はデフォルト0.888msで行われます。
