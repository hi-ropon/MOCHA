### テキスト
#### 他号機CPUユニットアクセス命令の同時実行
他号機CPUユニットアクセス命令は、指定された条件範囲内であれば同時に実行可能です。  
同時実行できる条件は「各号機で使用可能なブロック数」以上の合計ブロック数であることです。  
もし同時実行時に使用ブロック数がシステムエリアの総ブロック数を超える場合、そのスキャンでは命令は無処理となり、次のスキャンで再度実行されます。  
ただし、システムエリアの空きブロック数がSD796～SD799の設定値より少ない場合は異常完了となります。  
また、使用ブロック数や設定値が少ない場合の実行可否についても説明しています。

#### 他号機CPUユニットアクセス命令使用時のインタロック
インタロック用に特殊リレーSM796～SM799（マルチCPU間専用命令最大使用ブロック数設定）があります。  
複数の他号機CPUユニットアクセス命令を同時に実行する場合は、このインタロックを使用してください。  
Pointとして、SM796～SM799を使用する際はSD796～SD799に各号機で使用する命令の最大ブロック数を設定し、設定値以下になると対応する特殊リレーがONになることが示されています。

#### 注意事項
- 書込み/読出し対象のCPUユニットが立ち上がっている状態で命令を実行する必要があること。  
- 書込み/読出し対象のCPUユニットが立ち上がっていない状態で実行すると無処理となること。  
- 命令実行後、完了デバイスがONするまでの間に設定データの範囲変更を行うとデータ格納ができなくなること。  
- システム情報エリアを上書きしないように注意すること。  
- 使用ブロック数が設定値より多い場合、インタロックをとっていても命令が実行できないことがあること。  
- SD796～SD799は該当号機を対象とした命令実行前に設定することが推奨されていること。

### 表
#### SD設定値と空きブロック数の大小関係による命令実行可否

| SD設定値*3と空きブロック数*2の大小関係 | 命令使用ブロック数*1と空きブロック数の大小関係 | 実行結果 |
| :--- | :--- | :--- |
| SD設定値 ≤ 空きブロック数 | 命令使用ブロック数 ≤ 空きブロック数 | 実行する |
| SD設定値 ≤ 空きブロック数 | 命令使用ブロック数 > 空きブロック数 | 実行しない（無処理） |
| SD設定値 > 空きブロック数 | 命令使用ブロック数 ≤ 空きブロック数 | 異常完了 |

*1 他号機CPUユニットアクセス命令が使用するブロック数  
*2 システムエリアの空きブロック数  
*3 SD796～SD799の設定値
