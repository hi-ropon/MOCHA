### テキスト
■SM705（変換桁数切換）がONの場合の(s)の設定方法について説明しています。
(s)には任意の桁数（00H（NULLコード）まで）の10進アスキーデータを設定します。ただし整数部が最大桁数（10桁）の場合は00Hの設定は不要です。
(s)の設定方法を具体的に示しています。

- (s)～(s)+5で指定するアスキーデータの範囲は、DDABIN(P)命令の場合は-2147483648～2147483647、DDABIN(P)_U命令の場合は0～4294967295です。
- 変換するデータが負の数の場合は、(s)+0の下位バイトに符号データとして2DH（-）を設定します。0または正の数の場合は符号データは設定せず、最上位桁のアスキーコードを設定します。
- 各位に設定するアスキーコードは30H～39Hの範囲内です。
- 正の数で数値部10桁の場合、(s)+5以降に格納されているデータは無視されます。負の数で数値部10桁の場合、(s)+5の上位バイト以降に格納されているデータは無視されます。
- 各位に設定するアスキーコードが20Hのときは30Hとして処理されます。00Hのときは10進アスキーデータの終端として処理されます。
- 下記の場合、(d)には0が格納されます。
  ・1文字目が00H（NULL）
  ・1文字目が2DH（-）で、2文字目が00H（NULL）

### 表・図
#### (s)に設定する値と(s)～(s)+5の内容の対応表
- 0の場合：
  - (s)の上位バイトに00Hを設定し、(s)+1以降は無視される。
  - ビット配置図では、b15～b8b7に00H、b0にASCII 10^0が入る。

- 正の数（数値部1桁）の場合：
  - (s)に正の数（数値部2桁）または負の数（数値部1桁）を設定。
  - (s)+1の下位バイトに00Hを設定し、(s)+1の上位バイト以降は無視される。
  - ビット配置図では、ASCII 10^0や2DH（-）の配置を示す。

- 正の数（数値部9桁）と負の数（数値部8桁）の場合：
  - 正の数は(s)+4の上位バイトに00Hを設定し、(s)+5は無視。
  - 負の数は(s)+5の下位バイトに00Hを設定し、(s)+5の上位バイトは無視。
  - 各バイトにASCIIコード（10^7～10^0など）が割り当てられている。

- 正の数（数値部10桁）と負の数（数値部10桁）の場合：
  - 正の数は(s)+5は無視され、00Hの設定は不要。
  - 負の数は(s)+5の上位バイトは無視され、00Hの設定は不要。
  - 各バイトにASCIIコード（10^9～10^0など）が割り当てられている。

#### ASCIIコードの説明
- ASCII 10^0：一の位のアスキーコード
- ASCII 10^1：十の位のアスキーコード
- ASCII 10^9：十億の位のアスキーコード

### ポイント
- (s)～(s)+5に設定するアスキーデータは命令の種類により範囲が異なる。
- 符号データは負の数の場合のみ設定し、正の数は符号なし。
- アスキーコードは30H～39Hの範囲で、20Hは30Hとして処理。
- 00Hは10進アスキーデータの終端コードとして扱う。
- 特定条件下で(d)に0が格納される。
