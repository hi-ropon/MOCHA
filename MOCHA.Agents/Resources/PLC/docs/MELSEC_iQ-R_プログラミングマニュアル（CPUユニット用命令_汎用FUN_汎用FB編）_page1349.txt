### テキスト
#### 32.29 DINT型→WORD型変換
関数名：DINT_TO_WORD(_E)  
この関数はDINT型データをWORD型データへ変換します。

#### 対応CPU
RnCPU、RnENCPU、RnPCPU（三重化・一般）、RnSPFCPU（安全・一般）に対応しています。

#### ラダー、FBD/LD、STでの使用例
- ラダー、FBD/LDではEN/ENOなしとEN/ENO付きの2種類のブロック図が示されています。  
- ST言語ではEN/ENOなしの場合は `d:=DINT_TO_WORD(s);`、EN/ENO付きの場合は `d:=DINT_TO_WORD_E(EN,ENO,s);` と記述します。

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| :--- | :--- | :--- | :--- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | DINT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | WORD |

#### 機能
##### 演算処理
- 入力されたDINT型データ(s)をWORD型データ(d)に変換して出力します。  
- 入力値の上位16ビットの情報は捨てられます。  
- 図ではDINTの32ビットのうち下位16ビットがWORDの16ビットに変換される様子が示されています。  
- sへの入力値はDINT型のデータ値です。

##### 演算結果
1. EN/ENOなしファンクション  
   演算処理を実行し、dから演算出力値が出力されます。  
2. EN/ENO付きファンクション  
   実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
| :--- | :--- | :--- |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となります。その場合はdから出力されたデータを使用しないようプログラムで処理してください。
