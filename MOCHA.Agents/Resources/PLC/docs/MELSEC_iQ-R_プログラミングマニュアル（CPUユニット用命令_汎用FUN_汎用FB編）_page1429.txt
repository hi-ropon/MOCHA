本ページは主に表形式で構成されており、各種データ型（DINT型、WORD型、DWORD型、REAL型、LREAL型）に関する説明が記載されています。テキスト部分では各データ型の特徴や演算時の挙動について具体例を交えて解説しています。表はデータ型ごとに内容を整理しており、グラフや図は含まれていません。
</overview>

### 表
#### データ型と内容の説明

| データ型 | 内容 |
| :--- | :--- |
| DINT型 | アンダーフローおよびオーバーフローが発生しても演算エラーにはならない。ADD_E命令の場合、ENOからTRUEが出力される。<br>【例1】<br>2147483647 + 2 = -2147483647<br>(7FFFFFFFH) + (00000002H) = (80000001H)<br>最上位ビットが1になるため負の値になる。<br>【例2】<br>-2147483648 + (-2) = 2147483646<br>(80000000H) + (FFFEH) = (7FFFFFFFH)<br>最上位ビットが0になるため正の値になる。 |
| WORD型 | オーバーフローが発生しても演算エラーにはならない。ADD_E命令の場合、ENOからTRUEが出力される。<br>【例】<br>65535 + 1 = 0<br>(FFFFH) + (0001H) = (0000H) |
| DWORD型 | オーバーフローが発生しても演算エラーにはならない。ADD_E命令の場合、ENOからTRUEが出力される。<br>【例】<br>4294967295 + 1 = 0<br>(FFFFFFFFH) + (00000001H) = (00000000H) |
| REAL型 | 演算エラーとなり不定値を出力する。 |
| LREAL型 | （説明なし） |

### テキスト
- 各データ型の演算時のオーバーフローやアンダーフローの扱いについて説明。
- DINT型は符号付き32ビット整数で、オーバーフロー時に符号が反転する例を示している。
- WORD型、DWORD型は符号なし整数で、最大値を超えると0に戻ることを例示。
- REAL型は浮動小数点数で、演算エラー時に不定値を出力することを記載。
