### 表
#### データ型と内容の説明

| データ型 | 内容 |
| :--- | :--- |
| DINT型 | アンダーフローおよびオーバーフローが発生しても演算エラーにならず、SUB_Eの場合はENOからTRUEが出力されます。<br>【例1】<br>2147483647 - (-2) = 2147483647<br>(7FFFFFFFH) - (0000FFFEH) = (80000001H)<br>最上位ビットが1になるため負の値となる。<br>【例2】<br>-2147483648 - 2 = 2147483646<br>(80000000H) - (00000002H) = (7FFFFFFFH)<br>最上位ビットが0になるため正の値となる。 |
| WORD型 | アンダーフローが発生しても演算エラーにならず、ADD_Eの場合はENOからTRUEが出力されます。<br>【例】<br>0 - 1 = 65535<br>(0000H) - (0001H) = (FFFFH) |
| DWORD型 | アンダーフローが発生しても演算エラーにならず、ADD_Eの場合はENOからTRUEが出力されます。<br>【例】<br>0 - 1 = 4294967295<br>(00000000H) - (00000001H) = (FFFFFFFFH) |
| REAL型 | 演算エラーとなり不定値を出力します。 |
| LREAL型 | （説明なし） |

### テキスト
- 各データ型におけるアンダーフローやオーバーフローの扱いについて説明。
- DINT型は符号付き32ビット整数で、最上位ビットの状態により正負が決まることを例示。
- WORD型、DWORD型は符号なし整数で、アンダーフロー時に最大値に巻き戻る動作を例示。
- REAL型は浮動小数点数で、演算エラー時に不定値を出力することを記載。
- LREAL型については説明が空欄。
