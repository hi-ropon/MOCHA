### テキスト
#### M(P).DDWR命令の動作概要
- シーケンススキャンにおける命令実行の流れを示す図があり、命令実行時に他号機CPUユニットへ要求信号を送り、受付処理後に応答信号が返り、完了デバイス(d2)のON/OFFが制御される。
- D(P).DDWR命令およびM(P).DDWR命令の実行と正常/異常完了は、設定データで指定した完了デバイス(d2)の状態表示で確認可能。
- 完了デバイス(d2)は、命令完了したスキャンのEND処理でONになり、次のEND処理でOFFになる。
- 正常完了時は完了デバイスの状態がON/OFFで変化し、異常完了時は完了デバイスはOFFのまま変化しない。異常完了時は完了ステータスにエラーコードを格納。
- 命令で使用するブロック数は書込みデータ点数に依存し、詳細は1257ページの他号機CPUユニットアクセス命令を参照。
- 書込み対象先CPUユニットの相手先デバイスの指定可能範囲も1257ページを参照。
- システムエリアに空きブロックがない場合は異常完了となるが、SD796～SD799をインタロックとして使用することで防止可能。

### 表
#### エラーコード一覧 (SD0)
| エラーコード | 内容 |
| :--- | :--- |
| 2800H | 指定した他号機CPUユニットの先頭入出力番号(16進数4桁の上3桁)が3E0H～3E3Hの範囲外のとき。 |
| 2801H | 指定した他号機CPUユニットが誤っている。予約設定されている号機を指定、または未装着の号機を指定した場合。 |
| 2802H | 他号機CPUユニットがD(P).DDWR命令、M(P).DDWR命令をサポートしていない。 |
| 2803H | 指定した他号機CPUユニットに自身のCPUユニットを指定したとき。 |
| 2810H | 指定した他号機CPUユニットに命令を実行できないCPUユニットを指定したとき。 |
| 3404H | デバイスを指定する文字列が誤っているとき。 |
| 3405H | 書込みデータ点数(s1)+1が0～8192以外である。*2 |
| 3440H | CPU間定周期通信を使用しない設定でD(P).DDWR命令を実行した。 |
| 3441H | 各号機で使用可能なシステムエリアサイズを超えるデータ点数を指定した。 |

*2 2台のCPUユニットによるマルチCPUシステム構成時の最大設定範囲。システム構成により書込み可能なデータ点数が変わるため8192点以下となる場合あり。

#### エラーコード一覧 ((s1)+0)
| エラーコード | 内容 |
| :--- | :--- |
| 0010H | 対象先CPUユニットへの命令要求が許容値以上（システムエリアに空きブロックがない）。 |
| 1001H | (d1)で指定した他号機CPUユニットのデバイスが他号機CPUユニットで使用できないデバイス、またはデバイス範囲外。 |
| 1080H | D(P).DDWR命令、M(P).DDWR命令で設定した書込みデータ点数が0。 |
| 1090H | D(P).DDWR命令、M(P).DDWR命令で設定した書込み先のデバイスが外部からのデバイス書込み禁止に設定されている。 |
