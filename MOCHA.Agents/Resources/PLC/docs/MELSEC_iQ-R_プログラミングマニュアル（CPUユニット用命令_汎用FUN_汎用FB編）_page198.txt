### テキスト
#### マスタコントロール命令（MC命令）とマスタコントロール解除命令（MCR命令）
- MC命令は、指定したデバイスを変えることで同一ネスティング番号を何回でも使用可能。
- MC命令がONの時、指定デバイスのコイルがONになる。
- OUT命令などで同じデバイスを使うとダブルコイルになるため、MC命令で指定したデバイスは他の命令中で使わないこと。
- MCR命令はマスタコントロールの解除命令で、範囲の終了を示す。
- MCR命令の前に接点命令をつけてはいけない。
- 同一ネスティング番号のMC命令とMCR命令はセットで使用し、MCR命令が1箇所に集まっている入れ子構造の場合は、最も若いネスティング番号1つで全てのマスタコントロールを終了できる。

#### エラー
- 演算エラーはなし。

#### マスタコントロール命令の入れ子構造の使用方法
- マスタコントロール命令は入れ子構造にでき、区間はネスティング番号（N0～N14）で区別。
- 入れ子構造を使うことでプログラムの実行条件を順に制御可能。
- 入れ子構造を使った回路例を示す。

### 図（入れ子構造のマスタコントロール回路）
- 左側はエンジニアリングツール上の表示で、3つのMC命令（N0、N1、N2）と3つの入力（A、B、C）がそれぞれ対応。
- 右側は実際の動作回路で、各MC命令のON条件が並列・直列で組み合わされている。
- MCR命令はN2、N1、N0の順に配置されている。
- 実行条件は以下の通り：
  1. AがONのとき実行
  2. AかつBがONのとき実行
  3. AかつBかつCがONのとき実行
  4. A、B、Cに関係なし（MCR命令の解除）

この図は、入れ子構造により複雑な実行条件を順に制御できることを示している。
