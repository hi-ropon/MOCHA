### テキスト
#### 32.26 INT型→TIME型変換
INT_TO_TIME(_E)関数は、INT型データをTIME型データへ変換します。

対応CPUはRnCPU、RnENCPU（プロセス、二重化、一般、安全）で、RnSFCCPU（一般、安全）には対応していません。

#### ラダー、FBD/LDの構成
- EN/ENOなしの場合は、入力sから出力dへ変換。
- EN/ENO付きの場合は、EN入力、ENO出力を持ち、sからdへ変換。

ST言語では以下のように記述されます。
- EN/ENOなし: d := INT_TO_TIME(s);
- EN/ENO付き: d := INT_TO_TIME_E(EN, ENO, s);

#### 設定データ
| 引数 | 内容 | 種別 | データ型 |
| --- | --- | --- | --- |
| EN | 実行条件(TRUE:実行、FALSE:停止) | 入力変数 | BOOL |
| s(IN) | 入力 | 入力変数 | INT |
| ENO | 出力状態(TRUE:正常、FALSE:異常) | 出力変数 | BOOL |
| d | 出力 | 出力変数 | TIME |

#### 機能
- 演算処理として、入力されたINT型データ(s)をTIME型データに変換し、出力(d)します。
- sへの入力値はINT型データ値です。

#### 演算結果
1. EN/ENOなしファンクション
- 演算処理を実行し、dから演算出力値が出力されます。

2. EN/ENO付きファンクション
- 実行条件と演算結果は以下の通りです。

| 実行条件 EN | 演算結果 ENO | (d) |
| --- | --- | --- |
| TRUE(演算実行) | TRUE | 演算出力値 |
| FALSE(演算停止) | FALSE*1 | 不定値 |

*1 ENOからFALSEが出力された場合、dから出力されるデータは不定値となるため、その場合はdから出力されたデータを使用しないようプログラム処理を行う必要があります。

#### エラー
- 演算エラーはありません。
