### テキスト
#### Modbus RTUの仕様
Modbusプロトコルは仕様が一般に公開されており、簡単であるため産業分野で広く使用されています。  
通信方式はシングルクライアント／マルチサーバー方式で、クライアントのみがクエリ（問い合わせ）を発行し、サーバーは要求された処理を実行して応答メッセージを返します。  
AZシリーズはRTUモードのみをサポートし、ASCIIモードはサポートしていません。  
メッセージの送信方法は2種類あります。

- ユニキャストモード  
  クライアントはサーバー1台に対してクエリを送信し、サーバーは処理を実行してレスポンスを返します。

- ブロードキャストモード  
  クライアントがサーバーアドレス0を指定すると、すべてのサーバーにクエリを送信します。サーバーは処理を実行しますがレスポンスは返しません。

### 表
#### 通信仕様

| 項目       | 内容                                                                                         |
|------------|----------------------------------------------------------------------------------------------|
| 電気的特性 | EIA-485準拠、ストレートケーブル ツイストペア線（TIA/EIA-568B CAT5e以上を推奨）、総延長距離50mまで |
| 通信方式   | 半二重通信 調歩同期方式（データ：8ビット、ストップビット：1ビット／2ビット、パリティ：なし／偶数／奇数） |
| 伝送速度   | 9,600 bps、19,200 bps、38,400 bps、57,600 bps、115,200 bps、230,400 bpsから選択可能           |
| プロトコル | Modbus RTUモード                                                                             |
| 接続形態   | 上位システム1台に対して最大31台まで接続可能                                                 |

※ 配線・配置によりモーターケーブルや電源ケーブルから発生するノイズが問題になる場合は、シールドやフェライトコアの使用を推奨。

**図の説明:**  
- 上位システムからRS-485通信で複数の号機（1、2、31）に接続されている。

- 号機1と号機31に終端抵抗が設置されている。
