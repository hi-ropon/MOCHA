### テキスト
#### 積算負荷関連パラメータの説明

2. 手順1で決定した最大値をインフォメーションに設定する方法について説明しています。

3. 装置の稼動が始まり、モーターの積算負荷が設定値「5,000」に達するとインフォメーションが発生し、装置のメンテナンスを促す旨が記載されています。

---

■「積算負荷除数」パラメータについて  
- 積算負荷のカウント上限値は2,147,483,647。  
- 長時間の運転や結合ループの繰り返しで積算負荷が増えすぎる場合に管理が困難になるため、このパラメータを使用。  
- 「積算負荷除数」は積算負荷のカウント値を割る除数で、これによりカウント値の管理が容易になる。  

- 「積算負荷除数」パラメータが「1」の場合は増加率が1で、上限値に達すると積算負荷をカウントできなくなる。  
- 「積算負荷除数」パラメータが「5」の場合は増加率が1/5となり、増加が緩やかになる。

---

■「積算負荷自動クリア」パラメータについて  
- 「有効」に設定すると、MOVE出力がONになるたびに積算負荷が0にクリアされる。  
- 1回の運転ごとに積算負荷をリセット可能。  
- 「無効」に設定すると、MOVE出力がONでも積算負荷はリセットされず積算が継続。  
- 「無効」時はLAT-CLR入力で積算負荷をリセット可能。  
- 一定時間や一定条件で積算負荷をモニタできる。

### 図

#### 積算負荷除数パラメータの動作イメージ

```mermaid
flowchart LR
    subgraph 除数1の場合
        A1[積算負荷] -->|増加率=1| B1[上限値に達する]
        B1 --> C1[積算負荷カウント不可]
    end

    subgraph 除数5の場合
        A2[積算負荷] -->|増加率=1/5| B2[緩やかに増加]
    end
```

**説明:**  
- 除数1の場合は積算負荷が直線的に増加し、上限値に達するとカウントできなくなる。  
- 除数5の場合は積算負荷の増加が5分の1に抑えられ、増加が緩やかになる。

---

#### 積算負荷自動クリアパラメータの動作タイミングチャート

```mermaid
flowchart TD
    subgraph 有効時
        direction LR
        Speed1[速度] --> MOVE1[MOVE出力 ON/OFF]
        MOVE1 --> Load1[積算負荷 増減]
    end

    subgraph 無効時
        direction LR
        Speed2[速度] --> MOVE2[MOVE出力 ON/OFF]
        MOVE2 --> Load2[積算負荷 継続増加]
        LATCLR[LAT-CLR入力 ON/OFF] --> Load2
    end
```

**説明:**  
- 「有効」時はMOVE出力がONになるたびに積算負荷がリセットされる。  
- 「無効」時はMOVE出力がONでも積算負荷はリセットされず、LAT-CLR入力でリセット可能。
