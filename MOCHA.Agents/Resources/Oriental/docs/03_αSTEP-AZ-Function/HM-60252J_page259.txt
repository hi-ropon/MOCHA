### テキスト
#### 通信タイミング
ドライバが監視している通信時間、およびクライアントの通信タイミングについて説明しています。

- クライアントとサーバー（ドライバ）間の通信タイミングを示すタイムチャートがあり、クエリ送信、レスポンス受信の時間関係を示しています。
- 各タイミング要素（Tb1～Tb5、C3.5）の名称と内容を詳細に解説しています。

##### 各タイミング要素の説明
| 記号 | 名称 | 内容 |
| :--- | :--- | :--- |
| Tb1 | 通信タイムアウト（ドライバ） | ドライバは受信したクエリの間隔を監視。設定時間を過ぎてもクエリを受信できなければタイムアウトアラーム発生。正常メッセージ受信時は発生しない。 |
| Tb2 | 送信待ち時間（ドライバ） | クエリ受信後、レスポンス送信開始までの時間。パラメータで設定。 |
| Tb3 | ブロードキャスト間隔（クライアント） | ブロードキャスト時、次のクエリ送信までの時間。サイレントインターバル(C3.5)+5ms以上必要。 |
| Tb4 | 送信待ち時間（クライアント） | レスポンス受信後、次のクエリ送信までの時間。サイレントインターバル(C3.5)より長く設定推奨。 |
| Tb5 | クエリ処理時間（ドライバ） | 受信クエリ処理にかかる時間。メッセージ構成に応じて変動。 |
| C3.5 | サイレントインターバル | メッセージ終了判定のための間隔。C3.5以上の空き時間が必要。パラメータ設定により変動。 |

#### 「サイレントインターバル(Modbus)」パラメータを「0（自動）」に設定した場合の推奨値

| 通信速度 (bps) | サイレントインターバル (C3.5) | 送信待ち時間 (クライアント) (Tb4) の目安 |
| :------------- | :----------------------------- | :---------------------------------------- |
| 9,600          | 4.0 ms以上                     | 5.0 ms以上                               |
| 19,200以上     | 2.5 ms以上                     | 3.0 ms以上                               |

#### 重要事項
- クライアントの送信待ち時間(Tb4)がサイレントインターバルより短いと通信異常が発生するため、Tb4の設定を見直す必要がある。
- サイレントインターバル(C3.5)は製品シリーズにより異なるため、複数製品接続時はパラメータを適切に設定すること。
- 「サイレントインターバル(Modbus)」パラメータを持つ製品のみのシステムでは、共通設定により通信サイクルを向上可能。通常は「0（自動）」で使用。
