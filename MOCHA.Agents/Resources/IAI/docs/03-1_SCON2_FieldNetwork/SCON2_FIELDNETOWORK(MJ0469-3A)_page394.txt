<overview>
本ページはMECHATROLINK-Ⅲの基本通信に関する説明で、32バイトモードおよび48バイトモードのコマンド構成を示しています。構成は主に表形式で、各モードにおけるコマンドのバイト割り当てとレスポンスの対応関係を詳細に記載しています。テキスト部分は少なく、表が中心の構成です。グラフや図は含まれていません。
</overview>

<contents>
### 表
#### 32バイトモードのコマンド構成

| コマンド       | バイト | レスポンス               | バイト |
| -------------- | ------ | ------------------------ | ------ |
| コマンドコード (CMD) | 0      | コマンドコードレスポンス (RCMD) | 0      |
| ウォッチドッグデータ (WDT) | 1      | ウォッチドックステータス (RWDT) | 1      |
| コマンド制御 (CMD_CTRL) | 2, 3   | コマンドステータス (CMD_STAT) | 2, 3   |
| コマンドデータ (CMD_DATA) | 4～31  | レスポンスデータ (RSP_DATA)    | 4～31  |

- メインコマンドはコマンドコード、ウォッチドッグデータ、コマンド制御、コマンドデータで構成される。
- バイト4から31まではコマンドデータおよびレスポンスデータとして使用される。

#### 48バイトモードのコマンド構成

| コマンド       | バイト | レスポンス               | バイト |
| -------------- | ------ | ------------------------ | ------ |
| コマンドコード (CMD) | 0      | コマンドコードレスポンス (RCMD) | 0      |
| ウォッチドッグデータ (WDT) | 1      | ウォッチドックステータス (RWDT) | 1      |
| コマンド制御 (CMD_CTRL) | 2, 3   | コマンドステータス (CMD_STAT) | 2, 3   |
| コマンドデータ (CMD_DATA) | 4～47  | レスポンスデータ (RSP_DATA)    | 4～47  |

- 32バイトモードと同様の構成だが、コマンドデータおよびレスポンスデータのバイト数が増加している。
