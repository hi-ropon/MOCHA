### テキスト
#### 注意事項
- ピッチ送り動作でストロークエンドのソフトリミットに達すると、その位置で停止し、位置決め完了信号PENDがONになる。
- 押付け動作の直後（押付け状態時）にピッチ送りを行う場合、ピッチ送りの基点にピッチ送り量を加えた座標値へ直接移動するため注意が必要。
- 通常の位置決め動作中にピッチ送り動作のポジションNo.を起動（CSTR ON）すると、目標座標値にピッチ送り量を加えた座標値へ移動する。ピッチ送りの起動を繰り返すと、回数分だけピッチ送り量が目標位置に加算されるため、PLC側で完了位置の確認ができなくなるので避けること。
- 一時停止中にピッチ送りの起動（CSTR ON）を繰り返すと、起動回数に相当する移動が連続的に行われるため注意。対策として一時停止のままリセット信号RESをONにして残移動量をキャンセルするか、一時停止中に起動信号がONしないようインターロックすること。
- ピッチ送り動作でソフトリミット（ストロークエンド）に達した場合は減速して停止し、位置決め完了出力PENDを出力する。
- MOVEはPENDのOFFと同時にONし、ONと同時にOFFするため、位置決め幅が大きい場合はアクチュエーターが動作中でもOFFすることがある。
- ピッチ送り機能を使った押付け動作も可能。ただし、通常位置決め実行途中（PENDがONする前）にこの動作に切替える制御は行わないこと。起動信号CSTRがONした瞬間にピッチ送り機能を使った押付け動作が割り込まれ、PLC側でアクチュエーターの位置管理ができなくなる。

```mermaid
    subgraph 押付け動作
        A[アプローチ動作] --> B[押付け動作]
    end
    B --> C[現在の停止位置]
    B --> D[ピッチ送りの基点]
    C --> E[押付け動作と同方向のピッチ送り]
    D --> F[押付け動作と逆方向のピッチ送り]
    F --> G[2回目以降のピッチ送り = 設定値通り]
    E --> H[2回目以降のピッチ送り = 設定値通り]
    D --> I[ピッチ送りの基点までの移動 + ピッチ送り量]
    I --> J[ピッチ送りの基点 + ピッチ送り量の座標値へ移動]
```

**図の説明:**
- 押付け動作の流れとして、アプローチ動作から押付け動作へ進む。
- ピッチ送りの基点は押付けのポジションデータの「位置」に入力されている座標値。
- ピッチ送りは押付け動作と同方向または逆方向に行われ、2回目以降は設定値通りのピッチ送り量で移動。
- ピッチ送りの基点までの移動にピッチ送り量を加えた座標値へ直接移動する動作がある。
