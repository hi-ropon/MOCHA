### テキスト
#### 8.1 トラブル発生時の処理
迅速な復旧と再発防止のため、トラブル発生時には以下の点を確認することが推奨されています。

1. コントローラーの状態表示LEDおよびPIOの確認  
2. 上位コントローラー（PLCなど）のアラームの有無  
3. 主電源の電圧確認  
4. PIO電源（DC24V）の電圧確認  
5. ブレーキ電源の電圧（DC24V）確認（ブレーキ付きアクチュエーターの場合）  
6. アラームの確認（アラームコードはIA-OSなどのティーチングツールで確認）  
7. コネクター類の脱落または不完全接続  
8. ケーブル類の接続、断線や挟込みの確認  
　導通確認は主電源を切り、測定部の配線を外して行うこと  
9. 入出力信号の確認（上位コントローラーやティーチングツールを使用し、信号状態の矛盾を確認）  
10. ノイズ対策（接地線やノイズキラーの接続など）の確認  
11. トラブル発生までの経過および発生時の運転状況の確認  
12. 発生原因の解析  
13. 対策の実施

### 表
#### コントローラーの状態表示LEDおよびPIOの確認

| LED  | 表示     | 状態                                         |
| :--- | :------- | :------------------------------------------- |
| PWR  | 緑点灯   | システムレディー（CPU正常動作中）           |
|      | 消灯     | 電源OFF                                     |
| SV   | 緑点灯   | サーボON（運転可能）                         |
|      | 緑点滅   | 自動サーボOFF中                             |
|      | 消灯     | サーボOFF                                   |
| ALM  | 橙点灯   | アラーム発生中（動作解除またはコールドスタートレベルアラーム） |
| STOP | 赤点灯   | 非常停止中（アラームとは無関係）             |
| WRG  | 橙点滅   | メッセージレベルアラーム発生中               |
| SAFE | 赤点灯   | SS1-t入力中                                 |
