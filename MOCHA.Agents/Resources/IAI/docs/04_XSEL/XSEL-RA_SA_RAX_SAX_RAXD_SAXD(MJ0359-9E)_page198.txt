### テキスト
#### 4.1 運転の種類
XSELコントローラーは上位コントローラーがなくても運転可能なプログラミングコントローラーであり、専用プログラム言語（SEL言語）を使用します。SEL言語の詳細は別冊「SEL言語プログラミングマニュアル（MJ0224）」を参照してください。

運転方式は以下の通り分類されます。

### 図
```mermaid
flowchart TD
    A[運転方式]
    A --> B[スタンドアローン（単独）で運転]
    B --> B1[[4.4.1 電源投入による自動起動 参照]]
    A --> C[I/Oインターフェイスによる運転]
    C --> C1[PIO（24V入出力）による運転]
    C1 --> C1a[[4.4.2 プログラムNo.を指定して起動 参照]]
    C --> C2[フィールドネットワーク通信（オプション）による運転]
    C2 --> C2a[(DeviceNet、CC-Link、PROFIBUS-DP、EtherCAT、EtherNet/IP、CC-Link IE Fieldに対応可能)]
    C2a --> C2b[[別冊 各フィールドネットワーク取扱説明書 参照]]
    A --> D[シリアル通信で指令して運転]
    D --> D1[シリアル通信（RS-232C）による運転]
    D1 --> D1a[[別冊 SELプログラミングマニュアル 参照]]
    D --> D2[Ethernet通信による運転]
    D2 --> D2a[[別冊 Ethernet取扱説明書 参照]]
    A --> E[ティーチングツールでの運転]
```

**図の説明:**
- 運転方式は「スタンドアローン（単独）」「I/Oインターフェイスによる運転」「シリアル通信で指令して運転」「ティーチングツールでの運転」に分類される。
- I/Oインターフェイスによる運転はさらに「PIO（24V入出力）による運転」と「フィールドネットワーク通信（オプション）による運転」に分かれる。
- シリアル通信は「RS-232C通信」と「Ethernet通信」に分かれ、それぞれ別冊マニュアルを参照する。
