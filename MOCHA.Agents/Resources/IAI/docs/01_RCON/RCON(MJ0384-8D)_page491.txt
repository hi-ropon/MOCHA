### テキスト
#### PLC入力＝軸状態信号のアドレス割付内容
PLC入力としての軸状態信号の各通信プロトコルにおけるアドレス割付を示す。

### 表
#### 各通信プロトコルのアドレス割付

| 通信方式 | アドレス例（上位バイト） | アドレス例（下位バイト） |
| :--- | :--- | :--- |
| CC-Link, CC-Link IE Field, CC-Link IE TSN | m+0 ～ m+7 | - |
| DeviceNet | n+0 ～ n+7 | n+1 ～ n+15 |
| PROFIBUS-DP, EtherNet/IP, EtherCAT | p+0 ～ p+1 | p+0 ～ p+1 |

### 図の説明
- 1ワードは16ビットで構成されている。
- 各ワードは軸の現在位置（L・H）、現在電流値（L・H）、現在速度（L・H）、アラームコード、過負荷レベルモニターなどの情報を符号付き整数として割り当てている。
- 各通信プロトコルごとに対応するアドレスが異なり、CC-Link系はm+0～m+7、DeviceNetはn+0～n+7およびn+1～n+15、PROFIBUS-DPなどはp+0～p+1となっている。
