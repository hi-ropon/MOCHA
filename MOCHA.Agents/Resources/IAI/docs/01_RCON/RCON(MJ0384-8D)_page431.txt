### テキスト
#### 6.1 アドレス構成
RCONのアドレス構成は、フィールドネットワークの種類に関係なく、どのドライバーユニットも共通です。  
ネットワークに占有されるアドレスは、8ワードの固定領域と、動作モードと軸数によって変化するデータ領域、オプションユニット用データ領域で構成されます。  
各動作モードと占有データ領域は以下の通りです。  
直接数値指定モード、簡易直値モード、ポジショナー1～3、5モード、フル直値モード、フル直値2モードは混在して使用可能で、各軸に任意のモードを選択できます。  
EC接続ユニットは動作モードを選択できません。

#### 注意事項：MON信号について
- PLCとのリンク確立後、ゲートウェイ制御信号のMON信号をONにしてください。  
- EC接続ユニット以外はMON信号をONにしないとPLCからの制御ができません。  
- MON信号の詳細や有効/無効設定は以下を参照してください。  
  - 詳細：[6.2.1 固定領域の制御・状態信号【1】ゲートウェイユニットの制御信号・状態信号]  
  - 有効/無効設定：[5.4.5 特殊パラメーターの設定]

### 表（アドレス構成の固定領域とデータ領域）
| 領域名             | ワード数 | 備考                     |
|--------------------|----------|--------------------------|
| 固定領域           | 8ワード  | 固定領域                 |
| 直接数値指定モード | 8ワード  | × 軸数                   |
| 簡易直値モード     | 4ワード  | × 軸数                   |
| ポジショナー1モード| 4ワード  | × 軸数                   |
| ポジショナー2,5モード| 2ワード | × 軸数                   |
| ポジショナー3モード| 1ワード  | × 軸数                   |
| フル直値1,2モード  | 16ワード | × 軸数                   |
| オプションユニット | 1ワード  | × 軸数                   |

PLCからゲートウェイユニットを経由してドライバーユニットへ制御信号が送られるが、MON信号がOFFの場合、ドライバーユニットは制御信号をアクチュエーターに送れず制御ができない状態になることを示しています。
