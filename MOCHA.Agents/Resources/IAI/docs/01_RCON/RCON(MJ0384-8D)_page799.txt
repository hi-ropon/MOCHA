### テキスト
#### 軸構成変更画面の説明
“軸構成変更画面”が開きます。まず、ドライバーユニットで使用する軸数を選択してください。

#### 操作手順の概要
- 追加後の軸番号設定を「未割当番号」または「シフト」から選択可能。
- 「未割当番号」は未割当の番号を設定する。
- 「シフト」は追加した分のドライバーの軸番号を前からシフトする。

### 表
| 選択項目       | 軸番号の設定変更結果                         |
| :------------- | :------------------------------------------ |
| 未割当番号     | 未割当の番号を設定する。                     |
| シフト         | 追加した分のドライバーの軸番号を前からシフトする。 |

### 画像説明
- 画面キャプチャには「軸構成変更」画面が表示されており、追加後の軸番号設定を選択するドロップダウンメニューや現在値とプレビューの軸番号が示されている。
- 2つのスクリーンショットセットは、Unit1とUnit2に予約ユニットを追加した場合の軸番号の変化を示す。
  - 「未割当番号」を選択した場合、追加ユニットは未割当番号が割り当てられる。
  - 「シフト」を選択した場合、既存の軸番号が前からシフトされ、追加ユニットの軸番号が連続的に割り当てられる。

＜Unit1-Unit2に予約ユニットを追加し、「未割当番号」を選択した場合＞
（左）追加前 → （右）追加後の軸番号が変化

＜Unit1-Unit2に予約ユニットを追加し、「シフト」を選択した場合＞
（左）追加前 → （右）追加後の軸番号が変化
