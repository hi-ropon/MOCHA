### テキスト
#### 6.5.1 入出力信号のタイミング
PLCのシーケンスプログラムでアクチュエーターを動作させるため、いずれかの制御信号をONにし、その応答（状態）信号がPLCに戻るまでの最大応答時間は以下の式で表されます。

最大応答時間（ms）＝ Yt + (Tt×3) + 指令処理時間 + (Tt×2) + Xt

- Tt：伝送時間（軸の総数により異なる）  
  - 総数1～4軸：1ms  
  - 5～9軸：2ms  
  - 10～14軸：3ms  
  - 15～16軸：4ms  
- Yt：マスター局→リモートI/O局伝送遅れ時間  
- Xt：リモートI/O局→マスター局伝送遅れ時間  

YtとXtはフィールドネットワーク伝送遅れ時間を示します。

マスター局→リモートI/O局伝送遅れ時間（Yt）、リモートI/O局→マスター局伝送遅れ時間（Xt）については、各種フィールドネットワークマスター ユニットおよび搭載されるPLCの取扱説明書を参照してください。

また、EC接続ユニットを接続する場合は「6.4.3 EC接続ユニット接続時の注意事項」を参照してください。

伝送路の問題などで通信エラーが発生した場合、通信時間（Tt×3 / Tt×2）が伝送時間（Tt）×エラー発生回数分延びます。通信が正常に行えなくなった場合は、動作解除レベルのアラーム（アラーム8DE「ドライバーユニット通信異常」）が発生し、アクチュエーターが停止します。

### 図
```mermaid
flowchart TD
    subgraph PLCシーケンスプログラム
        A1[制御信号] -->|送信| B1[マスター局]
        B1 -->|状態信号| A2[状態信号]
    end

    subgraph ゲートウェイ
        C1[制御信号] -->|送信| D1[ゲートウェイ]
        D1 -->|状態信号| C2[状態信号]
    end

    subgraph ドライバーユニット
        E1[制御信号] -->|送信| F1[ドライバーユニット]
        F1 -->|状態信号| E2[状態信号]
    end

    A1 -->|Yt: マスター局→リモートI/O局伝送遅れ時間| C1
    C1 -->|Tt×3: 通信時間| E1
    E1 -->|指令処理時間| F1
    F1 -->|Tt×2: 通信時間| C2
    C2 -->|Xt: リモートI/O局→マスター局伝送遅れ時間| A2
```

**図の説明:**  
- 制御信号はPLCシーケンスプログラムからマスター局、ゲートウェイ、ドライバーユニットへ順に送信される。  
- 状態信号は逆方向に戻り、各区間の伝送遅れ時間（Yt、Tt×3、Tt×2、Xt）と指令処理時間が含まれる。  
- これらの遅延時間の合計が最大応答時間となる。
