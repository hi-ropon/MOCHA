### テキスト
#### 5.1 運転の基本
本コントローラーはフィールドネットワークで制御されます。アクチュエーターにはスライダータイプ、ロッドタイプ、ローリータイプ、グリッパータイプなど様々なタイプがありますが、本書内で特別な記載がない限り、運転制御方法は同一です。

### 図
```mermaid
flowchart LR
    PLC[PLC（マスター）]
    Controller[コントローラー（スレーブ）]
    Actuator[アクチュエーター]
    TeachingBox[ティーチングボックス]
    Slave1[スレーブ]
    Slave2[スレーブ]

    PLC <-->|フィールドネットワークでデータ送受信| Slave1
    PLC <-->|フィールドネットワークでデータ送受信| Slave2
    PLC <--> Controller
    Controller --> Actuator
    TeachingBox --> Controller

    subgraph 指令・確認
        direction TB
        Instruction[目標位置、速度などを指令]
        Confirmation[移動完了確認（状態信号読込み）]
    end

    PLC --> Instruction
    Confirmation --> PLC
```

**図の説明:**
- PLC（マスター）がフィールドネットワークを介して複数のスレーブ（コントローラー）とデータ送受信を行う。
- コントローラー（スレーブ）はアクチュエーターを制御し、ティーチングボックスからの操作も受ける。
- PLCは目標位置や速度などの指令を出し、移動完了の確認（状態信号の読込み）を行う。
</contents>
