### テキスト
#### PLC入力＝軸状態信号のアドレス割付内容
PLC入力としての軸状態信号のアドレス割付を、各種通信プロトコルごとに示しています。  
対象プロトコルはCC-Link（CC-Link IE Field、CC-Link IE TSN含む）、DeviceNet、PROFIBUS-DP、EtherNet/IP、EtherCAT、PROFINET-IOです。

### 表
| 通信プロトコル | アドレス例（上位バイト・下位バイト） |
| :------------- | :----------------------------------- |
| CC-Link, CC-Link IE Field, CC-Link IE TSN | m+0 ～ m+7 |
| DeviceNet | n+0 ～ n+7 |
| PROFIBUS-DP, EtherNet/IP, EtherCAT | n+0 ～ n+15 |
| PROFINET-IO | p+0, p+1 |

### グラフ（ビット割付図）
1ワード＝16ビット（ビット15b～0b）で構成され、各ビットの意味は以下の通りです。

| ワード | 内容 |
| :----- | :--- |
| m+0 / n+0 / p+0 | 現在位置（L）（符号付整数） |
| m+1 / n+1 / p+0 | 現在位置（H）（符号付整数） |
| m+2 / n+2 / p+0 | 現在電流値（L） |
| m+3 / n+3 / p+0 | 現在電流値（H） |
| m+4 / n+4 / p+1 | 現在速度（L）（符号付整数） |
| m+5 / n+5 / p+1 | 現在速度（H）（符号付整数） |
| m+6 / n+6 / p+1 | アラームコード |
| m+7 / n+7 / p+1 | 過負荷レベルモニター |
