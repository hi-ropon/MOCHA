### テキスト
#### 1.8.4 占有サイズ制限
占有サイズとは、“固定領域”と“データ領域”を合わせたデータサイズのことを指します。  
- 固定領域はゲートウェイユニットに割り当てられている8ワードの領域です。  
- データ領域はドライバーユニットの軸制御設定（動作モード）と接続台数、オプションユニットの接続台数によって変動します。

占有サイズにはフィールドネットワークごとに使用できるデータ量の制限があり、以下の表にネットワークごとの占有サイズを示しています。

```mermaid
    データ領域[データ領域]

    データ領域 --> ポジショナー1モード["ポジショナー1モード（4ワード）×軸数"]
    データ領域 --> ポジショナー2_5モード["ポジショナー2,5モード（2ワード）×軸数"]
    データ領域 --> ポジショナー3モード["ポジショナー3モード（1ワード）×軸数"]
    データ領域 --> フル直値1_2モード["フル直値1,2モード（16ワード）×軸数"]
    データ領域 --> オプションユニット["オプションユニット（1ワード）×ユニット数"]
    
    固定領域 --> データ領域
```
**図の説明:**  
固定領域（8ワード）に加え、動作モードや軸数、オプションユニット数に応じたデータ領域が加算され、占有サイズが決まる。

| フィールドネットワーク           | 占有サイズ【W】（バイト数） |
| :------------------------------ | :-------------------------- |
| CC-Link                         | 128（256）                  |
| CC-Link IE Field                | 256（512）                  |
| CC-Link IE TSN                  | 256（512）                  |
| DeviceNet                      | 72（144）                   |
| EtherCAT                       | 72（144）                   |
| EtherNet/IP                    | 72（144）                   |
| EtherNet/IP（フロントポート） | 128（256）                  |
| PROFIBUS-DP                   | 72（144）                   |
| PROFINET-IO                   | 72（144）                   |

※ CC-Link、CC-Link IE Field、CC-Link IE TSN、EtherNet/IP（フロントポート）は、GW制御領域＋予約領域（8W分）を含まない占有サイズを示しています。
