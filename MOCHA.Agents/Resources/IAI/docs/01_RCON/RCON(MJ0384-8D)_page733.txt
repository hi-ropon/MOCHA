### テキスト
#### 10.3.2 ゲートウェイのアラームコード別の原因と対策
- アラームコードはゲートウェイ状態信号0のALMC1～128（b0～b7）に読出される。  
- 詳細は「6.2.1 固定領域の制御・状態信号」を参照。  
- ゲートウェイパラメーター設定ツールに表示されるアラームコードは、下表のアラームコードの先頭に「8」が付加される（例：アラームコード43は843と表示）。

### 表
#### ゲートウェイのアラームコード別原因と対策

| アラームコード | アラーム名称                         | 原因/対策                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| :------------- | :--------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 4A (84A)       | リアルタイムクロック発振停止検出   | 原因：時刻データが失われた。時刻データはゲートウェイユニットの電源をOFFしてから10日程度保持できる。<br>対策：ゲートウェイパラメーター設定ツールから時刻を再度設定してください。                                                                                                                                                                                                                                                                                                   |
| 4B (84B)       | リアルタイムクロックアクセス異常   | 原因：RCON内部のエラー。内部で時刻データの取得に失敗。<br>対策：電源再投入してください。                                                                                                                                                                                                                                                                                                                                                                                               |
| 4C (84C)       | カレンダー機能用バックアップコンデンサー推定寿命超過 | 原因：ゲートウェイ内部のコンデンサーの静電容量が50%低下。<br>対策：できるだけ早くゲートウェイユニットを交換してください。                                                                                                                                                                                                                                                                                                                                                             |
| 50 (850)       | フィールドネットワーク通信異常（ERR-C） | 原因：フィールドネットワークのリンク異常。ゲートウェイパラメーター設定ツールで本エラー発生時ラッチを設定すると、エラーの状態でアクチュエーターの動作を停止し、解除信号を受付けるまで指令は無視される。<br>対策：1) 上位機器（PLCなど）がRUN状態（運転状態）もしくはモニター状態か確認。<br>2) フィールドネットワークの設定（ノードアドレス、通信速度など）や配線を確認。                                                                                                         |
| 61 (861)       | 従属軸通信内部エラー（送信）         | 原因：RCON内部の通信エラー。アクチュエーター各軸を接続するドライバーユニットと通信できなかった。<br>対策：電源再投入してください。                                                                                                                                                                                                                                                                                                                                                   |
| 62 (862)       | 従属軸通信内部エラー（受信）         | 原因：RCON内部の通信エラー。アクチュエーター各軸を接続するドライバーユニットと通信できなかった。<br>対策：電源再投入してください。再度発生する場合は問い合わせ。                                                                                                                                                                                                                                                                                                                   |
| 6D (86D)       | 電源ユニット通信異常                 | 原因：ゲートウェイユニットと当社製DC24V電源ユニット（型式：PSA-24(L)）の通信異常。<br>1) 通信ケーブルの断線やコネクター接触不良<br>2) 電源ユニットの接続台数設定が正しくない<br>3) 電源ユニットのロータリースイッチによるアドレス設定が正しくない<br>4) 1台の電源ユニットの通信ラインを複数のゲートウェイユニットに接続している<br>5) ノイズの影響<br>対策：1) ケーブルやコネクター確認<br>2) ゲートウェイ設定ツールで確認<br>3) アドレス重複確認<br>4) 1台の電源ユニットの通信ラインは1台のゲートウェイユニットのみ接続<br>5) ノイズ対策実施 |
