### 表：ドライバーユニット・簡易アブソユニットのアラームの原因と対策

| アラームコード | アラーム名称           | 原因/対策                                                                                                                                                                                                                   |
|:--------------:|:---------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 0A7            | 指令減速度異常       | **原因**: 移動中に減速度を低く変更した時、減速距離が不足し、変更後の減速度で現在位置から減速するとソフトリミットを超えた。<br>**対策**: 減速度変更のための移動指令のタイミングを早くする。<br>【図解】速度変更時のソフトリミット超過イメージ図あり。 |
| 0AB            | 指令速度異常         | **原因**: モーションネットワークにおいて、PLCからの指令がアクチュエーターの最高速度を超えている。<br>**対策**: PLCのパラメーター、プログラムを見直す。                                                                                   |
| 0AC            | 推定寿命超過エラー   | **原因**: ドライバーユニットのモーター電源用コンデンサーが計算上の寿命時間を超えた。<br>**対策**: できる限り早くドライバーユニットを交換する。                                                                                     |
| 0B5 A/Sドライバー限定 | Z相位置異常         | **原因**: 原点復帰時にメカエンドで反転後、規定の距離を移動してもZ相を検出できなかった。<br>**対策**: 当社まで問い合わせ。                                                                                                   |
| 0B6 Aドライバー限定 | Z相検出タイムアウト | **原因**: 簡易アブソ仕様の時、電源投入後の最初のサーボONまたは原点復帰時にZ相を検出できなかった。<br>1) アクチュエーター接続ケーブルのコネクター接触不良、断線<br>2) ブレーキ付きの場合、ブレーキが解除できない<br>3) 外力加わり、モーターの検出動作ができない<br>4) アクチュエーター自体の摺動抵抗が大きい<br>**対策**: 1) ケーブル配線状況確認 2) ブレーキ解除スイッチ確認 3) 組付け状態確認 4) 電源遮断後手動で摺動抵抗確認 |
| 0BA            | 原点センサー未検出   | **原因**: 原点センサー付きアクチュエーターの原点復帰動作が正常完了していない。<br>1) 原点復帰途中でワークが周囲と干渉<br>2) アクチュエーターの摺動抵抗が大きい<br>3) 原点センサーの取付不良、故障、断線<br>**対策**: 1) 干渉ワーク除去 2) 干渉なしなら2),3)の可能性あり。問い合わせ推奨。 |

### 図：指令減速度異常の原因説明図

```mermaid
flowchart TD
    A[ソフトリミットを超えないときの減速開始位置] --> B[速度減速]
    B --> C{この位置で指令するとソフトリミットを超える}
    C --> D[ソフトリミット]

    style A fill:#f9f,stroke:#333,stroke-width:1px
    style D fill:#fcc,stroke:#333,stroke-width:1px

    note right of C
      移動途中で速度変更する際の、
      次の移動指令を出すタイミングが遅いことが原因
    end note
```

**図の説明:**  
移動中に減速度を低く変更した場合、減速開始位置が遅れることでソフトリミットを超えてしまうことを示しています。  
このため、速度変更の指令タイミングを早める必要があります。
