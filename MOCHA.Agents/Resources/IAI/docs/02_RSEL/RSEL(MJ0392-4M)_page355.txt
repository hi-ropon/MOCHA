### テキスト
#### 16ビットデータごとに上位8ビット、下位8ビットを入れ替えて読書きする方法
16ビットデータごとに上位8ビット、下位8ビットを入れ替えて読書きを行う場合、RSELシステムでIN命令、OUT命令などの入出力ポート操作命令を実行する前に、FMIO命令でフォーマット種別を1（16ビットデータごとに、上位8ビット、下位8ビットを入れ替える）に設定してください。  
詳細は別冊「SEL言語プログラミングマニュアル取扱説明書（MJ0224）」を参照してください。  
以下にPLCマスター側とRSELシステム側の割付け例を示します。

### 図
#### PLCマスター側データとRSELシステム側データのビット入れ替え例

```mermaid
flowchart LR
    subgraph PLCマスター
        HEX1[HEX: 1] --> BIN1[BIN: 0001]
        HEX2[HEX: 2] --> BIN2[BIN: 0010]
        HEX3[HEX: 3] --> BIN3[BIN: 0011]
        HEX4[HEX: 4] --> BIN4[BIN: 0100]
    end

    subgraph RSELシステム
        HEX3_2[HEX: 3] --> BIN3_2[BIN: 0011]
        HEX4_2[HEX: 4] --> BIN4_2[BIN: 0100]
        HEX1_2[HEX: 1] --> BIN1_2[BIN: 0001]
        HEX2_2[HEX: 2] --> BIN2_2[BIN: 0010]
    end

    BIN1 -- 入れ替え --> BIN3_2
    BIN2 -- 入れ替え --> BIN4_2
    BIN3 -- 入れ替え --> BIN1_2
    BIN4 -- 入れ替え --> BIN2_2
```

**説明:**  
PLCマスター側の16進数データ「1234h」は、RSELシステム側では「3412h」となり、上位8ビットと下位8ビットが入れ替わっていることを示しています。
