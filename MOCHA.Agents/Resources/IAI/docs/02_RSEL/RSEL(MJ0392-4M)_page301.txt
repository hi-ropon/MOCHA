### テキスト
#### 5.8 運転の種類
RSELシステムは、上位コントローラー（PLC）がなくても運転が可能なプログラミングコントローラーです。  
プログラムは、当社専用プログラム言語（SEL言語）を使用します。  
詳細は別冊「SEL言語プログラミングマニュアル（MJ0224）」を参照してください。

### 図
```mermaid
flowchart TD
    A[運転方式] --> B[スタンドアローン（単独）で運転]
    B --> B1[[5.9.1 電源投入による自動起動 参照]]

    A --> C[I/Oインターフェイスによる運転]
    C --> C1[PIO（24V入出力）による運転]
    C1 --> C1a[[5.9.2 プログラムNo.を指定して起動 参照]]
    C --> C2[フィールドネットワーク通信による運転]
    C2 --> C2a[CC-Link、CC-Link IE Field、DeviceNet、EtherCAT、EtherNet/IP、PROFIBUS-DP、PROFINET IOに対応可能]
    C2a --> C2b[[第6章 フィールドネットワーク、PIO、SIO 参照]]

    A --> D[シリアル通信で指令して運転]
    D --> D1[シリアル通信（RS-232C）による運転]
    D1 --> D1a[[別冊 SEL言語プログラミングマニュアル 参照]]
    D --> D2[Ethernet通信（オプション）による運転]

    A --> E[ティーチングツールでの運転]
```

**図の説明:**  
- 運転方式は大きく4つに分類される。  
- スタンドアローン（単独）での運転は電源投入による自動起動が可能。  
- I/Oインターフェイスによる運転は、PIO（24V入出力）やフィールドネットワーク通信を利用。  
- シリアル通信ではRS-232CやオプションのEthernet通信が利用可能。  
- ティーチングツールを用いた運転も可能。
