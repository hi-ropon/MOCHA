### テキスト
#### PLCマスター側出力領域とRSELシステム側入力領域の対応
PLCマスター側の出力データ（例：1234h）は、RSELシステム側では同じ値（1234h）として扱われることを説明しています。

---

### 表
#### PLCマスター側出力領域とRSELシステム側入力領域のビット対応表

| アドレス | Bit7 (MSB) | 6 | 5 | 4 | 3 | 2 | 1 | 0 (LSB) |
| :------- | :--------- |:-:|:-:|:-:|:-:|:-:|:-:|:------- |
| RSELシステム側 入力ポート番号 | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
| PLCマスター側 出力ワードアドレス | 0 (下位バイト) |  |  |  |  |  |  |  |
| RSELシステム側 入力ポート番号 | 15 | 14 | 13 | 12 | 11 | 10 | 9 | 8 |
| PLCマスター側 出力ワードアドレス | 0 (上位バイト) |  |  |  |  |  |  |  |
| RSELシステム側 入力ポート番号 | 23 | 22 | 21 | 20 | 19 | 18 | 17 | 16 |
| PLCマスター側 出力ワードアドレス | 1 (下位バイト) |  |  |  |  |  |  |  |
| RSELシステム側 入力ポート番号 | 31 | 30 | 29 | 28 | 27 | 26 | 25 | 24 |
| PLCマスター側 出力ワードアドレス | 1 (上位バイト) |  |  |  |  |  |  |  |

### 図
#### PLCマスター側データとRSELシステム側データの対応例

```mermaid
flowchart TD
    A[PLCマスター HEX: 1 2 3 4] -->|BIN: 0001 0010 0011 0100| B[RSELシステム HEX: 1 2 3 4]
    B -->|BIN: 0001 0010 0011 0100| C[同じビットパターンで受信]
```

---

**説明**  
- PLCマスター側の16進数データ「1234h」は、2進数で「0001 0010 0011 0100」と表される。  
- RSELシステム側でも同じ16進数・2進数のビットパターンでデータを受け取る。  
- 表は各ビットの対応関係を示し、PLCマスター側のワードアドレスの下位・上位バイトがRSELシステム側の入力ポート番号のビットに対応していることを示す。
