### テキスト
#### 6.2.3 I/Oポートとデータの読書き
RSELシステムのI/O（入出力）ポート操作のためのSEL言語命令は、データの入替えを行わず読み込み/書き込みを行うのが初期値となっています。  
以下にRSELシステム側と各フィールドネットワークのマスター（PLCマスター）側の割付け例を示します。

### 表

| アドレス                 | Bit7 (MSB) | 6   | 5   | 4   | 3   | 2   | 1   | 0 (LSB) |
|--------------------------|------------|-----|-----|-----|-----|-----|-----|---------|
| RSELシステム側 出力ポート番号 | 307        | 306 | 305 | 304 | 303 | 302 | 301 | 300     |
| PLCマスター側 入力ワードアドレス |            |     |     |     | 0（下位バイト） |     |     |         |
| RSELシステム側 出力ポート番号 | 315        | 314 | 313 | 312 | 311 | 310 | 309 | 308     |
| PLCマスター側 入力ワードアドレス |            |     |     |     | 0（上位バイト） |     |     |         |
| RSELシステム側 出力ポート番号 | 323        | 322 | 321 | 320 | 319 | 318 | 317 | 316     |
| PLCマスター側 入力ワードアドレス |            |     |     |     | 1（下位バイト） |     |     |         |
| RSELシステム側 出力ポート番号 | 331        | 330 | 329 | 328 | 327 | 326 | 325 | 324     |

| ...                      |            |     |     |     |     |     |     |         |

### 図（データ変換例）
RSELシステムからのデータ「1234h」は、PLCマスター側でも同じく「1234h」となります。

```mermaid
flowchart TD
    A[RSELシステム] --> B[PLCマスター]

    subgraph RSELシステム
        direction LR
        HEX1[HEX: 1] --> HEX2[HEX: 2] --> HEX3[HEX: 3] --> HEX4[HEX: 4]
        BIN1[BIN: 0001] --> BIN2[BIN: 0010] --> BIN3[BIN: 0011] --> BIN4[BIN: 0100]
    end

    subgraph PLCマスター
        direction LR
        HEX1_p[HEX: 1] --> HEX2_p[HEX: 2] --> HEX3_p[HEX: 3] --> HEX4_p[HEX: 4]
        BIN1_p[BIN: 0001] --> BIN2_p[BIN: 0010] --> BIN3_p[BIN: 0011] --> BIN4_p[BIN: 0100]
    end
```

**説明:**  
- HEX表記とBIN表記の各桁がRSELシステム側からPLCマスター側へそのまま対応していることを示しています。
